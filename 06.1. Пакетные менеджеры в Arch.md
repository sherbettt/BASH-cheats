# **Руководство по пакетным менеджерам и управлению репозиториями в EndeavourOS (и Arch Linux) 🚀**

### **Введение**
EndeavourOS — это дистрибутив, основанный на Arch Linux. Это значит, что вы получаете всю мощь Arch (роллинг-релизы, свежее ПО, AUR), но с простым и дружелюбным установщиком и несколькими предустановленными инструментами, которые делают жизнь новичка проще. Главные герои этого руководства — **Pacman** (сердце системы), **Yay** (предустановленный AUR-хелпер) и **Paru** (отличная альтернатива, написанная на Rust).

---

## **Часть 1: Пакетные менеджеры в EndeavourOS**

### **Основные пакетные менеджеры**

В вашей системе уже есть всё необходимое:

1.  **Pacman** — это **низкоуровневый, основной пакетный менеджер**. Он работает с официальными репозиториями Arch. Быстрый и мощный, но требует ручного управления некоторыми вещами (например, обновлениями из AUR он не занимается). Команда: `pacman`.
2.  **Yay** («Yet Another Yogurt») — это **AUR-хелпер**, который **уже предустановлен в EndeavourOS** (в отличие от обычного Arch или Manjaro). Это консольная программа, которая умеет работать и с официальными репозиториями (через pacman), и с **AUR (Arch User Repository)**. Команда: `yay`. **Для повседневного использования рекомендуется использовать именно `yay`**, так как он закрывает 99% потребностей — от обновления системы до установки программ из AUR.
3.  **Paru** — это современный AUR-хелпер, написанный на Rust. Он полностью совместим с pacman (поддерживает все его флаги) и предлагает дополнительные удобные функции. Многие опытные пользователи предпочитают его Yay за скорость и чистоту кода. Команда: `paru`.
4.  **Pamac** — графический менеджер пакетов (как «Магазин приложений»). Он тоже умеет работать с AUR. В EndeavourOS он не установлен по умолчанию, но вы можете поставить его, если хотите графический интерфейс: `yay -S pamac-aur`.

## **🔥 PACMAN — Сердце системы (подробный разбор)**

**Pacman** (Package Manager) — это основной инструмент управления пакетами в Arch Linux и всех его производных. Все остальные программы (yay, paru, pamac) — это всего лишь "обёртки" (frontends), которые используют Pacman под капотом и добавляют свои функции (например, работу с AUR).

### **Структура команд Pacman (флаги)**

Все команды Pacman строятся по простой схеме: `pacman -<флаг_операции> <флаг_дополнительный> <пакет>`

**Основные флаги операций (что делаем):**

| Флаг | Значение | Пример |
|:---:|:---|:---|
| **`-S`** | **Sync** — синхронизация с репозиториями (установка, поиск, обновление) | `sudo pacman -S firefox` |
| **`-R`** | **Remove** — удаление пакетов | `sudo pacman -R firefox` |
| **`-Q`** | **Query** — запрос информации о локальной базе данных (установленные пакеты) | `pacman -Q firefox` |
| **`-F`** | **Files** — поиск по файлам (в репозиториях) | `pacman -F /bin/bash` |
| **`-U`** | **Upgrade** — установка из локального файла | `sudo pacman -U ~/Загрузки/пакет.pkg.tar.zst` |

**Дополнительные флаги (как делаем, уточняем действие):**

| Флаг | Значение | Пример |
|:---:|:---|:---|
| **`-y`** | Обновить базу данных пакетов из репозиториев | `sudo pacman -Sy` |
| **`-u`** | Обновить все пакеты в системе | `sudo pacman -Syu` (**главная команда!**) |
| **`-s`** | Поиск по названию или описанию | `pacman -Ss текстовый-редактор` |
| **`-i`** | Показать подробную информацию о пакете | `pacman -Qi bash` (установленном) / `pacman -Si bash` (в репозитории) |
| **`-l`** | Список всех файлов, установленных пакетом | `pacman -Ql bash` |
| **`-o`** | Найти пакет, которому принадлежит файл | `pacman -Qo /bin/bash` |
| **`-d`** | Только проверить зависимости (dry run) | `sudo pacman -Sd firefox` (пропустить проверку зависимостей — опасно!) |
| **`-w`** | Только скачать пакет, не устанавливать | `sudo pacman -Sw firefox` |
| **`-n`** | При удалении не трогать конфиги (с `-R`) | `sudo pacman -Rn firefox` |
| **`-c`** | При удалении удалить и зависимости (с `-R`) | `sudo pacman -Rc firefox` |

### **Полный справочник команд Pacman (с примерами)**

Вот полный набор команд, которые пригодятся в повседневной работе (адаптировано из cht.sh):

| Действие | Команда |
|:---|:---|
| **Поиск пакета** | `pacman -Ss <пакет>...` |
| **Обновить базу и все пакеты** | `sudo pacman -Syu` |
| **Установить пакет** | `sudo pacman -S <пакет>...` |
| **Удалить пакет** | `sudo pacman -R <пакет>...` |
| **Удалить пакет + зависимости + сирот** | `sudo pacman -Rcs <пакет>...` |
| **Информация о пакете в репозитории** | `pacman -Si <пакет>...` |
| **Установить из локального файла** | `sudo pacman -U <файл.tar.xz или .zst>` |
| **Список всех файлов установленного пакета** | `pacman -Ql <пакет>...` |
| **Список явно установленных пакетов** | `pacman -Qe` |
| **Список последних установленных пакетов (не из групп base)** | `expac --timefmt='%Y-%m-%d %T' '%l\t%n' $(comm -23 <(pacman -Qeq\|sort) <(pacman -Qqg base base-devel\|sort)) \| sort -r \| head -20` |
| **Список пакетов-сирот (не нужны, но висят)** | `pacman -Qdt` |
| **Список установленных пакетов, отсортированных по размеру** | `pacman -Qi \| awk '/^Name/ {name=$3} /^Installed Size/ {print name, $4 substr($5,1,1)}' \| column -t \| sort -rhk2 \| cat -n \| tac` |
| **Очистить весь кэш pacman** | `sudo pacman -Scc` |

### **Важные особенности и советы по Pacman**

1.  **Обновление системы (`-Syu`)** — это одна команда, которая и синхронизирует базу данных, и обновляет пакеты. **Всегда делайте это перед установкой нового ПО**, чтобы избежать конфликтов зависимостей.

2.  **Pacman не работает с AUR** — это важно помнить. Если вы хотите установить что-то из AUR, используйте `yay` или `paru`.

3.  **Очистка кэша** — Pacman хранит все скачанные пакеты в `/var/cache/pacman/pkg/`. Со временем это может занять много места:
    ```bash
    # Удалить ВСЕ кэшированные пакеты (кроме последних версий)
    sudo pacman -Sc
    # Удалить ВСЕ пакеты из кэша (полная очистка)
    sudo pacman -Scc
    ```

4.  **Установка пакетов вручную из файла** — иногда нужно установить пакет, который вы скачали отдельно (например, с зеркала):
    ```bash
    sudo pacman -U /путь/к/пакету.pkg.tar.zst
    ```

## **🔥 PARU — Современная альтернатива Yay**

**Paru** — это AUR-хелпер нового поколения, написанный на Rust. Он полностью совместим с pacman (поддерживает все его флаги) и предлагает дополнительные возможности. Многие пользователи Arch и EndeavourOS переходят на него с Yay.

### **Установка Paru в EndeavourOS**

Так как Paru сам находится в AUR, установим его через Yay:

```bash
# Установка Paru
yay -S paru

# Или можно установить через git (более ручной способ)
sudo pacman -S --needed base-devel git
git clone https://aur.archlinux.org/paru.git
cd paru
makepkg -si
```

### **Сравнение Paru и Yay**

| Особенность | Yay | Paru |
|:---|:---|:---|
| **Язык программирования** | Go | Rust |
| **Совместимость с pacman** | Полная | Полная (включая все флаги) |
| **Цветной вывод** | Да | Да |
| **Интерактивный выбор пакетов** | Да | Да |
| **Режим "только AUR"** | `yay -Syu --aur` | `paru -Syu --aur` |
| **Поиск с рейтингом** | `yay -Ss` | `paru -Ss` (с улучшенным выводом) |
| **Обновление системы** | `yay` | `paru` |

### **Основные команды Paru**

| Действие | Команда Paru |
|:---|:---|
| **Обновить систему (репозитории + AUR)** | `paru` или `paru -Syu` |
| **Установить пакет** | `paru -S <пакет>` |
| **Удалить пакет** | `paru -R <пакет>` |
| **Удалить пакет с зависимостями** | `paru -Rns <пакет>` |
| **Поиск пакета** | `paru -Ss <запрос>` |
| **Информация о пакете** | `paru -Si <пакет>` |
| **Очистка кэша** | `paru -Sc` |
| **Показать статистику** | `paru -P --stats` |

### **Уникальные возможности Paru**

1.  **Режим `--devel`** — обновление пакетов, собранных из git-исходников (аналог `yay -Syu --devel`):
    ```bash
    paru -Syu --devel
    ```

2.  **Улучшенный поиск** — Paru показывает более структурированную информацию при поиске:
    ```bash
    paru -Ss yandex-browser
    ```

3.  **Интеграция с системой** — Paru уважает настройки pacman из `/etc/pacman.conf`, включая игнорирование пакетов и настройки цветов.

4.  **Проверка PKGBUILD** — Paru всегда показывает вам PKGBUILD перед сборкой (если не указан флаг `--noconfirm`), что повышает безопасность.

## **Сравнение команд (Pacman/Yay/Paru vs Apt)**

Для тех, кто переходит с Ubuntu/Debian. **Для EndeavourOS в колонке "Рекомендуемая команда" можно использовать как `yay`, так и `paru` — выбирайте тот, который больше нравится.**

| Действие | В Ubuntu/Debian (apt) | В EndeavourOS (Pacman) | **В EndeavourOS (Yay/Paru — РЕКОМЕНДУЕТСЯ)** |
| :--- | :--- | :--- | :--- |
| **Обновить систему** | `sudo apt update && sudo apt upgrade` | `sudo pacman -Syu` | **`yay`** или **`paru`** (просто команда без аргументов) |
| **Установить пакет** | `sudo apt install <пакет>` | `sudo pacman -S <пакет>` | **`yay -S <пакет>`** или **`paru -S <пакет>`** |
| **Удалить пакет** | `sudo apt remove <пакет>` | `sudo pacman -R <пакет>` | **`yay -R <пакет>`** или **`paru -R <пакет>`** |
| **Удалить пакет с зависимостями** | `sudo apt autoremove` | `sudo pacman -Rns <пакет>` | **`yay -Rns <пакет>`** или **`paru -Rns <пакет>`** |
| **Поиск пакета** | `apt search <запрос>` | `pacman -Ss <запрос>` | **`yay -Ss <запрос>`** или **`paru -Ss <запрос>`** |
| **Показать инфо о пакете** | `apt show <пакет>` | `pacman -Si <пакет>` | **`yay -Si <пакет>`** или **`paru -Si <пакет>`** |

## **🔥 Расширенный список команд пакетного менеджера (для Yay и Paru)**

Здесь собраны самые полезные команды для повседневной работы и тонкой настройки системы.

### **Базовые операции (Установка, Удаление, Поиск)**

*   **`yay -S <имя_пакета>`** или **`paru -S <имя_пакета>`** — установить пакет (из репозитория или AUR).
*   **`yay -Ss <ключевое_слово>`** или **`paru -Ss <ключевое_слово>`** — поиск пакета по ключевому слову во всех репозиториях и AUR.
*   **`yay -Rns <имя_пакета>`** или **`paru -Rns <имя_пакета>`** — удалить пакет, его ненужные зависимости и конфигурационные файлы.
*   **`yay -R <имя_пакета>`** или **`paru -R <имя_пакета>`** — удалить только пакет, оставив зависимости и конфиги.
*   **`yay -G <имя_пакета_из_AUR>`** или **`paru -G <имя_пакета_из_AUR>`** — только скачать PKGBUILD из AUR, не собирая пакет.
*   **`yay -Y --gendb`** — сгенерировать базу данных для AUR (полезно, если вы только что установили `yay`).

### **Обновление системы (Rolling Release)**

*   **`yay`** или **`paru`** (просто команда) — **главная команда**. Синхронизирует репозитории, обновляет официальные пакеты и проверяет обновления для AUR-пакетов.
*   **`yay -Syu --devel`** или **`paru -Syu --devel`** — обновить систему, включая пакеты из AUR, которые собираются из "живых" исходников (например, из Git-репозиториев).
*   **`yay -Syu --nodiffmenu --nocleanmenu`** — обновление без лишних вопросов (автоматическое принятие стандартных действий).

### **Информация о пакетах и системе**

*   **`yay -Qi <имя_пакета>`** или **`paru -Qi <имя_пакета>`** — показать подробную информацию об *установленном* пакете (версия, зависимости, описание, дата установки).
*   **`yay -Si <имя_пакета>`** или **`paru -Si <имя_пакета>`** — показать подробную информацию о пакете в *репозитории*.
*   **`yay -Ql <имя_пакета>`** или **`paru -Ql <имя_пакета>`** — список всех файлов, которые установил данный пакет.
*   **`yay -Qo /путь/до/файла`** или **`paru -Qo /путь/до/файла`** — узнать, какому пакету принадлежит файл в системе (аналог `dpkg -S`).
*   **`yay -Qdt`** или **`paru -Qdt`** — показать список "осиротевших" пакетов (зависимости, которые больше не нужны ни одному установленному пакету).
*   **`yay -Y -Pg`** — вывести статистику по пакетам из AUR (только для Yay).
*   **`paru -P --stats`** — вывести статистику по пакетам (для Paru).
*   **`pacman -Q | wc -l`** — посчитать общее количество установленных пакетов в системе.

### **Очистка системы и кэша**

*   **`yay -Sc`** или **`paru -Sc`** — удалить из кэша (`/var/cache/pacman/pkg/`) все старые версии пакетов, кроме текущих установленных.
*   **`yay -Scc`** или **`paru -Scc`** — **очистить кэш полностью** (удалить вообще все скачанные пакеты). Делайте это, только если уверены, что не придется откатываться до старых версий.
*   **`yay -Y -c`** — удалить неиспользуемые зависимости (осиротевшие пакеты) и очистить кэш сборки AUR-пакетов (`~/.cache/yay/`).
*   **`paru -c`** — аналогичная очистка для Paru.

## **Часть 2: Управление репозиториями в EndeavourOS**

### **Структура репозиториев (Arch Linux)**

EndeavourOS использует стандартные репозитории Arch Linux. Основные из них:

```
core      - самые базовые пакеты системы (ядро, glibc, systemd)
extra     - остальные пакеты (браузеры, офисные пакеты, игры)
community - пакеты, собранные и поддерживаемые сообществом Arch Linux
multilib  - 32-битные библиотеки для 64-битной системы (нужно для игр и Steam)
```

### **Конфигурационные файлы**

*   **Главный файл:** `/etc/pacman.conf`
    Здесь настраиваются параметры pacman (цветной вывод, параллельные загрузки, игнорирование пакетов) и подключаются репозитории.
*   **Файл зеркал:** `/etc/pacman.d/mirrorlist`
    Список серверов, откуда скачиваются пакеты. EndeavourOS имеет свой инструмент для управления этим списком.

### **Управление зеркалами в EndeavourOS (`pacman-mirrors`)**

EndeavourOS поставляется с собственной утилитой `pacman-mirrors`, которая настраивает список зеркал специально для этого дистрибутива. **Не редактируйте mirrorlist вручную!** Используйте эту команду:

*   **`sudo pacman-mirrors --fasttrack`** — автоматически выбрать 10 самых быстрых зеркал.
*   **`sudo pacman-mirrors --country Russia,Germany --fasttrack 5`** — выбрать 5 самых быстрых зеркал только из России и Германии.
*   **`sudo pacman-mirrors --geoip`** — автоматически определить вашу страну по IP и выбрать зеркала оттуда.
*   **`sudo pacman-mirrors --status`** — показать текущий статус и скорость всех настроенных зеркал.
*   **`sudo pacman-mirrors --api --set-branch unstable`** — переключиться на нестабильную ветку (только для опытных!).
*   **`sudo pacman-mirrors --interactive`** — интерактивный режим выбора зеркал в псевдографическом интерфейсе.
*   **`sudo pacman-mirrors --generate`** — сгенерировать новый список зеркал на основе текущих настроек.
*   **`sudo pacman-mirrors --list`** — показать список всех доступных зеркал.

**Важно:** После изменения зеркал всегда выполняйте **`sudo pacman -Syy`** (принудительное обновление баз данных), чтобы применить изменения.

### **Ручное управление зеркалами (продвинутый уровень)**

Если вы хотите отредактировать список зеркал вручную, используйте:

```bash
# Сделайте резервную копию текущего mirrorlist
sudo cp /etc/pacman.d/mirrorlist /etc/pacman.d/mirrorlist.backup

# Отредактируйте файл (например, через nano)
sudo nano /etc/pacman.d/mirrorlist
```

В файле зеркала располагаются в порядке приоритета: чем выше зеркало в списке, тем выше его приоритет. Раскомментируйте (удалите `#`) нужные строки.

**Совет:** Используйте `reflector` для автоматической генерации списка зеркал (альтернатива `pacman-mirrors`):
```bash
# Установка reflector
sudo pacman -S reflector

# Создать список из 10 самых быстрых зеркал из России и Германии, отсортированных по скорости
sudo reflector --country Russia,Germany --latest 10 --sort rate --save /etc/pacman.d/mirrorlist
```

## **🔥 Управление ядром в EndeavourOS**

### **Просмотр информации о ядре**

*   **`uname -r`** — показать текущую версию запущенного ядра.
*   **`uname -a`** — показать полную информацию о системе и ядре.
*   **`pacman -Q | grep linux`** — показать все установленные пакеты, связанные с ядром.
*   **`ls /boot`** — посмотреть установленные образы ядра и initramfs.

### **Обновление ядра**

Ядро в Arch Linux обновляется вместе с системой. **При выполнении `yay` или `sudo pacman -Syu` ядро автоматически обновится, если вышла новая версия.**

После обновления ядра **обязательно перезагрузите систему**, чтобы изменения вступили в силу:
```bash
sudo reboot
```

### **Поиск и установка альтернативных ядер**

В официальных репозиториях и AUR доступны различные версии ядер:

| Ядро | Описание | Установка |
|:---|:---|:---|
| **linux** | Стандартное ядро (рекомендуется для большинства пользователей) | `sudo pacman -S linux` |
| **linux-lts** | Долгосрочная поддержка (Long Term Support) — более стабильная версия | `sudo pacman -S linux-lts` |
| **linux-zen** | Оптимизированное для производительности и отзывчивости ядро | `sudo pacman -S linux-zen` |
| **linux-hardened** | Ядро с усиленными механизмами безопасности | `sudo pacman -S linux-hardened` |
| **linux-ck** | Ядро с патчами Con Kolivas для лучшей производительности на десктопах | `yay -S linux-ck` |
| **linux-xanmod** | Ядро с оптимизациями для игр и десктопа | `yay -S linux-xanmod` |

### **Установка и настройка нескольких ядер**

Вы можете установить несколько ядер параллельно и выбирать нужное при загрузке через GRUB:

```bash
# Установить стандартное ядро и LTS-ядро
sudo pacman -S linux linux-lts

# После установки обновите GRUB
sudo grub-mkconfig -o /boot/grub/grub.cfg
```

### **Удаление старого ядра**

Если вы установили несколько ядер и хотите удалить одно из них:

```bash
# Просмотр установленных ядер
pacman -Q | grep linux

# Удаление конкретного ядра (например, linux-zen)
sudo pacman -R linux-zen
```

**⚠️ Внимание:** Никогда не удаляйте ядро, на котором работает система в данный момент. Перед удалением убедитесь, что у вас есть запасное рабочее ядро.

### **Специальные команды для управления ядром**

*   **Обновление модулей ядра после обновления:** обычно происходит автоматически, но при необходимости:
    ```bash
    sudo mkinitcpio -P
    ```
    Эта команда пересобирает initramfs для всех установленных ядер.

*   **Просмотр загруженных модулей ядра:**
    ```bash
    lsmod
    ```

*   **Получение информации о конкретном модуле:**
    ```bash
    modinfo <имя_модуля>
    ```

*   **Загрузка/выгрузка модуля вручную:**
    ```bash
    sudo modprobe <имя_модуля>   # загрузить
    sudo modprobe -r <имя_модуля> # выгрузить
    ```

### **Решение проблем с ядром**

*   **Если система не загружается после обновления ядра:**
    1. При загрузке в GRUB выберите **"Advanced options for EndeavourOS"**.
    2. Выберите загрузку с предыдущей версией ядра.
    3. После входа в систему можно откатить обновление или исправить конфигурацию.

*   **Откат обновления ядра (если не загружается):**
    ```bash
    # Просмотр истории пакетов
    grep "upgraded linux" /var/log/pacman.log
    
    # Установка предыдущей версии из кэша
    sudo pacman -U /var/cache/pacman/pkg/linux-<старая_версия>.pkg.tar.zst
    ```

## **Часть 3: AUR (Arch User Repository) — Главная суперсила**

AUR — это хранилище пользовательских скриптов (PKGBUILD), которые позволяют установить практически любую программу под Arch, даже если её нет в официальных репозиториях.

**В EndeavourOS работа с AUR максимально проста благодаря предустановленному `yay` (или вы можете установить `paru`).**

### **Как установить программу из AUR (например, Яндекс.Браузер)**

1.  **Поиск пакета:**
    ```bash
    yay -Ss yandex-browser
    # или
    paru -Ss yandex-browser
    ```
    Вы увидите список пакетов, включая `yandex-browser` (стабильный), `yandex-browser-beta` и т.д.

2.  **Установка:**
    ```bash
    yay -S yandex-browser
    # или
    paru -S yandex-browser
    ```
    Хелпер скачает PKGBUILD, покажет вам его для проверки (можно нажать Enter), а затем соберет и установит пакет, автоматически разрешив все зависимости.

### **Ручная установка из AUR (без хелпера)**

Иногда полезно знать, как установить пакет из AUR вручную — например, если вы не доверяете хелперам или хотите понять процесс:

```bash
# Убедитесь, что установлены base-devel и git
sudo pacman -S --needed base-devel git

# Склонируйте репозиторий пакета
git clone https://aur.archlinux.org/<package>.git
cd <package>

# Соберите пакет
makepkg -s

# Установите собранный пакет
sudo pacman -U <package>.pkg.tar.zst
```

### **Популярные AUR-пакеты для EndeavourOS**

*   `google-chrome` — Google Chrome.
*   `visual-studio-code-bin` — Microsoft Visual Studio Code.
*   `spotify` — официальный клиент Spotify.
*   `teamviewer` — TeamViewer.
*   `slack-desktop` — Slack.
*   `discord` — Discord (есть и в официальных репозиториях, но из AUR часто новее).
*   `enpass-bin` — Enpass Password Manager.
*   `paru` — сам Paru (мета-ирония — хелпер в AUR).

## **Часть 4: Практические примеры и решения проблем**

### **Пример: Установка Яндекс.Браузера (пошагово)**

1.  Откройте терминал.
2.  Выполните поиск (необязательно, но полезно для проверки названия):
    ```bash
    yay -Ss yandex-browser
    ```
3.  Установите браузер:
    ```bash
    yay -S yandex-browser
    ```
4.  Yay может попросить вас ввести пароль (для sudo) и подтвердить установку. Если пакет требует ручного вмешательства (например, принятия лицензии), он сообщит об этом. Готово! Ярлык появится в меню приложений Budgie.

### **Решение типичных проблем**

*   **Ошибка: "failed to commit transaction (conflicting files)"**
    Конфликт файлов. Обычно это значит, что пакет пытается перезаписать файл, принадлежащий другому пакету. Решение:
    ```bash
    yay -Syu --overwrite="путь/к/файлу"  # Перезаписать конкретный файл
    # Или для всех конфликтов (с осторожностью!)
    yay -Syu --overwrite="*"
    ```

*   **Ошибка: "invalid or corrupted package (PGP signature)"**
    Проблема с GPG-ключами при сборке AUR-пакета. Решение:
    ```bash
    # Обновить ключи
    sudo pacman-key --refresh-keys
    # Или установить недостающий ключ вручную (id ключа будет в ошибке)
    sudo pacman-key --recv-keys <ID_ключа>
    sudo pacman-key --lsign-key <ID_ключа>
    ```

*   **Ошибка: "nothing to do" при попытке установить AUR-пакет**
    Хелпер не находит пакет. Проверьте название командой `yay -Ss <примерное_название>` или `paru -Ss <примерное_название>`. Возможно, пакет называется иначе.

*   **Пропала скорость интернета при загрузке пакетов**
    Обновите зеркала:
    ```bash
    sudo pacman-mirrors --fasttrack && sudo pacman -Syy
    ```

*   **Paru не найден после установки**
    Если вы установили Paru, но команда `paru` не работает, возможно, путь не добавлен в PATH. Попробуйте перелогиниться или выполнить:
    ```bash
    source ~/.bashrc
    ```

## **Чек-лист и советы для новичка в EndeavourOS**

1.  **Первое, что нужно сделать после установки EndeavourOS:**
    ```bash
    yay  # Полностью обновить систему
    # или
    paru  # если вы установили Paru
    ```

2.  **Регулярное обслуживание:**
    *   Раз в неделю выполняйте **`yay`** или **`paru`**.
    *   Иногда чистите систему: **`yay -Y -c`** (для Yay) или **`paru -c`** (для Paru) — удалит ненужные зависимости и мусор из кэша.
    *   Проверяйте "осиротевшие" пакеты: **`yay -Qdt`** или **`paru -Qdt`**. Если список не пуст, удалите их командой **`yay -Rns $(yay -Qdtq)`** или **`paru -Rns $(paru -Qdtq)`**.

3.  **Золотое правило Arch/EndeavourOS:** **НИКОГДА** не выполняйте частичное обновление. Если вы хотите обновить один пакет (`yay -S firefox`), система все равно проверит целостность всех зависимостей. Но если у вас давно не было полного обновления (`yay`), установка отдельного пакета может привести к конфликтам. Поэтому **перед установкой любого ПО всегда делайте полное обновление системы**.

4.  **Yay или Paru?** Оба отличные. Yay предустановлен и работает "из коробки". Paru — современная альтернатива на Rust с чуть более чистым кодом. Попробуйте оба и решите, какой вам больше нравится.

5.  **Если что-то сломалось:** Не паникуйте! У вас есть мощное сообщество EndeavourOS (форум, Telegram) и Arch Wiki. Помните про **Timeshift** — сделайте снимок системы после успешной установки и настройки, чтобы можно было легко откатиться назад в случае проблем.

## **Приложение: Шпаргалка по командам (Quick Reference)**

### **Pacman (основные)**
| Команда | Описание |
|:---|:---|
| `sudo pacman -Syu` | Полное обновление системы |
| `sudo pacman -S пакет` | Установить пакет |
| `sudo pacman -Rns пакет` | Удалить пакет с зависимостями |
| `pacman -Ss запрос` | Поиск пакета |
| `pacman -Qi пакет` | Инфо об установленном пакете |

### **Yay (дополнительно к pacman)**
| Команда | Описание |
|:---|:---|
| `yay` | Обновить систему (включая AUR) |
| `yay -S пакет` | Установить из репозитория или AUR |
| `yay -Ss запрос` | Поиск везде (включая AUR) |
| `yay -Y -c` | Очистить кэш и сирот |

### **Paru (дополнительно к pacman)**
| Команда | Описание |
|:---|:---|
| `paru` | Обновить систему (включая AUR) |
| `paru -S пакет` | Установить из репозитория или AUR |
| `paru -Ss запрос` | Поиск везде (включая AUR) |
| `paru -c` | Очистить кэш и сирот |
| `paru -P --stats` | Статистика по пакетам |

### **Ядро**
| Команда | Описание |
|:---|:---|
| `uname -r` | Версия текущего ядра |
| `pacman -Q \| grep linux` | Список установленных ядер |
| `sudo pacman -S linux-lts` | Установить LTS-ядро |
| `sudo mkinitcpio -P` | Пересобрать initramfs для всех ядер |
| `sudo grub-mkconfig -o /boot/grub/grub.cfg` | Обновить конфигурацию GRUB |

### **Зеркала**
| Команда | Описание |
|:---|:---|
| `sudo pacman-mirrors --fasttrack` | Выбрать 10 быстрых зеркал |
| `sudo pacman-mirrors --country Russia --fasttrack 5` | Выбрать 5 быстрых зеркал из России |
| `sudo pacman-mirrors --geoip` | Выбрать зеркала по геолокации |
| `sudo pacman-mirrors --status` | Показать статус зеркал |
| `sudo pacman -Syy` | Принудительно обновить базы после смены зеркал |
