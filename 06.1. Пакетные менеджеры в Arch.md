# **🚀 Руководство по пакетным менеджерам и управлению репозиториями в EndeavourOS (и Arch Linux)**

### **Введение**
EndeavourOS — это дистрибутив, основанный на Arch Linux. Это значит, что вы получаете всю мощь Arch (роллинг-релизы, свежее ПО, AUR), но с простым и дружелюбным установщиком и несколькими предустановленными инструментами, которые делают жизнь новичка проще. Главные герои этого руководства — **Pacman** (сердце системы) и **Yay** (лучший друг для установки программ из AUR).

---

## **Часть 1: Пакетные менеджеры в EndeavourOS**

### **Основные пакетные менеджеры**

В вашей системе уже есть всё необходимое:

1.  **Pacman** — это **низкоуровневый, основной пакетный менеджер**. Он работает с официальными репозиториями Arch. Быстрый и мощный, но требует ручного управления некоторыми вещами (например, обновлениями из AUR он не занимается). Команда: `pacman`.
2.  **Yay** («Yet Another Yogurt») — это **AUR-хелпер**, который **уже предустановлен в EndeavourOS** (в отличие от обычного Arch или Manjaro). Это консольная программа, которая умеет работать и с официальными репозиториями (через pacman), и с **AUR (Arch User Repository)**. Команда: `yay`. **Для повседневного использования рекомендуется использовать именно `yay`**, так как он закрывает 99% потребностей — от обновления системы до установки программ из AUR.

3.  **Pamac** — графический менеджер пакетов (как «Магазин приложений»). Он тоже умеет работать с AUR. В EndeavourOS он не установлен по умолчанию, но вы можете поставить его, если хотите графический интерфейс: `yay -S pamac-aur`.

### **Сравнение команд (Pacman/Yay vs Apt)**

Для тех, кто переходит с Ubuntu/Debian. **Для EndeavourOS в колонке "Рекомендуемая команда" используется `yay`.**

| Действие | В Ubuntu/Debian (apt) | В EndeavourOS/Arch (Pacman) | **В EndeavourOS (Yay — РЕКОМЕНДУЕТСЯ)** |
| :--- | :--- | :--- | :--- |
| **Обновить систему** | `sudo apt update && sudo apt upgrade` | `sudo pacman -Syu` | **`yay`** (просто `yay` без аргументов) |
| **Установить пакет** | `sudo apt install <пакет>` | `sudo pacman -S <пакет>` | **`yay -S <пакет>`** |
| **Удалить пакет** | `sudo apt remove <пакет>` | `sudo pacman -R <пакет>` | **`yay -R <пакет>`** |
| **Удалить пакет с зависимостями** | `sudo apt autoremove` | `sudo pacman -Rns <пакет>` | **`yay -Rns <пакет>`** |
| **Поиск пакета** | `apt search <запрос>` | `pacman -Ss <запрос>` | **`yay -Ss <запрос>`** |
| **Показать инфо о пакете** | `apt show <пакет>` | `pacman -Si <пакет>` | **`yay -Si <пакет>`** |

## **🔥 Расширенный список команд пакетного менеджера (Yay и Pacman)**

Здесь собраны самые полезные команды для повседневной работы и тонкой настройки системы.

### **Базовые операции (Установка, Удаление, Поиск)**

*   **`yay -S <имя_пакета>`** — установить пакет (из репозитория или AUR).
*   **`yay -Ss <ключевое_слово>`** — поиск пакета по ключевому слову во всех репозиториях и AUR.
*   **`yay -Rns <имя_пакета>`** — удалить пакет, его ненужные зависимости и конфигурационные файлы.
*   **`yay -R <имя_пакета>`** — удалить только пакет, оставив зависимости и конфиги.
*   **`yay -G <имя_пакета_из_AUR>`** — только скачать PKGBUILD из AUR, не собирая пакет.
*   **`yay -Y --gendb`** — сгенерировать базу данных для AUR (полезно, если вы только что установили `yay`).

### **Обновление системы (Rolling Release)**

*   **`yay`** или **`yay -Syu`** — **главная команда**. Синхронизирует репозитории (`-y`), обновляет официальные пакеты (`-u`) и проверяет обновления для AUR-пакетов.
*   **`yay -Syu --devel`** — обновить систему, включая пакеты из AUR, которые собираются из "живых" исходников (например, из Git-репозиториев). Включает флаги `--git`, `--hg` и т.д. для пересборки таких пакетов при наличии обновлений в исходном коде.
*   **`yay -Syu --nodiffmenu --nocleanmenu`** — обновление без лишних вопросов (автоматическое принятие стандартных действий).

### **Информация о пакетах и системе**

*   **`yay -Qi <имя_пакета>`** — показать подробную информацию об *установленном* пакете (версия, зависимости, описание, дата установки).
*   **`yay -Si <имя_пакета>`** — показать подробную информацию о пакете в *репозитории*.
*   **`yay -Ql <имя_пакета>`** — список всех файлов, которые установил данный пакет.
*   **`yay -Qo /путь/до/файла`** — узнать, какому пакету принадлежит файл в системе (аналог `dpkg -S`).
*   **`yay -Qdt`** — показать список "осиротевших" пакетов (зависимости, которые больше не нужны ни одному установленному пакету).
*   **`yay -Y -Pg`** — вывести статистику по пакетам из AUR.
*   **`pacman -Q | wc -l`** — посчитать общее количество установленных пакетов в системе.
*   **`yay -Ps`** — показать статистику системы (количество пакетов, размер кэша и т.д.).

### **Очистка системы и кэша**

*   **`yay -Sc`** — удалить из кэша (`/var/cache/pacman/pkg/`) все старые версии пакетов, кроме текущих установленных.
*   **`yay -Scc`** — **очистить кэш полностью** (удалить вообще все скачанные пакеты). Делайте это, только если уверены, что не придется откатываться до старых версий.
*   **`yay -Y -c`** — удалить неиспользуемые зависимости (осиротевшие пакеты) и очистить кэш сборки AUR-пакетов (`~/.cache/yay/`).

## **Часть 2: Управление репозиториями в EndeavourOS**

### **Структура репозиториев (Arch Linux)**

EndeavourOS использует стандартные репозитории Arch Linux. Основные из них:

```
core      - самые базовые пакеты системы (ядро, glibc, systemd)
extra     - остальные пакеты (браузеры, офисные пакеты, игры)
community - пакеты, собранные и поддерживаемые сообществом Arch Linux
multilib  - 32-битные библиотеки для 64-битной системы (нужно для игр и Steam)
```

### **Конфигурационные файлы**

*   **Главный файл:** `/etc/pacman.conf`
    Здесь настраиваются параметры pacman (цветной вывод, параллельные загрузки, игнорирование пакетов) и подключаются репозитории.
*   **Файл зеркал:** `/etc/pacman.d/mirrorlist`
    Список серверов, откуда скачиваются пакеты. EndeavourOS имеет свой инструмент для управления этим списком.

### **Управление зеркалами в EndeavourOS (`pacman-mirrors`)**

EndeavourOS поставляется с собственной утилитой `pacman-mirrors`, которая настраивает список зеркал специально для этого дистрибутива. **Не редактируйте mirrorlist вручную!** Используйте эту команду:

*   **`sudo pacman-mirrors --fasttrack`** — автоматически выбрать 10 самых быстрых зеркал.
*   **`sudo pacman-mirrors --country Russia,Germany --fasttrack 5`** — выбрать 5 самых быстрых зеркал только из России и Германии.
*   **`sudo pacman-mirrors --geoip`** — автоматически определить вашу страну по IP и выбрать зеркала оттуда.
*   **`sudo pacman-mirrors --status`** — показать текущий статус и скорость всех настроенных зеркал.
*   **`sudo pacman-mirrors --api --set-branch unstable`** — переключиться на нестабильную ветку (только для опытных!).
*   **`sudo pacman-mirrors --interactive`** — интерактивный режим выбора зеркал в псевдографическом интерфейсе.

**Важно:** После изменения зеркал всегда выполняйте **`sudo pacman -Syy`** (принудительное обновление баз данных), чтобы применить изменения.

## **Часть 3: AUR (Arch User Repository) — Главная суперсила**

AUR — это хранилище пользовательских скриптов (PKGBUILD), которые позволяют установить практически любую программу под Arch, даже если её нет в официальных репозиториях.

**В EndeavourOS работа с AUR максимально проста благодаря предустановленному `yay`.**

### **Как установить программу из AUR (например, Яндекс.Браузер)**

1.  **Поиск пакета:**
    ```bash
    yay -Ss yandex-browser
    ```
    Вы увидите список пакетов, включая `yandex-browser` (стабильный), `yandex-browser-beta` и т.д.

2.  **Установка:**
    ```bash
    yay -S yandex-browser
    ```
    Yay скачает PKGBUILD, покажет вам его для проверки (можно нажать Enter), а затем соберет и установит пакет, автоматически разрешив все зависимости.

### **Популярные AUR-пакеты для EndeavourOS**

*   `google-chrome` — Google Chrome.
*   `visual-studio-code-bin` — Microsoft Visual Studio Code.
*   `spotify` — официальный клиент Spotify.
*   `teamviewer` — TeamViewer.
*   `slack-desktop` — Slack.
*   `discord` — Discord (есть и в официальных репозиториях, но из AUR часто новее).
*   `enpass-bin` — Enpass Password Manager.

## **Часть 4: Практические примеры и решения проблем**

### **Пример: Установка Яндекс.Браузера (пошагово)**

1.  Откройте терминал.
2.  Выполните поиск (необязательно, но полезно для проверки названия):
    ```bash
    yay -Ss yandex-browser
    ```
3.  Установите браузер:
    ```bash
    yay -S yandex-browser
    ```
4.  Yay может попросить вас ввести пароль (для sudo) и подтвердить установку. Если пакет требует ручного вмешательства (например, принятия лицензии), он сообщит об этом. Готово! Ярлык появится в меню приложений Budgie.

### **Решение типичных проблем**

*   **Ошибка: "failed to commit transaction (conflicting files)"**
    Конфликт файлов. Обычно это значит, что пакет пытается перезаписать файл, принадлежащий другому пакету. Решение:
    ```bash
    yay -Syu --overwrite="путь/к/файлу"  # Перезаписать конкретный файл
    # Или для всех конфликтов (с осторожностью!)
    yay -Syu --overwrite="*"
    ```

*   **Ошибка: "invalid or corrupted package (PGP signature)"**
    Проблема с GPG-ключами при сборке AUR-пакета. Решение:
    ```bash
    # Обновить ключи
    sudo pacman-key --refresh-keys
    # Или установить недостающий ключ вручную (id ключа будет в ошибке)
    sudo pacman-key --recv-keys <ID_ключа>
    sudo pacman-key --lsign-key <ID_ключа>
    ```

*   **Ошибка: "nothing to do" при попытке установить AUR-пакет**
    Yay не находит пакет. Проверьте название командой `yay -Ss <примерное_название>`. Возможно, пакет называется иначе.

*   **Пропала скорость интернета при загрузке пакетов**
    Обновите зеркала:
    ```bash
    sudo pacman-mirrors --fasttrack && sudo pacman -Syy
    ```

## **Чек-лист и советы для новичка в EndeavourOS**

1.  **Первое, что нужно сделать после установки EndeavourOS:**
    ```bash
    yay  # Полностью обновить систему
    ```

2.  **Регулярное обслуживание:**
    *   Раз в неделю выполняйте **`yay`**.
    *   Иногда чистите систему: **`yay -Y -c`** (удалит ненужные зависимости и мусор из кэша AUR).
    *   Проверяйте "осиротевшие" пакеты: **`yay -Qdt`**. Если список не пуст, удалите их командой **`yay -Rns $(yay -Qdtq)`**.

3.  **Золотое правило Arch/EndeavourOS:** **НИКОГДА** не выполняйте частичное обновление. Если вы хотите обновить один пакет (`yay -S firefox`), система все равно проверит целостность всех зависимостей. Но если у вас давно не было полного обновления (`yay`), установка отдельного пакета может привести к конфликтам. Поэтому **перед установкой любого ПО всегда делайте полное обновление системы**.

4.  **Если что-то сломалось:** Не паникуйте! У вас есть мощное сообщество EndeavourOS (форум, Telegram) и Arch Wiki. Помните про **Timeshift** — сделайте снимок системы после успешной установки и настройки, чтобы можно было легко откатиться назад в случае проблем.
