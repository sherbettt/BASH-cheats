## Способ 1: Проверка HTTP статуса (надежный)

```bash
# Проверка доступности GitHub
if curl -s --head https://github.com | head -n 1 | grep "HTTP.*200" > /dev/null; then
    echo "GitHub доступен"
else
    echo "GitHub недоступен"
fi
```

## Способ 2: Проверка с таймаутом

```bash
# С таймаутом 10 секунд
if curl -s --head --connect-timeout 10 https://github.com > /dev/null; then
    echo "GitHub доступен"
else
    echo "GitHub недоступен (таймаут или ошибка)"
fi
```

## Способ 3: Детальная проверка

```bash
# Проверка с выводом деталей
response=$(curl -s -o /dev/null -w "%{http_code}" --connect-timeout 10 https://github.com)

if [ "$response" = "200" ]; then
    echo "GitHub доступен (HTTP 200)"
elif [ "$response" = "000" ]; then
    echo "GitHub недоступен (нет соединения)"
else
    echo "GitHub ответил с кодом: $response"
fi
```

## Способ 4: Проверка DNS + HTTP

```bash
# Сначала проверяем DNS, потом HTTP
if nslookup github.com > /dev/null 2>&1; then
    echo "DNS github.com разрешается"
    if curl -s --head --connect-timeout 5 https://github.com > /dev/null; then
        echo "GitHub доступен по HTTPS"
    else
        echo "DNS работает, но HTTPS недоступен"
    fi
else
    echo "Не удается разрешить github.com"
fi
```

## Способ 5: Простая проверка скачивания

```bash
# Пробуем скачать маленький файл
if wget -q --spider https://github.com/anchore/syft/releases/latest/download/syft_linux_amd64.tar.gz; then
    echo "Доступ к релизам GitHub есть"
else
    echo "Нет доступа к релизам GitHub"
fi
```
