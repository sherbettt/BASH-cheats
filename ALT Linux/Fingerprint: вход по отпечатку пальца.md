## 📋 Настройка входа по отпечатку пальца (fingerprint)

Да, для входа в систему по отпечатку пальца в вашей ОС (Etersoft Ximper на базе ALT Linux) такая возможность существует. 
Она реализована с помощью стандартного для Linux набора инструментов `fprintd` .

Например, ноутбук Lenovo ThinkPad P14s Gen 6 AMD полностью подходит для этой задачи. 
Согласно информации об этой модели, сканер отпечатков пальцев должен работать в Linux .

### 🔧 Как настроить вход по отпечатку пальца

1.  **Установка необходимых пакетов**. Откройте терминал и выполните команду для установки `libfprint2`, `fprintd` и модуля для PAM (`pam_fprintd`), который и отвечает за вход в систему :
    ```bash
    sudo apt-get update
    sudo apt-get install libfprint2 fprintd pam_fprintd
    ```
    *Примечание: В репозиториях вашей версии ОС могут быть актуальные версии этих пакетов, например `fprintd-1.94.5-alt1`  или `libfprint2-1.94.7-alt2` .*

2.  **Добавление отпечатка пальца**. Запустите утилиту для записи вашего отпечатка. По умолчанию она запросит правый указательный палец для текущего пользователя :
    ```bash
    fprintd-enroll
    ```
    Следуйте инструкциям на экране: несколько раз приложите палец к сканеру. Если потребуется, можно указать другой палец с помощью опции `--finger` .

3.  **Настройка входа в систему**. После записи отпечатка, вход по пальцу должен работать на экране входа в систему (GDM), так как пакет `pam_fprintd` автоматически добавляет эту возможность .

### 🔍 Важный момент: проверка совместимости

Прежде чем начать, стоит убедиться, что ваш конкретный сканер поддерживается библиотекой `libfprint`. Ваш сканер имеет идентификатор `27c6:6594` . 
Этот идентификатор упоминается как работающий на вашей модели ноутбука , а также в списке совместимости для похожей модели T14s .

Для абсолютной уверенности вы можете выполнить в терминале команду `lsusb` и найти в её выводе строку с `27c6:6594` . Это означает, что ваше устройство поддерживается.

### 1. Сканер уже записан ✅
Ваш палец (`right-index-finger`) успешно сохранён. Проверка `fprintd-verify` показала совпадение.

### 2. Включить вход по отпечатку

В вашем файле `/etc/pam.d/gdm-password` **нет строки** `pam_fprintd.so`. Нужно её добавить:

```bash
sudo micro /etc/pam.d/gdm-password
```

Добавьте эту строку **после** `auth substack common-login` (примерно между 8-й и 9-й строками):

```
auth        sufficient    pam_fprintd.so
```

Должно получиться так:
```
auth            required        pam_shells.so
auth            required        pam_succeed_if.so quiet uid ne 0
auth            sufficient      pam_succeed_if.so user ingroup nopasswdlogin
auth            substack        common-login
auth            sufficient      pam_fprintd.so          <-- ЭТУ СТРОКУ ДОБАВИТЬ
auth            optional        pam_gnome_keyring.so
account         required        pam_nologin.so
...
```

Сохраните (`Ctrl+O`, Enter, `Ctrl+X`).

### 3. Проверьте в GNOME

Откройте **Настройки → Пользователи** и найдите раздел "Вход по отпечатку пальца" (может потребоваться разблокировать кнопкой в правом верхнем углу).

### 4. Перезагрузитесь

```bash
sudo reboot
```

После перезагрузки на экране входа появится возможность приложить палец вместо ввода пароля.

## 🔧 Если что-то пошло не так

**Вернуть как было** (удалить добавленную строку):
```bash
sudo micro /etc/pam.d/gdm-password
# Удалить строку с pam_fprintd.so и сохранить
```

**Проверить статус**:
```bash
fprintd-list $USER          # показать записанные пальцы
sudo systemctl status fprintd  # служба активна ✓
```
