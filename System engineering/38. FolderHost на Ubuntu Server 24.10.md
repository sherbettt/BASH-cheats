# 📚 Полная инструкция по установке FolderHost на Ubuntu Server 24.10

## 🎯 Цель
Установить FolderHost - легковесное self-hosted облачное хранилище файлов с веб-интерфейсом, работающее как единый бинарный файл без зависимостей.

---

## Часть 1: Скачивание и установка

### 1.1 Создаем директорию и скачиваем бинарник

```bash
# Создаем директорию в /usr/local/src (как вы любите)
sudo mkdir -p /usr/local/src/folderhost
cd /usr/local/src/folderhost

# Скачиваем последнюю версию (на момент установки - v26.5.0)
wget https://github.com/MertJSX/folderhost/releases/download/v26.5.0/folderhost-linux-amd64.zip

# Распаковываем архив
unzip folderhost-linux-amd64.zip

# Делаем файл исполняемым
chmod +x folderhost

# Копируем в системную директорию
sudo cp folderhost /usr/local/bin/
```

### 1.2 Создаем рабочую директорию для данных

```bash
# Создаем директорию для хранения конфигов и данных
sudo mkdir -p /usr/local/share/folderhost

# Переходим в нее
cd /usr/local/share/folderhost
```

## Часть 2: Первичная настройка

### 2.1 Первый запуск (генерация файлов)

```bash
# Запускаем вручную для генерации config.yml, database.db и папок
sudo /usr/local/bin/folderhost
```

**Что произошло:**
- Создалась папка `/tmp`
- Создался файл `config.yml`
- Создалась папка `host` (для хранения файлов)
- Создался файл `services.yml`
- Создалась папка `/recovery_bin` (корзина)
- Сгенерировался JWT ключ
- Создалась база данных SQLite (`database.db`)
- Создалась таблица пользователей
- Создалась таблица логов

**Прерываем запуск:** `Ctrl + C`

### 2.3 Настройка systemd сервиса

```bash
# Создаем файл сервиса
sudo mcedit /etc/systemd/system/folderhost.service
```

**Содержимое сервиса:**

```ini
[Unit]
Description=FolderHost - Self-hosted Cloud Platform
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/usr/local/share/folderhost
ExecStart=/usr/local/bin/folderhost
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

## Часть 3: Запуск и управление сервисом

### 3.1 Запуск сервиса

```bash
# Перезагружаем systemd (чтобы он увидел новый сервис)
sudo systemctl daemon-reload

# Включаем автозапуск при старте системы
sudo systemctl enable folderhost

# Запускаем сервис
sudo systemctl start folderhost

# Проверяем статус
sudo systemctl status folderhost --no-pager -l
```

**Ожидаемый вывод:**
```
● folderhost.service - FolderHost - Self-hosted Cloud Platform
     Loaded: loaded (/etc/systemd/system/folderhost.service; enabled)
     Active: active (running) since ...
   Main PID: 3134 (folderhost)
      Tasks: 7
     Memory: 2M
```

### 3.2 Команды управления

```bash
# Остановить сервис
sudo systemctl stop folderhost

# Перезапустить сервис
sudo systemctl restart folderhost

# Посмотреть логи
sudo journalctl -u folderhost -f

# Посмотреть последние 50 строк логов
sudo journalctl -u folderhost -n 50 --no-pager
```

## 📝 Дополнительные настройки (по желанию)

### Изменить лимит хранилища

Раскомментируйте в `config.yml`:
```yaml
storage_limit: "50 GB"
```

### Добавить дополнительных пользователей

Через веб-интерфейс: **Settings → Users → Create User**

### Включить сервисы

Отредактируйте `services.yml`:
```yaml
services:
  ftp: true      # FTP сервер
  sftp: true     # SFTP сервер
  webdav: true   # WebDAV
```

**Не забудьте перезапустить после изменений:**
```bash
sudo systemctl restart folderhost
```
