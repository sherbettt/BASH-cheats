## **Для безопасности** лучше использовать переменные окружения:
```bash
export REDISCLI_AUTH="RedisComplexPass"
```

##  **Проверка пинга**
```bash
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" ping
redis-cli -h 127.0.0.1 -p 6380 AUTH "RedisComplexPass"
```

##  **Информационные и диагностические команды**

```bash
# Основная информация о сервере
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" INFO

# Конкретные разделы INFO
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" INFO server
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" INFO memory
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" INFO clients
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" INFO stats

# Статистика в реальном времени
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --stat

# Мониторинг задержек
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --latency
```

##  **Анализ данных**

```bash
# Поиск больших ключей
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --bigkeys

# Поиск ключей, потребляющих много памяти
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --memkeys

# Поиск "горячих" ключей (требует политики LFU)
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --hotkeys

# Сканирование всех ключей по шаблону
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --scan --pattern "user:*"
```

##  **Администрирование**

```bash
# Список всех команд
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" COMMAND

# Информация о конкретной команде
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" COMMAND INFO get set

# Мониторинг команд в реальном времени
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" MONITOR

# Сброс статистики
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" CONFIG RESETSTAT

# Сохранение данных на диск
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" SAVE
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" BGSAVE
```

##  **Работа с ключами**

```bash
# Получить все ключи (осторожно на больших БД!)
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" KEYS "*"

# Подсчет количества ключей
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" DBSIZE

# Получить случайный ключ
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" RANDOMKEY

# Проверить существование ключа
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" EXISTS mykey

# Узнать тип значения
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" TYPE mykey
```

##  **Кластерные команды (если используется кластер)**

```bash
# Информация о кластере
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --cluster info

# Проверка состояния кластера
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" --cluster check

# Узлы кластера
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" CLUSTER NODES
redis-cli -h 127.0.0.1 -p 6380 -a "RedisComplexPass" CLUSTER INFO
```

##  **Работа с репликацией**
```bash
# Проверить статус репликации
redis-cli -h 127.0.0.1 -p 6380 INFO replication

# Подробная информация о репликации
redis-cli -h 127.0.0.1 -p 6380 ROLE

# Принудительно переподключиться к мастеру
redis-cli -h 127.0.0.1 -p 6380 SLAVEOF NO ONE  # Стать мастером
redis-cli -h 127.0.0.1 -p 6380 SLAVEOF 192.168.87.66 6380  # Снова стать репликой

# Проверить задержку репликации
redis-cli -h 127.0.0.1 -p 6380 --latency
```

##  **Диагностика соединения с мастером**
```bash
# Попробовать подключиться к мастеру напрямую
redis-cli -h 192.168.87.66 -p 6380 PING

# Проверить, слушает ли мастер на порту 6380
nc -zv 192.168.87.66 6380

# Посмотреть логи Redis для диагностики
tail -f /var/log/redis/redis-server.log
```

