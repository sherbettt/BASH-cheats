# 📚 Полная инструкция установки Sync‑in + OnlyOffice на Ubuntu 24.04

## 🎯 Цель
Установить Sync‑in (Self-hosted облачное хранилище) с интеграцией OnlyOffice для редактирования документов и поддержкой SSO (OIDC).

---

## Часть 1: Подготовка системы

### 1.1 Обновляем систему

```bash
sudo apt update && sudo apt upgrade -y
```

### 1.2 Устанавливаем Node.js 22.x

```bash
# Добавляем репозиторий NodeSource
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -

# Устанавливаем Node.js
sudo apt install -y nodejs

# Проверяем версии
node -v   # Должно быть v22.x
npm -v    # Должно быть 10.x
```

### 1.3 Устанавливаем MariaDB

```bash
sudo apt install -y mariadb-server mariadb-client

# Проверяем версию
mariadb --version
```

### 1.4 Устанавливаем Docker (для OnlyOffice)

```bash
sudo apt install -y docker.io docker-compose
sudo systemctl enable --now docker
sudo usermod -aG docker $USER
```

---

## Часть 2: Установка Sync‑in

### 2.1 Создаем директорию и устанавливаем

```bash
# Создаем рабочую директорию
sudo mkdir -p /opt/sync-in
cd /opt/sync-in

# Устанавливаем Sync‑in через npm
sudo npm install @sync-in/server
```

### 2.2 Создаем базу данных

```bash
# Заходим в MySQL
sudo mysql -u root
```

```sql
-- Создаем базу данных и пользователя
CREATE DATABASE sync_in CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'sync_in_user'@'localhost' IDENTIFIED BY 'SyncInPass123';
GRANT ALL PRIVILEGES ON sync_in.* TO 'sync_in_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 2.3 Создаем конфигурацию

```bash
# Копируем пример конфига
sudo npx sync-in-server init
```

**Редактируем конфиг:**

```bash
sudo nano /opt/sync-in/environment.yaml
```

**Полный рабочий конфиг:**

```yaml
auth:
  provider: mysql
  encryptionKey: "f7a3e8b2c1d5f6a9b8c7d6e5f4a3b2c1d0e9f8a7b6c5d4e3f2a1b0c9d8e7f6a5"
  token:
    access:
      secret: "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2"
      expiresIn: 900
    refresh:
      secret: "b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2c3"
      expiresIn: 604800

mysql:
  url: mysql://sync_in_user:SyncInPass123@localhost:3306/sync_in

applications:
  files:
    dataPath: /opt/sync-in/data

server:
  port: 8080
  host: 0.0.0.0
  cors:
    enabled: true
```

### 2.4 Создаем директорию для данных

```bash
sudo mkdir -p /opt/sync-in/data
sudo chmod 755 /opt/sync-in/data
```

### 2.5 Запускаем миграции и создаем админа

```bash
cd /opt/sync-in

# Миграции базы данных
sudo npx sync-in-server migrate-db

# Создаем администратора
sudo npx sync-in-server create-user \
  --role admin \
  --login "admin" \
  --password "RunTELOxiCld78"
```

## Часть 3: Systemd сервис для Sync‑in

### 3.1 Создаем сервис

```bash
sudo nano /etc/systemd/system/sync-in.service
```

```ini
[Unit]
Description=Sync-in Server
After=network.target mariadb.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/sync-in
Environment=NODE_ENV=production
ExecStart=/usr/bin/npx sync-in-server start
Restart=on-failure
RestartSec=10
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

### 3.2 Запускаем сервис

```bash
sudo systemctl daemon-reload
sudo systemctl enable sync-in
sudo systemctl start sync-in
sudo systemctl status sync-in --no-pager -l
```

## Часть 4: Установка OnlyOffice через Docker

### 4.1 Запускаем OnlyOffice

```bash
sudo docker run -d \
  --name onlyoffice \
  --restart unless-stopped \
  -p 8081:80 \
  -e JWT_ENABLED=true \
  -e JWT_SECRET="SyncInPass123" \
  onlyoffice/documentserver:latest
```

### 4.2 Проверяем работу

```bash
# Проверяем, что контейнер запущен
sudo docker ps | grep onlyoffice

# Проверяем healthcheck (должно вернуть "true")
curl http://localhost:8081/healthcheck
```

---

## Часть 5: Подключаем OnlyOffice к Sync‑in

### 5.1 Редактируем конфиг Sync‑in

```bash
sudo nano /opt/sync-in/environment.yaml
```

**Добавляем в конец файла:**

```yaml
collaborative:
  enabled: true
  editors:
    - type: onlyoffice
      url: http://localhost:8081
      jwtSecret: "SyncInPass123"
```

### 5.2 Перезапускаем Sync‑in

```bash
sudo systemctl restart sync-in
sudo systemctl status sync-in --no-pager
```

## Часть 8: Настройка SSO (OIDC) — опционально

### 8.1 Редактируем конфиг

```bash
sudo nano /opt/sync-in/environment.yaml
```

**Меняем секцию `auth`:**

```yaml
auth:
  provider: oidc
  oidc:
    issuerUrl: https://ваш-sso-провайдер.com/realms/ваш-realm
    clientId: "ваш-client-id"
    clientSecret: "ваш-client-secret"
    redirectUri: http://192.168.87.7:8080/api/auth/oidc/callback
    autoCreateUser: true
    options:
      adminRoleOrGroup: "sync-in-admin"
  encryptionKey: "f7a3e8b2c1d5f6a9b8c7d6e5f4a3b2c1d0e9f8a7b6c5d4e3f2a1b0c9d8e7f6a5"
  token:
    access:
      secret: "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2"
      expiresIn: 900
    refresh:
      secret: "b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2c3"
      expiresIn: 604800
```

### 8.2 Перезапускаем

```bash
sudo systemctl restart sync-in
```

## 🔧 Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Sync‑in не запускается | `sudo journalctl -u sync-in -n 50` |
| OnlyOffice не отвечает | `curl http://localhost:8081/healthcheck` |
| Ошибка подключения к БД | `mysql -u sync_in_user -pSyncInPass123` |
| Документы не открываются | Проверьте JWT_SECRET в конфигах |
