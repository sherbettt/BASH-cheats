# Установка Python 3.9.17 как системного на Debian 13 (Trixie)

## Введение

Данная инструкция предназначена для компиляции и установки Python 3.9.17 в качестве системного интерпретатора на Debian 13 (Trixie). 
Это необходимо для использования в качестве сборочной ноды Jenkins, где требуется именно эта версия Python. 
Инструкция учитывает, что в системе уже установлен Python 3.13.5, который будет сохранен как запасной вариант.


## Часть 1. Установка зависимостей

Перед компиляцией Python необходимо установить все необходимые библиотеки и инструменты:

```bash
sudo apt update
sudo apt install -y wget build-essential libssl-dev zlib1g-dev \
libbz2-dev libreadline-dev libsqlite3-dev libncursesw5-dev \
xz-utils tk-dev libxml2-dev libxmlsec1-dev libffi-dev liblzma-dev \
llvm libgdbm-dev libnss3-dev libdb-dev uuid-dev \
libc6-dev libpcre2-dev libbluetooth-dev
```

Эти пакеты включают:
- Компиляторы и утилиты сборки (build-essential)
- Библиотеки для работы с SSL (libssl-dev)
- Библиотеки для сжатия и архивации (zlib1g-dev, xz-utils)
- Библиотеки для работы с базами данных и терминалами
- Библиотеки для графического интерфейса (tk-dev)

## Часть 2. Скачивание исходного кода

Перейдите в директорию для исходников и скачайте архив с Python 3.9.17:

```bash
cd /usr/src
sudo wget https://www.python.org/ftp/python/3.9.17/Python-3.9.17.tgz
sudo tar -xzf Python-3.9.17.tgz
cd Python-3.9.17
```

## Часть 3. Конфигурация сборки

Настройка параметров компиляции с указанием системных путей и оптимизаций:

```bash
sudo ./configure --enable-optimizations \
--prefix=/usr \
--with-openssl=/usr \
--enable-shared \
--with-system-expat \
--with-system-ffi \
--with-ensurepip=install \
LDFLAGS="-Wl,-rpath /usr/lib:/usr/lib/x86_64-linux-gnu"
```

Пояснение ключей:
- `--enable-optimizations`: Включает оптимизации производительности
- `--prefix=/usr`: Устанавливает в системную директорию
- `--enable-shared`: Создает разделяемую библиотеку
- `--with-system-expat`: Использует системный expat
- `--with-system-ffi`: Использует системный libffi
- `--with-ensurepip=install`: Автоматически устанавливает pip
- `LDFLAGS`: Задает пути для поиска библиотек во время выполнения

## Часть 4. Компиляция

Запустите компиляцию, используя все доступные ядра процессора для ускорения:

```bash
sudo make -j $(nproc)
```

Процесс компиляции может занять 10-30 минут в зависимости от мощности системы.

## Часть 5. Установка

Установите скомпилированный Python в систему:

```bash
sudo make install
```

После установки проверьте, что Python 3.9.17 установлен:

```bash
/usr/bin/python3.9 --version
# Должно вывести: Python 3.9.17
```

## Часть 6. Настройка pip

Создайте символические ссылки для pip:

```bash
sudo ln -sf /usr/bin/pip3.9 /usr/bin/pip3
sudo ln -sf /usr/bin/pip3.9 /usr/bin/pip
```

Проверьте работу pip:

```bash
pip3 --version
# Должно вывести: pip 23.0.1 from /usr/lib/python3.9/site-packages/pip (python 3.9)
```

## Часть 7. Настройка приоритетов через update-alternatives

Этот шаг критически важен для корректной работы системы. Мы настраиваем систему выбора версии Python по умолчанию:

```bash
# Добавляем Python 3.9 как основной вариант с высоким приоритетом
sudo update-alternatives --install /usr/bin/python python /usr/bin/python3.9 100
sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.9 100

# Добавляем Python 3.13 как запасной вариант с низким приоритетом
sudo update-alternatives --install /usr/bin/python python /usr/bin/python3.13 50
sudo update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.13 50

# Устанавливаем Python 3.9 как основной
sudo update-alternatives --set python /usr/bin/python3.9
sudo update-alternatives --set python3 /usr/bin/python3.9
```

## Часть 8. Проверка установки

Выполните проверку, что Python 3.9.17 стал системным по умолчанию:

```bash
# Проверяем версии
python --version
# Должно быть: Python 3.9.17

python3 --version
# Должно быть: Python 3.9.17

# Проверяем пути
which python
# Должно быть: /usr/bin/python

which python3
# Должно быть: /usr/bin/python3

# Проверяем, что Python 3.13 все еще доступен
python3.13 --version
# Должно быть: Python 3.13.5
```

## Часть 9. Тестирование системной совместимости

Важно проверить, что системные утилиты продолжают работать с новой версией Python:

```bash
# Проверяем работу apt
apt --version
# Должно вывести версию apt без ошибок

# Проверяем работу других системных утилит
lsb_release -a

# Тестируем установку пакетов через apt
sudo apt update
sudo apt install -y python3-venv python3-dev

# Проверяем, что установились пакеты для Python 3.9
ls -la /usr/lib/python3.9/
```

## Часть 10. Обновление pip и установка базовых пакетов

Обновите pip и установите основные пакеты для разработки:

```bash
# Обновление pip
python3 -m pip install --upgrade pip setuptools wheel

# Проверка обновленной версии
pip3 --version
```

## Часть 11. Установка пакетов для Jenkins

Установите пакеты, необходимые для работы сборочной ноды Jenkins:

```bash
pip3 install pytest pylint coverage tox virtualenv
```

## Часть 12. Создание диагностического скрипта

Создайте скрипт для проверки окружения, который будет полезен при диагностике:

```bash
sudo tee /opt/check-python-env.sh << 'EOF'
#!/bin/bash
echo "=== Python Environment Check ==="
echo "System: $(lsb_release -ds)"
echo "Date: $(date)"
echo ""
echo "--- Python Versions ---"
echo "Default python: $(python --version 2>&1)"
echo "Default python3: $(python3 --version 2>&1)"
echo "Python 3.9 path: $(which python3.9)"
echo "Python 3.9 version: $(python3.9 --version 2>&1)"
echo "Python 3.13 path: $(which python3.13 2>/dev/null || echo 'Not found in PATH')"
echo "Python 3.13 version: $(python3.13 --version 2>&1 2>/dev/null || echo 'Not available')"
echo ""
echo "--- Pip ---"
echo "Pip version: $(pip3 --version)"
echo "Pip path: $(which pip3)"
echo ""
echo "--- Installed Python Packages ---"
pip3 list --format=columns
echo ""
echo "--- Python Path ---"
python3 -c "import sys; print('\n'.join(sys.path))"
EOF

sudo chmod +x /opt/check-python-env.sh
```

Запустите скрипт для проверки:

```bash
sudo /opt/check-python-env.sh
```

## Часть 13. Настройка для Jenkins (опционально)

Если Jenkins уже установлен на системе, настройте для него окружение:

```bash
# Создание директории для пользователя Jenkins
sudo mkdir -p /var/lib/jenkins/.local/bin

# Создание ссылок для Jenkins
sudo ln -sf /usr/bin/python3.9 /var/lib/jenkins/.local/bin/python
sudo ln -sf /usr/bin/pip3.9 /var/lib/jenkins/.local/bin/pip

# Установка правильных прав
sudo chown -R jenkins:jenkins /var/lib/jenkins/.local

# Добавление в PATH для Jenkins (добавьте в /etc/environment или профиль jenkins)
echo 'export PATH=/var/lib/jenkins/.local/bin:$PATH' | sudo tee -a /var/lib/jenkins/.profile
```

## Часть 14. Сохранение информации об установке

Создайте файл с информацией об установке для последующего использования:

```bash
sudo tee /opt/python-install-info.txt << EOF
Python 3.9.17 Installation Information
=======================================
Installation Date: $(date)
System: $(lsb_release -ds)
Python Version: $(python3 --version)
Pip Version: $(pip3 --version)
Python Path: $(which python3)

Alternative Configurations:
$(sudo update-alternatives --display python 2>/dev/null)
$(sudo update-alternatives --display python3 2>/dev/null)

Installed Packages:
$(pip3 list --format=columns)

Python 3.13.5 is available as python3.13
EOF

cat /opt/python-install-info.txt
```

## Часть 15. Финальная проверка

Выполните комплексную проверку установки:

```bash
# Проверка всех символических ссылок
ls -la /usr/bin/python* | grep -E "python|pip"

# Проверка альтернатив
sudo update-alternatives --display python
sudo update-alternatives --display python3

# Проверка импорта системных модулей
python3 -c "import sys, os, json, re, hashlib, sqlite3, ssl; print('All modules imported successfully')"

# Проверка компиляции простой программы
python3 -c "print('Hello from Python 3.9.17')"
```

## Устранение неполадок

### Проблемы с apt

Если apt перестает работать с ошибками, связанными с Python:

```bash
# Временное переключение на Python 3.13
sudo update-alternatives --config python3
# Выберите /usr/bin/python3.13 из списка

# Или переустановка apt
sudo apt --fix-broken install
sudo apt install --reinstall apt
```

### Восстановление Python 3.13 как основного

Если требуется вернуть Python 3.13 как основной:

```bash
sudo update-alternatives --config python
sudo update-alternatives --config python3
# Выберите соответствующий вариант с python3.13
```

### Полная переустановка системного Python

В случае серьезных проблем:

```bash
# Удаление альтернатив
sudo update-alternatives --remove python /usr/bin/python3.9
sudo update-alternatives --remove python3 /usr/bin/python3.9

# Переустановка системных пакетов
sudo apt install --reinstall python3 python3-minimal python3-dev

# Удаление вручную установленного Python 3.9
sudo rm -rf /usr/bin/python3.9 /usr/lib/python3.9 /usr/bin/pip3.9
```

## Заключение

После выполнения всех шагов ваш Debian 13 будет использовать Python 3.9.17 как системный интерпретатор по умолчанию. Python 3.13.5 останется доступным для вызова через команду `python3.13`. Система готова к использованию в качестве сборочной ноды Jenkins.

Для применения изменений в текущей сессии рекомендуется выполнить:

```bash
exec bash -l
```

Или перезагрузить систему:

```bash
sudo reboot
```
