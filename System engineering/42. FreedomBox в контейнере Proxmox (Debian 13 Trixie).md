# Инструкция по установке FreedomBox в контейнере Proxmox (Debian 13 Trixie)


## ⚙️ ШАГ 1: Создание LXC-контейнера

### 1.1 Создание шаблона Debian 13 (Trixie)

```bash
# На хосте Proxmox:
apt update && apt install -y dab wget

# Создаём рабочую директорию
mkdir ~/dab-debian13 && cd ~/dab-debian13

# Загружаем конфигурацию для Debian 13
wget -O dab.conf "https://git.proxmox.com/?p=dab-pve-appliances.git;a=blob_plain;f=debian-13-trixie-std-64/dab.conf;hb=HEAD"
wget -O Makefile "https://git.proxmox.com/?p=dab-pve-appliances.git;a=blob_plain;f=debian-13-trixie-std-64/Makefile;hb=HEAD"

# Собираем шаблон
make all

# Перемещаем шаблон в каталог Proxmox
mv *.zst /var/lib/vz/template/cache/
```

### 1.2 Создание контейнера

Через веб-интерфейс Proxmox:

1. Нажмите **Create CT**
2. **Hostname**: `freedombox`
3. **Password**: установите надёжный пароль
4. **Template**: выберите `debian-13-standard_13.0.20250715-1_amd64.tar.zst`
5. **Disk**: 8-10 GB (минимум)
6. **CPU**: 2 ядра
7. **Memory**: 1024-2048 MB
8. **Network**: выберите мост, статический IP (например, 192.168.87.199/24)
9. **DNS**: укажите ваш DNS-сервер

> **Важно**: Дайте контейнеру статический IP, чтобы не искать его каждый раз.

### 1.3 Заходим в контейнер

```bash
# На хосте Proxmox:
pct enter <ID_контейнера>
```

## 🔑 ШАГ 3: Установка FreedomBox

### 3.1 Создаём фиктивные LDAP-скрипты (ВАЖНО!)

Перед установкой создаём "заглушки" для LDAP-команд, чтобы обойти ошибки:

```bash
# Создаём директорию для фиктивных скриптов
mkdir -p /usr/local/bin

# Создаём фиктивный ldapaddgroup
cat > /usr/local/bin/ldapaddgroup <<'EOF'
#!/bin/bash
echo "Fake LDAP: group $1 created" >> /tmp/ldap-fake.log
exit 0
EOF

# Создаём фиктивный ldapadduser
cat > /usr/local/bin/ldapadduser <<'EOF'
#!/bin/bash
echo "Fake LDAP: user $1 added to group $2" >> /tmp/ldap-fake.log
exit 0
EOF

# Создаём фиктивный ldapaddmember
cat > /usr/local/bin/ldapaddmember <<'EOF'
#!/bin/bash
echo "Fake LDAP: member $1 added to group $2" >> /tmp/ldap-fake.log
exit 0
EOF

# Создаём фиктивный ldapsearch
cat > /usr/local/bin/ldapsearch <<'EOF'
#!/bin/bash
echo "dn: dc=thisbox" >> /tmp/ldap-fake.log
exit 0
EOF

# Делаем их исполняемыми
chmod +x /usr/local/bin/ldap*
```

### 3.2 Установка FreedomBox

```bash
# Устанавливаем FreedomBox
DEBIAN_FRONTEND=noninteractive apt install -y freedombox
```

**Что будет происходить:**
- Появятся окна с настройкой `nslcd` и `libnss-ldapd` — **нажимайте Cancel** (отмена)
- Появится окно popularity-contest — выберите **No** (можно и Yes)
- В окне libnss-ldapd — **оставьте все пункты пустыми** и нажмите Ok

### 3.3 После установки — фикс LDAP-зависимостей

```bash
# Останавливаем и отключаем LDAP-сервисы
systemctl stop nslcd 2>/dev/null
systemctl disable nslcd 2>/dev/null
systemctl mask nslcd 2>/dev/null

# Удаляем проблемные пакеты (если есть)
apt remove --purge nslcd libpam-ldapd libnss-ldapd nslcd-utils -y 2>/dev/null

# Устанавливаем фиктивный пакет-заглушку для libnss-ldapd
cat > /tmp/fake-nss-ldapd <<EOF
Section: misc
Priority: optional
Standards-Version: 3.9.2
Package: fake-libnss-ldapd
Version: 1.0
Architecture: all
Provides: libnss-ldapd
Description: Fake package to satisfy FreedomBox dependency
EOF

cd /tmp
equivs-build fake-nss-ldapd 2>/dev/null || apt install -y equivs && equivs-build fake-nss-ldapd
dpkg -i fake-libnss-ldapd_1.0_all.deb 2>/dev/null

# Очищаем PAM от LDAP
for file in /etc/pam.d/*; do
    sed -i '/pam_ldap/d' "$file"
done

# Удаляем LDAP из nsswitch.conf
sed -i 's/ ldap//g' /etc/nsswitch.conf
sed -i 's/ldap //g' /etc/nsswitch.conf

# Завершаем настройку пакетов
dpkg --configure -a
apt install --fix-broken -y
```

## 🌐 ШАГ 4: Настройка веб-сервера Apache

FreedomBox работает через **Apache** как прокси для Plinth (веб-интерфейс).

```bash
# Создаём конфигурацию для FreedomBox в Apache
cat > /etc/apache2/sites-available/freedombox.conf <<'EOF'
<VirtualHost *:80>
    ServerName _default_
    Redirect permanent / https://192.168.87.199/
</VirtualHost>
EOF

# Настраиваем HTTPS с самоподписанным сертификатом
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/ssl/private/freedombox-selfsigned.key \
    -out /etc/ssl/certs/freedombox-selfsigned.crt \
    -subj "/C=RU/ST=State/L=City/O=Organization/CN=192.168.87.199"

cat > /etc/apache2/sites-available/freedombox-ssl.conf <<'EOF'
<VirtualHost *:443>
    ServerName 192.168.87.199
    ServerAdmin webmaster@localhost

    SSLEngine on
    SSLCertificateFile /etc/ssl/certs/freedombox-selfsigned.crt
    SSLCertificateKeyFile /etc/ssl/private/freedombox-selfsigned.key

    SSLProxyEngine On
    SSLProxyVerify none
    SSLProxyCheckPeerCN off
    SSLProxyCheckPeerName off

    ProxyPreserveHost On
    ProxyPass / http://127.0.0.1:8000/
    ProxyPassReverse / http://127.0.0.1:8000/

    ErrorLog ${APACHE_LOG_DIR}/freedombox-error.log
    CustomLog ${APACHE_LOG_DIR}/freedombox-access.log combined
</VirtualHost>
EOF

# Включаем нужные модули
a2enmod proxy proxy_http ssl

# Отключаем дефолтный сайт, включаем наш
a2dissite 000-default
a2ensite freedombox freedombox-ssl

# Перезапускаем Apache
systemctl restart apache2
```

## ⚠️ ШАГ 6: ПРАВКА КОДА PLINTH (КЛЮЧЕВОЙ МОМЕНТ!)

Plinth пытается установить LDAP-пакеты. Нужно закомментировать эту проверку:

```bash
# Создаём резервную копию
cp /usr/lib/python3/dist-packages/plinth/modules/users/__init__.py /tmp/users_init.py.bak

# Редактируем файл
mcedit /usr/lib/python3/dist-packages/plinth/modules/users/__init__.py
```

**Найдите и закомментируйте строки 69-73** (добавьте `#` в начале каждой строки):

```python
#        packages = Packages('packages-users', [
#            'ldapscripts', 'ldap-utils', 'libnss-ldapd', 'libpam-ldapd',
#            'nslcd', 'samba-common-bin', 'slapd', 'tdb-tools'
#        ])
#        self.add(packages)
```

**Сохраните** (F2) и **выйдите** (F10).

```bash
# Очищаем кэш Python
find /usr/lib/python3/dist-packages/plinth/ -name "*.pyc" -delete

# Перезапускаем Plinth
systemctl restart plinth

# Проверяем статус
systemctl status plinth
```

## 📋 ЧТО МЫ ВКЛЮЧИЛИ / ОТКЛЮЧИЛИ / СОЗДАЛИ

| Компонент | Действие | Причина |
|-----------|----------|---------|
| **Apache** | Включили как прокси | FreedomBox требует веб-сервер |
| **SSL** | Включили самоподписанный | Для HTTPS-доступа |
| **Plinth** | Исправили код, убрали LDAP-проверку | Чтобы установка не падала |
| **nslcd** | Остановили и замаскировали | Не нужен без LDAP-сервера |
| **PAM** | Очистили от LDAP | Чтобы sudo работал |
| **nsswitch.conf** | Удалили ldap из поиска | Чтобы система не искала пользователей в LDAP |
| **Фиктивные LDAP-скрипты** | Создали заглушки (`/usr/local/bin/ldap*`) | Чтобы Plinth думал, что LDAP есть |
| **fake-libnss-ldapd** | Установили фиктивный пакет | Чтобы удовлетворить зависимость |
| **Группа freedombox-share** | Создали вручную | Чтобы Plinth не падал при создании |

## 🚨 Что делать, если что-то пошло не так

### Plinth не запускается
```bash
journalctl -u plinth -n 30 --no-pager
systemctl restart plinth
```

### Apache не запускается
```bash
apache2ctl configtest
systemctl restart apache2
```

### Страница висит (спиннер)
- Обновите код Plinth (ШАГ 6)
- Проверьте логи: `journalctl -u plinth -f`

### Sudo не работает
```bash
# Проверяем PAM
grep -r pam_ldap /etc/pam.d/
# Если есть — удаляем: sed -i '/pam_ldap/d' /etc/pam.d/*
```
