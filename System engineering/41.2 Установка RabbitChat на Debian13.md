# Инструкция по установке rabbitChat на Debian 13

## Введение

`rabbitChat` — это чат-сервер на основе RabbitMQ, использующий AMQP протокол. Проект написан на Python 2.7, который не входит в состав Debian 13 по умолчанию, поэтому нам пришлось установить его через альтернативный интерпретатор — PyPy.

## Часть 2: Установка PyPy (Python 2.7)

Поскольку Python 2.7 отсутствует в Debian 13, мы используем PyPy — альтернативный интерпретатор, совместимый с Python 2.7.

### 2.1 Скачивание PyPy

```bash
cd ~/programs
wget https://downloads.python.org/pypy/pypy2.7-v7.3.16-linux64.tar.bz2
tar -xjf pypy2.7-v7.3.16-linux64.tar.bz2
```

### 2.2 Настройка PATH

```bash
# Добавляем PyPy в PATH
export PATH="$HOME/programs/pypy2.7-v7.3.16-linux64/bin:$PATH"
echo 'export PATH="$HOME/programs/pypy2.7-v7.3.16-linux64/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 2.3 Проверка установки PyPy

```bash
pypy2 --version
# Должно быть: Python 2.7.18 (PyPy 7.3.16)
```

## Часть 3: Установка pip и зависимостей

### 3.1 Установка pip для PyPy

```bash
pypy2 -m ensurepip
pypy2 -m pip install --upgrade pip
```

### 3.2 Установка необходимых библиотек

```bash
pip2.7 install tornado
pip2.7 install sockjs-tornado
pip2.7 install pika==0.13.1
```

### 3.3 Установка rabbitChat

```bash
pip2.7 install rabbitChat
```

## Часть 5: Альтернативный способ — консольный чат

Если rabbitChat не работает, можно использовать простой консольный чат:

### 5.1 Создание скрипта чата

```bash
### ~/chat.py
import pika
import sys
import threading
import time

username = sys.argv[1] if len(sys.argv) > 1 else 'Guest'

connection = pika.BlockingConnection(
    pika.ConnectionParameters(
        host='localhost',
        credentials=pika.PlainCredentials('admin', 'runtelorg')
    )
)
channel = connection.channel()
channel.queue_declare(queue='chat_room', durable=True)

def send_message(message):
    data = '[{}] {}: {}'.format(time.strftime('%H:%M:%S'), username, message)
    channel.basic_publish(exchange='', routing_key='chat_room', body=data)

def callback(ch, method, properties, body):
    print('\n{}'.format(body))
    sys.stdout.write('{}> '.format(username))
    sys.stdout.flush()

def receive():
    channel.basic_consume(callback, queue='chat_room', no_ack=True)
    channel.start_consuming()

print('Welcome to chat, {}!'.format(username))
print('Type /exit to quit')

t = threading.Thread(target=receive)
t.daemon = True
t.start()

while True:
    try:
        msg = raw_input('{}> '.format(username))
        if msg == '/exit':
            break
        send_message(msg)
    except KeyboardInterrupt:
        break

connection.close()
```

### 5.2 Запуск консольного чата

**В первом терминале:**
```bash
python2 ~/simple_chat.py Alice
```

**Во втором терминале:**
```bash
python2 ~/simple_chat.py Bob
```

## Часть 6: Управление сервисами

### 6.1 Управление RabbitMQ

```bash
# Проверка статуса
sudo systemctl status rabbitmq-server

# Остановка
sudo systemctl stop rabbitmq-server

# Запуск
sudo systemctl start rabbitmq-server

# Перезапуск
sudo systemctl restart rabbitmq-server
```

### 6.2 Просмотр логов

```bash
# Логи RabbitMQ
sudo journalctl -u rabbitmq-server -f

# Логи rabbitChat (если запущен в foreground)
# Видны в терминале
```

## Часть 7: Устранение проблем

### 7.1 Ошибка "Not_Authorized" в rabbitmqadmin

```bash
# Создать конфиг для rabbitmqadmin
cat > ~/.rabbitmqadmin.conf <<EOF
[default]
hostname = localhost
port = 15672
username = admin
password = runtelorg
vhost = /
EOF
chmod 600 ~/.rabbitmqadmin.conf
```

### 7.2 Ошибка при установке pika

```bash
# Используйте совместимую версию
pip2.7 install pika==0.13.1
```

### 7.3 Если rabbitChat не запускается

Проверьте, что RabbitMQ работает:
```bash
sudo systemctl status rabbitmq-server
curl http://localhost:15672/
```

| Команда | Описание |
|---------|----------|
| `sudo systemctl start rabbitmq-server` | Запустить RabbitMQ |
