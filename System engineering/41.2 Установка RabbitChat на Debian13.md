# Инструкция по установке и использованию RabbitMQ и rabbitChat на Debian 13

## Введение

`rabbitChat` — это чат-сервер на основе RabbitMQ, использующий AMQP протокол. Проект написан на Python 2.7, который не входит в состав Debian 13 по умолчанию, поэтому нам пришлось установить его через альтернативный интерпретатор — PyPy.

В данной инструкции мы не только установим RabbitMQ и rabbitChat, но и настроим консольный чат, а также разберём, как работает брокер сообщений на практике.

## Часть 2: Установка PyPy (Python 2.7)

Поскольку Python 2.7 отсутствует в Debian 13, мы используем PyPy — альтернативный интерпретатор, совместимый с Python 2.7.

### 2.1 Скачивание PyPy

```bash
cd ~/programs
wget https://downloads.python.org/pypy/pypy2.7-v7.3.16-linux64.tar.bz2
tar -xjf pypy2.7-v7.3.16-linux64.tar.bz2
```

### 2.2 Настройка PATH

```bash
# Добавляем PyPy в PATH
export PATH="$HOME/programs/pypy2.7-v7.3.16-linux64/bin:$PATH"
echo 'export PATH="$HOME/programs/pypy2.7-v7.3.16-linux64/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 2.3 Проверка установки PyPy

```bash
pypy2 --version
# Должно быть: Python 2.7.18 (PyPy 7.3.16)
```

## Часть 5: Альтернативный способ — консольный чат

Если rabbitChat не работает, можно использовать простой консольный чат. Этот чат показывает, как работает обмен сообщениями через RabbitMQ на самом низком уровне.

### 5.1 Создание скрипта чата

Создайте файл `~/chat.py` со следующим содержимым:

```python
import pika
import sys
import threading
import time

username = sys.argv[1] if len(sys.argv) > 1 else 'Guest'

connection = pika.BlockingConnection(
    pika.ConnectionParameters(
        host='localhost',
        credentials=pika.PlainCredentials('admin', 'runtelorg')
    )
)
channel = connection.channel()
channel.queue_declare(queue='chat_room', durable=True)

def send_message(message):
    data = '[{}] {}: {}'.format(time.strftime('%H:%M:%S'), username, message)
    channel.basic_publish(exchange='', routing_key='chat_room', body=data)

def callback(ch, method, properties, body):
    print('\n{}'.format(body))
    sys.stdout.write('{}> '.format(username))
    sys.stdout.flush()

def receive():
    channel.basic_consume(callback, queue='chat_room', no_ack=True)
    channel.start_consuming()

print('Welcome to chat, {}!'.format(username))
print('Type /exit to quit')

t = threading.Thread(target=receive)
t.daemon = True
t.start()

while True:
    try:
        msg = raw_input('{}> '.format(username))
        if msg == '/exit':
            break
        send_message(msg)
    except KeyboardInterrupt:
        break

connection.close()
```

### 5.2 Запуск консольного чата

**В первом терминале:**
```bash
python2 ~/chat.py Alice
```

**Во втором терминале:**
```bash
python2 ~/chat.py Bob
```

### 5.3 Пример работы чата

```
Терминал 1 (Alice):
Welcome to chat, Alice!
Type /exit to quit
Alice> Привет, Боб!
Alice> 
[13:36:34] Bob: Hi, Алиса!
Alice> Как дела?

Терминал 2 (Bob):
Welcome to chat, Bob!
Type /exit to quit
Bob> Hi, Алиса!
Bob> 
[13:36:42] Alice: Привет, Боб!
Bob> Как дела?
```

## Часть 6: Как работает брокер сообщений (RabbitMQ)

### 6.1 Основные понятия

| Термин | Описание | Аналогия |
|--------|----------|----------|
| **Producer (Отправитель)** | Приложение, которое отправляет сообщения | Отправитель письма |
| **Consumer (Получатель)** | Приложение, которое получает сообщения | Получатель письма |
| **Queue (Очередь)** | Хранилище сообщений до их получения | Почтовый ящик |
| **Exchange (Обменник)** | Маршрутизатор сообщений | Почтовое отделение |
| **Routing Key** | Адрес доставки | Индекс на конверте |
| **Binding (Привязка)** | Правило маршрутизации | Инструкция для почтальона |

### 6.2 Схема работы

```
┌─────────────────────────────────────────────────────────────────┐
│                     Producer (Отправитель)                     │
│  Alice пишет: "Привет!"                                       │
└─────────────────────────┬──────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                       Exchange (Обменник)                     │
│  amq.default (стандартный обменник)                           │
│  - Получает сообщение                                        │
│  - Смотрит на routing_key="chat_room"                        │
│  - Отправляет в очередь с таким же именем                    │
└─────────────────────────┬──────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                        Queue (Очередь)                        │
│  chat_room                                                    │
│  - Хранит сообщение до доставки                               │
│  - Отдаёт сообщение первому свободному потребителю           │
│  - Удаляет сообщение после подтверждения (ACK)               │
└─────────────────────────┬──────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Consumer (Получатель)                     │
│  Bob видит: "[13:36:42] Alice: Привет!"                      │
└─────────────────────────────────────────────────────────────────┘
```

### 6.3 Как работает очередь `chat_room`

Когда вы запускаете консольный чат, происходит следующее:

1. **Подключение к RabbitMQ**
   ```python
   connection = pika.BlockingConnection(
       pika.ConnectionParameters(
           host='localhost',
           credentials=pika.PlainCredentials('admin', 'runtelorg')
       )
   )
   ```

2. **Создание канала и очереди**
   ```python
   channel = connection.channel()
   channel.queue_declare(queue='chat_room', durable=True)
   ```
   - Если очередь `chat_room` не существует — она создаётся
   - `durable=True` означает, что очередь сохранится после перезагрузки

3. **Отправка сообщения**
   ```python
   channel.basic_publish(
       exchange='', 
       routing_key='chat_room', 
       body=data
   )
   ```
   - `exchange=''` — используем стандартный обменник `amq.default`
   - `routing_key='chat_room'` — отправляем в очередь с этим именем

4. **Получение сообщения**
   ```python
   channel.basic_consume(callback, queue='chat_room', no_ack=True)
   ```
   - `callback` — функция, которая вызывается при получении сообщения
   - `no_ack=True` — сообщение удаляется сразу после доставки

### 6.4 Мониторинг очередей через веб-консоль

После запуска чата откройте веб-консоль RabbitMQ:
```
http://192.168.87.199:15672/
```

На вкладке **"Queues"** вы увидите информацию о очереди `chat_room`:

| Показатель | Значение | Что значит |
|------------|----------|------------|
| **Ready** | 0 | Сообщений, ожидающих доставки, нет |
| **Unacked** | 0 | Все сообщения подтверждены |
| **Total** | 0 | В очереди нет сообщений |
| **Consumers** | 2 | Подключены два потребителя (Alice и Bob) |

### 6.5 Отправка тестового сообщения через веб-консоль

1. Откройте очередь `chat_room` в веб-консоли
2. В разделе **"Publish message"** введите:
   - **Payload:** `Hello from web!`
   - **Delivery mode:** `2 - Persistent`
3. Нажмите **"Publish message"**
4. Сообщение появится в консольном чате у всех участников

### 6.6 Команды для мониторинга

```bash
# Список всех очередей
rabbitmqadmin list queues

# Проверка подключений
rabbitmqadmin list connections

# Проверка каналов
rabbitmqadmin list channels

# Отправка тестового сообщения через API
curl -u admin:runtelorg \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"properties":{},"routing_key":"chat_room","payload":"Hello, RabbitMQ!","payload_encoding":"string"}' \
  http://localhost:15672/api/exchanges/%2F/amq.default/publish
```

## Часть 8: Устранение проблем

### 8.1 Ошибка "Not_Authorized" в rabbitmqadmin

```bash
# Создать конфиг для rabbitmqadmin
cat > ~/.rabbitmqadmin.conf <<EOF
[default]
hostname = localhost
port = 15672
username = admin
password = runtelorg
vhost = /
EOF
chmod 600 ~/.rabbitmqadmin.conf
```

### 8.2 Ошибка при установке pika

```bash
# Используйте совместимую версию
pip2.7 install pika==0.13.1
```

### 8.3 Если rabbitChat не запускается

Проверьте, что RabbitMQ работает:
```bash
sudo systemctl status rabbitmq-server
curl http://localhost:15672/
```
