# Полная инструкция по установке и использованию RabbitMQ и rabbitChat на Debian 13

## Введение

`rabbitChat` — это чат-сервер на основе RabbitMQ, использующий AMQP протокол. Проект написан на Python 2.7, который не входит в состав Debian 13 по умолчанию, поэтому нам пришлось установить его через альтернативный интерпретатор — PyPy.

В данной инструкции мы не только установим RabbitMQ и rabbitChat, но и настроим консольный чат, а также разберём, как работает брокер сообщений на практике.

## Часть 2: Установка PyPy (Python 2.7)

Поскольку Python 2.7 отсутствует в Debian 13, мы используем PyPy — альтернативный интерпретатор, совместимый с Python 2.7.

### 2.1 Скачивание PyPy

```bash
cd ~/programs
wget https://downloads.python.org/pypy/pypy2.7-v7.3.16-linux64.tar.bz2
tar -xjf pypy2.7-v7.3.16-linux64.tar.bz2
```

### 2.2 Настройка PATH

```bash
# Добавляем PyPy в PATH
export PATH="$HOME/programs/pypy2.7-v7.3.16-linux64/bin:$PATH"
echo 'export PATH="$HOME/programs/pypy2.7-v7.3.16-linux64/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 2.3 Проверка установки PyPy

```bash
pypy2 --version
# Должно быть: Python 2.7.18 (PyPy 7.3.16)
```

## Часть 5: Альтернативный способ — консольный чат

Если rabbitChat не работает, можно использовать простой консольный чат. Этот чат показывает, как работает обмен сообщениями через RabbitMQ на самом низком уровне.

### 5.1 Создание скрипта чата (Round-Robin)

Создайте файл `~/chat.py` со следующим содержимым:

```python
import pika
import sys
import threading
import time

username = sys.argv[1] if len(sys.argv) > 1 else 'Guest'

connection = pika.BlockingConnection(
    pika.ConnectionParameters(
        host='localhost',
        credentials=pika.PlainCredentials('admin', 'runtelorg')
    )
)
channel = connection.channel()
channel.queue_declare(queue='chat_room', durable=True)

def send_message(message):
    data = '[{}] {}: {}'.format(time.strftime('%H:%M:%S'), username, message)
    channel.basic_publish(exchange='', routing_key='chat_room', body=data)

def callback(ch, method, properties, body):
    print('\n{}'.format(body))
    sys.stdout.write('{}> '.format(username))
    sys.stdout.flush()

def receive():
    channel.basic_consume(callback, queue='chat_room', no_ack=True)
    channel.start_consuming()

print('Welcome to chat, {}!'.format(username))
print('Type /exit to quit')

t = threading.Thread(target=receive)
t.daemon = True
t.start()

while True:
    try:
        msg = raw_input('{}> '.format(username))
        if msg == '/exit':
            break
        send_message(msg)
    except KeyboardInterrupt:
        break

connection.close()
```

### 5.2 Запуск консольного чата

**В первом терминале:**
```bash
python2 ~/chat.py Alice
```

**Во втором терминале:**
```bash
python2 ~/chat.py Bob
```

**В третьем терминале:**
```bash
python2 ~/chat.py Charlie
```

### 5.3 Пример работы чата (Round-Robin)

## Часть 6: Broadcast-чат — доставка всем участникам

### 6.1 В чём отличие?

При использовании обычной очереди сообщения распределяются **по очереди (Round-Robin)** между потребителями. Это значит, что если у вас три участника, сообщения будут доставляться по очереди: первое — Alice, второе — Bob, третье — Charlie, четвёртое — Alice и т.д.

Для чата это неудобно, потому что все участники должны видеть все сообщения. Для этого используется **Fanout Exchange (широковещательная рассылка)**.

### 6.2 Создание скрипта broadcast-чата

Создайте файл `~/broadcast_chat.py`:

```python
import pika
import sys
import threading
import time

username = sys.argv[1] if len(sys.argv) > 1 else 'Guest'

connection = pika.BlockingConnection(
    pika.ConnectionParameters(
        host='localhost',
        credentials=pika.PlainCredentials('admin', 'runtelorg')
    )
)
channel = connection.channel()

# Объявляем fanout exchange для широковещательной рассылки
channel.exchange_declare(exchange='chat_broadcast', exchange_type='fanout', durable=True)

# Создаём временную очередь для каждого участника
# exclusive=True означает, что очередь удалится, когда участник отключится
result = channel.queue_declare(queue='', exclusive=True)
queue_name = result.method.queue

# Привязываем очередь к exchange
channel.queue_bind(exchange='chat_broadcast', queue=queue_name)

print('Welcome to broadcast chat, {}!'.format(username))
print('Type /exit to quit')

def send_message(message):
    data = '[{}] {}: {}'.format(time.strftime('%H:%M:%S'), username, message)
    # Отправляем в fanout exchange
    channel.basic_publish(exchange='chat_broadcast', routing_key='', body=data)

def callback(ch, method, properties, body):
    # Проверяем, что сообщение не от самого себя
    if username not in body:
        print('\n{}'.format(body))
        sys.stdout.write('{}> '.format(username))
        sys.stdout.flush()
    else:
        # Если сообщение своё — просто показываем, что отправлено
        sys.stdout.write('{}> '.format(username))
        sys.stdout.flush()

def receive():
    channel.basic_consume(callback, queue=queue_name, no_ack=True)
    channel.start_consuming()

# Запускаем поток для получения сообщений
t = threading.Thread(target=receive)
t.daemon = True
t.start()

# Основной цикл отправки сообщений
while True:
    try:
        msg = raw_input('{}> '.format(username))
        if msg == '/exit':
            break
        send_message(msg)
    except KeyboardInterrupt:
        break

connection.close()
```

### 6.3 Запуск broadcast-чата

**Терминал 1 (Alice):**
```bash
python2 ~/broadcast_chat.py Alice
```

**Терминал 2 (Bob):**
```bash
python2 ~/broadcast_chat.py Bob
```

**Терминал 3 (Charlie):**
```bash
python2 ~/broadcast_chat.py Charlie
```

## Часть 7: Как работает брокер сообщений (RabbitMQ)

### 7.1 Основные понятия

| Термин | Описание | Аналогия |
|--------|----------|----------|
| **Producer (Отправитель)** | Приложение, которое отправляет сообщения | Отправитель письма |
| **Consumer (Получатель)** | Приложение, которое получает сообщения | Получатель письма |
| **Queue (Очередь)** | Хранилище сообщений до их получения | Почтовый ящик |
| **Exchange (Обменник)** | Маршрутизатор сообщений | Почтовое отделение |
| **Routing Key** | Адрес доставки | Индекс на конверте |
| **Binding (Привязка)** | Правило маршрутизации | Инструкция для почтальона |
| **Fanout Exchange** | Широковещательная рассылка | Оповещение всех подписчиков |

### 7.2 Схема работы Round-Robin (обычная очередь)

```
┌─────────────────────────────────────────────────────────────────┐
│                     Producer (Отправитель)                     │
│  Alice пишет: "Привет!"                                       │
└─────────────────────────┬──────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                       Exchange (Обменник)                     │
│  amq.default (стандартный обменник)                           │
│  - Получает сообщение                                        │
│  - Смотрит на routing_key="chat_room"                        │
│  - Отправляет в очередь с таким же именем                    │
└─────────────────────────┬──────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                        Queue (Очередь)                        │
│  chat_room                                                    │
│  - Хранит сообщение до доставки                               │
│  - Отдаёт сообщение по Round-Robin:                           │
│    M1 → Alice, M2 → Bob, M3 → Charlie, M4 → Alice...         │
└─────────────────────────┬──────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Consumers (Получатели)                    │
│  Alice получает M1, M4...                                    │
│  Bob получает M2, M5...                                      │
│  Charlie получает M3, M6...                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 7.3 Схема работы Broadcast (fanout exchange)

```
┌─────────────────────────────────────────────────────────────────┐
│                     Producer (Отправитель)                     │
│  Alice пишет: "Привет всем!"                                  │
└─────────────────────────┬──────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│              Fanout Exchange (chat_broadcast)                 │
│                                                               │
│  1. Сообщение от Alice попадает в exchange                   │
│  2. Exchange копирует сообщение во все привязанные очереди    │
│                                                               │
│  ┌─────────────┬─────────────┬─────────────┬─────────────┐   │
│  │ Очередь     │ Очередь     │ Очередь     │ Очередь     │   │
│  │ Alice       │ Bob         │ Charlie     │ ...         │   │
│  └─────────────┴─────────────┴─────────────┴─────────────┘   │
│       │              │              │                         │
│       ▼              ▼              ▼                         │
│   Alice видит    Bob видит     Charlie видит                 │
│   сообщение      сообщение      сообщение                    │
└─────────────────────────────────────────────────────────────────┘
```

### 7.4 Как работает очередь `chat_room`

Когда вы запускаете консольный чат, происходит следующее:

1. **Подключение к RabbitMQ**
   ```python
   connection = pika.BlockingConnection(
       pika.ConnectionParameters(
           host='localhost',
           credentials=pika.PlainCredentials('admin', 'runtelorg')
       )
   )
   ```

2. **Создание канала и очереди**
   ```python
   channel = connection.channel()
   channel.queue_declare(queue='chat_room', durable=True)
   ```
   - Если очередь `chat_room` не существует — она создаётся
   - `durable=True` означает, что очередь сохранится после перезагрузки

3. **Отправка сообщения**
   ```python
   channel.basic_publish(
       exchange='', 
       routing_key='chat_room', 
       body=data
   )
   ```
   - `exchange=''` — используем стандартный обменник `amq.default`
   - `routing_key='chat_room'` — отправляем в очередь с этим именем

4. **Получение сообщения**
   ```python
   channel.basic_consume(callback, queue='chat_room', no_ack=True)
   ```
   - `callback` — функция, которая вызывается при получении сообщения
   - `no_ack=True` — сообщение удаляется сразу после доставки

### 7.5 Как работает broadcast-чат

1. **Создание fanout exchange**
   ```python
   channel.exchange_declare(exchange='chat_broadcast', exchange_type='fanout', durable=True)
   ```

2. **Создание временной очереди для каждого участника**
   ```python
   result = channel.queue_declare(queue='', exclusive=True)
   queue_name = result.method.queue
   ```

3. **Привязка очереди к exchange**
   ```python
   channel.queue_bind(exchange='chat_broadcast', queue=queue_name)
   ```

4. **Отправка в fanout exchange**
   ```python
   channel.basic_publish(exchange='chat_broadcast', routing_key='', body=data)
   ```

### 7.6 Мониторинг очередей через веб-консоль

После запуска чата откройте веб-консоль RabbitMQ:
```
http://192.168.87.199:15672/
```

На вкладке **"Queues"** вы увидите информацию о очереди `chat_room`:

| Показатель | Значение | Что значит |
|------------|----------|------------|
| **Ready** | 0 | Сообщений, ожидающих доставки, нет |
| **Unacked** | 0 | Все сообщения подтверждены |
| **Total** | 0 | В очереди нет сообщений |
| **Consumers** | 2 | Подключены два потребителя (Alice и Bob) |

На вкладке **"Exchanges"** вы увидите fanout exchange `chat_broadcast`.

### 7.7 Отправка тестового сообщения через веб-консоль

1. Откройте очередь `chat_room` в веб-консоли
2. В разделе **"Publish message"** введите:
   - **Payload:** `Hello from web!`
   - **Delivery mode:** `2 - Persistent`
3. Нажмите **"Publish message"**
4. Сообщение появится в консольном чате у всех участников

### Что мы получили в итоге:

- ✅ **Работающий RabbitMQ** на Debian 13
- ✅ **PyPy 2.7** для запуска legacy-приложений
- ✅ **rabbitChat** с веб-интерфейсом

### Практические выводы:

1. **Очереди создаются автоматически** при первом использовании
2. **Сообщения доставляются мгновенно** при наличии потребителей
3. **Веб-консоль** позволяет мониторить работу в реальном времени
4. **RabbitMQ гарантирует доставку** даже если потребитель временно недоступен
5. **AMQP протокол** обеспечивает стандартизированный обмен сообщениями
6. **Round-Robin** — стандартное поведение очередей для балансировки нагрузки
7. **Fanout Exchange** — используется для широковещательной рассылки всем участникам
8. **Временные очереди** (`exclusive=True`) автоматически удаляются при отключении участника

### Когда какой тип использовать:

| Тип доставки | Описание | Когда использовать |
|--------------|----------|---------------------|
| **Round-Robin** | Сообщения распределяются по очереди между потребителями | Обработка задач, балансировка нагрузки |
| **Broadcast** | Каждое сообщение получают все потребители | Чаты, уведомления, трансляции |
