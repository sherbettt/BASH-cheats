# 📚 Полная инструкция по установке FreeDrive на Ubuntu Server

## 🎯 Цель
Установить FreeDrive — легковесное self-hosted облачное хранилище на Go с веб-интерфейсом в стиле Google Drive.

---

## Часть 1: Скачивание бинарника

### 1.1 Создаем директорию и скачиваем

```bash
# Создаем директорию для скачивания
mkdir -p ~/programs/FreeDrive
cd ~/programs/FreeDrive

# Скачиваем последнюю версию (проверьте актуальную на https://github.com/abdullaabdullazade/freedrive/releases)
wget https://github.com/abdullaabdullazade/freedrive/releases/latest/download/freedrive-linux-amd64

# Делаем исполняемым
chmod +x freedrive-linux-amd64
```

### 1.2 Устанавливаем в систему

```bash
# Копируем в системную директорию
sudo cp freedrive-linux-amd64 /usr/local/bin/freedrive

# Проверяем
/usr/local/bin/freedrive --version 2>&1 | head -5
```

---

## Часть 2: Создание конфигурации

### 2.1 Создаем директории

```bash
# Директория для данных
sudo mkdir -p /opt/freedrive/data

# Директория для конфигов
sudo mkdir -p /etc/freedrive
```

### 2.2 Генерируем ключи безопасности

```bash
# Генерируем JWT секрет (для подписи токенов)
openssl rand -hex 32

# Генерируем ключ шифрования (для шифрования файлов)
openssl rand -hex 32
```

**Сохраните эти ключи!** Они понадобятся для конфига.

### 2.3 Создаем файл окружения

```bash
sudo mcedit /etc/freedrive/freedrive.env
```

**Содержимое (подставьте свои ключи):**

```env
# Основные настройки
FREEDRIVE_PORT=8080
FREEDRIVE_DATA_DIR=/opt/freedrive/data
FREEDRIVE_ADMIN_EMAIL=admin@runtel.ru
FREEDRIVE_ADMIN_PASSWORD=RunTELOxiCld78!

# Безопасность (используйте сгенерированные реальные ключи!)
FREEDRIVE_JWT_SECRET=e39ced643cffddd3bf194c3ee981587a20861eb38991214b0ce47ca47efe9ff0
FREEDRIVE_ENCRYPTION_KEY=46f1d859927891600ea804fcd5a354a14f2d1ed14b49f3526fa668b2d66ba45b
```

**⚠️ Важно:**
- Ключи должны быть **реальными hex-строками**, а не командами `$(openssl rand -hex 32)`
- `JWT_SECRET` и `ENCRYPTION_KEY` должны быть **разными**
- Пароль администратора задайте свой

## Часть 3: Создание systemd сервиса

### 3.1 Создаем файл сервиса

```bash
sudo mcedit /etc/systemd/system/freedrive.service
```

**Содержимое:**

```ini
[Unit]
Description=FreeDrive - Self-hosted Cloud Drive
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/freedrive
EnvironmentFile=/etc/freedrive/freedrive.env
ExecStart=/usr/local/bin/freedrive
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 3.2 Запускаем сервис

```bash
# Перезагружаем systemd
sudo systemctl daemon-reload

# Включаем автозапуск
sudo systemctl enable freedrive

# Запускаем сервис
sudo systemctl start freedrive

# Проверяем статус
sudo systemctl status freedrive --no-pager -l
```

## Часть 6: Управление сервисом

```bash
# Остановить
sudo systemctl stop freedrive

# Запустить
sudo systemctl start freedrive

# Перезапустить
sudo systemctl restart freedrive

# Посмотреть логи
sudo journalctl -u freedrive -f

# Посмотреть последние 50 строк логов
sudo journalctl -u freedrive -n 50 --no-pager

# Проверить статус
sudo systemctl status freedrive --no-pager -l
```

## 🔧 Устранение неполадок

### Проблема: Порт 8080 уже занят

```bash
# Проверяем, кто на порту
sudo lsof -i :8080

# Меняем порт в конфиге
sudo mcedit /etc/freedrive/freedrive.env
# Измените: FREEDRIVE_PORT=8085

# Перезапускаем
sudo systemctl restart freedrive
```

### Проблема: Ошибка доступа к данным

```bash
# Исправляем права
sudo chmod 755 /opt/freedrive/data
sudo chown -R root:root /opt/freedrive
```

### Проблема: Сервис не запускается

```bash
# Запускаем вручную для диагностики
```
