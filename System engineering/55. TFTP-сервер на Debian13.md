## 1. Установка и настройка TFTP-сервера 

### 📦 Установка сервера

```bash
# Обновление списка пакетов
sudo apt update

# Установка TFTP-сервера и клиента (опционально)
sudo apt install tftpd-hpa tftp-hpa
```

### ⚙️ Настройка конфигурации

Файл конфигурации: `/etc/default/tftpd-hpa`

```bash
# Редактируем конфигурацию
sudo mcedit /etc/default/tftpd-hpa
```

Содержимое файла:
```ini
# /etc/default/tftpd-hpa

TFTP_USERNAME="tftp"
TFTP_DIRECTORY="/srv/tftp"
TFTP_ADDRESS="0.0.0.0:69"
TFTP_OPTIONS="--secure --create"
```

**Параметры:**
- `TFTP_USERNAME` — пользователь, от которого работает сервер
- `TFTP_DIRECTORY` — корневая папка сервера (куда класть файлы)
- `TFTP_ADDRESS` — IP и порт для прослушивания (`0.0.0.0:69` = все интерфейсы)
- `TFTP_OPTIONS`:
  - `--secure` — ограничивает доступ только к `TFTP_DIRECTORY`
  - `--create` — разрешает загрузку файлов на сервер (опционально)

### 📁 Настройка прав доступа

```bash
# Создаем директорию (если не существует)
sudo mkdir -p /srv/tftp

# Меняем владельца на пользователя tftp
sudo chown -R tftp:tftp /srv/tftp

# Устанавливаем права (чтение/запись для владельца, чтение для всех)
sudo chmod -R 755 /srv/tftp
```

### 🚀 Запуск и проверка

```bash
# Перезапуск сервера после изменений
sudo systemctl restart tftpd-hpa

# Проверка статуса
sudo systemctl status tftpd-hpa

# Включение автозапуска при старте системы
sudo systemctl enable tftpd-hpa

# Проверка, что сервер слушает порт 69
sudo ss -uln | grep 69
```

**Ожидаемый вывод:**
```
UNCONN 0 0 0.0.0.0:69 0.0.0.0:*
```

## 2. Управление сервером

### Команды для управления сервисом

```bash
# Запуск сервера
sudo systemctl start tftpd-hpa

# Остановка сервера
sudo systemctl stop tftpd-hpa

# Перезапуск сервера
sudo systemctl restart tftpd-hpa

# Проверка статуса
sudo systemctl status tftpd-hpa

# Просмотр логов в реальном времени
sudo journalctl -u tftpd-hpa -f

# Просмотр последних 50 строк логов
sudo journalctl -u tftpd-hpa -n 50

# Отключение автозапуска
sudo systemctl disable tftpd-hpa
```

### Работа с файлами в `/srv/tftp`

```bash
# Копирование файла на сервер (с сохранением прав)
sudo cp /path/to/file /srv/tftp/
sudo chown tftp:tftp /srv/tftp/file

# Создание тестового файла
echo "Hello TFTP!" | sudo tee /srv/tftp/test.txt
sudo chown tftp:tftp /srv/tftp/test.txt

# Удаление файла
sudo rm /srv/tftp/file.bin

# Просмотр содержимого
ls -la /srv/tftp/
```

## 3. Работа с клиентами

### 🖥️ Установка клиента на разные ОС

**Debian/Ubuntu:**
```bash
sudo apt install tftp-hpa
```

**Arch/Manjaro:**
```bash
sudo pacman -S tftp-hpa
# или
sudo pacman -S atftp
```

**RHEL/CentOS/Fedora:**
```bash
sudo dnf install tftp
```

**Windows:**
- Встроенный клиент: `tftp` (через командную строку)
- Или установить [Tftpd64](https://pjo2.github.io/tftpd64/) (графический интерфейс)

### 📥 Скачивание файлов с сервера

**Интерактивный режим (для отладки):**
```bash
tftp 192.168.87.153
tftp> get test.txt
tftp> quit
```

**Пакетный режим (для скриптов):**
```bash
# Через tftp-hpa
tftp 192.168.87.153 -c get test.txt

# Через atftp
atftp -g -r test.txt -l test.txt 192.168.87.153
```

### 📤 Загрузка файлов на сервер (если включен `--create`)

```bash
# Интерактивный режим
tftp 192.168.87.153
tftp> put localfile.txt
tftp> quit

# Пакетный режим
tftp 192.168.87.153 -c put localfile.txt remotefile.txt

# Через atftp
atftp -p -l localfile.txt -r remotefile.txt 192.168.87.153
```

### 🔧 Параметры клиента

```bash
# tftp-hpa
tftp [options] host
  -c         # Выполнить команду в пакетном режиме
  -m         # Режим передачи (netascii, octet, mail)
  -v         # Вербозный режим (показывать прогресс)

# Примеры
tftp 192.168.87.153 -c get firmware.bin
tftp 192.168.87.153 -v -c get test.txt  # С отображением прогресса
```

## 4. Полезные команды

### 📊 Мониторинг и диагностика

```bash
# Проверка, слушает ли сервер порт
sudo ss -uln | grep 69

# Просмотр активных соединений
sudo ss -un | grep 69

# Проверка открытых файлов сервера
sudo lsof -i UDP:69

# Проверка логов в реальном времени
sudo journalctl -u tftpd-hpa -f

# Проверка прав на директорию
ls -la /srv/tftp/

# Проверка пользователя tftp
getent passwd tftp
```

### 🔥 Firewall (брандмауэр)

```bash
# Если используется ufw
sudo ufw allow 69/udp
sudo ufw status

# Если используется iptables
sudo iptables -A INPUT -p udp --dport 69 -j ACCEPT
sudo iptables -L -n -v | grep 69

# Если используется nftables
sudo nft add rule inet filter input udp dport 69 accept
```

### 🧪 Тестирование локально

```bash
# Скачивание с самого себя (через localhost)
tftp 127.0.0.1 -c get test.txt

# Скачивание через IP-адрес
tftp 192.168.87.153 -c get test.txt

# Тест через atftp
atftp -g -r test.txt -l test.txt 127.0.0.1
```

### 📦 Полезные утилиты

```bash
# Установка di-netboot-assistant (для PXE с Debian)
sudo apt install di-netboot-assistant

# Подготовка образов Debian для PXE
sudo di-netboot-assistant install stable

# Список доступных версий
sudo di-netboot-assistant list

# Удаление образов
sudo di-netboot-assistant remove stable
```

## 5. Настройка PXE-загрузки

### 📋 Что нужно для PXE

1. **TFTP-сервер** (уже есть) — раздает загрузчики
2. **DHCP-сервер** — выдает IP и указывает путь к загрузчику
3. **Файлы загрузчика** — `pxelinux.0`, `grub.efi` и т.д.
4. **Образы ОС** — ядро, initrd, корневая файловая система

### 📥 Установка компонентов PXE

```bash
# Установка необходимых пакетов
sudo apt install pxelinux syslinux-common ipxe

# Копирование загрузчиков в /srv/tftp
sudo cp /usr/lib/PXELINUX/pxelinux.0 /srv/tftp/
sudo cp /usr/lib/syslinux/modules/bios/*.c32 /srv/tftp/
sudo cp /usr/lib/ipxe/ipxe.efi /srv/tftp/  # Для UEFI
```

### 📁 Структура TFTP-директории для PXE

```
/srv/tftp/
├── pxelinux.0                 # Основной загрузчик (BIOS)
├── ipxe.efi                   # Загрузчик iPXE (UEFI)
├── pxelinux.cfg/              # Папка с конфигурациями
│   └── default                # Конфигурация по умолчанию
├── debian/                    # Образы Debian
│   ├── vmlinuz                # Ядро
│   └── initrd.img             # Initrd
├── firmware.bin               # Прошивка для устройств
└── test.txt                   # Тестовый файл
```

### 📝 Конфигурация PXE (pxelinux.cfg/default)

```bash
sudo mcedit /srv/tftp/pxelinux.cfg/default
```

Пример конфигурации для загрузки Debian:

```conf
DEFAULT debian-installer

LABEL debian-installer
    MENU LABEL Install Debian 13 (Trixie)
    KERNEL debian/vmlinuz
    APPEND initrd=debian/initrd.img url=http://192.168.87.153/debian/preseed.cfg
    IPAPPEND 2

LABEL local
    MENU LABEL Boot from local disk
    LOCALBOOT 0

LABEL firmware
    MENU LABEL Update UniFi Firmware
    KERNEL firmware.bin
    APPEND -
```

### 🖥️ Настройка DHCP для PXE

Если у вас есть DHCP-сервер (например, `isc-dhcp-server`), добавьте:

```bash
sudo mcedit /etc/dhcp/dhcpd.conf
```

```conf
subnet 192.168.87.0 netmask 255.255.255.0 {
    range 192.168.87.100 192.168.87.200;
    option routers 192.168.87.1;
    option domain-name-servers 8.8.8.8;
    
    # PXE настройки
    next-server 192.168.87.153;          # IP TFTP-сервера
    filename "pxelinux.0";                # Файл загрузчика
    
    # Для UEFI
    option arch code 93 = unsigned integer 16;
    if option arch = 00:07 {
        filename "ipxe.efi";
    }
}
```

### 🚀 Автоматическая загрузка образов Debian

```bash
# Скачивание и подготовка образов Debian
sudo di-netboot-assistant install stable

# Все файлы появятся в /srv/tftp/debian/
ls -la /srv/tftp/debian/
```

## 6. Диагностика и устранение проблем

### ❌ Проблема: Клиент зависает/не отвечает

**Причина:** Таймаут ожидания ответа.

**Решение:**
```bash
# Используйте пакетный режим вместо интерактивного
tftp 192.168.87.153 -c get test.txt

# Или установите таймаут (в секундах)
tftp 192.168.87.153 -c get test.txt -t 5
```

### ❌ Ошибка: Permission denied

**Причина:** Неправильные права на файлы или папку.

**Решение:**
```bash
# Проверка прав
ls -la /srv/tftp/

# Исправление прав
sudo chown -R tftp:tftp /srv/tftp
sudo chmod -R 755 /srv/tftp
```

### ❌ Ошибка: File not found

**Причина:** Файл отсутствует или нечитаем.

**Решение:**
```bash
# Проверка существования файла
ls -la /srv/tftp/test.txt

# Проверка прав на файл
sudo chown tftp:tftp /srv/tftp/test.txt
sudo chmod 644 /srv/tftp/test.txt
```

### ❌ Ошибка: Connection refused

**Причина:** Сервер не запущен или слушает другой интерфейс.

**Решение:**
```bash
# Проверка статуса
sudo systemctl status tftpd-hpa

# Проверка, что слушает все интерфейсы
sudo ss -uln | grep 69

# Проверка конфигурации
cat /etc/default/tftpd-hpa
# Должно быть: TFTP_ADDRESS="0.0.0.0:69"

# Перезапуск
sudo systemctl restart tftpd-hpa
```

### ❌ Проблема: Файл скачивается, но поврежден

**Причина:** Режим передачи (ASCII вместо BINARY).

**Решение:**
```bash
# Используйте бинарный режим (octet)
tftp 192.168.87.153
tftp> mode octet
tftp> get firmware.bin
tftp> quit
```

### 🧪 Полная диагностика

```bash
# 1. Проверка сети
ping -c 3 192.168.87.153

# 2. Проверка порта (с ноутбука)
nc -vuz 192.168.87.153 69

# 3. Проверка порта (с сервера)
sudo ss -uln | grep 69

# 4. Проверка прав
ls -la /srv/tftp/

# 5. Проверка логов
sudo journalctl -u tftpd-hpa -f

# 6. Проверка конфигурации
cat /etc/default/tftpd-hpa

# 7. Ручной запуск с отладкой
sudo systemctl stop tftpd-hpa
sudo /usr/sbin/in.tftpd --listen --user tftp --address 0.0.0.0:69 --secure --verbose /srv/tftp
```
