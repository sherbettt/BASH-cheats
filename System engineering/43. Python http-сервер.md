Для запуска HTTP-сервера без отрыва от сессии есть несколько способов:

## 1. Использование `nohup` (простой способ)
```bash
nohup python3 -m http.server 8000 > http.log 2>&1 &
```
- Сервер запустится в фоне
- Вывод будет направлен в файл `http.log`
- Процесс продолжит работу после закрытия терминала

## 2. Использование `screen` (рекомендуемый способ)
```bash
# Установка screen (если не установлен)
apt-get install screen  # для Debian/Ubuntu
yum install screen      # для CentOS/RHEL

# Запуск в screen сессии
screen -S http_server
python3 -m http.server 8000
# Нажмите Ctrl+A, затем D для отсоединения от сессии

# Для возврата к сессии:
screen -r http_server
```

## 3. Использование `tmux` (альтернатива screen)
```bash
tmux new -s http_server
python3 -m http.server 8000
# Ctrl+B, затем D для отсоединения

# Для возврата:
tmux attach -t http_server
```

## 4. Создание systemd-сервиса (самый надежный способ)
Создайте файл `/etc/systemd/system/http-server.service`:
```ini
[Unit]
Description=Python HTTP Server
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root
ExecStart=/usr/bin/python3 -m http.server 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

Затем выполните:
```bash
systemctl daemon-reload
systemctl enable http-server.service
systemctl start http-server.service
systemctl status http-server.service  # проверить статус
```

## Разрешение доступа извне

Сервер уже слушает на `0.0.0.0:8000`, что означает все интерфейсы. Но нужно настроить:

### 1. Настройка firewall (если включен)
```bash
# Для UFW (Ubuntu/Debian)
ufw allow 8000/tcp

# Для firewalld (CentOS/RHEL)
firewall-cmd --permanent --add-port=8000/tcp
firewall-cmd --reload

# Для iptables
iptables -A INPUT -p tcp --dport 8000 -j ACCEPT
```

### 2. Настройка облачного провайдера
Если сервер в облаке (AWS, GCP, DigitalOcean и т.д.):
- Откройте порт 8000 в **Security Group** или **Firewall** вашего облачного провайдера
- Добавьте правило для входящего трафика TCP на порт 8000

### 3. Проверка доступности
С внешнего компьютера:
```bash
curl http://<ВАШ_IP_АДРЕС>:8000
```

### 4. Дополнительная безопасность (опционально)
```bash
# Если хотите ограничить доступ по IP
python3 -m http.server 8000 --bind 192.168.97.209  # только локальная сеть

# Или использовать более безопасный SimpleHTTPSServer для HTTPS
# (потребуется генерация сертификатов)
```

**Важно**: Python HTTP-сервер не предназначен для production-использования. Для продакшена используйте nginx, Apache или gunicorn.
