# Установка Nextcloud 33 на Debian 13 (Trixie) без Docker

## Предисловие

Debian 13 (Trixie) использует актуальные версии пакетов. В отличие от более старых руководств, здесь **PHP 8.4**, а не 8.2. 

## Что потребуется

- Сервер с Debian 13 (Trixie)
- Скачать исходники сервера с https://download.nextcloud.com/server/releases/
- опциоанально смотри https://github.com/nextcloud/all-in-one/releases/tag/v12.9.2

## Шаг 1. Установка необходимых пакетов

```bash
apt update
apt install -y apache2 mariadb-server \
  php8.4 php8.4-{fpm,mysql,zip,xml,gd,mbstring,curl,simplexml,intl,bcmath,redis} \
  unzip wget certbot python3-certbot-apache libapache2-mod-php8.4
apt install -y php8.4-pgsql php8.4-pq php8.4-ps php8.4-redis
```

**Что установится:**
- Apache 2.4 — веб-сервер
- MariaDB 11.8 — база данных
- PHP 8.4 + необходимые модули
- certbot — для бесплатных SSL-сертификатов

## Шаг 2. Настройка PHP (увеличение лимитов)

```bash
sed -i 's/upload_max_filesize = 2M/upload_max_filesize = 10G/' /etc/php/8.4/apache2/php.ini
sed -i 's/post_max_size = 8M/post_max_size = 10G/' /etc/php/8.4/apache2/php.ini
sed -i 's/memory_limit = 128M/memory_limit = 512M/' /etc/php/8.4/apache2/php.ini
sed -i 's/;date.timezone =/date.timezone = UTC/' /etc/php/8.4/apache2/php.ini

systemctl restart apache2
```

> **Важно для Debian 13:** пути с `php8.4`, а не с `php8.2`!

## Шаг 3. Распаковка Nextcloud

```bash
cd /путь/к/архиву
unzip nextcloud-33.0.2.zip -d /var/www/html/
chown -R www-data:www-data /var/www/html/nextcloud
```

## Шаг 4. Настройка базы данных MariaDB

```bash
mysql_secure_installation
```

Запустите настройку, отвечая: Enter → N → Y → Y → Y → Y

```bash
mysql -u root -p
```

В интерактивном режиме выполните:

```sql
CREATE DATABASE nextcloud CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
CREATE USER 'nextcloud'@'localhost' IDENTIFIED BY 'ваш_пароль';
GRANT ALL PRIVILEGES ON nextcloud.* TO 'nextcloud'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

## Шаг 5. Настройка Apache

Создайте конфигурацию виртуального хоста:

```bash
cat > /etc/apache2/sites-available/nextcloud.conf << 'EOF'
<VirtualHost *:80>
    ServerName localhost
    DocumentRoot /var/www/html/nextcloud
    
    <Directory /var/www/html/nextcloud/>
        Options +FollowSymlinks
        AllowOverride All
        Require all granted
    </Directory>
    
    ErrorLog ${APACHE_LOG_DIR}/nextcloud_error.log
    CustomLog ${APACHE_LOG_DIR}/nextcloud_access.log combined
</VirtualHost>
EOF

a2dissite 000-default.conf
a2ensite nextcloud.conf
a2enmod rewrite headers env dir mime setenvif
systemctl restart apache2
```

## Шаг 6. Настройка фоновых задач (cron)

```bash
echo '*/5 * * * * www-data php -f /var/www/html/nextcloud/cron.php' > /etc/cron.d/nextcloud
chmod 644 /etc/cron.d/nextcloud
```

## Шаг 7. Завершение установки через веб-интерфейс

Откройте браузер и перейдите по адресу:

```
http://IP-вашего-сервера
```

Вас встретит мастер установки. Заполните:

| Поле | Значение |
|------|----------|
| Имя администратора | придумайте |
| Пароль | надёжный пароль |
| Папка данных | оставьте `/var/www/html/nextcloud/data` |
| База данных | **MySQL/MariaDB** |
| Пользователь БД | `nextcloud` |
| Пароль БД | тот что задали в шаге 4 |
| Имя БД | `nextcloud` |
| Хост | `localhost` |

## Шаг 8 (опционально). Настройка HTTPS через certbot

Если у вас есть домен, направленный на сервер:

```bash
systemctl stop apache2
certbot certonly --standalone -d ваш-домен.ru
systemctl start apache2
```

Затем создайте конфигурацию для HTTPS:

```bash
cat > /etc/apache2/sites-available/nextcloud-ssl.conf << EOF
<VirtualHost *:443>
    ServerName ваш-домен.ru
    DocumentRoot /var/www/html/nextcloud
    
    SSLEngine on
    SSLCertificateFile /etc/letsencrypt/live/ваш-домен.ru/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/ваш-домен.ru/privkey.pem
    
    <Directory /var/www/html/nextcloud/>
        Options +FollowSymlinks
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
EOF

a2ensite nextcloud-ssl.conf
a2enmod ssl
systemctl restart apache2
```

## После установки (полезные команды)

```bash
# Проверка статуса Nextcloud
sudo -u www-data php /var/www/html/nextcloud/occ status

# Добавление доверенного домена (если используете домен)
sudo -u www-data php /var/www/html/nextcloud/occ config:system:set trusted_domains 1 --value="ваш-домен.ru"

# Оптимизация базы данных
sudo -u www-data php /var/www/html/nextcloud/occ db:add-missing-indices
```

## Заключение

Nextcloud 33 установлен и готов к работе. Система использует актуальные версии пакетов Debian 13, что гарантирует стабильность и безопасность.

**Особенности для Debian 13 (Trixie):**
- PHP 8.4 вместо 8.2
- MariaDB 11.8
- Все пакеты доступны из официальных репозиториев
