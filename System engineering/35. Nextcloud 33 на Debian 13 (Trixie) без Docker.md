# Установка Nextcloud 33 на Debian 13 (Trixie) с Nginx и PostgreSQL

## Предисловие

Debian 13 (Trixie) использует актуальные версии пакетов. Здесь **PHP 8.4**, а не 8.2.  
Инструкция построена на связке **Nginx + PostgreSQL** – производительной и надёжной.

## Что потребуется

- Сервер с Debian 13 (Trixie)
- Nextcloud сервер: https://download.nextcloud.com/server/releases/
- (опционально) https://github.com/nextcloud/all-in-one/releases/tag/v12.9.2

---

## Шаг 1. Установка необходимых пакетов

```bash
apt update
apt install -y nginx postgresql postgresql-contrib \
  php8.4-fpm php8.4-pgsql \
  php8.4-{zip,xml,gd,mbstring,curl,intl,bcmath,redis} \
  unzip wget certbot python3-certbot-nginx
```

## Шаг 2. Настройка PHP

```bash
sed -i 's/upload_max_filesize = 2M/upload_max_filesize = 10G/' /etc/php/8.4/fpm/php.ini
sed -i 's/post_max_size = 8M/post_max_size = 10G/' /etc/php/8.4/fpm/php.ini
sed -i 's/memory_limit = 128M/memory_limit = 512M/' /etc/php/8.4/fpm/php.ini
sed -i 's/;date.timezone =/date.timezone = UTC/' /etc/php/8.4/fpm/php.ini

systemctl restart php8.4-fpm
```

## Шаг 3. Распаковка Nextcloud

```bash
unzip nextcloud-33.0.2.zip -d /var/www/html/
chown -R www-data:www-data /var/www/html/nextcloud
```

## Шаг 4. Настройка базы данных PostgreSQL

```bash
sudo -u postgres psql
```

```sql
CREATE DATABASE nextcloud;
CREATE USER nextcloud_user WITH PASSWORD 'ваш_пароль';
GRANT ALL PRIVILEGES ON DATABASE nextcloud TO nextcloud_user;
ALTER DATABASE nextcloud OWNER TO nextcloud_user;
\q
```

## Шаг 5. Полный рабочий конфиг Nginx

См. пример https://docs.nextcloud.com/server/19/admin_manual/installation/nginx.html

Итоговый файл `/etc/nginx/sites-available/nextcloud` (на основе `nextcloud.hard`):

```nginx
# ============================================================
# Конфигурация Nginx для Nextcloud 33
# Debian 13 (Trixie) | PHP 8.4 FPM | PostgreSQL 17
# ============================================================
#
# Ссылки на документацию (закомментированы, т.к. nginx не понимает URL без #)
# https://nginx.org/ru/docs/beginners_guide.html
# https://nginx.org/ru/docs/dirindex.html
# https://nginx.org/ru/docs/varindex.html
# https://docs.nextcloud.com/server/19/admin_manual/installation/nginx.html
#
# ============================================================

# Проверка занятости порта перед запуском:
# lsof -i :80

# ------------------------------------------------------------
# Блок upstream определяет, куда nginx будет передавать PHP-запросы
# ------------------------------------------------------------
upstream php-handler {
    # Через Unix-сокет (рекомендуется, быстрее TCP)
    server unix:/var/run/php/php8.4-fpm.sock;
    
    # Альтернатива: через TCP-порт (если сокет не работает)
    # server 127.0.0.1:9000;
    # server 192.168.87.5:9000;
}

# ------------------------------------------------------------
# Первый блок server: обрабатывает HTTP (порт 80)
# Перенаправляет весь трафик на HTTPS
# ------------------------------------------------------------
server {
    listen 80;
    listen [::]:80;
    server_name nextcloud.runtel.ru;
    
    # Принудительный редирект на HTTPS
    return 301 https://$server_name:443$request_uri;
}

# ------------------------------------------------------------
# Второй блок server: обрабатывает HTTPS (порт 443)
# Основная рабочая конфигурация
# ------------------------------------------------------------
server {
    # Слушаем порт 443 (HTTPS) на IPv4 и IPv6
    listen 443 ssl;
    listen [::]:443 ssl;
    
    # Включаем HTTP/2 для ускорения загрузки
    http2 on;
    
    # Имя сервера (ваш домен)
    server_name nextcloud.runtel.ru;

    # --------------------------------------------------------
    # Путь к корню установки Nextcloud
    # --------------------------------------------------------
    root /var/www/html/nextcloud;

    # --------------------------------------------------------
    # Настройки SSL-сертификатов
    # --------------------------------------------------------
    # Вариант 1: Let's Encrypt сертификаты (реальные)
    ssl_certificate /etc/nginx/letsencyprt-certs/fullchain1.pem;
    ssl_certificate_key /etc/nginx/letsencyprt-certs/privkey1.pem;

    # Вариант 2: Самодельный сертификат (закомментирован)
    # ssl_certificate /etc/nginx/ssl-certs/runtel.pem;
    # ssl_certificate_key /etc/nginx/ssl-certs/runtel.pem;

    # Дополнительные настройки SSL (безопасность)
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # --------------------------------------------------------
    # Безопасные HTTP-заголовки
    # Защищают от XSS, кликджекинга и других атак
    # --------------------------------------------------------
    add_header Referrer-Policy "no-referrer" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Download-Options "noopen" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Permitted-Cross-Domain-Policies "none" always;
    add_header X-Robots-Tag "none" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # Убираем заголовок X-Powered-By (информационная безопасность)
    fastcgi_hide_header X-Powered-By;

    # --------------------------------------------------------
    # Лимиты на загрузку файлов
    # --------------------------------------------------------
    client_max_body_size 10G;      # Максимальный размер загружаемого файла
    fastcgi_buffers 64 4K;         # Буферы для FastCGI

    # --------------------------------------------------------
    # Включаем Gzip сжатие для уменьшения трафика
    # --------------------------------------------------------
    gzip on;
    gzip_vary on;
    gzip_comp_level 4;
    gzip_min_length 256;
    gzip_proxied expired no-cache no-store private no_last_modified no_etag auth;
    gzip_types application/atom+xml application/javascript application/json 
               application/ld+json application/manifest+json application/rss+xml 
               application/vnd.geo+json application/vnd.ms-fontobject 
               application/x-font-ttf application/x-web-app-manifest+json 
               application/xhtml+xml application/xml font/opentype image/bmp 
               image/svg+xml image/x-icon text/cache-manifest text/css text/plain 
               text/vcard text/vnd.rim.location.xloc text/vtt text/x-component 
               text/x-cross-domain-policy;

    # --------------------------------------------------------
    # Перенаправления для CalDAV и CardDAV (синхронизация календарей/контактов)
    # --------------------------------------------------------
    location = /.well-known/carddav {
        return 301 $scheme://$host/remote.php/dav;
    }
    location = /.well-known/caldav {
        return 301 $scheme://$host/remote.php/dav;
    }

    # --------------------------------------------------------
    # Основной location: все запросы перенаправляем на index.php
    # --------------------------------------------------------
    location / {
        rewrite ^ /index.php;
    }

    # --------------------------------------------------------
    # Запрет доступа к служебным папкам
    # --------------------------------------------------------
    location ~ ^\/(?:build|tests|config|lib|3rdparty|templates|data)\/ {
        deny all;
    }
    location ~ ^\/(?:\.|autotest|occ|issue|indie|db_|console) {
        deny all;
    }

    # --------------------------------------------------------
    # Обработка PHP-скриптов (ядро Nextcloud)
    # --------------------------------------------------------
    location ~ ^\/(?:index|remote|public|cron|core\/ajax\/update|status|ocs\/v[12]|updater\/.+|ocs-provider\/.+)(?:\.php)(?:$|\/) {
        fastcgi_split_path_info ^(.+?\.php)(\/.*|)$;
        set $path_info $fastcgi_path_info;
        try_files $fastcgi_script_name =404;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $path_info;
        fastcgi_param HTTPS on;
        fastcgi_param modHeadersAvailable true;
        fastcgi_param front_controller_active true;
        fastcgi_pass php-handler;
        fastcgi_intercept_errors on;
        fastcgi_request_buffering off;
    }

    # --------------------------------------------------------
    # Статические файлы с долгим кешированием (CSS, JS, шрифты)
    # --------------------------------------------------------
    location ~ \.(?:css|js|woff2?|svg|gif|map)$ {
        try_files $uri /index.php$request_uri;
        add_header Cache-Control "public, max-age=15778463";
        add_header Referrer-Policy "no-referrer" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-Download-Options "noopen" always;
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Permitted-Cross-Domain-Policies "none" always;
        add_header X-Robots-Tag "none" always;
        add_header X-XSS-Protection "1; mode=block" always;
        access_log off;  # Не логируем запросы к статике (экономия места)
    }

    # --------------------------------------------------------
    # Изображения и медиафайлы (тоже кешируем, но меньше)
    # --------------------------------------------------------
    location ~ \.(?:png|html|ttf|ico|jpg|jpeg|bcmap|mp4|webm)$ {
        try_files $uri /index.php$request_uri;
        access_log off;
    }
}
```

### Примечания по конфигу:

- **`rewrite ^ /index.php;`** – именно так требует официальная документация Nextcloud для корневого location. Именно **этот** вариант работает без циклических редиректов, если статические файлы обрабатываются отдельными location.
- **`try_files $uri /index.php$request_uri;`** – используется только для статики, что безопасно.
- **`fastcgi_param HTTPS on;`** – обязательно, чтобы Nextcloud знал, что соединение защищённое.

## Шаг 6. Активация конфигурации

```bash
ln -sf /etc/nginx/sites-available/nextcloud /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
```

## Шаг 7. Откуда брать сертификаты (развёрнуто)

### 7.1. Let’s Encrypt (рекомендуется)

```bash
certbot --nginx -d nextcloud.runtel.ru
```

Сертификаты появятся в:  
`/etc/letsencrypt/live/nextcloud.runtel.ru/`

### 7.2. Самоподписанный (для тестов)

```bash
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/ssl/private/nextcloud-selfsigned.key \
  -out /etc/ssl/certs/nextcloud-selfsigned.crt \
  -subj "/CN=nextcloud.runtel.ru"
```

### 7.3. Корпоративный / платный сертификат

Вы покупаете `.crt` и `.key`, кладёте в любую безопасную директорию (например, `/etc/ssl/private/`) и указываете пути в конфиге Nginx.

### 7.4. Какие сертификаты использовались в процессе отладки

- На сервере Teleport использовались реальные **Let’s Encrypt** сертификаты для `teleport.runtel.org`.
- На сервере Nextcloud временно лежали самоподписанные сертификаты (файлы `fullchain1.pem`, `privkey1.pem`), полученные ранее для теста HTTPS.
- После наладки прямого HTTP-доступа Nextcloud был запущен через HTTP, но **конфиг полностью готов к HTTPS** — достаточно добавить актуальные сертификаты.

## Шаг 8. Настройка прав для Nextcloud

```bash
cd /var/www/html/nextcloud
find . -type d -exec chmod 750 {} \;
find . -type f -exec chmod 640 {} \;
chmod +x occ
chmod 750 config
chown -R www-data:www-data .
```

## Шаг 9. Настройка cron

```bash
echo '*/5 * * * * www-data php -f /var/www/html/nextcloud/cron.php' > /etc/cron.d/nextcloud
chmod 644 /etc/cron.d/nextcloud
```

## Шаг 10. Завершение установки через веб-интерфейс

Откройте `http://nextcloud.runtel.ru` (если HTTPS ещё не готов) или сразу `https://nextcloud.runtel.ru`.

**Параметры установки:**

| Поле | Значение |
|------|----------|
| Имя администратора | (придумать) |
| Пароль администратора | (надёжный) |
| Папка данных | `/var/www/html/nextcloud/data` |
| База данных | PostgreSQL |
| Пользователь БД | `nextcloud_user` |
| Пароль БД | (тот, что задали) |
| Имя БД | `nextcloud` |
| Хост | `127.0.0.1` |
| Порт | `5432` |

## Шаг 11. Пост-установочные действия

```bash
sudo -u www-data php /var/www/html/nextcloud/occ status
sudo -u www-data php /var/www/html/nextcloud/occ db:add-missing-indices
sudo -u www-data php /var/www/html/nextcloud/occ background:cron
```
