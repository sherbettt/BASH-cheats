# Установка Nextcloud 33 на Debian 13 (Trixie) с PostgreSQL и Nginx

## Предисловие

Debian 13 (Trixie) использует актуальные версии пакетов. В отличие от более старых руководств, здесь **PHP 8.4**, а не 8.2. Данная инструкция использует связку **Nginx + PostgreSQL** — более производительную и надёжную, чем классическая Apache + MariaDB.

## Что потребуется

- Сервер с Debian 13 (Trixie)
- Скачать исходники сервера с https://download.nextcloud.com/server/releases/
- опционально смотри https://github.com/nextcloud/all-in-one/releases/tag/v12.9.2

## Шаг 1. Установка необходимых пакетов

```bash
apt update
apt install -y nginx postgresql postgresql-contrib \
  php8.4-fpm php8.4-pgsql \
  php8.4-{zip,xml,gd,mbstring,curl,intl,bcmath,redis} \
  unzip wget certbot python3-certbot-nginx
```

**Что установится:**
- Nginx — веб-сервер
- PostgreSQL 17 — реляционная база данных
- PHP 8.4 FPM + необходимые модули
- certbot — для бесплатных SSL-сертификатов

## Шаг 2. Настройка PHP (увеличение лимитов)

```bash
sed -i 's/upload_max_filesize = 2M/upload_max_filesize = 10G/' /etc/php/8.4/fpm/php.ini
sed -i 's/post_max_size = 8M/post_max_size = 10G/' /etc/php/8.4/fpm/php.ini
sed -i 's/memory_limit = 128M/memory_limit = 512M/' /etc/php/8.4/fpm/php.ini
sed -i 's/;date.timezone =/date.timezone = UTC/' /etc/php/8.4/fpm/php.ini

systemctl restart php8.4-fpm
```

> **Важно для Debian 13:** пути с `php8.4`, а не с `php8.2`!

## Шаг 3. Распаковка Nextcloud

```bash
cd /путь/к/архиву
unzip nextcloud-33.0.2.zip -d /var/www/html/
chown -R www-data:www-data /var/www/html/nextcloud
```

## Шаг 4. Настройка базы данных PostgreSQL

```bash
sudo -u postgres psql
```

В интерактивном режиме выполните:

```sql
CREATE DATABASE nextcloud;
CREATE USER nextcloud_user WITH PASSWORD 'ваш_пароль';
GRANT ALL PRIVILEGES ON DATABASE nextcloud TO nextcloud_user;
ALTER DATABASE nextcloud OWNER TO nextcloud_user;
\q
```

## Шаг 5. Настройка Nginx

Создайте конфигурацию виртуального хоста:

```bash
cat > /etc/nginx/sites-available/nextcloud << 'EOF'
upstream php-handler {
    server unix:/var/run/php/php8.4-fpm.sock;
}

server {
    listen 80;
    listen [::]:80;
    server_name _;
    
    root /var/www/html/nextcloud;
    
    add_header X-Content-Type-Options nosniff;
    add_header X-XSS-Protection "1; mode=block";
    add_header X-Robots-Tag none;
    add_header X-Download-Options noopen;
    add_header X-Permitted-Cross-Domain-Policies none;
    add_header Referrer-Policy no-referrer;
    
    client_max_body_size 10G;
    fastcgi_buffers 64 4K;
    
    gzip on;
    gzip_vary on;
    gzip_comp_level 4;
    gzip_min_length 256;
    
    index index.php index.html /index.php$request_uri;
    
    location / {
        try_files $uri $uri/ /index.php$request_uri;
    }
    
    location ~ ^/(?:build|tests|config|lib|3rdparty|templates|data) {
        deny all;
    }
    
    location ~ /\.(?!well-known).* {
        deny all;
    }
    
    location ~ ^/(?:index|remote|public|cron|core/ajax/update|status|ocs/v[12]|updater/.+|ocs-provider/.+)\.php(?:$|/) {
        fastcgi_split_path_info ^(.+?\.php)(/.*)$;
        set $path_info $fastcgi_path_info;
        try_files $fastcgi_script_name =404;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $path_info;
        fastcgi_param HTTPS off;
        fastcgi_param modHeadersAvailable true;
        fastcgi_param front_controller_active true;
        fastcgi_pass php-handler;
        fastcgi_intercept_errors on;
        fastcgi_request_buffering off;
    }
    
    location ~ \.(?:css|js|svg|gif|png|jpg|ico|wasm|tflite|woff2?)$ {
        try_files $uri /index.php$request_uri;
        expires 6M;
        access_log off;
        add_header Cache-Control "public, immutable";
    }
    
    location = /.well-known/carddav {
        return 301 $scheme://$host/remote.php/dav;
    }
    
    location = /.well-known/caldav {
        return 301 $scheme://$host/remote.php/dav;
    }
    
    access_log /var/log/nginx/nextcloud_access.log;
    error_log /var/log/nginx/nextcloud_error.log;
}
EOF

rm -f /etc/nginx/sites-enabled/default
ln -s /etc/nginx/sites-available/nextcloud /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

## Шаг 6. Настройка прав для Nginx

```bash
find /var/www/html/nextcloud/ -type d -exec chmod 750 {} \;
find /var/www/html/nextcloud/ -type f -exec chmod 640 {} \;
chmod 750 /var/www/html/nextcloud/data
```

## Шаг 7. Настройка фоновых задач (cron)

```bash
echo '*/5 * * * * www-data php -f /var/www/html/nextcloud/cron.php' > /etc/cron.d/nextcloud
chmod 644 /etc/cron.d/nextcloud
```

## Шаг 8. Завершение установки через веб-интерфейс

Откройте браузер и перейдите по адресу:

```
http://IP-вашего-сервера
```

Вас встретит мастер установки. Заполните:

| Поле | Значение |
|------|----------|
| Имя администратора | придумайте |
| Пароль | надёжный пароль |
| Папка данных | оставьте `/var/www/html/nextcloud/data` |
| База данных | **PostgreSQL** |
| Пользователь БД | `nextcloud_user` |
| Пароль БД | тот что задали в шаге 4 |
| Имя БД | `nextcloud` |
| Хост | `localhost` |
| Порт | `5432` |

## Шаг 9 (опционально). Настройка HTTPS через certbot

Если у вас есть домен, направленный на сервер:

```bash
systemctl stop nginx
certbot certonly --standalone -d ваш-домен.ru
systemctl start nginx
```

Затем создайте конфигурацию для HTTPS (замените `ваш-домен.ru`):

```bash
cat > /etc/nginx/sites-available/nextcloud-ssl << 'EOF'
upstream php-handler {
    server unix:/var/run/php/php8.4-fpm.sock;
}

server {
    listen 80;
    listen [::]:80;
    server_name ваш-домен.ru;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name ваш-домен.ru;
    
    ssl_certificate /etc/letsencrypt/live/ваш-домен.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/ваш-домен.ru/privkey.pem;
    
    root /var/www/html/nextcloud;
    
    # Остальные директивы — такие же, как в HTTP конфиге
    add_header X-Content-Type-Options nosniff;
    add_header X-XSS-Protection "1; mode=block";
    add_header X-Robots-Tag none;
    add_header X-Download-Options noopen;

rm -f /etc/nginx/sites-enabled/nextcloud
nginx -t
systemctl restart nginx
```

## После установки (полезные команды)

```bash
# Проверка статуса Nextcloud
sudo -u www-data php /var/www/html/nextcloud/occ status

# Добавление доверенного домена (если используете домен)
sudo -u www-data php /var/www/html/nextcloud/occ config:system:set trusted_domains 1 --value="ваш-домен.ru"

# Оптимизация базы данных
sudo -u www-data php /var/www/html/nextcloud/occ db:add-missing-indices
```

## Заключение

Nextcloud 33 установлен и готов к работе. Система использует актуальные версии пакетов Debian 13, что гарантирует стабильность и безопасность.

**Особенности для Debian 13 (Trixie):**
- PHP 8.4 вместо 8.2
- PostgreSQL 17 вместо старых версий
- Nginx 1.26 с поддержкой HTTP/2
