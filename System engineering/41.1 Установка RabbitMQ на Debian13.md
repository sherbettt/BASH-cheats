# Инструкция по установке и настройке RabbitMQ на Debian 13

### Ресурс источник
```txt
https://www.rabbitmq.com/docs/install-debian
```

## Часть 1: Установка RabbitMQ

### 1.1 Установка ключа GPG для репозитория RabbitMQ

```bash
sudo apt-get install curl gnupg -y
curl -1sLf "https://keys.openpgp.org/vks/v1/by-fingerprint/0A9AF2115F4687BD29803A206B73A36E6026DFCA" | sudo gpg --dearmor | sudo tee /usr/share/keyrings/com.rabbitmq.team.gpg > /dev/null
```

### 1.2 Добавление репозиториев RabbitMQ

Создайте файл `/etc/apt/sources.list.d/rabbitmq.sources`:

```bash
sudo tee /etc/apt/sources.list.d/rabbitmq.sources <<EOF
Types: deb
URIs: https://deb1.rabbitmq.com/rabbitmq-server/debian/trixie
Suites: trixie
Components: main
Signed-By: /usr/share/keyrings/com.rabbitmq.team.gpg

Types: deb
URIs: https://deb2.rabbitmq.com/rabbitmq-server/debian/trixie
Suites: trixie
Components: main
Signed-By: /usr/share/keyrings/com.rabbitmq.team.gpg

# Репозиторий Erlang (критически важно для совместимости!)
Types: deb
URIs: https://deb1.rabbitmq.com/rabbitmq-erlang/debian/trixie
Suites: trixie
Components: main
Signed-By: /usr/share/keyrings/com.rabbitmq.team.gpg

Types: deb
URIs: https://deb2.rabbitmq.com/rabbitmq-erlang/debian/trixie
Suites: trixie
Components: main
Signed-By: /usr/share/keyrings/com.rabbitmq.team.gpg
EOF
```

### 1.3 Установка RabbitMQ и Erlang

```bash
sudo apt update
sudo apt install -y erlang-base \
  erlang-asn1 erlang-crypto erlang-eldap erlang-ftp erlang-inets \
  erlang-mnesia erlang-os-mon erlang-parsetools erlang-public-key \
  erlang-runtime-tools erlang-snmp erlang-ssl \
  erlang-syntax-tools erlang-tftp erlang-tools erlang-xmerl \
  rabbitmq-server
```

### 1.4 Запуск и автозагрузка

```bash
sudo systemctl enable rabbitmq-server
sudo systemctl start rabbitmq-server
```

### 1.5 Проверка установки

```bash
rabbitmqctl version
rabbitmqctl status | grep "Erlang"
```

## Часть 2: Настройка пользователей и прав доступа

### 2.1 Создание административного пользователя

```bash
# Создаём пользователя
sudo rabbitmqctl add_user admin ВашПароль

# Назначаем права администратора
sudo rabbitmqctl set_user_tags admin administrator

# Даём полные права на виртуальный хост "/"
sudo rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"
```

### 2.2 Проверка пользователей

```bash
sudo rabbitmqctl list_users
```

### 2.3 Удаление гостевого пользователя (рекомендуется для безопасности)

```bash
sudo rabbitmqctl delete_user guest
```

### 3.1 Активация плагина
```bash
sudo rabbitmq-plugins enable rabbitmq_management
```

```
http://IP-вашего-сервера:15672/
```

читай https://www.rabbitmq.com/docs/plugins

## Часть 4: Работа с виртуальными хостами (vhosts)

Виртуальные хосты позволяют изолировать очереди и обменники для разных проектов.

### 4.1 Создание виртуального хоста

```bash
sudo rabbitmqctl add_vhost /название_проекта
```

**Пример:**
```bash
sudo rabbitmqctl add_vhost /my_project
```

### 4.2 Назначение прав пользователю на виртуальный хост

```bash
sudo rabbitmqctl set_permissions -p /название_проекта admin ".*" ".*" ".*"
```

### 4.3 Просмотр всех виртуальных хостов

```bash
sudo rabbitmqctl list_vhosts
```

### 4.4 Удаление виртуального хоста

```bash
sudo rabbitmqctl delete_vhost /название_проекта
```

## Часть 5: Управление правами пользователей

### 5.1 Команда `set_permissions` - подробное объяснение

```
rabbitmqctl set_permissions [--vhost <vhost>] <username> <conf> <write> <read>
```

**Аргументы:**
- `<username>` - имя пользователя
- `<conf>` - права на конфигурацию (создание/удаление очередей и обменников)
- `<write>` - права на запись (публикация сообщений)
- `<read>` - права на чтение (получение сообщений)

**Шаблоны разрешений:**
- `".*"` - полный доступ (ко всем очередям и обменникам)
- `"^queue_name$"` - доступ только к конкретной очереди
- `"^amq\."` - доступ к обменникам, начинающимся с "amq."

### 5.2 Примеры настройки прав

```bash
# Полный доступ ко всему
sudo rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"

# Доступ только к очередям, начинающимся с "app_"
sudo rabbitmqctl set_permissions -p / developer "^app_.*" "^app_.*" "^app_.*"

# Только чтение из очередей, начинающихся с "logs_"
sudo rabbitmqctl set_permissions -p / monitor ".*" ".*" "^logs_.*"

# Только запись в очереди, начинающиеся с "task_"
sudo rabbitmqctl set_permissions -p / producer ".*" "^task_.*" ".*"
```

### 5.3 Просмотр прав

```bash
# Права пользователя на всех виртуальных хостах
sudo rabbitmqctl list_user_permissions admin

# Все пользователи и их права на конкретном виртуальном хосте
sudo rabbitmqctl list_permissions -p /my_project
```

## Часть 6: Оптимизация для продакшена

### 6.1 Настройка лимитов файловых дескрипторов

Создайте файл `/etc/systemd/system/rabbitmq-server.service.d/limits.conf`:

```bash
sudo mkdir -p /etc/systemd/system/rabbitmq-server.service.d
sudo tee /etc/systemd/system/rabbitmq-server.service.d/limits.conf <<EOF
[Service]
LimitNOFILE=65536
EOF
```

Примените изменения:
```bash
sudo systemctl daemon-reload
sudo systemctl restart rabbitmq-server
```

### 6.2 Проверка лимитов

```bash
cat /proc/$(pgrep -f "beam.smp" | head -1)/limits | grep "open files"
```

### 6.3 Основные команды для мониторинга

```bash
# Статус ноды
sudo rabbitmqctl status

# Информация о кластере
sudo rabbitmqctl cluster_status

# Список очередей
sudo rabbitmqctl list_queues

# Список обменников
sudo rabbitmqctl list_exchanges

# Список подключений
sudo rabbitmqctl list_connections

# Список всех очередей с дополнительной информацией
sudo rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

## Часть 7: Работа с веб-консолью

После входа в консоль по адресу `http://IP:15672/`:

1. **Вкладка "Overview"** - общая информация о сервере
2. **Вкладка "Connections"** - активные подключения
3. **Вкладка "Channels"** - активные каналы
4. **Вкладка "Exchanges"** - обменники (создание/удаление)
5. **Вкладка "Queues"** - очереди (создание/удаление/просмотр сообщений)
6. **Вкладка "Admin"** - управление пользователями и виртуальными хостами

## Часть 8: Пример создания и использования очереди

### 8.1 Через веб-консоль

1. Перейдите в "Queues"
2. Нажмите "Add a new queue"
3. Введите имя очереди (например, `test_queue`)
4. Нажмите "Add queue"

### 8.2 Через командную строку

```bash
# Создать очередь (через политику)
sudo rabbitmqctl set_policy test-policy "^test_" '{"max-length":1000}' --apply-to queues

# Опубликовать тестовое сообщение (через веб-консоль удобнее)
```

### 8.3 Пример на Python для подключения

```python
import pika

connection = pika.BlockingConnection(
    pika.ConnectionParameters(
        host='192.168.87.199',
        port=5672,
        credentials=pika.PlainCredentials('admin', 'ВашПароль')
    )
)
channel = connection.channel()

# Создаём очередь
channel.queue_declare(queue='test_queue')

# Отправляем сообщение
channel.basic_publish(
    exchange='',
    routing_key='test_queue',
    body='Hello, RabbitMQ!'
)

print("Сообщение отправлено")
connection.close()
```

## Важные заметки

1. **Никогда не используйте пакеты из стандартных репозиториев Debian** - они устаревшие
2. **Всегда устанавливайте Erlang из репозитория RabbitMQ** - это гарантирует совместимость
3. **В продакшене удалите пользователя `guest`** - это стандартная практика безопасности
4. **Настраивайте лимиты файловых дескрипторов** для высоких нагрузок
5. **Используйте виртуальные хосты** для изоляции разных проектов
