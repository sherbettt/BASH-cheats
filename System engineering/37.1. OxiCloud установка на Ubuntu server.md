## Инструкция по установке OxiCloud на Ubuntu Server 24.10

### 🎯 Цель
Установить OxiCloud (Self-Hosted Cloud Storage) с интеграцией ONLYOFFICE через WOPI.

---

## Часть 1: Подготовка системы

### 1.1 Обновляем систему
```bash
sudo apt update && sudo apt upgrade -y
```

### 1.2 Устанавливаем зависимости
```bash
# Базовые пакеты для сборки
sudo apt install -y curl wget git mcedit mc build-essential

# База данных PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# Кэш и очередь (для ONLYOFFICE)
sudo apt install -y redis-server rabbitmq-server

# Зависимости для сборки Rust-проекта
sudo apt install -y pkg-config libssl-dev libpq-dev
```

### 1.3 Запускаем сервисы
```bash
sudo systemctl enable --now postgresql redis rabbitmq-server
```

---

## Часть 2: Установка Rust и Cargo

```bash
# Устанавливаем Rust через rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y

# Применяем переменные окружения
source "$HOME/.cargo/env"

# Проверяем установку
rustc --version
cargo --version
```

---

## Часть 3: Сборка OxiCloud

### 3.1 Скачиваем исходники
```bash
# Создаём директорию
mkdir -p ~/programs/OxiCloud
cd ~/programs/OxiCloud

# Клонируем репозиторий
git clone https://github.com/DioCrafts/OxiCloud.git
cd OxiCloud
```

### 3.2 Создаём файл окружения `.env`
```bash
cp example.env .env
mcedit .env
```

**Содержимое `.env` (минимальное для старта):**
```bash
# База данных
OXICLOUD_DB_CONNECTION_STRING=postgres://oxicloud:oxicloud@localhost:5432/oxicloud
DATABASE_URL=postgres://oxicloud:oxicloud@localhost:5432/oxicloud

# Сервер
OXICLOUD_SERVER_HOST=0.0.0.0
OXICLOUD_SERVER_PORT=8085
OXICLOUD_BASE_URL=http://192.168.87.7:8085

# Безопасность
OXICLOUD_JWT_SECRET=$(openssl rand -hex 32)

# WOPI (ONLYOFFICE) — пока отключено
OXICLOUD_WOPI_ENABLED=false
```

### 3.3 Собираем проект
```bash
cargo build --release
```

---

## Часть 4: Настройка базы данных

### 4.1 Создаём пользователя и базу для OxiCloud
```bash
sudo -u postgres psql << EOF
CREATE USER oxicloud WITH PASSWORD 'oxicloud';
CREATE DATABASE oxicloud OWNER oxicloud;
GRANT ALL PRIVILEGES ON DATABASE oxicloud TO oxicloud;
ALTER USER oxicloud WITH REPLICATION;
EOF
```

### 4.2 Запускаем миграции
```bash
DATABASE_URL=postgres://oxicloud:oxicloud@localhost:5432/oxicloud ./target/release/oxicloud migrate
```

**Ожидаемый вывод:** `Database migrations complete`

---

## Часть 5: Перенос OxiCloud в /usr/local (production-установка)

### 5.1 Останавливаем сервис (если запущен)
```bash
sudo systemctl stop oxicloud 2>/dev/null
```

### 5.2 Создаём структуру директорий
```bash
sudo mkdir -p /usr/local/share/oxicloud
sudo mkdir -p /usr/local/share/oxicloud/storage
sudo mkdir -p /usr/local/share/oxicloud/static
sudo mkdir -p /var/log/oxicloud
```

### 5.3 Копируем бинарник
```bash
sudo cp target/release/oxicloud /usr/local/bin/oxicloud
sudo chmod +x /usr/local/bin/oxicloud
```

### 5.4 Копируем статические файлы
```bash
sudo cp -r static/* /usr/local/share/oxicloud/static/
```

### 5.5 Копируем и настраиваем .env
```bash
sudo cp .env /usr/local/share/oxicloud/.env
sudo mcedit /usr/local/share/oxicloud/.env
```

### 5.6 Если папка migrations существует и в ней есть файлы *.sql, их тоже нужно скопировать
```bash
# Копируем миграции (если есть)
sudo cp -r ~/programs/OxiCloud/OxiCloud-0.5.6/migrations /usr/local/share/oxicloud/

# Копируем документацию (не обязательно)
sudo cp ~/programs/OxiCloud/OxiCloud-0.5.6/README.md /usr/local/share/oxicloud/
```

**Итоговое содержимое `/usr/local/share/oxicloud/.env`:**
```bash
# База данных
OXICLOUD_DB_CONNECTION_STRING=postgres://oxicloud:oxicloud@localhost:5432/oxicloud
DATABASE_URL=postgres://oxicloud:oxicloud@localhost:5432/oxicloud

# Сервер
OXICLOUD_SERVER_HOST=0.0.0.0
OXICLOUD_SERVER_PORT=8085
OXICLOUD_BASE_URL=http://192.168.87.7:8085

# Пути для хранения данных
OXICLOUD_STORAGE_PATH=/usr/local/share/oxicloud/storage
OXICLOUD_STATIC_PATH=/usr/local/share/oxicloud/static

# JWT секрет (сгенерируйте свой)
OXICLOUD_JWT_SECRET=$(openssl rand -hex 32)

# WOPI (ONLYOFFICE) — пока отключено
OXICLOUD_WOPI_ENABLED=false
```

### 5.6 Создаём systemd-сервис
```bash
sudo mcedit /etc/systemd/system/oxicloud.service
```

**Содержимое:**
```ini
[Unit]
Description=OxiCloud Server
After=network.target postgresql.service
Wants=postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/usr/local/share/oxicloud
EnvironmentFile=/usr/local/share/oxicloud/.env
ExecStart=/usr/local/bin/oxicloud serve
Restart=on-failure
RestartSec=10
LimitNOFILE=65536

# Логирование
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

### 5.7 Запускаем сервис
```bash
sudo systemctl daemon-reload
sudo systemctl enable oxicloud
sudo systemctl start oxicloud
sudo systemctl status oxicloud
```

### 5.8 Проверяем работу
```bash
# Проверяем порт
ss -tlnp | grep 8085

# Смотрим логи
sudo journalctl -u oxicloud -f
```

## Часть 7: Установка ONLYOFFICE (интеграция)

### 7.1 Подготовка базы для ONLYOFFICE
```bash
sudo -u postgres psql << EOF
CREATE USER onlyoffice WITH PASSWORD 'onlyoffice';
CREATE DATABASE onlyoffice OWNER onlyoffice;
GRANT ALL PRIVILEGES ON DATABASE onlyoffice TO onlyoffice;
EOF
```

### 7.2 Добавление репозитория и ключа ONLYOFFICE
```bash
# Устанавливаем software-properties-common
sudo apt install -y software-properties-common

# Добавляем GPG-ключ ONLYOFFICE
sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys CB2DE8E5

# Добавляем репозиторий
sudo add-apt-repository "deb http://download.onlyoffice.com/repo/debian squeeze main"

# Обновляем список пакетов
sudo apt update
```

### 7.3 Установка ONLYOFFICE
```bash
# Проверяем, что пакет доступен
apt search onlyoffice | grep documentserver

# Устанавливаем ONLYOFFICE Document Server
sudo apt install -y onlyoffice-documentserver
```

### 7.4 Настройка порта и конфигурации ONLYOFFICE

**Важно:** DocService по умолчанию пытается слушать порт 8000, но этот порт мы будем использовать для Nginx. Нужно перенастроить DocService на порт 8080.

```bash
# Редактируем конфиг ONLYOFFICE
sudo mcedit /etc/onlyoffice/documentserver/local.json
```

Добавьте секцию `"server": { "port": 8080 }` в `"CoAuthoring"`:

```json
{
  "services": {
    "CoAuthoring": {
      "server": {
        "port": 8080
      },
      "sql": {
        "type": "postgres",
        "dbHost": "localhost",
        "dbPort": "5432",
        "dbName": "onlyoffice",
        "dbUser": "onlyoffice",
        "dbPass": "onlyoffice"
      },
      "token": {
        "enable": {
          "request": {
            "inbox": true,
            "outbox": true
          },
          "browser": true
        }
      }
    }
  },
  "wopi": {
    "enable": true
  }
}
```

### 7.5 Создаём Nginx конфиг для ONLYOFFICE

```bash
sudo mcedit /etc/nginx/sites-available/onlyoffice
```

Содержимое:
```nginx
server {
    listen 8000;
    server_name _;
    
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 7.6 Активируем сайт и перезапускаем

```bash
sudo ln -s /etc/nginx/sites-available/onlyoffice /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/ds.conf 2>/dev/null
sudo nginx -t
sudo systemctl reload nginx
```

### 7.7 Перезапускаем ONLYOFFICE сервисы

```bash
sudo systemctl restart ds-docservice ds-converter
```

### 7.8 Проверка ONLYOFFICE

```bash
# Прямой запрос к docservice (минуя Nginx)
curl http://localhost:8080/healthcheck
# Должно вернуть "true"

# Через Nginx на порту 8000
curl http://192.168.87.7:8000/healthcheck
# Должно вернуть "true"

# WOPI discovery
curl http://192.168.87.7:8000/hosting/discovery
# Должен вернуть XML
```

## ⚠️ Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| `apt search onlyoffice` ничего не находит | Убедитесь, что репозиторий добавлен и `apt update` выполнен |
| Ошибка GPG-ключа | Выполните `sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys CB2DE8E5` |
| DocService падает с `EADDRINUSE :::8000` | Добавьте `"server": { "port": 8080 }` в `local.json` и настройте Nginx прокси |
| WOPI не работает | Проверьте `curl http://192.168.87.7:8000/hosting/discovery` |
| JWT ошибки при открытии документа | Синхронизируйте `OXICLOUD_WOPI_SECRET` с секретом из `local.json` |
| ONLYOFFICE не запускается после установки | Проверьте логи: `journalctl -u ds-docservice -f` |
| OxiCloud не стартует после переноса | Проверьте права на папки и содержимое `.env` |
