## Инструкция по установке OxiCloud на Ubuntu Server

### 🎯 Цель
Установить OxiCloud (Self-Hosted Cloud Storage) с интеграцией ONLYOFFICE через WOPI.

---

## Часть 1: Подготовка системы

### 1.1 Обновляем систему
```bash
sudo apt update && sudo apt upgrade -y
```

### 1.2 Устанавливаем зависимости
```bash
# Базовые пакеты для сборки
sudo apt install -y curl wget git nano mc build-essential

# База данных PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# Кэш и очередь (для ONLYOFFICE)
sudo apt install -y redis-server rabbitmq-server

# Зависимости для сборки Rust-проекта
sudo apt install -y pkg-config libssl-dev libpq-dev
```

### 1.3 Запускаем сервисы
```bash
sudo systemctl enable --now postgresql redis rabbitmq-server
```

---

## Часть 2: Установка Rust и Cargo

```bash
# Устанавливаем Rust через rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y

# Применяем переменные окружения
source "$HOME/.cargo/env"

# Проверяем установку
rustc --version
cargo --version
```

---

## Часть 3: Сборка OxiCloud

### 3.1 Скачиваем исходники
```bash
# Создаём директорию
mkdir -p ~/programs/OxiCloud
cd ~/programs/OxiCloud

# Клонируем репозиторий (или используем архив с релизом)
git clone https://github.com/DioCrafts/OxiCloud.git
cd OxiCloud

# Или для стабильной версии:
# wget https://github.com/DioCrafts/OxiCloud/archive/refs/tags/v0.5.6.tar.gz
# tar -xzf v0.5.6.tar.gz
# cd OxiCloud-0.5.6
```

### 3.2 Создаём файл окружения `.env`
```bash
cp example.env .env
nano .env
```

**Содержимое `.env` (минимальное для старта):**
```bash
# База данных
OXICLOUD_DB_CONNECTION_STRING=postgres://oxicloud:oxicloud@localhost:5432/oxicloud
DATABASE_URL=postgres://oxicloud:oxicloud@localhost:5432/oxicloud

# Сервер
OXICLOUD_SERVER_HOST=0.0.0.0
OXICLOUD_SERVER_PORT=8085
OXICLOUD_BASE_URL=http://192.168.87.5:8085

# Безопасность
OXICLOUD_JWT_SECRET=$(openssl rand -hex 32)

# WOPI (ONLYOFFICE) — пока отключено
OXICLOUD_WOPI_ENABLED=false
```

### 3.3 Собираем проект
```bash
# Сборка в release-режиме (занимает 5-10 минут)
cargo build --release
```

---

## Часть 4: Настройка базы данных

### 4.1 Создаём пользователя и базу для OxiCloud
```bash
sudo -u postgres psql << EOF
CREATE USER oxicloud WITH PASSWORD 'oxicloud';
CREATE DATABASE oxicloud OWNER oxicloud;
GRANT ALL PRIVILEGES ON DATABASE oxicloud TO oxicloud;
ALTER USER oxicloud WITH REPLICATION;
EOF
```

### 4.2 Запускаем миграции
```bash
DATABASE_URL=postgres://oxicloud:oxicloud@localhost:5432/oxicloud ./target/release/oxicloud migrate
```

**Ожидаемый вывод:** `Database migrations complete`

---

## Часть 5: Запуск OxiCloud

### 5.1 Пробный запуск
```bash
./target/release/oxicloud serve
```
Сервер доступен по адресу: `http://IP_СЕРВЕРА:8085`

### 5.2 Создание systemd-сервиса
```bash
sudo nano /etc/systemd/system/oxicloud.service
```

**Содержимое:**
```ini
[Unit]
Description=OxiCloud Server
After=network.target postgresql.service
Wants=postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/root/programs/OxiCloud/OxiCloud
EnvironmentFile=/root/programs/OxiCloud/OxiCloud/.env
ExecStart=/root/programs/OxiCloud/OxiCloud/target/release/oxicloud serve
Restart=on-failure
RestartSec=10
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

### 5.3 Запуск сервиса
```bash
sudo systemctl daemon-reload
sudo systemctl enable oxicloud
sudo systemctl start oxicloud
sudo systemctl status oxicloud
```

## Часть 7: Установка ONLYOFFICE (интеграция)

### 7.1 Подготовка базы для ONLYOFFICE
```bash
sudo -u postgres psql << EOF
CREATE USER onlyoffice WITH PASSWORD 'onlyoffice';
CREATE DATABASE onlyoffice OWNER onlyoffice;
GRANT ALL PRIVILEGES ON DATABASE onlyoffice TO onlyoffice;
EOF
```

### 7.2 Установка ONLYOFFICE (выберите один способ)

**Способ А — Snap (Ubuntu):**
```bash
sudo apt install -y snapd
sudo snap install onlyoffice-ds
sudo snap set onlyoffice-ds port=8000
sudo snap run onlyoffice-ds.config -s wopi.enable true
sudo snap restart onlyoffice-ds
```

**Способ Б — Docker (любая система):**
```bash
sudo apt install -y docker.io
sudo systemctl enable --now docker
docker run -d --name onlyoffice --restart always -p 8000:80 onlyoffice/documentserver
docker exec -it onlyoffice bash -c "echo '{\"wopi\": {\"enable\": true}}' > /etc/onlyoffice/documentserver/local.json"
docker restart onlyoffice
```

### 7.3 Проверка ONLYOFFICE
```bash
curl -k http://192.168.87.5:8000/healthcheck
# Должно вернуть "true"

curl -k http://192.168.87.5:8000/hosting/discovery
# Должен вернуть XML
```

## Часть 8: Включение WOPI в OxiCloud

### 8.1 Редактируем `.env`
```bash
nano /root/programs/OxiCloud/OxiCloud/.env
```

**Изменяем:**
```bash
OXICLOUD_WOPI_ENABLED=true
OXICLOUD_WOPI_DISCOVERY_URL=http://192.168.87.5:8000/hosting/discovery
OXICLOUD_WOPI_SECRET=9623f71b105c68b7b153a54889b42b350efcd46191dad6a31f9ecee77ef849de
```

### 8.2 Перезапускаем OxiCloud
```bash
sudo systemctl restart oxicloud
```

```bash
sudo systemctl status oxicloud

# Логи OxiCloud
sudo journalctl -u oxicloud -f

sudo snap services onlyoffice-ds

# Статус ONLYOFFICE (Docker)
docker ps | grep onlyoffice

# Перезапуск ONLYOFFICE (Snap)
sudo snap restart onlyoffice-ds

# Перезапуск ONLYOFFICE (Docker)
docker restart onlyoffice
```
