## 🔄 **ПРОЦЕСС ВОССТАНОВЛЕНИЯ БАЗЫ ДАННЫХ ИЗ ДАМПА**

### **Проблема при восстановлении:**
При попытке восстановить дамп поверх существующей базы возникали ошибки:
```
ОШИБКА: ограничение "verto_profile_local_network_fkey" уже существует
ОШИБКА: повторяющееся значение ключа нарушает ограничение уникальности
```
**Причина:** В базе уже были таблицы, индексы и ограничения, а дамп пытался их создать заново.

### **Правильный процесс восстановления:**

#### **Шаг 1: Переключение на пользователя postgres**
```bash
# Все операции с PostgreSQL должны выполняться от пользователя postgres
sudo -u postgres bash
# или
su - postgres
```
**Почему:** В PostgreSQL используется аутентификация `peer` - системный пользователь должен соответствовать пользователю БД.

#### **Шаг 2: Завершение всех подключений к базе**
```bash
# Проверяем активные подключения
psql -c "SELECT pid, usename, application_name, state FROM pg_stat_activity WHERE datname='rt_pbx_v2';"

# Принудительно завершаем все подключения к базе
psql << EOF
SELECT pg_terminate_backend(pid) 
FROM pg_stat_activity 
WHERE datname = 'rt_pbx_v2' AND pid <> pg_backend_pid();
EOF
```
**Почему:** Нельзя удалить базу, если есть активные подключения.

#### **Шаг 3: Удаление существующей базы**
```bash
# Удаляем базу (только после завершения всех подключений)
psql -c "DROP DATABASE IF EXISTS rt_pbx_v2;"
```
**Результат:** База полностью удалена вместе со всеми данными.

#### **Шаг 4: Создание новой базы**
```bash
# Создаем базу заново с правильным владельцем
psql -c "CREATE DATABASE rt_pbx_v2 OWNER rt_pbx;"
```
**Почему:** Важно указать владельца `rt_pbx`, чтобы приложение могло работать с базой.

#### **Шаг 5: Восстановление из дампа**
```bash
# Восстанавливаем базу из SQL дампа
psql -d rt_pbx_v2 -f /var/lib/postgresql/rt_pbx_v2_2026-03-31.sql
```
**Результат:** Все таблицы, данные, индексы и ограничения создаются заново без ошибок.

```bash

    tablename,
    pg_size_pretty(pg_total_relation_size('public.' || tablename)) as total_size,
    pg_size_pretty(pg_relation_size('public.' || tablename)) as table_size,
    pg_size_pretty(pg_indexes_size('public.' || tablename)) as indexes_size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size('public.' || tablename) DESC
LIMIT 10;

    tablename,
    count(indexname) as indexes_count
FROM pg_indexes
WHERE schemaname = 'public'
GROUP BY tablename
ORDER BY indexes_count DESC
LIMIT 10;

-- Проверить наличие foreign keys
SELECT 
    conname as constraint_name,
    conrelid::regclass as table_name,
    confrelid::regclass as references_table
FROM pg_constraint 
WHERE contype = 'f' 
AND connamespace = 'public'::regnamespace;
EOF
```
