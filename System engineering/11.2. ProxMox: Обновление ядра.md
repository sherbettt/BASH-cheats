# Обновление Proxmox VE 8.4 с разрешением конфликта пакетов

## Введение

При обновлении Proxmox VE может возникнуть конфликт между пакетами `pve-firmware` и `firmware-linux-free`. Эта проблема проявляется при выполнении `apt upgrade` сообщением:

```
Следующие пакеты имеют неудовлетворённые зависимости:
 pve-firmware : Конфликтует: firmware-linux-free но 20200122-1 должен быть установлен
E: Сломанные пакеты
```

В данной статье описан проверенный способ решения этого конфликта и успешного обновления системы.

---

## Предварительная проверка

Перед началом обновления необходимо оценить текущее состояние системы.

### 1. Проверка версии Proxmox

```bash
pveversion -v
```

### 2. Проверка загруженного ядра

```bash
uname -r
```

### 3. Проверка свободного места на диске

```bash
df -h
```

**Важно:** Для успешного обновления необходимо минимум 5-10 ГБ свободного места на корневом разделе.

### 4. Проверка установленных ядер Proxmox

```bash
dpkg -l | grep "proxmox-kernel" | grep -v "helper"
```

### 5. Проверка наличия конфликтующих пакетов

```bash
dpkg -l | grep firmware-linux-free
apt-cache policy pve-firmware firmware-linux-free
```

---

## Решение конфликта пакетов

Если при попытке обновления возникает конфликт между `pve-firmware` и `firmware-linux-free`, выполните следующие шаги.

### Шаг 1: Установка aptitude

```bash
# Установка aptitude для автоматического разрешения зависимостей
apt install aptitude -y
```

### Шаг 2: Обновление через aptitude

```bash
# Обновление списка пакетов
aptitude update

# Безопасное обновление с автоматическим разрешением конфликтов
aptitude safe-upgrade -y
```

Aptitude автоматически разрешит конфликт, выбрав подходящее решение (обычно сохраняет `pve-firmware` и игнорирует конфликт).

### Шаг 3: Проверка результата

```bash
# Проверить, что обновление прошло успешно
apt update
apt upgrade -y
```

Если команда выполняется без ошибок — конфликт разрешен.

---

## Полное обновление системы

### Шаг 4: Обновление всех пакетов

```bash
# Обновление списка репозиториев
apt update

# Обновление установленных пакетов
apt upgrade -y

# Полное обновление с разрешением зависимостей
apt dist-upgrade -y
```

### Шаг 5: Установка последнего ядра

```bash
# Установка последней версии ядра для Proxmox VE 8
apt install proxmox-kernel-6.8 -y
```

### Шаг 6: Проверка установленных ядер

```bash
# Просмотр установленных ядер Proxmox
dpkg -l | grep "proxmox-kernel" | grep -v "helper"

# Просмотр файлов ядра в /boot
ls -la /boot/vmlinuz-*
```

Ожидаемый результат: должно быть установлено ядро версии **6.8.12-20-pve** или новее.

### Шаг 7: Удаление старых ядер и ненужных пакетов

```bash
# Автоматическое удаление ненужных пакетов и старых ядер
apt autoremove --purge -y

# Очистка кэша APT
apt clean
```

## Перезагрузка сервера

### Шаг 8: Проверка работающих виртуальных машин

```bash
# Просмотр работающих QEMU виртуальных машин
qm list

# Просмотр работающих LXC контейнеров
pct list
```

Если есть работающие ВМ/контейнеры, их необходимо остановить или мигрировать (если сервер в кластере):

```bash
# Остановка виртуальной машины (пример)
qm stop <VM_ID>

# Остановка контейнера (пример)
pct stop <CT_ID>

# Миграция на другой узел (для кластера)
qm migrate <VM_ID> <target_node> --online
```

### Шаг 9: Перезагрузка сервера

```bash
# Перезагрузка системы
reboot
```

После перезагрузки дождитесь полной загрузки системы (1-3 минуты).

---

## Проверка после обновления

### Шаг 10: Проверка загруженного ядра

```bash
# Проверка текущего ядра
uname -r
```

Ожидаемый результат: `6.8.12-20-pve` или новее.

### Шаг 11: Проверка версии Proxmox

```bash
# Просмотр версии Proxmox
pveversion -v
```

### Шаг 12: Проверка работы служб

```bash
# Проверка статуса кластерной службы
systemctl status pve-cluster -l --no-pager

# Проверка статуса демона Proxmox
systemctl status pvedaemon -l --no-pager

# Проверка статуса веб-прокси
systemctl status pveproxy -l --no-pager
```

Все службы должны быть в статусе `active (running)`.

### Шаг 13: Проверка виртуальных машин

```bash
# Просмотр всех ВМ и их статусов
qm list
pct list

# Запуск остановленных ВМ (если необходимо)
qm start <VM_ID>
pct start <CT_ID>
```

### Шаг 14: Проверка кластера (если сервер в кластере)

```bash
# Проверка статуса кластера
pvecm status
```

Все узлы должны быть в статусе `online`.

## Полный пример для одного сервера

Ниже приведен полный набор команд для одного сервера:

```bash
# 1. Предварительная проверка
pveversion -v
uname -r
df -h

# 2. Решение конфликта через aptitude
apt install aptitude -y
aptitude update
aptitude safe-upgrade -y

# 3. Обновление системы
apt update
apt upgrade -y
apt dist-upgrade -y

# 4. Установка последнего ядра
apt install proxmox-kernel-6.8 -y

# 5. Проверка установленных ядер
dpkg -l | grep "proxmox-kernel" | grep -v "helper"

# 6. Очистка
apt autoremove --purge -y
apt clean

# 7. Проверка ВМ перед перезагрузкой
qm list
pct list

# 8. Перезагрузка
reboot

# --- ПОСЛЕ ПЕРЕЗАГРУЗКИ ---

# 9. Проверка ядра
uname -r

# 10. Проверка версии
pveversion -v

# 11. Проверка служб
systemctl status pve-cluster pvedaemon pveproxy

# 12. Проверка ВМ
qm list
pct list
```

## Возможные проблемы и их решение

### Проблема 1: Недостаточно места на диске

Если при обновлении возникает ошибка "No space left on device":

```bash
# Очистка кэша APT
apt clean

# Очистка старых логов
journalctl --vacuum-size=500M

# Удаление старых ядер
apt autoremove --purge -y

# Поиск больших файлов
du -sh /var/* | sort -hr | head -10
```

### Проблема 2: Конфликт не разрешился через aptitude

Если aptitude не смог автоматически разрешить конфликт:

```bash
# Временная блокировка pve-firmware
apt-mark hold pve-firmware

# Обновление системы
apt update
apt upgrade -y
apt dist-upgrade -y

# Снятие блокировки и переустановка
apt-mark unhold pve-firmware
apt install --reinstall pve-firmware -y
```

### Проблема 3: Не загружается новое ядро после перезагрузки

```bash
# Проверить конфигурацию GRUB
update-grub

# Проверить загрузочный раздел
efibootmgr -v  # для UEFI
grub-install /dev/sda  # для BIOS
```

## Заключение

После выполнения всех шагов:
- Proxmox VE обновлен до последней версии в ветке 8.x
- Установлено новое ядро 6.8.12-20-pve
- Конфликт между pve-firmware и firmware-linux-free разрешен
- Система очищена от старых ядер и ненужных пакетов

Для перехода на Proxmox VE 9 (Debian 13 Trixie) потребуется отдельная процедура с обновлением репозиториев и полным обновлением дистрибутива.
