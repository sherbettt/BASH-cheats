# Шпаргалка по командам pveam

| Команда | Описание |
|---------|----------|
| `pveam update` | Обновить список доступных шаблонов |
| `pveam available` | Показать все доступные шаблоны |
| `pveam available --section system` | Показать только системные шаблоны |
| `pveam download <storage> <template>` | Скачать шаблон в указанное хранилище |
| `pveam list` | Показать все шаблоны в локальных хранилищах |
| `pveam list <storage>` | Показать шаблоны в конкретном хранилище |
| `pveam remove <template>` | Удалить шаблон из хранилища |
| `pveam help` | Показать справку по командам |

## Примеры использования

```bash
# Обновить список доступных для скачивания шаблонов
pveam update

# Показать все доступные шаблоны, отфильтровать по Ubuntu
pveam available | grep ubuntu

# Скачать шаблоны различных ОС в хранилище 'local'
pveam download local debian-13-standard_13.1-2_amd64.tar.zst
pveam download local ubuntu-25.04-standard_25.04-1.1_amd64.tar.zst
pveam download local gentoo-current-openrc_20250508_amd64.tar.xz
pveam download local archlinux-base_20240911-1_amd64.tar.zst

# Показать все скачанные шаблоны в хранилище local
pveam list local

# Удалить шаблон из хранилища local
pveam remove local:ubuntu-22.04-standard_22.04-1_amd64.tar.zst
```

**Примечание:** скачанные шаблоны по умолчанию хранятся в `/var/lib/vz/template/cache/`

## Управление хранилищами и шаблонами

### Просмотр всех хранилищ и их статуса

```bash
pvesm status
```
Вывод показывает все настроенные хранилища, их тип, статус, размер и точку монтирования.

### Просмотр конфигурации хранилищ

```bash
cat /etc/pve/storage.cfg
```
```c
dir: home
        path /home
        content images,vztmpl,rootdir,iso,backup
        nodes proxmox5,pmx3,prox4,pmx6,prox5,pmx4,pmx
        prune-backups keep-last=5
        shared 0

dir: local
        path /var/lib/vz
        content backup,iso,rootdir,images,vztmpl
        prune-backups keep-last=1
        shared 0

dir: home_pmx3

        content backup,iso,rootdir,images,vztmpl

glusterfs: data
```

Файл содержит настройки каждого хранилища: путь, тип контента, список узлов и параметры ротации бэкапов.

### Особенности работы с несколькими хранилищами

В Proxmox может быть настроено несколько хранилищ для шаблонов (тип `dir`). Пример конфигурации:

| Хранилище | Реальный путь | Назначение |
|-----------|---------------|------------|
| `local` | `/var/lib/vz/template/cache/` | Стандартное хранилище |
| `home` | `/home/template/cache/` | Дополнительное хранилище |
| `ssd_1tb` | `/stg/1tb` | Быстрое SSD-хранилище |

**Важно:** команда `pveam list` без аргумента не работает — всегда нужно указывать имя хранилища:

```bash
# Показать шаблоны в разных хранилищах
pveam list home      # шаблоны в /home/template/cache/
pveam list local     # шаблоны в /var/lib/vz/template/cache/
pveam list ssd_1tb   # шаблоны в /stg/1tb
```

### Поиск и удаление старых шаблонов

```bash
# Посмотреть все шаблоны в хранилище home
pveam list home

# Удалить конкретный шаблон
pveam remove home:debian-11-standard_11.7-1_amd64.tar.zst

# Удалить все шаблоны из хранилища home (осторожно!)
pveam list home | awk '{print $1}' | xargs -I {} pveam remove home:{}
```

### Определение фактического расположения файлов шаблонов

Путь к файлам зависит от конфигурации хранилища в `/etc/pve/storage.cfg`:

```bash
# Для хранилища 'local' — стандартный путь
ls -la /var/lib/vz/template/cache/

# Для хранилища 'home' — путь из конфига
ls -la /home/template/cache/
```

## Решение проблем при удалении шаблонов

### Ошибка: `unable to parse directory volume name`

Если `pveam remove` выдаёт такую ошибку, это означает, что шаблон был добавлен вручную (не через `pveam download`) или нарушена структура хранилища.

**Альтернативные способы удаления:**

### Способ 1 — через `pvesm free` (рекомендуется)

```bash
# Показать все шаблоны в хранилище с их полными именами (Volid)
pvesm list local --content vztmpl

# Удалить конкретный шаблон по Volid
pvesm free local:vztmpl/имя_файла.tar.xz

# Пример удаления
pvesm free local:vztmpl/alt-p11-rootfs-systemd-etcnet-x86_64.tar.xz
```

### Способ 2 — прямое удаление через `rm`

```bash
# Удалить файл вручную из каталога шаблонов
rm -f /var/lib/vz/template/cache/имя_файла.tar.xz

# Обновить кэш Proxmox после ручного удаления
pveam update
```

### Способ 3 — массовое удаление всех шаблонов из хранилища

```bash
# Удалить все шаблоны из хранилища local
pvesm list local --content vztmpl | awk '{print $1}' | xargs -I {} pvesm free {}
```

```bash
# Просмотр конфигурации всех хранилищ
cat /etc/pve/storage.cfg

pvesm status

# Список шаблонов в читаемом формате
pveam list local

# Список шаблонов с техническими деталями (Volid) для удаления через pvesm
pvesm list local --content vztmpl

# Проверка владельца файлов шаблонов
ls -la /var/lib/vz/template/cache/
```

---

## Важно: владельцы файлов

При просмотре каталога с шаблонами (`ls -la`) обращайте внимание на владельца файлов:

```bash
ls -la /var/lib/vz/template/cache/
# Пример вывода, где часть файлов принадлежит пользователю debian:
# -rw-r--r-- 1 debian debian 69M alt-sisyphus-rootfs-systemd-x86_64.tar.xz
```

Если файлы принадлежат не root, это может вызывать проблемы при удалении через `pveam`. Исправляется командой:

```bash
# Изменить владельца всех шаблонов на root
chown root:root /var/lib/vz/template/cache/*
```

## Создание шаблона из контейнера 111

Контейнер 111 (`pwdns1`) работает на узле `pmx6`. Выполните следующие команды **на узле pmx6**:

### Способ 1 — через `pct template` (простой)

```bash
# Остановить контейнер
pct stop 111

# Создать шаблон
pct template 111 --storage local

# Шаблон автоматически сохранится в /var/lib/vz/template/cache/
# Имя будет вида: local:vztmpl/pwdns1.tar.zst
```

### Способ 2 — через `vzdump` (с контролем имени файла)

```bash
# Остановить контейнер
pct stop 111

# Перейти в каталог шаблонов
cd /var/lib/vz/template/cache/

# Создать дамп с сжатием zstd
vzdump 111 --mode suspend --compress zstd --dumpdir .

# После завершения появится файл вида:
# vzdump-lxc-111-2025_04_14-15_30_45.tar.zst

# Переименовать в понятное имя
mv vzdump-lxc-111-*.tar.zst pwdns1-template.tar.zst
```

### Проверка результата

```bash
# Посмотреть созданные шаблоны
pveam list local | grep -E "pwdns1|111"

# Или через ls
ls -la /var/lib/vz/template/cache/ | grep -E "pwdns1|111|vzdump"
```

## Важные замечания

| Параметр | `pct template` | `vzdump` |
|----------|----------------|----------|
| Имя файла | Автоматическое (`pwdns1.tar.zst`) | Можно задать вручную |
| Сжатие | По умолчанию | Указывается (`--compress zstd`) |
| Дополнительные опции | Минимум | Больше контроля |
| Рекомендация | Для быстрого создания | Для production с бэкапами |

## Запуск контейнера после создания шаблона

```bash
# Контейнер 111 после создания шаблона останется остановленным
# Его можно запустить снова (шаблон не удаляет оригинал)
pct start 111

# Или удалить оригинал, если не нужен
pct destroy 111
```

## Создание нового контейнера из шаблона

```bash
# Из шаблона, созданного через pct template
pct create 300 /var/lib/vz/template/cache/pwdns1.tar.zst \
    --storage local \
    --hostname pwdns1-clone \
    --password new_password \
    --net0 name=eth0,bridge=vmbr0,ip=dhcp

# Или из дампа vzdump
pct restore 301 /var/lib/vz/template/cache/vzdump-lxc-111-*.tar.zst \
    --storage local
```

## Рекомендация

Для контейнера 111 (`pwdns1`) используйте `pct template`, так как это DNS-сервер и важно сохранить все настройки сети и конфигурации Bind9.
