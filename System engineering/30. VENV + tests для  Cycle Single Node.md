### **1. Установлен Python 3.13.12 из исходников**

**Результат:** Python 3.13.12 установлен в `/usr/local/bin/python3.13` и `/usr/local/bin/pip3.13`.  
Системный Python 3.11.2 остался нетронутым в `/usr/bin/python3`.
```bash
cd /tmp
wget https://www.python.org/ftp/python/3.13.12/Python-3.13.12.tar.xz
tar -xf Python-3.13.12.tar.xz
cd Python-3.13.12
./configure --prefix=/usr/local --enable-optimizations --enable-loadable-sqlite-extensions --with-ensurepip=install --enable-shared LDFLAGS="-Wl,-rpath /usr/local/lib"
make -j$(nproc)
sudo make altinstall
```


### **2. Создано виртуальное окружение на Python 3.13.12**
```bash
/usr/local/bin/python3.13 -m venv .py3.13_venv
source .py3.13_venv/bin/activate


**Важно:** После активации команда `python` указывает на интерпретатор из виртуального окружения:

which python
# /root/ansible-proj/runtel_auto_tests/.py3.13_venv/bin/python
python --version
# Python 3.13.12
```


### Когда виртуальное окружение **активировано** (в начале строки есть `(.py3.13_venv)`):

```bash
# Просто pytest (без python -m)
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml --collect-only -q

# Или через python -m (но без полного пути)
python -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml --collect-only -q

# Проверим, что pytest доступен
which pytest
# Должно показать: /root/ansible-proj/runtel_auto_tests/.py3.13_venv/bin/pytest
```

### ❌ Неправильно (то, что вы делали):

```bash
# Неправильно — использует глобальный Python 3.13
/usr/local/bin/python3.13 -m pytest ./tests/api_tests/ ...

# Неправильно — пытается запустить скрипт как модуль
/usr/local/bin/python3.13 -m /usr/bin/pytest ...
```

### 📝 Почему так происходит?

- `python` в активированном окружении → указывает на `.py3.13_venv/bin/python`
- `pytest` в активированном окружении → указывает на `.py3.13_venv/bin/pytest`
- `/usr/local/bin/python3.13` → глобальный интерпретатор, установленный из исходников
- Пакеты установлены в `.py3.13_venv/lib/python3.13/site-packages`
- Глобальный Python 3.13 не видит пакеты из виртуального окружения

## **Запуск тестов с Python 3.13.12**

### **Важное правило:**
Когда виртуальное окружение **активировано** (`source .py3.13_venv/bin/activate`), используйте **прямые команды без указания путей**.

### **Сбор информации о тестах:**
```bash
# Активируем окружение (если ещё не активировано)
source .py3.13_venv/bin/activate

# Просто pytest
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml --collect-only -q

# Или через python -m
python -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml --collect-only -q
```

### **Запуск всех тестов:**
```bash
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v
```

### **Запуск с подробным выводом:**
```bash
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v -s
```

### **Запуск конкретного тестового файла:**
```bash
pytest ./tests/api_tests/test_base_user_crud_operations.py -c vars/pyproject-auto_tests.toml -v
```

### **Запуск с остановкой на первой ошибке:**
```bash
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v -x
```

### **Запуск тестов с глобальным Python 3.13 (без виртуального окружения):**
```bash
# Только если пакеты установлены глобально (не рекомендуется)
/usr/local/bin/python3.13 -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v
```
Но так как пакеты установлены в виртуальное окружение, этот способ **не сработает** — pytest не будет найден.

## **Полезные команды для дальнейшей работы:**

```bash
# Активировать виртуальное окружение
source .py3.13_venv/bin/activate

# Выйти из виртуального окружения
deactivate

# Проверить, какой Python используется
which python
python --version

# Проверить, какие пакеты установлены в виртуальном окружении
pip list

# Установить новый пакет (после активации окружения)
pip install <package-name>

# Удалить виртуальное окружение (если нужно пересоздать)
rm -rf .py3.13_venv
/usr/local/bin/python3.13 -m venv .py3.13_venv
source .py3.13_venv/bin/activate
pip install -r requirements.txt
pip install runtel-aioesl -i https://gitlab.runtel.org/api/v4/groups/10/-/packages/pypi/simple
pip install runtel_logger -i https://gitlab.runtel.org/api/v4/groups/10/-/packages/pypi/simple

# Обновить pip в виртуальном окружении
pip install --upgrade pip

# Проверить, что pytest установлен и доступен
python -c "import pytest; print(pytest.__version__)"
# Должно вывести: 9.0.2
```

## **Решение проблемы "No module named pytest":**

Если вы видите ошибку `/usr/local/bin/python3.13: No module named pytest`, значит вы используете **глобальный Python 3.13**, а не интерпретатор из виртуального окружения.

**Исправление:**
1. Убедитесь, что виртуальное окружение активировано:
   ```bash
   source .py3.13_venv/bin/activate
   ```
2. Проверьте, что вы в правильном окружении:
   ```bash
   which python
   # Должно показывать путь внутри .py3.13_venv
   ```
3. Запускайте тесты без указания полных путей:
   ```bash
   pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v
   ```
```
