## **Краткий итог проделанной работы:**


### **1. Создано виртуальное окружение**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### **2. Установлены зависимости из PyPI**
```bash
pip install -r requirements.txt
```
Установлены: `allure-pytest`, `playwright`, `pytest`, `requests`, `ujson` и другие.

### **3. Установлены корпоративные пакеты из GitLab**
```bash
pip install runtel-aioesl -i https://gitlab.runtel.org/api/v4/groups/10/-/packages/pypi/simple
pip install runtel_logger -i https://gitlab.runtel.org/api/v4/groups/10/-/packages/pypi/simple
```

**ВАЖНО! Если у нас Python 3.11.2, то придётся форматировать строки, убирать переносы, меня кавычки**
<br/> **Лучше скомплировать Python 3.13**
### **4. Исправлены синтаксические ошибки в файлах:**
- `utils/data_generator.py` - проблема с f-строками и кавычками
- `utils/logger.py` - проблема с f-строками
- `tests/api_tests/test_base_user_crud_operations.py` - многострочные f-строки
- `tests/api_tests/test_domain_queue_crud_operations.py` - отступы и дубликаты строк

## **Теперь можно запускать тесты!**

### **Сбор информации о тестах:**
```bash
python3 -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml --collect-only -q
```

### **Запуск всех тестов:**
```bash
python3 -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v
```

### **Запуск с подробным выводом:**
```bash
python3 -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v -s
```

### **Запуск конкретного тестового файла:**
```bash
python3 -m pytest ./tests/api_tests/test_base_user_crud_operations.py -c vars/pyproject-auto_tests.toml -v
```

### **Запуск с остановкой на первой ошибке:**
```bash
python3 -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v -x
```

## **Полезные команды для дальнейшей работы:**

```bash
# Активировать виртуальное окружение (если вышли)
source .venv/bin/activate

# Выйти из виртуального окружения
deactivate

# Удалите папку .venv
rm -rf .venv

# Убедитесь, что папка удалена
ls -alF | grep .venv
# Если ничего не вывело — окружение удалено

# Обновить список пакетов
pip list

# Установить новый пакет
pip install <package-name>
```
