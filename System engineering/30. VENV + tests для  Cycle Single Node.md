## **1. Установка Python 3.13.12 из исходников**

Системный Python в Debian 12 имеет версию 3.11.2, которая не поддерживает современный синтаксис, используемый в автотестах (многострочные f-строки, новые аннотации типов). Поэтому требуется установить Python 3.13.12.

**Результат установки:**
- Python 3.13.12 устанавливается в `/usr/local/bin/python3.13`
- Pip 3.13 устанавливается в `/usr/local/bin/pip3.13`
- Системный Python 3.11.2 остаётся нетронутым в `/usr/bin/python3`

**Команды для установки:**
```bash
cd /tmp
wget https://www.python.org/ftp/python/3.13.12/Python-3.13.12.tar.xz
tar -xf Python-3.13.12.tar.xz
cd Python-3.13.12

./configure \
    --prefix=/usr/local \
    --enable-optimizations \
    --enable-loadable-sqlite-extensions \
    --with-ensurepip=install \
    --enable-shared \
    LDFLAGS="-Wl,-rpath /usr/local/lib"

make -j$(nproc)
sudo make altinstall
```

**Пояснение параметров configure:**
- `--prefix=/usr/local` — установка в отдельную директорию, не конфликтующую с системным Python
- `--enable-optimizations` — включает оптимизации производительности (PGO)
- `--enable-loadable-sqlite-extensions` — поддержка расширений SQLite
- `--with-ensurepip=install` — автоматическая установка pip
- `--enable-shared` — создание разделяемой библиотеки libpython
- `LDFLAGS="-Wl,-rpath /usr/local/lib"` — добавление пути к библиотеке в поиск загрузчика

**Важно:** Используется `make altinstall`, а не `make install`, чтобы не перезаписывать системные файлы `python3` и `pip3`.

## **2. Создание виртуального окружения**

Виртуальное окружение создаётся на базе установленного Python 3.13.12. Это позволяет изолировать зависимости проекта от глобальной системы.

**Команды:**
```bash
cd /root/ansible-proj/runtel_auto_tests
/usr/local/bin/python3.13 -m venv .py3.13_venv
source .py3.13_venv/bin/activate
```

**После активации окружения:**
- Команда `python` начинает указывать на интерпретатор внутри виртуального окружения
- Команда `pip` начинает указывать на pip внутри виртуального окружения
- Все устанавливаемые пакеты попадают в `.py3.13_venv/lib/python3.13/site-packages/`

**Проверка:**
```bash
which python
# Вывод: /root/ansible-proj/runtel_auto_tests/.py3.13_venv/bin/python

python --version
# Вывод: Python 3.13.12
```

## **4. Запуск тестов**

### **Основное правило**
Когда виртуальное окружение активировано (в начале строки терминала отображается `(.py3.13_venv)`), необходимо использовать прямые команды без указания полных путей к интерпретатору.

### **Почему это важно**
- В активированном окружении `python` указывает на `.py3.13_venv/bin/python`
- В активированном окружении `pytest` указывает на `.py3.13_venv/bin/pytest`
- Все пакеты установлены в `.py3.13_venv/lib/python3.13/site-packages/`
- Глобальный интерпретатор `/usr/local/bin/python3.13` не видит пакеты из виртуального окружения

### **Правильные команды для запуска**

**Сбор информации о тестах (dry-run):**
```bash
# Активация окружения (если ещё не активирована)
source .py3.13_venv/bin/activate

# Запуск сбора тестов
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml --collect-only -q

# Альтернативный вариант через python -m
python -m pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml --collect-only -q
```

**Запуск всех тестов:**
```bash
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v
```

**Запуск с подробным выводом (включая print):**
```bash
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v -s
```

**Запуск конкретного тестового файла:**
```bash
pytest ./tests/api_tests/test_base_user_crud_operations.py -c vars/pyproject-auto_tests.toml -v
```

**Запуск с остановкой при первой ошибке:**
```bash
pytest ./tests/api_tests/ -c vars/pyproject-auto_tests.toml -v -x
```

### **Решение типичной ошибки "No module named pytest"**

```bash
# Ошибка: используется глобальный Python 3.13, который не видит пакеты из виртуального окружения
/usr/local/bin/python3.13 -m pytest ./tests/api_tests/ ...

# Ошибка: попытка запустить скрипт как модуль
/usr/local/bin/python3.13 -m /usr/bin/pytest ...
```

## **8. Важное примечание о версиях Python**

Исходный код автотестов использует синтаксис, требующий Python версии 3.13 или выше:
- Многострочные f-строки
- Современные аннотации типов
- Новые возможности синтаксиса

**Проверка совместимости:**
- Python 3.11.2 (системный) — **не подходит**, вызывает синтаксические ошибки
- Python 3.13.12 (скомпилированный) — **полностью совместим**
