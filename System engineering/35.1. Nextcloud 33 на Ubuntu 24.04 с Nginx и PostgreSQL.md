# Полная инструкция по настройке Nextcloud 33 на Ubuntu 24.04 с Nginx, PostgreSQL, OnlyOffice и SSO Keycloak

## Предисловие

Ubuntu 24.04 (Noble Numbat) использует стабильные версии пакетов. Здесь **PHP 8.3**, а не 8.4.  
Инструкция построена на связке **Nginx + PostgreSQL** – производительной и надёжной.  
В конце добавим **OnlyOffice** для онлайн-редактирования документов прямо в браузере и **Keycloak SSO** для централизованной аутентификации.

## Что потребуется

- Сервер с Ubuntu 24.04 LTS
- Nextcloud сервер: https://download.nextcloud.com/server/releases/
- Домен (или поддомен) для OnlyOffice, например `only-office.runtel.ru`
- Домен (или поддомен) для Keycloak SSO, например `sso.runtel.ru`
- (опционально) https://github.com/nextcloud/all-in-one/releases/tag/v12.9.2
- Инструкция https://docs.nextcloud.com/server/33/admin_manual/installation/index.html

## Шаг 5. Полный рабочий конфиг Nginx

См. пример https://docs.nextcloud.com/server/19/admin_manual/installation/nginx.html

Итоговый файл `/etc/nginx/sites-available/nextcloud`:

```nginx
# ============================================================
# Конфигурация Nginx для Nextcloud 33
# Ubuntu 24.04 | PHP 8.3 FPM | PostgreSQL
# ============================================================

upstream php-handler {
    server unix:/var/run/php/php8.3-fpm.sock;
}

# ------------------------------------------------------------
# Перенаправление HTTP на HTTPS
# ------------------------------------------------------------
server {
    listen 80;
    listen [::]:80;
    server_name nextcloud.runtel.ru;
    return 301 https://$server_name$request_uri;
}

# ------------------------------------------------------------
# Основной блок HTTPS
# ------------------------------------------------------------
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    
    server_name nextcloud.runtel.ru;
    root /var/www/html/nextcloud;

    # SSL сертификаты (настройте под себя)
    ssl_certificate /etc/nginx/runtel.pem;
    ssl_certificate_key /etc/nginx/runtel.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # Безопасные заголовки
    add_header Referrer-Policy "no-referrer" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Download-Options "noopen" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Permitted-Cross-Domain-Policies "none" always;
    add_header X-Robots-Tag "none" always;
    add_header X-XSS-Protection "1; mode=block" always;

    fastcgi_hide_header X-Powered-By;
    client_max_body_size 10G;
    fastcgi_buffers 64 4K;

    # Gzip сжатие
    gzip on;
    gzip_vary on;
    gzip_comp_level 4;
    gzip_min_length 256;
    gzip_proxied expired no-cache no-store private no_last_modified no_etag auth;
    gzip_types application/atom+xml application/javascript application/json 
               application/ld+json application/manifest+json application/rss+xml 
               application/vnd.geo+json application/vnd.ms-fontobject 
               application/x-font-ttf application/x-web-app-manifest+json 
               application/xhtml+xml application/xml font/opentype image/bmp 
               image/svg+xml image/x-icon text/cache-manifest text/css text/plain 
               text/vcard text/vnd.rim.location.xloc text/vtt text/x-component 
               text/x-cross-domain-policy;

    # CalDAV / CardDAV
    location = /.well-known/carddav {
        return 301 $scheme://$host/remote.php/dav;
    }
    location = /.well-known/caldav {
        return 301 $scheme://$host/remote.php/dav;
    }

    # Основной location
    location / {
        rewrite ^ /index.php;
    }

    # Запрет доступа к служебным папкам
    location ~ ^\/(?:build|tests|config|lib|3rdparty|templates|data)\/ {
        deny all;
    }
    location ~ ^\/(?:\.|autotest|occ|issue|indie|db_|console) {
        deny all;
    }

    # Обработка PHP
    location ~ ^\/(?:index|remote|public|cron|core\/ajax\/update|status|ocs\/v[12]|updater\/.+|ocs-provider\/.+)(?:\.php)(?:$|\/) {
        fastcgi_split_path_info ^(.+?\.php)(\/.*|)$;
        set $path_info $fastcgi_path_info;
        try_files $fastcgi_script_name =404;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $path_info;
        fastcgi_param HTTPS on;
        fastcgi_param modHeadersAvailable true;
        fastcgi_param front_controller_active true;
        fastcgi_pass php-handler;
        fastcgi_intercept_errors on;
        fastcgi_request_buffering off;
    }

    # Статические файлы с кешированием
    location ~ \.(?:css|js|woff2?|svg|gif|map)$ {
        try_files $uri /index.php$request_uri;
        add_header Cache-Control "public, max-age=15778463";
        access_log off;
    }

    # Изображения и медиа
    location ~ \.(?:png|html|ttf|ico|jpg|jpeg|bcmap|mp4|webm)$ {
        try_files $uri /index.php$request_uri;
        access_log off;
    }
}
```

### 12.4. Добавление репозитория и установка OnlyOffice Document Server

```bash

# Добавляем репозиторий

# Предустанавливаем пароль в debconf
echo "onlyoffice-documentserver onlyoffice/db/password password OfficePass99rntl" | sudo debconf-set-selections
echo "onlyoffice-documentserver onlyoffice/db/user string onlyoffice" | sudo debconf-set-selections
echo "onlyoffice-documentserver onlyoffice/db/dbname string onlyoffice" | sudo debconf-set-selections
echo "onlyoffice-documentserver onlyoffice/db/host string 127.0.0.1" | sudo debconf-set-selections

# Обновляем и устанавливаем

```

### 12.5. Настройка Nginx для OnlyOffice

См. инструкцию от разработчика https://docs.nextcloud.com/server/33/admin_manual/installation/nginx.html

Создайте конфиг `/etc/nginx/sites-available/onlyoffice`:

```nginx
server {
    listen 80;
    server_name only-office.runtel.ru;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name only-office.runtel.ru;

    ssl_certificate /etc/nginx/runtel.pem;
    ssl_certificate_key /etc/nginx/runtel.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    # Правило для статики: убираем версию из URL
    location ~ ^/([\d]+\.[\d]+\.[\d]+[\.|-][\w]+)/(?<path>.*)$ {
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_pass http://127.0.0.1:8000/$path$is_args$args;
        proxy_http_version 1.1;
    }

    # Прокси для всех остальных запросов
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        client_max_body_size 100M;
        proxy_read_timeout 3600s;
        proxy_connect_timeout 3600s;
        proxy_send_timeout 3600s;
        proxy_buffering off;
    }
}
```

Активируйте конфиг:

```bash
sudo ln -sf /etc/nginx/sites-available/onlyoffice /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## Шаг 13. Настройка SSO (Keycloak + Nextcloud)

**Что такое SSO?**  
Single Sign-On позволяет пользователю один раз войти в Keycloak и получить доступ ко всем подключённым сервисам (Nextcloud, OnlyOffice и другие) без повторного ввода пароля.

---

### Часть 13.1. Настройка Keycloak (Identity Provider)

**Где делаем:** Веб-интерфейс Keycloak (https://sso.runtel.ru:8443)

#### 13.1.1. Создание клиента в Keycloak

**Навигация:**
1. Зайдите в Admin Console Keycloak
2. Выберите realm (например, `runtel`)
3. Перейдите в раздел **Clients**
4. Нажмите **Create client**

**Параметры создания клиента (первый экран):**

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| Client ID | `nextcloud` | Уникальный идентификатор клиента |
| Client protocol | `OpenID Connect` | Протокол аутентификации |
| Root URL | `https://nextcloud.runtel.ru` | Базовый URL Nextcloud |

Нажмите **Next**.

#### 13.1.2. Детальные настройки созданного клиента

**Навигация:** Clients → nextcloud → вкладка **Settings**

**Основные настройки (General Settings):**

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| Client ID | `nextcloud` | Уникальный идентификатор |
| Name | `nextcloud` | Отображаемое имя |
| Description | `For https://nextcloud.runtel.ru/apps/dashboard/` | Описание (необязательно) |
| Always display in UI | `OFF` | Показывать ли клиент в интерфейсе входа |

**Настройки доступа (Access Settings):**

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| Root URL | `https://nextcloud.runtel.ru` | Базовый URL (без звёздочки!) |
| Home URL | `https://nextcloud.runtel.ru/` | Домашняя страница после входа |
| **Valid redirect URIs** | `https://nextcloud.runtel.ru/apps/user_oidc/code` | **Критически важно!** URL для перенаправления после входа |
| Valid post logout redirect URIs | `https://nextcloud.runtel.ru/*` | Куда перенаправлять после выхода |
| Web origins | `https://nextcloud.runtel.ru/` | Разрешённые источники для CORS |
| Admin URL | *(оставить пустым)* | Не требуется |

**Настройки аутентификации (Authentication Settings):**

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| Client authentication | `ON` | Включает аутентификацию клиента (генерируется Client Secret) |
| Authorization | `OFF` | Тонкая настройка прав (не требуется) |
| Standard flow | `ON` | Разрешает стандартный OIDC поток |
| Implicit flow | `OFF` | Не используется |
| Direct access grants | `OFF` | Не используется |

Нажмите **Save**.

#### 13.1.3. Получение Client Secret

**Навигация:** Clients → nextcloud → вкладка **Credentials**

**Что получаем:** 
```
Client Secret: 21Vbk2sgr2qj99Lyqud0GOgiLBar0Z4X
```

#### 13.1.4. Настройка ролей для прав администратора

**Создание клиентской роли `admin`:**

**Навигация:**
- **Clients** → выберите **nextcloud** → вкладка **Roles**
- Нажмите **Create role**
- **Role name:** `admin`
- Нажмите **Save**

**Назначение роли пользователю:**

**Навигация:**
- **Users** → выберите нужного пользователя (например, `k@runtel.ru`)
- Откройте вкладку **Role mapping**
- Нажмите **Assign role** → выберите **Filter by clients** → укажите **nextcloud**
- Выберите роль **`admin`** и нажмите **Assign**

**Добавление маппера для передачи роли в токен:**

**Навигация:**
- **Clients** → выберите **nextcloud**
- Перейдите на вкладку **Client scopes**
- Нажмите на **`nextcloud-dedicated`**
- Нажмите **Add mapper** → **By configuration**
- Выберите тип **`User Client Role`**

**Заполните поля маппера:**

| Поле | Значение | Пояснение |
|------|----------|-----------|
| Name | `Client Roles for Nextcloud` | Имя маппера |
| Client ID | `nextcloud` | Выберите из списка |
| Token Claim Name | `nextcloud-roles` | Имя поля для групп |
| Claim JSON Type | `String` | Тип данных |
| Add to ID token | `ON` | Добавлять в ID токен |
| Add to access token | `ON` | Добавлять в access токен |
| Add to userinfo | `ON` | Добавлять в userinfo |

Нажмите **Save**.

**Результат:** В токене появится поле `"nextcloud-roles": ["admin"]`

### Часть 13.2. Проверка токена через Evaluate (важный шаг!)

**Где делаем:** Keycloak Admin Console → Clients → nextcloud → Client scopes → **Evaluate**

2. В поле **`User`** выберите пользователя `k@runtel.ru`
3. В поле **`Client`** оставьте `nextcloud`
4. Нажмите **`Generate`**

**Проверяем сгенерированные токены:**

**Access Token:**

```json
  "realm_access": {
    "roles": ["default-roles-runtel", "offline_access"]
  },
  "resource_access": {
    "nextcloud": {
      "roles": ["admin"]
    }
  }

```

**User Info:**
```json
{
  "sub": "cca59452-aa46-420d-8a5e-d27882ebcea6",
  "email_verified": true,
  "name": "Кирилл Кораблин",
  "nextcloud-roles": ["admin"],
  "preferred_username": "kkorablin",
  "given_name": "Кирилл",
  "family_name": "Кораблин",
  "email": "k@runtel.ru"
}
```

**Что проверяем:**
- ✅ Наличие поля `"nextcloud-roles": ["admin"]` - это самое важное!
- ✅ Правильность `preferred_username` - будет использоваться как логин
- ✅ Правильность `email` и `name`

**Если поле `nextcloud-roles` отсутствует или пустое:**
- Проверьте настройки маппера (шаг 13.1.4)
- Проверьте, что роль `admin` назначена пользователю
- Проверьте, что в маппере указан `Token Claim Name: nextcloud-roles`

**Если поле есть, но значение не `["admin"]`:**
- Проверьте, что роль называется именно `admin` (а не `Администратор` или другое)
- Проверьте, что пользователю назначена эта роль во вкладке `Role mapping`

```bash
  --discoveryuri="https://sso.runtel.ru:8443/realms/runtel/.well-known/openid-configuration" \
  --scope="openid profile email" \
  "Keycloak"
```

**Разбор параметров:**

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| `--clientid` | `nextcloud` | Должен совпадать с Client ID в Keycloak |
| `--clientsecret` | `21Vbk2sgr2qj99Lyqud0GOgiLBar0Z4X` | Секретный ключ из Keycloak |
| `--discoveryuri` | `https://sso.runtel.ru:8443/realms/runtel/.well-known/openid-configuration` | Discovery endpoint для автоматического получения URL |
| `--scope` | `openid profile email` | Запрашиваемые данные |
| `"Keycloak"` | (последний аргумент) | Имя провайдера в Nextcloud |

#### 13.3.4. Включение групповой синхронизации
***ВНИМАНИЕ!*** Включение групповой поликити group-provisioning может привести к обратному эффекту, когда после логина у пользователя не сохраняются права админа.

```bash
sudo -u www-data php /var/www/html/nextcloud/occ user_oidc:provider \
  --group-provisioning="1" \
  --mapping-groups="nextcloud-roles" \
  "Keycloak"
```

**Что делают параметры:**
- `--group-provisioning="1"` - включает автоматическую синхронизацию групп
- `--mapping-groups="nextcloud-roles"` - указывает поле в токене с группами

## Шаг 14. Устранение распространённых проблем

### 14.1. Ошибка 403 Forbidden

```bash
sudo chmod 755 /var/www/html/nextcloud
sudo chmod 755 /var/www/html/nextcloud/config
sudo systemctl restart nginx php8.3-fpm
```

### 14.2. Ошибка подключения к PostgreSQL

```bash
sudo -u postgres psql -c "ALTER USER nextcloud_user WITH PASSWORD 'новый_пароль';"
```

### 14.3. Циклический редирект (HTTP → HTTP)

Убедитесь, что в конфиге Nginx есть `fastcgi_param HTTPS on;`

### 14.4. OnlyOffice не открывает документы (JWT ошибка)

```bash
# Проверяем JWT секрет
sudo -u www-data php /var/www/html/nextcloud/occ config:app:get onlyoffice jwt_secret

# Если пусто - устанавливаем
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice jwt_secret --value="<JWT-secret>"
```

### 14.5. Ошибка SSL сертификата для Keycloak

```bash
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set user_oidc allow_self_signed --value="1"
```

### 14.6. Кнопка входа через Keycloak не появляется

```bash
sudo -u www-data php /var/www/html/nextcloud/occ maintenance:repair
sudo systemctl restart php8.3-fpm nginx
```

### 14.7. Пользователь не получает права администратора

**Решение:** Добавить пользователя в группу admin вручную:

```bash
# Найти uid пользователя
sudo -u www-data php /var/www/html/nextcloud/occ user:list

# Добавить в группу
sudo -u www-data php /var/www/html/nextcloud/occ group:adduser admin "хешированный_uid_пользователя"
```

## Основные команды occ (синтаксис)

```bash
sudo -u www-data php /var/www/html/nextcloud/occ [команда] [параметры]
```

**Пояснение:**
- `sudo -u www-data` - выполняет команду от имени пользователя `www-data` (владельца файлов Nextcloud)
- `php /var/www/html/nextcloud/occ` - запускает консольный менеджер Nextcloud
- `[команда]` - конкретная команда для выполнения

```bash
sudo -u www-data php /var/www/html/nextcloud/occ maintenance:dump --output /backup/nextcloud_structure.sql
```

## 14. Команды для отладки и логирования

### Установка уровня логирования
```bash
# 0 - DEBUG, 1 - INFO, 2 - WARN, 3 - ERROR, 4 - FATAL
sudo -u www-data php /var/www/html/nextcloud/occ config:system:set loglevel --value="0"
```

### Просмотр лога в реальном времени
```bash
sudo tail -f /var/www/html/nextcloud/data/nextcloud.log
```

### Очистка лога
```bash
sudo truncate -s 0 /var/www/html/nextcloud/data/nextcloud.log
```

### Включение отладки для OIDC
```bash
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set user_oidc debug --value="1"
```

## Условные обозначения в командах

| Обозначение | Значение |
|-------------|----------|
| `<username>` | Имя пользователя в Nextcloud (например, `admin`, `kkorablin`) |
| `<groupname>` | Название группы (например, `admin`, `users`) |
| `<app_name>` | Название приложения (например, `user_oidc`, `onlyoffice`) |
| `<identifier>` | Идентификатор провайдера OIDC (например, `Keycloak`) |
| `<client_id>` | Client ID из Keycloak (например, `nextcloud`) |
| `<client_secret>` | Client Secret из Keycloak |
| `<discovery_url>` | Discovery endpoint Keycloak |
| `<claim_name>` | Имя поля в JWT токене для групп (например, `nextcloud-roles`) |

## Важные замечания

1. **Все команды выполняются от пользователя `www-data`** с использованием `sudo`
2. **Occ должен находиться в корневой директории Nextcloud** (`/var/www/html/nextcloud/occ`)
3. **Перед выполнением критических операций** (удаление пользователей, изменение БД) рекомендуется сделать бэкап
4. **После изменения конфигурации** часто требуется перезапуск сервисов:
   ```bash
   sudo systemctl restart php8.3-fpm nginx
   ```

5. **Для отладки проблем** используйте:
   ```bash
   sudo -u www-data php /var/www/html/nextcloud/occ status
   sudo tail -f /var/www/html/nextcloud/data/nextcloud.log
   ```
