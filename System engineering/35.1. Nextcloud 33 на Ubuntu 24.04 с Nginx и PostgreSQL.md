# Установка Nextcloud 33 на Ubuntu 24.04 с Nginx и PostgreSQL + OnlyOffice + SSO Keycloack

## Предисловие

Ubuntu 24.04 (Noble Numbat) использует стабильные версии пакетов. Здесь **PHP 8.3**, а не 8.4.  
Инструкция построена на связке **Nginx + PostgreSQL** – производительной и надёжной.  
В конце добавим **OnlyOffice** для онлайн-редактирования документов прямо в браузере.

## Что потребуется

- Сервер с Ubuntu 24.04 LTS
- Nextcloud сервер: https://download.nextcloud.com/server/releases/
- Домен (или поддомен) для OnlyOffice, например `only-office.runtel.ru`
- (опционально) https://github.com/nextcloud/all-in-one/releases/tag/v12.9.2

## Шаг 5. Полный рабочий конфиг Nginx

См. пример https://docs.nextcloud.com/server/19/admin_manual/installation/nginx.html

Итоговый файл `/etc/nginx/sites-available/nextcloud`:

```nginx
# ============================================================
# Конфигурация Nginx для Nextcloud 33
# Ubuntu 24.04 | PHP 8.3 FPM | PostgreSQL
# ============================================================

upstream php-handler {
    server unix:/var/run/php/php8.3-fpm.sock;
}

# ------------------------------------------------------------
# Перенаправление HTTP на HTTPS
# ------------------------------------------------------------
server {
    listen 80;
    listen [::]:80;
    server_name nextcloud.runtel.ru;
    return 301 https://$server_name$request_uri;
}

# ------------------------------------------------------------
# Основной блок HTTPS
# ------------------------------------------------------------
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    
    server_name nextcloud.runtel.ru;
    root /var/www/html/nextcloud;

    # SSL сертификаты (настройте под себя)
    ssl_certificate /etc/nginx/runtel.pem;
    ssl_certificate_key /etc/nginx/runtel.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # Безопасные заголовки
    add_header Referrer-Policy "no-referrer" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Download-Options "noopen" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Permitted-Cross-Domain-Policies "none" always;
    add_header X-Robots-Tag "none" always;
    add_header X-XSS-Protection "1; mode=block" always;

    fastcgi_hide_header X-Powered-By;
    client_max_body_size 10G;
    fastcgi_buffers 64 4K;

    # Gzip сжатие
    gzip on;
    gzip_vary on;
    gzip_comp_level 4;
    gzip_min_length 256;
    gzip_proxied expired no-cache no-store private no_last_modified no_etag auth;
    gzip_types application/atom+xml application/javascript application/json 
               application/ld+json application/manifest+json application/rss+xml 
               application/vnd.geo+json application/vnd.ms-fontobject 
               application/x-font-ttf application/x-web-app-manifest+json 
               application/xhtml+xml application/xml font/opentype image/bmp 
               image/svg+xml image/x-icon text/cache-manifest text/css text/plain 
               text/vcard text/vnd.rim.location.xloc text/vtt text/x-component 
               text/x-cross-domain-policy;

    # CalDAV / CardDAV
    location = /.well-known/carddav {
        return 301 $scheme://$host/remote.php/dav;
    }
    location = /.well-known/caldav {
        return 301 $scheme://$host/remote.php/dav;
    }

    # Основной location
    location / {
        rewrite ^ /index.php;
    }

    # Запрет доступа к служебным папкам
    location ~ ^\/(?:build|tests|config|lib|3rdparty|templates|data)\/ {
        deny all;
    }
    location ~ ^\/(?:\.|autotest|occ|issue|indie|db_|console) {
        deny all;
    }

    # Обработка PHP
    location ~ ^\/(?:index|remote|public|cron|core\/ajax\/update|status|ocs\/v[12]|updater\/.+|ocs-provider\/.+)(?:\.php)(?:$|\/) {
        fastcgi_split_path_info ^(.+?\.php)(\/.*|)$;
        set $path_info $fastcgi_path_info;
        try_files $fastcgi_script_name =404;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $path_info;
        fastcgi_param HTTPS on;
        fastcgi_param modHeadersAvailable true;
        fastcgi_param front_controller_active true;
        fastcgi_pass php-handler;
        fastcgi_intercept_errors on;
        fastcgi_request_buffering off;
    }

    # Статические файлы с кешированием
    location ~ \.(?:css|js|woff2?|svg|gif|map)$ {
        try_files $uri /index.php$request_uri;
        add_header Cache-Control "public, max-age=15778463";
        access_log off;
    }

    # Изображения и медиа
    location ~ \.(?:png|html|ttf|ico|jpg|jpeg|bcmap|mp4|webm)$ {
        try_files $uri /index.php$request_uri;
        access_log off;
    }
}
```

### 12.4. Настройка Nginx для OnlyOffice

Встроенный Nginx OnlyOffice не используется, и вы должны проксировать запросы через ваш основной Nginx.

Создайте конфиг `/etc/nginx/sites-available/onlyoffice`:

```nginx
server {
    listen 80;
    server_name only-office.runtel.ru;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name only-office.runtel.ru;

    ssl_certificate /etc/nginx/runtel.pem;
    ssl_certificate_key /etc/nginx/runtel.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    # ПРАВИЛО ДЛЯ СТАТИКИ: убираем версию из URL
#    location ~ ^/\d+\.\d+\.\d+-[a-f0-9]+/(.*)$ {
#        proxy_pass http://127.0.0.1:8000/$1;
#        proxy_http_version 1.1;
#        proxy_set_header Host $host;
#        proxy_set_header X-Real-IP $remote_addr;

#        proxy_set_header X-Forwarded-Proto $scheme;
#       proxy_set_header Upgrade $http_upgrade;
#       proxy_set_header Connection "upgrade";
#        expires 30d;
#        add_header Cache-Control "public, immutable";
#    }

    location ~ ^/([\d]+\.[\d]+\.[\d]+[\.|-][\w]+)/(?<path>.*)$ {
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_pass http://127.0.0.1:8000/$path$is_args$args;
        proxy_http_version 1.1;
    }


    # Прокси для всех остальных запросов
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        client_max_body_size 100M;
        proxy_read_timeout 3600s;
        proxy_connect_timeout 3600s;
        proxy_send_timeout 3600s;
        proxy_buffering off;
    }
}
```

Активируйте конфиг:

```bash
sudo ln -sf /etc/nginx/sites-available/onlyoffice /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 12.5. Настройка JWT (безопасность)

***Если установка OnlyOffice прошла с предыдущими шагами верно, то JWT секрет автоматически подставился.*** 
Однако, **узнать его необходимо** для дальнейших шагов ниже. 

#### Узнать JWT секрет (обязательно!)
```bash
# Смотрим сгенерированный JWT секрет
sudo documentserver-jwt-status.sh
```
```
Your JWT settings:
JWT enabled -  true
JWT secret  -  <JWT-secret>
JWT header  -  Authorization
```

#### НЕВЕРОЯТНО ВАЖНО!
Подставить данный JWT токен нужно будет в настройках самого NextCloud для OnlyOffice: https://nextcloud.runtel.ru/settings/admin/onlyoffice


#### Отредактируйте файл `/etc/onlyoffice/documentserver/local.json`:
```bash
sudo mcedit /etc/onlyoffice/documentserver/local.json
```

Найдите секцию `"token"` и приведите к виду:
```json
"token": {
    "enable": {
        "request": {
            "inbox": true,
            "outbox": true
        },
        "browser": true
    },
    "inbox": {
        "header": "Authorization",
        "inBody": false
    },
    "outbox": {
        "header": "Authorization",
        "inBody": false
    },
    "secret": {
        "inbox": {
            "string": "ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
        },
        "outbox": {
            "string": "ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
        },
        "browser": {
            "string": "ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
        }
    }
}
```
Замените `ВАШ_СЕКРЕТНЫЙ_КЛЮЧ` на надёжный пароль (сгенерируйте: `openssl rand -base64 32`).

#### Перезапустите OnlyOffice:
```bash
sudo systemctl restart onlyoffice-documentserver
```

### 12.6. Установка и настройка приложения OnlyOffice в Nextcloud
```bash
# 1. Устанавливаем приложение OnlyOffice (если ещё не установлено)
sudo -u www-data php /var/www/html/nextcloud/occ app:install onlyoffice

# 2. Настраиваем адрес сервера документов
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice DocumentServerUrl --value="https://only-office.runtel.ru/"

# 3. Указываем JWT секрет (тот, что сгенерировался автоматически)
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice jwt_secret --value="<JWT-secret>"

# 4. Отключаем проверку SSL (для самоподписанных сертификатов)
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice verify_peer_off --value="true"

# 5. Проверяем соединение
sudo -u www-data php /var/www/html/nextcloud/occ onlyoffice:check

# 6. Проверяем установку приложения через список
sudo -u www-data php /var/www/html/nextcloud/occ app:list | grep onlyoffice
```

#### Если команда onlyoffice:check не найдена
Возможно, приложение только что установлено и нужно обновить список команд:
```bash
# Очищаем кеш Nextcloud
sudo -u www-data php /var/www/html/nextcloud/occ app:update --all

# Или просто проверяем статус
sudo -u www-data php /var/www/html/nextcloud/occ app:list | grep onlyoffice
```

### 12.8. Проверка портов
ВАЖНО:
- ***`docservice`*** (основной сервис OnlyOffice) слушает на порту 8000. Это Python/Node.js процесс, который обрабатывает запросы на редактирование документов.
- Встроенный Nginx OnlyOffice (ds.conf) настроен на порт 80, но его сервис (nginx-ds) не запущен и даже не существует в systemd. Поэтому ваш внешний Nginx должен проксировать запросы напрямую на docservice:8000.

```bash
ss -tlnp | grep -E ":(80|443|8080|8000|3000)" | grep -v nginx
ss -tlnp | grep -E ":(80|443|8080|8000|3000)"
grep -r "listen" /etc/onlyoffice/documentserver/nginx/

sudo ss -tlnp | grep 8000
# Показал: LISTEN *:8000 - users:(("docservice",pid=4188,fd=24))
```

#### Проверяем напрямую и через Nginx
```bash
sudo nginx -t
sudo systemctl restart nginx
```
```bash
# Прямой доступ к docservice (должен работать)
curl http://127.0.0.1:8000/welcome/

# Доступ через ваш Nginx (должен работать)
curl -k https://only-office.runtel.ru/welcome/

# Смотрим заголовки ответа
curl -I -k https://only-office.runtel.ru/welcome/
```


### 12.9. Установка MIME типов
Скорее всего MIME типы по умолчанию у вас не установлены. Они нужны именно для того, чтобы открывать документы прямо в браузере.
```bash
# Проверяем, что Nextcloud видит OnlyOffice
sudo -u www-data php /var/www/html/nextcloud/occ onlyoffice:documentserver

# Проверяем версию приложения
sudo -u www-data php /var/www/html/nextcloud/occ app:list | grep onlyoffice

# Проверяем настройки OnlyOffice в Nextcloud
sudo -u www-data php /var/www/html/nextcloud/occ config:app:get onlyoffice formats

# Если пусто - устанавливаем форматы
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice formats --value='{
```

## Шаг 13. Устранение распространённых проблем

### 13.1. Ошибка 403 Forbidden

```bash
sudo chmod 755 /var/www/html/nextcloud
sudo chmod 755 /var/www/html/nextcloud/config
sudo systemctl restart nginx php8.3-fpm
```

### 13.2. Ошибка подключения к PostgreSQL

```bash
sudo -u postgres psql -c "ALTER USER nextcloud_user WITH PASSWORD 'новый_пароль';"
```

### 13.3. Циклический редирект (HTTP → HTTP)

Убедитесь, что в конфиге Nginx есть `fastcgi_param HTTPS on;`

### 13.4. Белая страница после установки

```bash
sudo chown -R www-data:www-data /var/www/html/nextcloud
sudo chmod -R 755 /var/www/html/nextcloud
sudo systemctl restart nginx php8.3-fpm
```

### 13.6. OnlyOffice не открывает документы (JWT ошибка)

Убедитесь, что секретный ключ совпадает в `/etc/onlyoffice/documentserver/local.json` и в настройках Nextcloud:

```bash
# Проверяем, что JWT секрет установлен
sudo -u www-data php /var/www/html/nextcloud/occ config:app:get onlyoffice jwt_secret

# Если пусто - устанавливаем
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice jwt_secret --value="vIvmWbYpTx4smJf3sm0thfkw0im7XTvz"

# Проверяем отключение проверки сертификата
sudo -u www-data php /var/www/html/nextcloud/occ config:app:get onlyoffice verify_peer_off

# Если не true - устанавливаем
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice verify_peer_off --value="true"

# Очистка кеша
sudo -u www-data php /var/www/html/nextcloud/occ files:scan --all

# Перезапуск OnlyOffice после изменений
sudo systemctl restart onlyoffice-documentserver
sudo systemctl restart nginx
```

### 13.7. OnlyOffice показывает "Download with invalid jwt"

Добавьте в конфиг Nginx для OnlyOffice дополнительный заголовок:

```nginx
proxy_set_header AuthorizationJwt $http_authorization;
```

И перезапустите Nginx.
