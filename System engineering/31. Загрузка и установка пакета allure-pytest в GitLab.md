# Подробная инструкция: Загрузка и установка пакета allure-pytest в GitLab

## Цель
Загрузить пакет `allure-pytest` в корпоративный GitLab и установить его без доступа к PyPI.

---

## Часть 0. Получение пакета из PyPI

### 0.1. Скачивание пакета с PyPI (без зависимостей)

Перед загрузкой в GitLab, пакет был скачан с официального репозитория PyPI (Python Package Index).

**Источник:** https://pypi.org/project/allure-pytest/

```bash
# Создаем временную директорию для загрузки
mkdir /tmp/packages

# Скачиваем только целевой пакет без зависимостей
pip download allure-pytest==2.15.3 --dest /tmp/packages --no-deps
```

## Часть 1. Подготовка и загрузка пакета в GitLab

### 1.1. Создание Token

Требуется зайти на страницу gitLab вашей организации, перейти по Edit **Profile → Access Token**. 
<br/> Создаём новый и перед сохранением ОБЯЗАТЕЛЬНО нужно временно скопировать его перед дальнейшим добавлением в конфигурацию.

**Необходимые права для токена:**
- `api` (полный доступ к API GitLab)
- `write_repository` (права на запись в репозиторий)

### 1.2. Создание конфигурационного файла `.pypirc`

Создали файл `/home/kkorablin/.pypirc` для аутентификации в GitLab:

```ini
[distutils]
index-servers = gitlab

[gitlab]
## https://gitlab.runtel.org/runtel/runtel_auto_tests/edit - ID: 113
repository = https://gitlab.runtel.org/api/v4/projects/113/packages/pypi
username = __token__
password = glpat-fVfyK2aSL23hnCZaEsSA5W86MQp1OjFqCA.01.0y03yeb0c
```

**Неудачные варианты (которые пробовали, но они не сработали):**
```ini
[distutils]
index-servers = gitlab

[gitlab]
# Неправильный URL - без /api/v4
repository = https://gitlab.runtel.org/packages/pypi

# Попытка загрузить в группу (не поддерживается для загрузки)
repository = https://gitlab.runtel.org/api/v4/groups/10/-/packages/pypi

# Использование обычного username вместо __token__
username = kkorablin
```

**Важно:** 
- Использовали `__token__` в качестве username (стандарт для GitLab)
- Указали правильный `project_id = 113` (проект `runtel_auto_tests`)
- Токен должен иметь права `api` и `write_repository`
- URL должен быть в формате: `https://gitlab.domain/api/v4/projects/{project_id}/packages/pypi`

## Выводы

1. **Источник пакета:** Пакет был скачан с PyPI (https://pypi.org) в виде двух артефактов: `.whl` и `.tar.gz`

7. **Оптимальный подход:** Скачать пакет со всеми зависимостями одной командой, загрузить все пакеты в GitLab, затем устанавливать с флагом `--no-index`
