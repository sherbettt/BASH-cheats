# Подробная инструкция: Загрузка и установка пакета allure-pytest в GitLab

## Цель
Загрузить пакет `allure-pytest` в корпоративный GitLab и установить его без доступа к PyPI.

---

## Часть 0. Получение пакета из PyPI

### 0.1. Скачивание пакета с PyPI

Перед загрузкой в GitLab, пакет был скачан с официального репозитория PyPI (Python Package Index).

**Источник:** https://pypi.org/project/allure-pytest/

```bash
# Создаем временную директорию для загрузки
mkdir /tmp/packages

# Скачиваем пакет из PyPI
pip download allure-pytest==2.15.3 --dest /tmp/packages --no-deps
```

**Результат:**
```
Collecting allure-pytest==2.15.3
  Downloading allure_pytest-2.15.3-py3-none-any.whl (12.2 kB)
  Downloading allure_pytest-2.15.3.tar.gz (17.3 kB)
Saved /tmp/packages/allure_pytest-2.15.3-py3-none-any.whl
Saved /tmp/packages/allure_pytest-2.15.3.tar.gz
Successfully downloaded allure-pytest
```

**Какие артефакты были скачаны:**
1. **allure_pytest-2.15.3-py3-none-any.whl** (12.2 KB) - бинарный wheel-пакет
2. **allure_pytest-2.15.3.tar.gz** (17.3 KB) - исходный код в виде tar-архива

**Почему два файла:**
- `.whl` - готовый к установке бинарный пакет (предпочтительный для установки)
- `.tar.gz` - исходный код (используется для сборки или если wheel недоступен)

**Проверка содержимого:**
```bash
ls -la /tmp/packages/
```

**Результат:**
```
-rw-r--r-- 1 kkorablin kkorablin 18k allure_pytest-2.15.3.tar.gz
-rw-r--r-- 1 kkorablin kkorablin 13k allure_pytest-2.15.3-py3-none-any.whl
```

## Часть 1. Подготовка и загрузка пакета в GitLab

### 1.1. Создание Token

Требуется зайти на страницу gitLab вашей организации, перейти по Edit **Profile → Access Token**. 
<br/> Создаём новый и перед сохранением ОБЯЗАТЕЛЬНО нужно временно скопировать его перед дальнейшим добавлением в конфигурацию.

**Необходимые права для токена:**
- `api` (полный доступ к API GitLab)
- `write_repository` (права на запись в репозиторий)

### 1.2. Создание конфигурационного файла `.pypirc`

Создали файл `/home/kkorablin/.pypirc` для аутентификации в GitLab:

```ini
[distutils]
index-servers = gitlab

[gitlab]
## https://gitlab.runtel.org/runtel/runtel_auto_tests/edit - ID: 113
repository = https://gitlab.runtel.org/api/v4/projects/113/packages/pypi
username = __token__
password = glpat-fVfyK2aSL23hnCZaEsSA5W86MQp1OjFqCA.01.0y03yeb0c
```

**Неудачные варианты (которые пробовали, но они не сработали):**
```ini
[distutils]
index-servers = gitlab

[gitlab]
# Неправильный URL - без /api/v4
repository = https://gitlab.runtel.org/packages/pypi

# Попытка загрузить в группу (не поддерживается для загрузки)
repository = https://gitlab.runtel.org/api/v4/groups/10/-/packages/pypi

# Использование обычного username вместо __token__
username = kkorablin
```

**Важно:** 
- Использовали `__token__` в качестве username (стандарт для GitLab)
- Указали правильный `project_id = 113` (проект `runtel_auto_tests`)
- Токен должен иметь права `api` и `write_repository`
- URL должен быть в формате: `https://gitlab.domain/api/v4/projects/{project_id}/packages/pypi`

Проверили работоспособность токена через API:

### 1.4. Загрузка пакета в GitLab

Загружаем скачанные ранее артефакты в GitLab:

```bash
twine upload --verbose --repository gitlab /tmp/packages/*.whl /tmp/packages/*.tar.gz
```

**Что произошло:**

## Часть 2. Исследование зависимостей пакета

### 2.1. Зачем нужно исследовать зависимости?

При установке `allure-pytest` pip пытается автоматически установить зависимости. Чтобы понять, какие пакеты нужны, мы распаковали архив и изучили его содержимое.

### 2.2. Скачивание пакета из GitLab для анализа

```bash

pip download allure-pytest \
  --index-url https://gitlab.runtel.org/api/v4/projects/113/packages/pypi/simple \
  --dest /tmp/check --no-deps
```

### 2.4. Поиск зависимостей в METADATA

В директории `*.dist-info` находится файл `METADATA`, который содержит информацию о зависимостях.


**Результат:**

✅ **Узнали зависимости:**
- `pytest` версии 4.5.0 или выше
- `allure-python-commons` точной версии 2.15.3

## Дальнейшие шаги

Чтобы установить `allure-pytest` полностью без доступа к PyPI, нужно:

### Вариант 1: Загрузить зависимости в GitLab

```bash
# 1. Скачать зависимости из PyPI
pip download pytest allure-python-commons==2.15.3 \
  --dest /tmp/deps

# 2. Загрузить в GitLab
twine upload --repository gitlab /tmp/deps/*.whl

# 3. Установить всё вместе
pip install allure-pytest \
  --no-index \
  --index-url https://gitlab.runtel.org/api/v4/projects/113/packages/pypi/simple
```

### Вариант 2: Использовать локальный репозиторий

```bash
# Скачать все пакеты с зависимостями
pip download allure-pytest pytest allure-python-commons \
  --dest /tmp/full_repo

# Установить из локальной папки
pip install --no-index --find-links /tmp/full_repo allure-pytest
```
