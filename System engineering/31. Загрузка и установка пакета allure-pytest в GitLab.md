# Подробная инструкция: Загрузка и установка пакета allure-pytest в GitLab

## Цель
Загрузить пакет `allure-pytest` в корпоративный GitLab и установить его без доступа к PyPI.

---

## Часть 1. Подготовка и загрузка пакета в GitLab

### 1.1. Создание конфигурационного файла `.pypirc`

Создали файл `/home/kkorablin/.pypirc` для аутентификации в GitLab:

```ini
[distutils]
index-servers = gitlab

[gitlab]
repository = https://gitlab.runtel.org/api/v4/projects/113/packages/pypi
username = __token__
password = glpat-fVfyK2aSL23hnCZaEsSA5W86MQp1OjFqCA.01.0y03yeb0c
```

**Важно:** 
- Использовали `__token__` в качестве username (стандарт для GitLab)
- Указали правильный `project_id = 113` (проект `runtel_auto_tests`)
- Токен должен иметь права `api` и `write_repository`

---

### 1.2. Проверка токена

Проверили работоспособность токена через API:

```bash
curl --header "PRIVATE-TOKEN: glpat-fVfyK2aSL23hnCZaEsSA5W86MQp1OjFqCA.01.0y03yeb0c" \
  "https://gitlab.runtel.org/api/v4/projects/113"
```

**Результат:** успешный ответ с данными проекта, токен работает.

### 1.3. Загрузка пакета в GitLab

## Часть 2. Исследование зависимостей пакета

### 2.1. Зачем нужно исследовать зависимости?

При установке `allure-pytest` pip пытается автоматически установить зависимости. Чтобы понять, какие пакеты нужны, мы распаковали архив и изучили его содержимое.

### 2.2. Распаковка .whl файла

```bash
# Скачали пакет в временную директорию
pip download allure-pytest \
  --index-url https://gitlab.runtel.org/api/v4/projects/113/packages/pypi/simple \
  --dest /tmp/check --no-deps

# Перешли в директорию
cd /tmp/check

# Распаковали .whl файл (это обычный zip-архив)
unzip allure_pytest-2.15.3-py3-none-any.whl
```

**Результат распаковки:**
```
Archive:  allure_pytest-2.15.3-py3-none-any.whl
  inflating: allure_pytest/__init__.py  
  inflating: allure_pytest/compat.py  
  inflating: allure_pytest/helper.py  
  inflating: allure_pytest/listener.py  
  inflating: allure_pytest/plugin.py  
  inflating: allure_pytest/stash.py  
  inflating: allure_pytest/utils.py  
  inflating: allure_pytest-2.15.3.dist-info/METADATA  
  inflating: allure_pytest-2.15.3.dist-info/WHEEL  
  inflating: allure_pytest-2.15.3.dist-info/entry_points.txt  
  inflating: allure_pytest-2.15.3.dist-info/top_level.txt  
  inflating: allure_pytest-2.15.3.dist-info/RECORD  
```

**Что мы узнали:** В .whl файле содержится директория `*.dist-info` с метаданными пакета.

---

### 2.3. Поиск зависимостей в METADATA

В директории `*.dist-info` находится файл `METADATA`, который содержит информацию о зависимостях.

```bash
# Посмотрели содержимое METADATA
cat allure_pytest-2.15.3.dist-info/METADATA | grep -E "Requires|Depends"
```

**Результат:**
```
Requires-Dist: pytest>=4.5.0
Requires-Dist: allure-python-commons==2.15.3
```

✅ **Узнали зависимости:**
- `pytest` версии 4.5.0 или выше
- `allure-python-commons` точной версии 2.15.3

---

### 2.4. Альтернативный способ: через pip show

После установки пакета зависимости можно посмотреть командой:

```bash
pip install /tmp/check/allure_pytest-2.15.3-py3-none-any.whl --no-deps
pip show allure-pytest
```

**Результат:**
```
Name: allure-pytest
Version: 2.15.3
Requires: allure-python-commons, pytest
```

## Дальнейшие шаги

Чтобы установить `allure-pytest` полностью без доступа к PyPI, нужно:

### Вариант 1: Загрузить зависимости в GitLab

```bash
# 1. Скачать зависимости
pip download pytest allure-python-commons==2.15.3 \
  --dest /tmp/deps

# 2. Загрузить в GitLab
twine upload --repository gitlab /tmp/deps/*.whl

# 3. Установить всё вместе
pip install allure-pytest \
  --no-index \
  --index-url https://gitlab.runtel.org/api/v4/projects/113/packages/pypi/simple
```

### Вариант 2: Использовать локальный репозиторий

```bash
# Скачать все пакеты с зависимостями
pip download allure-pytest pytest allure-python-commons \
  --dest /tmp/full_repo

# Установить из локальной папки
pip install --no-index --find-links /tmp/full_repo allure-pytest
```

## Выводы

1. **Для исследования зависимостей пакета** нужно распаковать `.whl` файл и посмотреть `METADATA` в директории `*.dist-info`
2. **Загрузка в GitLab** работает только через endpoint с `project_id`, не через группу
3. **Установка без зависимостей** возможна с флагом `--no-deps`, но пакет не будет работать
4. **Для полноценной установки** нужно загрузить все зависимости в GitLab или использовать локальный репозиторий
