## Полная инструкция установки Nextcloud на ALT Linux Server 11

Основная статья https://www.altlinux.org/Nextcloud

<br/>



### Часть 1. Установка Nextcloud через deploy

```bash
# Устанавливаем deploy
apt-get install deploy

# Запускаем установку Nextcloud (всё сделает автоматически)
deploy nextcloud
```

После выполнения будет сообщение: `Deploy complete successful.`

------------------------------------

### Часть 2. Настройка MariaDB (база данных)

#### 2.1. Узнаём пароль root MariaDB

При установке через deploy пароль root MariaDB не выводится. Если нужно сбросить:

```bash
# Останавливаем MariaDB
systemctl stop mariadb

# Запускаем в безопасном режиме
mysqld_safe --skip-grant-tables --skip-networking &

# Подключаемся без пароля
mysql -u root
```

**В MySQL выполни:**

```sql
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED BY 'НовыйПароль123';
EXIT;
```

```bash
# Завершаем процесс и запускаем нормально
pkill mysqld
systemctl start mariadb

# Проверяем вход
mysql -u root -p
```

#### 2.2. Создание пользователя и базы данных (если нужно)

```bash
mysql -u root -p
```

```sql
-- Создать БД
CREATE DATABASE mydb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Создать пользователя
CREATE USER 'myuser'@'localhost' IDENTIFIED BY 'mypassword';

-- Дать права
GRANT ALL PRIVILEGES ON mydb.* TO 'myuser'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

#### 2.3. Полезные команды MariaDB

```sql
-- Посмотреть все базы данных
SHOW DATABASES;

-- Посмотреть всех пользователей
SELECT host, user FROM mysql.user;

-- Посмотреть права пользователя
SHOW GRANTS FOR 'myuser'@'localhost';

-- Удалить пользователя
DROP USER 'myuser'@'localhost';

-- Удалить базу данных
DROP DATABASE mydb;
```

### Часть 3. Настройка Nextcloud

#### 3.1. Добавление доверенных доменов

```bash
mcedit /var/www/webapps/nextcloud/config/config.php
```

**Найди строку `'trusted_domains'` и приведи к виду:**

```php
'trusted_domains' => 
array (
  0 => '192.168.87.160',
  1 => 'localhost',
  2 => 'nextcloud.runtel.ru',
),
```

#### 3.2. Пароль администратора Nextcloud

Пароль для пользователя `ncadmin` можно узнать командой:

```bash
journalctl -b | grep admin-pass
```

Вывод будет примерно такой:
```
--admin-pass 'yah3ahShua3eeM5d'
```

**Логин:** `ncadmin`  
**Пароль:** `yah3ahShua3eeM5d` (или тот, что в твоём логе)

#### 3.3. Сброс пароля администратора (если забыл)

```bash
cd /var/www/webapps/nextcloud
php occ user:resetpassword ncadmin
```

Введи новый пароль дважды.

#### 3.4. Создание нового администратора

```bash
cd /var/www/webapps/nextcloud
php occ user:add --display-name="Администратор" --group="admin" admin
```

Введи пароль для нового пользователя.

### Часть 4. Настройка Apache

#### 4.1. Проверка конфига Nextcloud

```bash
ccat /etc/httpd2/conf/sites-available/nextcloud.conf
```

Должно быть:

```apache
Alias /nextcloud "/var/www/webapps/nextcloud/"

<Directory "/var/www/webapps/nextcloud/">
    Require all granted
    Options FollowSymlinks MultiViews
    AllowOverride All
    <IfModule mod_dav.c>
        Dav off
    </IfModule>
    SetEnv HOME /var/www/webapps/nextcloud
    SetEnv HTTP_HOME /var/www/webapps/nextcloud
</Directory>
```

#### 4.2. Включение сайта и перезапуск

```bash
a2ensite nextcloud.conf
systemctl restart httpd2
```

### Часть 6. Полезные команды для управления

```bash
# Статус сервисов
systemctl status httpd2 mariadb

# Перезапуск Apache
systemctl restart httpd2

# Перезапуск MariaDB
systemctl restart mariadb

# Логи Apache
tail -f /var/log/httpd2/error_log

# Логи Nextcloud
tail -f /var/www/webapps/nextcloud/data/nextcloud.log

# Очистка кеша Nextcloud
cd /var/www/webapps/nextcloud
php occ maintenance:repair

# Обновление приложений
php occ app:update --all
```

### Часть 7. Устранение типичных ошибок

#### Ошибка: "Following symlinks is not allowed"

```bash
# Удаляем символическую ссылку и создаём реальную папку
rm -f /var/www/webapps/nextcloud/config
mkdir -p /var/www/webapps/nextcloud/config
cp -r /etc/nextcloud/config/* /var/www/webapps/nextcloud/config/
chown -R apache2:apache2 /var/www/webapps/nextcloud/config
```

#### Ошибка: "Temporary directory not writable"

```bash
mkdir -p /tmp/.private/root
chmod 700 /tmp/.private/root
```

#### Ошибка: "Access denied for user 'root'@'localhost'"

Сбрось пароль MariaDB по инструкции из Части 2.1.
