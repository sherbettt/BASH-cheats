# Установка и настройка ONLYOFFICE Docs для OpenCloud

## 1. Подготовка сервера

Убедитесь, что сервер обновлён и установлены базовые зависимости:

```bash
apt update && apt upgrade -y
apt install -y curl wget gnupg2 postgresql redis-server nginx-extras
```

## 2. Настройка PostgreSQL

Создайте базу данных и пользователя для ONLYOFFICE:

```bash
sudo -u postgres psql -c "CREATE USER onlyoffice WITH PASSWORD 'onlyoffice';"
sudo -u postgres psql -c "CREATE DATABASE onlyoffice OWNER onlyoffice;"
```

> *Вы можете изменить пароль `onlyoffice` на более надёжный, но его нужно запомнить.*

## 3. Установка ONLYOFFICE Docs

Добавьте официальный репозиторий:

```bash
mkdir -p -m 700 ~/.gnupg
curl -fsSL https://download.onlyoffice.com/GPG-KEY-ONLYOFFICE | gpg --no-default-keyring --keyring gnupg-ring:/tmp/onlyoffice.gpg --import
chmod 644 /tmp/onlyoffice.gpg
chown root:root /tmp/onlyoffice.gpg
mv /tmp/onlyoffice.gpg /usr/share/keyrings/onlyoffice.gpg

echo "deb [signed-by=/usr/share/keyrings/onlyoffice.gpg] https://download.onlyoffice.com/repo/debian squeeze main" | tee /etc/apt/sources.list.d/onlyoffice.list
apt update
```

Установите сам пакет:

```bash
apt install -y onlyoffice-documentserver
```

**Во время установки:**

- Будет задан вопрос о пароле PostgreSQL для пользователя `onlyoffice`. Введите тот же пароль, что вы задали в п. 2.
- Если пароль не спросили (что бывает редко), то он уже прописан в конфигах `/etc/onlyoffice/documentserver/local.json`.

После успешной установки вы увидите сообщение:

```
Congratulations, the ONLYOFFICE documentserver has been installed successfully!
JWT is enabled by default. A random secret is generated automatically.
```

## 4. Получение JWT-секрета (обязательно для интеграции)

Секрет нужен, чтобы OpenCloud и ONLYOFFICE могли безопасно обмениваться данными.

```bash
/usr/bin/documentserver-jwt-status.sh
```

Пример вывода:
```
Your JWT settings:
JWT enabled -  true
JWT secret  -  ompzEBlSyxLavWydAvQHibBO6DAyCVfV
JWT header  -  Authorization
```

**Сохраните этот секрет!** Он понадобится при настройке OpenCloud.

Если скрипт не сработал, посмотрите секрет вручную:

```bash
grep -i "secret" /etc/onlyoffice/documentserver/local.json | head -1
```

## 5. Настройка проксирования через Nginx

Создайте конфигурационный файл `/etc/nginx/sites-available/office`:

```nginx
server {
    listen 443 ssl;
    server_name office.runtel.ru;

    ssl_certificate /etc/nginx/ssl/oc.runtel.ru.crt;
    ssl_certificate_key /etc/nginx/ssl/oc.runtel.ru.key;

    location / {
        proxy_pass http://127.0.0.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

Активируйте сайт:

```bash
ln -s /etc/nginx/sites-available/office /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

## 6. Настройка OpenCloud для работы с ONLYOFFICE

### 6.1. Включите службу `collaboration` (мост WOPI)

Создайте systemd‑юнит `/etc/systemd/system/opencloud-collaboration.service`:

```ini
[Unit]
Description=OpenCloud Collaboration (ONLYOFFICE) Service
After=network.target opencloud.service

[Service]
Type=simple
User=opencloud
Group=opencloud
Environment="HOME=/home/opencloud"
Environment="OC_INSECURE=true"
Environment="OC_URL=https://oc.runtel.ru"
ExecStart=/usr/local/bin/opencloud collaboration server
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Запустите и включите автозапуск:

```bash
systemctl daemon-reload
systemctl enable --now opencloud-collaboration.service
```

### 6.2. Добавьте секцию `collaboration` в конфиг OpenCloud

Отредактируйте `/home/opencloud/.opencloud/config/opencloud.yaml` и добавьте в конец:

```yaml
collaboration:
  wopi:
    secret: "TuFVcNHa2YJzBciUh6lXqIbFAnTM5HpR"   # ваш JWT-секрет
  app:
    product: "OnlyOffice"
    address: "https://office.runtel.ru"
    insecure: true
```

> **Важно:** замените секрет на тот, который вы получили на шаге 4.

### 6.3. Перезапустите OpenCloud

```bash
systemctl restart opencloud.service
```

## 7. Проверка работы

- Загрузите в OpenCloud любой файл `.docx`, `.xlsx` или `.pptx`.
- Нажмите на него. Он должен открыться во встроенном редакторе ONLYOFFICE.
- Проверьте создание новых документов через меню «Создать» (`+`).

## 8. Диагностика при ошибках

- Логи ONLYOFFICE:
  ```bash
  journalctl -u onlyoffice-documentserver -f
  tail -f /var/log/onlyoffice/documentserver/converter/out.log
  ```

- Проверка WOPI‑эндпоинта OpenCloud:
  ```bash
  curl -k https://oc.runtel.ru/wopi/files/
  ```

- Проверка, что веб‑сокеты работают (в консоли браузера при открытии документа не должно быть ошибок WebSocket).
