# Установка Nextcloud 33 на Ubuntu 24.04 с Nginx и PostgreSQL + OnlyOffice

## Предисловие

Ubuntu 24.04 (Noble Numbat) использует стабильные версии пакетов. Здесь **PHP 8.3**, а не 8.4.  
Инструкция построена на связке **Nginx + PostgreSQL** – производительной и надёжной.  
В конце добавим **OnlyOffice** для онлайн-редактирования документов прямо в браузере.

## Что потребуется

- Сервер с Ubuntu 24.04 LTS
- Nextcloud сервер: https://download.nextcloud.com/server/releases/
- Домен (или поддомен) для OnlyOffice, например `only-office.runtel.ru`
- (опционально) https://github.com/nextcloud/all-in-one/releases/tag/v12.9.2

## Шаг 1. Установка необходимых пакетов

```bash
sudo apt update
sudo apt upgrade -y
sudo apt install -y nginx postgresql postgresql-contrib \
  php8.3-fpm php8.3-pgsql \
  php8.3-{zip,xml,gd,mbstring,curl,intl,bcmath,redis,imagick} \
  unzip wget certbot python3-certbot-nginx
```

## Шаг 2. Настройка PHP

```bash
sudo sed -i 's/upload_max_filesize = 2M/upload_max_filesize = 10G/' /etc/php/8.3/fpm/php.ini
sudo sed -i 's/post_max_size = 8M/post_max_size = 10G/' /etc/php/8.3/fpm/php.ini
sudo sed -i 's/memory_limit = 128M/memory_limit = 512M/' /etc/php/8.3/fpm/php.ini
sudo sed -i 's/;date.timezone =/date.timezone = UTC/' /etc/php/8.3/fpm/php.ini

sudo systemctl restart php8.3-fpm
```

## Шаг 3. Скачивание и распаковка Nextcloud

```bash
cd /tmp
wget https://download.nextcloud.com/server/releases/latest.zip
unzip latest.zip
sudo mv nextcloud /var/www/html/
sudo chown -R www-data:www-data /var/www/html/nextcloud
```

## Шаг 4. Настройка базы данных PostgreSQL

```bash
sudo -u postgres psql
```

```sql
CREATE DATABASE nextcloud;
CREATE USER nextcloud_user WITH PASSWORD 'ваш_пароль';
GRANT ALL PRIVILEGES ON DATABASE nextcloud TO nextcloud_user;
ALTER DATABASE nextcloud OWNER TO nextcloud_user;
\q
```

## Шаг 5. Полный рабочий конфиг Nginx

См. пример https://docs.nextcloud.com/server/19/admin_manual/installation/nginx.html

Итоговый файл `/etc/nginx/sites-available/nextcloud`:

```nginx
# ============================================================
# Конфигурация Nginx для Nextcloud 33
# Ubuntu 24.04 | PHP 8.3 FPM | PostgreSQL
# ============================================================

upstream php-handler {
    server unix:/var/run/php/php8.3-fpm.sock;
}

# ------------------------------------------------------------
# Перенаправление HTTP на HTTPS
# ------------------------------------------------------------
server {
    listen 80;
    listen [::]:80;
    server_name nextcloud.runtel.ru;
    return 301 https://$server_name$request_uri;
}

# ------------------------------------------------------------
# Основной блок HTTPS
# ------------------------------------------------------------
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    
    server_name nextcloud.runtel.ru;
    root /var/www/html/nextcloud;

    # SSL сертификаты (настройте под себя)
    ssl_certificate /etc/nginx/runtel.pem;
    ssl_certificate_key /etc/nginx/runtel.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # Безопасные заголовки
    add_header Referrer-Policy "no-referrer" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Download-Options "noopen" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Permitted-Cross-Domain-Policies "none" always;
    add_header X-Robots-Tag "none" always;
    add_header X-XSS-Protection "1; mode=block" always;

    fastcgi_hide_header X-Powered-By;
    client_max_body_size 10G;
    fastcgi_buffers 64 4K;

    # Gzip сжатие
    gzip on;
    gzip_vary on;
    gzip_comp_level 4;
    gzip_min_length 256;
    gzip_proxied expired no-cache no-store private no_last_modified no_etag auth;
    gzip_types application/atom+xml application/javascript application/json 
               application/ld+json application/manifest+json application/rss+xml 
               application/vnd.geo+json application/vnd.ms-fontobject 
               application/x-font-ttf application/x-web-app-manifest+json 
               application/xhtml+xml application/xml font/opentype image/bmp 
               image/svg+xml image/x-icon text/cache-manifest text/css text/plain 
               text/vcard text/vnd.rim.location.xloc text/vtt text/x-component 
               text/x-cross-domain-policy;

    # CalDAV / CardDAV
    location = /.well-known/carddav {
        return 301 $scheme://$host/remote.php/dav;
    }
    location = /.well-known/caldav {
        return 301 $scheme://$host/remote.php/dav;
    }

    # Основной location
    location / {
        rewrite ^ /index.php;
    }

    # Запрет доступа к служебным папкам
    location ~ ^\/(?:build|tests|config|lib|3rdparty|templates|data)\/ {
        deny all;
    }
    location ~ ^\/(?:\.|autotest|occ|issue|indie|db_|console) {
        deny all;
    }

    # Обработка PHP
    location ~ ^\/(?:index|remote|public|cron|core\/ajax\/update|status|ocs\/v[12]|updater\/.+|ocs-provider\/.+)(?:\.php)(?:$|\/) {
        fastcgi_split_path_info ^(.+?\.php)(\/.*|)$;
        set $path_info $fastcgi_path_info;
        try_files $fastcgi_script_name =404;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $path_info;
        fastcgi_param HTTPS on;
        fastcgi_param modHeadersAvailable true;
        fastcgi_param front_controller_active true;
        fastcgi_pass php-handler;
        fastcgi_intercept_errors on;
        fastcgi_request_buffering off;
    }

    # Статические файлы с кешированием
    location ~ \.(?:css|js|woff2?|svg|gif|map)$ {
        try_files $uri /index.php$request_uri;
        add_header Cache-Control "public, max-age=15778463";
        access_log off;
    }

    # Изображения и медиа
    location ~ \.(?:png|html|ttf|ico|jpg|jpeg|bcmap|mp4|webm)$ {
        try_files $uri /index.php$request_uri;
        access_log off;
    }
}
```

## Шаг 6. Активация конфигурации

```bash
sudo ln -sf /etc/nginx/sites-available/nextcloud /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx php8.3-fpm
```

## Шаг 7. Настройка сертификатов Let's Encrypt

```bash
sudo certbot --nginx -d nextcloud.runtel.ru
```

Для тестового окружения можно использовать самоподписанный сертификат:

```bash
sudo mkdir -p /etc/nginx/ssl
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/nextcloud.key \
  -out /etc/nginx/ssl/nextcloud.crt \
  -subj "/CN=nextcloud.runtel.ru"
```

## Шаг 8. Настройка прав для Nextcloud

```bash
cd /var/www/html/nextcloud
sudo chown -R www-data:www-data .
sudo find . -type d -exec chmod 755 {} \;
sudo find . -type f -exec chmod 644 {} \;
sudo chmod +x occ
sudo chmod 750 config
```

## Шаг 9. Настройка cron

```bash
echo '*/5 * * * * www-data php -f /var/www/html/nextcloud/cron.php' | sudo tee /etc/cron.d/nextcloud
sudo chmod 644 /etc/cron.d/nextcloud
```

## Шаг 12. Установка и настройка OnlyOffice для онлайн-редактирования

Этот шаг позволит открывать и редактировать документы (DOCX, XLSX, PPTX) прямо в браузере, прямо как в Google Docs.

### 12.1. Установка зависимостей для OnlyOffice

```bash
sudo apt update
sudo apt install -y postgresql rabbitmq-server
```

### 12.2.1 Создание базы данных для OnlyOffice

```
sudo -u postgres psql
```
```sql
CREATE USER onlyoffice WITH PASSWORD 'OfficePass99rntl';
CREATE DATABASE onlyoffice OWNER onlyoffice;
GRANT ALL PRIVILEGES ON DATABASE onlyoffice TO onlyoffice;
ALTER DATABASE onlyoffice OWNER TO onlyoffice;
```
#### Если были созданы пользователи ранее неправлиьно, то удаляем
```sql
-- 2. Удаляем старых пользователей и базу, если они есть
DROP DATABASE IF EXISTS onlyoffice;
DROP USER IF EXISTS onlyoffice;
DROP USER IF EXISTS onlyoffice_user;

-- 3. Создаём пользователя (именно 'onlyoffice')
CREATE USER onlyoffice WITH PASSWORD 'OfficePass99rntl';

-- 4. Создаём базу данных (именно 'onlyoffice') и назначаем владельца
CREATE DATABASE onlyoffice OWNER onlyoffice;

-- 5. Даём все привилегии на базу данных новому пользователю
GRANT ALL PRIVILEGES ON DATABASE onlyoffice TO onlyoffice;

-- 6. Выходим из psql
\q
```

или
```bash
sudo -u postgres psql << EOF
CREATE USER onlyoffice WITH PASSWORD 'OfficePass99rntl';
CREATE DATABASE onlyoffice OWNER onlyoffice;
GRANT ALL PRIVILEGES ON DATABASE onlyoffice TO onlyoffice;
ALTER DATABASE onlyoffice OWNER TO onlyoffice;
EOF
```
иди ещё можно так
```bash
# Создаём пользователя и базу данных
sudo -i -u postgres psql -c "CREATE USER onlyoffice WITH PASSWORD 'strong_password_here';"
sudo -i -u postgres psql -c "CREATE DATABASE onlyoffice OWNER onlyoffice;"
sudo -i -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE onlyoffice TO onlyoffice;"
sudo -i -u postgres psql -c "ALTER DATABASE onlyoffice OWNER TO onlyoffice;"

# Проверяем создание
sudo -i -u postgres psql -c "\l" | grep onlyoffice
```


### 12.2.2 Добавляем правила аутентификации в pg_hba.conf
```bash
sudo tee -a /etc/postgresql/16/main/pg_hba.conf << 'EOF'
# OnlyOffice specific rule (allow password auth)
host    onlyoffice     onlyoffice     127.0.0.1/32            md5
host    onlyoffice     onlyoffice     ::1/128                 md5
EOF
```
#### Перезапускаем PostgreSQL
```bash
sudo systemctl restart postgresql
```

### 12.2.3 Создаём файл .pgpass для автоматической авторизации
```bash
echo "127.0.0.1:5432:onlyoffice:onlyoffice:OfficePass99rntl" | sudo tee /root/.pgpass
sudo chmod 600 /root/.pgpass
```

#### Проверяем подключение (должно работать без запроса пароля)
```bash
psql -h 127.0.0.1 -U onlyoffice -d onlyoffice -c "SELECT 1"
```

### 12.3. Добавление репозитория и установка OnlyOffice Document Server

```bash
# Добавляем GPG ключ
curl -fsSL https://download.onlyoffice.com/GPG-KEY-ONLYOFFICE | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/onlyoffice.gpg

# Добавляем репозиторий
echo "deb [signed-by=/etc/apt/trusted.gpg.d/onlyoffice.gpg] https://download.onlyoffice.com/repo/debian squeeze main" | sudo tee /etc/apt/sources.list.d/onlyoffice.list
```

### 12.3.1. Прямяая устанвока из пакета
```
# Обновляем и устанавливаем
sudo apt update
sudo apt install -y onlyoffice-documentserver
```

**В процессе установки:**
- Введите пароль для пользователя `onlyoffice` PostgreSQL (тот, что задали выше)
- Примите лицензию `ttf-mscorefonts-installer` (клавиша Tab → OK → Enter)
- Примите EULA OnlyOffice (выберите Yes)

### 12.3.2. Установка с предустанвокой в debconf-set-selections
```bash
# 1. Добавляем репозиторий OnlyOffice (уже есть, но проверим)
curl -fsSL https://download.onlyoffice.com/GPG-KEY-ONLYOFFICE | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/onlyoffice.gpg
echo "deb [signed-by=/etc/apt/trusted.gpg.d/onlyoffice.gpg] https://download.onlyoffice.com/repo/debian squeeze main" | sudo tee /etc/apt/sources.list.d/onlyoffice.list

# 2. Предустанавливаем пароль в debconf (это ВАЖНО!)
echo "onlyoffice-documentserver onlyoffice/db/password password OfficePass99rntl" | sudo debconf-set-selections
echo "onlyoffice-documentserver onlyoffice/db/user string onlyoffice" | sudo debconf-set-selections
echo "onlyoffice-documentserver onlyoffice/db/dbname string onlyoffice" | sudo debconf-set-selections
echo "onlyoffice-documentserver onlyoffice/db/host string 127.0.0.1" | sudo debconf-set-selections
echo "onlyoffice-documentserver onlyoffice/db/port string 5432" | sudo debconf-set-selections

# 3. Обновляем список пакетов
sudo apt update

# 4. Устанавливаем OnlyOffice
sudo apt install -y onlyoffice-documentserver
```

### 12.4. Настройка Nginx для OnlyOffice

Создайте конфиг `/etc/nginx/sites-available/onlyoffice`:

```nginx
server {
    listen 80;
    server_name only-office.runtel.ru;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name only-office.runtel.ru;

    ssl_certificate /etc/nginx/runtel.pem;
    ssl_certificate_key /etc/nginx/runtel.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    location / {
        proxy_pass http://127.0.0.1:80;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        client_max_body_size 100M;
        proxy_read_timeout 3600s;
    }
}
```

Активируйте конфиг:

```bash
sudo ln -sf /etc/nginx/sites-available/onlyoffice /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 12.5. Настройка JWT (безопасность)

Отредактируйте файл `/etc/onlyoffice/documentserver/local.json`:

```bash
sudo nano /etc/onlyoffice/documentserver/local.json
```

Найдите секцию `"token"` и приведите к виду:

```json
"token": {
    "enable": {
        "request": {
            "inbox": true,
            "outbox": true
        },
        "browser": true
    },
    "inbox": {
        "header": "Authorization",
        "inBody": false
    },
    "outbox": {
        "header": "Authorization",
        "inBody": false
    },
    "secret": {
        "inbox": {
            "string": "ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
        },
        "outbox": {
            "string": "ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
        },
        "browser": {
            "string": "ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
        }
    }
}
```

Замените `ВАШ_СЕКРЕТНЫЙ_КЛЮЧ` на надёжный пароль (сгенерируйте: `openssl rand -base64 32`).

Перезапустите OnlyOffice:

```bash
sudo systemctl restart onlyoffice-documentserver
```

### 12.6. Установка и настройка приложения OnlyOffice в Nextcloud

```bash
# Устанавливаем приложение (если ещё не установлено)
sudo -u www-data php /var/www/html/nextcloud/occ app:install onlyoffice

# Настраиваем подключение к серверу документов
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice DocumentServerUrl --value="https://only-office.runtel.ru/"

# Указываем внутренний адрес (если всё на одном сервере)
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice DocumentServerInternalUrl --value="http://127.0.0.1/"

# Устанавливаем JWT секрет (тот же, что в local.json)
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice jwt_secret --value="ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"

# Отключаем проверку SSL для внутренних запросов (если сертификаты самоподписанные)
sudo -u www-data php /var/www/html/nextcloud/occ config:app:set onlyoffice verify_peer_off --value="true"
```

### 12.7. Проверка работы OnlyOffice

```bash
# Проверка соединения из Nextcloud
sudo -u www-data php /var/www/html/nextcloud/occ onlyoffice:check

# Откройте в браузере проверочную страницу OnlyOffice
# https://only-office.runtel.ru/welcome/
```

Теперь зайдите в Nextcloud, создайте новый документ Word, Excel или PowerPoint — он должен открыться в редакторе прямо в браузере.

## Шаг 13. Устранение распространённых проблем

### 13.1. Ошибка 403 Forbidden

```bash
sudo chmod 755 /var/www/html/nextcloud
sudo chmod 755 /var/www/html/nextcloud/config
sudo systemctl restart nginx php8.3-fpm
```

### 13.2. Ошибка подключения к PostgreSQL

```bash
sudo -u postgres psql -c "ALTER USER nextcloud_user WITH PASSWORD 'новый_пароль';"
```

### 13.3. Циклический редирект (HTTP → HTTP)

Убедитесь, что в конфиге Nginx есть `fastcgi_param HTTPS on;`

### 13.4. Белая страница после установки

```bash
sudo chown -R www-data:www-data /var/www/html/nextcloud
sudo chmod -R 755 /var/www/html/nextcloud
sudo systemctl restart nginx php8.3-fpm
```

### 13.5. Убрать предупреждение о pq.so

```bash
sudo rm -f /etc/php/8.3/mods-available/pq.ini 2>/dev/null
sudo systemctl restart php8.3-fpm
```

### 13.6. OnlyOffice не открывает документы (JWT ошибка)

Убедитесь, что секретный ключ совпадает в `/etc/onlyoffice/documentserver/local.json` и в настройках Nextcloud:

```bash
# Проверка секрета в Nextcloud
sudo -u www-data php /var/www/html/nextcloud/occ config:app:get onlyoffice jwt_secret

# Перезапуск OnlyOffice после изменений
sudo systemctl restart onlyoffice-documentserver
sudo systemctl restart nginx
```

### 13.7. OnlyOffice показывает "Download with invalid jwt"

Добавьте в конфиг Nginx для OnlyOffice дополнительный заголовок:

```nginx
proxy_set_header AuthorizationJwt $http_authorization;
```

И перезапустите Nginx.
