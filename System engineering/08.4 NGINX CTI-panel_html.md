## Часть 1: Конфиг Nginx (`registry.runtel.ru`) с комментариями в коде

```nginx
# ============================================================
# БЛОК 1: HTTP → HTTPS редирект
# ============================================================
server {
    server_name registry.runtel.org;          # Домен, на который реагируем
    listen 80;                                # Слушаем порт 80 (обычный HTTP)
    return 301 https://registry.runtel.org$request_uri;
    # Постоянный редирект (код 301) на тот же URL, но по HTTPS
    # $request_uri — сохраняет полный путь (например, /cti/ или /api/cti/)
}

# ============================================================
# БЛОК 2: ОСНОВНОЙ HTTPS-СЕРВЕР
# ============================================================
server {
    server_name registry.runtel.org;          # Домен
    listen 443 ssl;                           # Слушаем порт 443 (HTTPS)
    charset utf-8;                            # Кодировка UTF-8
    client_max_body_size 5M;                  # Максимальный размер тела запроса (для загрузки)
    
    # Пути к логам
    access_log /var/log/nginx/registry.runtel.org-access.log;
    error_log /var/log/nginx/registry.runtel.org-error.log;
    
    # SSL-сертификаты
    ssl_certificate /etc/nginx/ssl/runtelorg.pem;
    ssl_certificate_key /etc/nginx/ssl/runtelorg.pem;

    # ==========================================================
    # БЛОК 3: API-ПРОКСИ (возвращает JSON)
    # ==========================================================
    location /api/cti/ {
        limit_except GET HEAD { deny all; }
        # Разрешаем только GET и HEAD (запрещаем POST, PUT, DELETE)

        # Проксируем на внутренний GitLab API
        # ВАЖНО: теперь без /310/package_files — универсально!
        proxy_pass https://192.168.87.10/api/v4/projects/86/packages/;
        # Если клиент запросит /api/cti/310/package_files,
        # Nginx проксирует на .../packages/310/package_files
        
        # Передаём токен для авторизации в GitLab
        proxy_set_header Authorization "Bearer glpat-uyy-DXxy2iXh5eFZxWyB";
        
        # Подменяем Host, чтобы GitLab "думал", что запрос пришёл на его имя
        proxy_set_header Host gitlab.runtel.org;
        
        # Передаём реальный IP клиента
        proxy_set_header X-Real-IP $remote_addr;
        
        # Передаём цепочку прокси (если есть)
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # Передаём протокол (https или http)
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # Используем HTTP/1.1 для keep-alive
        proxy_http_version 1.1;
    }

    # ==========================================================
    # БЛОК 4: CGI-СКРИПТ (генерирует красивую HTML-страницу)
    # ==========================================================
    location /cti/ {
        include fastcgi_params;
        # Подключаем стандартные параметры FastCGI
        
        # Передаём токен в CGI-скрипт через переменную окружения
        fastcgi_param  GITLAB_TOKEN   "glpat-uyy-DXxy2iXh5eFZxWyB";
        
        # Передаём имя проекта для универсальности
        fastcgi_param  GITLAB_PROJECT "runtel/cti-panel";
        
        # Передаём запрос в fcgiwrap через Unix-сокет
        fastcgi_pass unix:/var/run/fcgiwrap.socket;
        
        # Указываем путь к CGI-скрипту
        fastcgi_param SCRIPT_FILENAME /usr/lib/cgi-bin/cti-panel;
    }

    # ==========================================================
    # БЛОК 5: РЕДИРЕКТ /cti → /cti/
    # ==========================================================
    location = /cti {
        return 301 /cti/;
        # Если пользователь заходит на /cti (без слеша),
        # перенаправляем на /cti/ (со слешем) для единообразия
    }

    # ==========================================================
    # БЛОК 6: СКАЧИВАНИЕ ФАЙЛОВ
    # ==========================================================
    location ~ ^/cti/download/(\d+)$ {
        limit_except GET HEAD { deny all; }
        # Разрешаем только GET и HEAD
        
        # Проксируем на GitLab API для скачивания
        # $1 — это ID файла, захваченный из URL
        proxy_pass https://192.168.87.10/api/v4/projects/86/packages/310/package_files/$1/download;
        
        # Заголовки авторизации и хоста
        proxy_set_header Authorization "Bearer glpat-uyy-DXxy2iXh5eFZxWyB";
        proxy_set_header Host gitlab.runtel.org;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
    }

    # ==========================================================
    # БЛОК 7: ВСЁ ОСТАЛЬНОЕ — 404
    # ==========================================================
    location / {
        return 404;
        # Любой запрос, не попавший ни в один из блоков выше, получает 404
    }
}
```

## Часть 2: CGI-скрипт `/usr/lib/cgi-bin/cti-panel` с комментариями в коде

```bash
#!/bin/bash
# Указываем интерпретатор — bash

# ============================================================
# ОБЪЯВЛЕНИЕ ПЕРЕМЕННЫХ
# ============================================================

# Базовый URL GitLab API
GL="https://gitlab.runtel.org/api/v4"

# Путь к лог-файлу для отладки
LOG="/var/log/cti.log"

# ============================================================
# ФУНКЦИЯ ЛОГИРОВАНИЯ
# ============================================================
log() {
    # Выводит сообщение с датой, временем и PID процесса
    # Пример: 2026-07-17 06:33:52 [pid 12345] === запрос: REQUEST_URI=/cti/ ===
    printf '%s [pid %s] %s\n' "$(date '+%F %T')" "$$" "$*" >> "$LOG" 2>/dev/null
}

# ============================================================
# ФУНКЦИЯ ЗАПРОСА К API
# ============================================================
api() {
    local path="$1" code tmp
    tmp=$(mktemp)  # Создаём временный файл для ответа
    
    # Выполняем запрос к GitLab API
    # -s: тихий режим
    # -o "$tmp": сохраняем ответ во временный файл
    # -w '%{http_code}': выводим HTTP-код ответа
    code=$(curl -s -o "$tmp" -w '%{http_code}' \
        -H "PRIVATE-TOKEN: ${GITLAB_TOKEN}" "${GL}${path}")
    
    # Логируем URL и HTTP-код
    log "API GET ${GL}${path} -> HTTP ${code}"
    
    # Если код не 200 — логируем ошибку
    if [ "$code" != "200" ]; then
        log "API ERROR тело: $(head -c 500 "$tmp" | tr '\n' ' ')"
    else
        log "API OK, тело (первые 300): $(head -c 300 "$tmp" | tr '\n' ' ')"
    fi
    
    # Выводим содержимое временного файла
    cat "$tmp"
    
    # Удаляем временный файл
    rm -f "$tmp"
}

# ============================================================
# ПАРСИНГ QUERY_STRING (параметров URL)
# ============================================================

# Объявляем ассоциативный массив для параметров
declare -A Q

# Функция URL-декодирования
urldecode() { printf '%b' "${1//%/\\x}"; }

# Разбираем QUERY_STRING (например, "pkg=310&dl=1")
IFS='&' read -ra _pairs <<< "$QUERY_STRING"
for _p in "${_pairs[@]}"; do
    k="${_p%%=*}"   # Берём часть до =
    v="${_p#*=}"    # Берём часть после =
    [ -n "$k" ] && Q["$k"]=$(urldecode "${v//+/ }")
done

# ============================================================
# ФУНКЦИИ ЭКРАНИРОВАНИЯ ДЛЯ HTML
# ============================================================

# Экранирует спецсимволы для безопасного вывода в HTML
esc() {
    local s="$1"
    s="${s//&/&amp;}"   # & → &amp;
    s="${s//</&lt;}"    # < → &lt;
    s="${s//>/&gt;}"    # > → &gt;
    s="${s//\"/&quot;}" # " → &quot;
    printf '%s' "$s"
}

# URL-кодирует строку для использования в ссылках
enc() {
    printf '%s' "$1" | jq -sRr @uri
}

# ============================================================
# ФУНКЦИИ ГЕНЕРАЦИИ HTML
# ============================================================

# Выводит заголовки HTTP и начало HTML-страницы
html_head() {
    # Content-Type — обязательный заголовок для CGI
    printf 'Content-Type: text/html; charset=utf-8\r\n\r\n'
    
    # Выводим HTML с тёмным дизайном
    cat <<'HEAD'
<!DOCTYPE html>
<html lang="ru"><head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<title>Registry</title>
<style>
  :root { --bg:#0f1420; --card:#1a2130; --line:#2a3446; --accent:#6ea8fe; --text:#e6e9ef; --muted:#8b97ab; }
  * { box-sizing:border-box; }
  body { margin:0; font:15px/1.5 -apple-system,Segoe UI,Roboto,Ubuntu,sans-serif;
         background:var(--bg); color:var(--text); }
  .wrap { max-width:860px; margin:0 auto; padding:32px 20px 60px; }
  header { display:flex; align-items:center; gap:12px; margin-bottom:24px; }
  header .logo { font-size:28px; }
  header h1 { font-size:22px; margin:0; font-weight:600; }
  .crumbs { color:var(--muted); font-size:13px; margin-bottom:18px; }
  .crumbs a { color:var(--accent); text-decoration:none; }
  .crumbs a:hover { text-decoration:underline; }
  ul.list { list-style:none; margin:0; padding:0; display:flex; flex-direction:column; gap:8px; }
  ul.list li { margin:0; }
  ul.list a { display:flex; align-items:center; gap:12px; padding:14px 16px;
              background:var(--card); border:1px solid var(--line); border-radius:12px;
              color:var(--text); text-decoration:none; transition:.15s; }
  ul.list a:hover { border-color:var(--accent); transform:translateY(-1px); }
  .ico { font-size:20px; flex:none; }
  .name { font-weight:500; flex:1 1 auto; word-break:break-all; }
  .ver  { color:var(--muted); font-size:13px; flex:none; }
  .size { color:var(--muted); font-size:13px; flex:none; }
  .empty { color:var(--muted); text-align:center; padding:48px 0; }
  footer { margin-top:32px; color:var(--muted); font-size:12px; text-align:center; }
</style>
</head><body><div class="wrap">
<header><span class="logo">&#128230;</span><h1>Package Registry</h1></header>
HEAD
}

# Выводит закрывающие теги HTML
html_foot() {
    printf '<footer>GitLab Package Registry &middot; %s</footer></div></body></html>' "$(esc "$GITLAB_PROJECT")"
}

# ============================================================
# ФУНКЦИЯ ПЕРЕВОДА БАЙТОВ В ЧЕЛОВЕЧЕСКИЙ ФОРМАТ
# ============================================================

human() {
    local b="$1"
    if   [ "$b" -ge 1073741824 ] 2>/dev/null; then awk "BEGIN{printf \"%.1f ГБ\", $b/1073741824}"
    elif [ "$b" -ge 1048576    ] 2>/dev/null; then awk "BEGIN{printf \"%.1f МБ\", $b/1048576}"
    elif [ "$b" -ge 1024       ] 2>/dev/null; then awk "BEGIN{printf \"%.1f КБ\", $b/1024}"
    else printf '%s Б' "${b:-?}"; fi
}

# ============================================================
# ОСНОВНАЯ ЛОГИКА СКРИПТА
# ============================================================

# Логируем начало запроса
log "=== запрос: REQUEST_URI=${REQUEST_URI} QUERY_STRING=${QUERY_STRING} ==="
if [ -z "$GITLAB_TOKEN" ]; then
    log "ВНИМАНИЕ: GITLAB_TOKEN пуст!"
else
    log "GITLAB_TOKEN задан (длина ${#GITLAB_TOKEN})"
fi
log "GITLAB_PROJECT=${GITLAB_PROJECT}"

# Кодируем имя проекта для URL
PROJECT=$(printf '%s' "${GITLAB_PROJECT:-group/project}" | jq -sRr @uri)

# ============================================================
# СЛУЧАЙ 1: СКАЧИВАНИЕ ФАЙЛА (?dl=1)
# ============================================================

if [ -n "${Q[dl]}" ]; then
    name=$(enc "${Q[name]}")      # Имя пакета (CTI-desktop)
    version=$(enc "${Q[version]}") # Версия (2.3.0)
    file=$(enc "${Q[file]}")       # Имя файла (app-v2.3.0.620.zip)
    
    # Формируем путь к файлу в GitLab
    dlpath="/projects/${PROJECT}/packages/generic/${name}/${version}/${file}"
    
    log "СКАЧИВАНИЕ ${GL}${dlpath} (file=${Q[file]})"
    
    # Выводим заголовки для скачивания
    printf 'Content-Type: application/octet-stream\r\n'
    printf 'Content-Disposition: attachment; filename="%s"\r\n\r\n' "${Q[file]}"
    
    # Скачиваем файл из GitLab и передаём его клиенту
    curl -s -H "PRIVATE-TOKEN: ${GITLAB_TOKEN}" "${GL}${dlpath}"
    
    log "СКАЧИВАНИЕ завершено (curl exit=$?)"
    exit 0
fi

# ============================================================
# СЛУЧАЙ 2: ПРОСМОТР ПАКЕТА (?pkg=310)
# ============================================================

if [ -n "${Q[pkg]}" ]; then
    pid=$(enc "${Q[pkg]}")  # ID пакета (310)
    
    # Получаем информацию о пакете
    info=$(api "/projects/${PROJECT}/packages/${pid}")
    pname=$(jq -r '.name'    <<< "$info")
    pver=$(jq -r '.version'  <<< "$info")
    log "пакет ${pid}: name=${pname} version=${pver}"
    
    # Получаем список файлов в пакете
    files=$(api "/projects/${PROJECT}/packages/${pid}/package_files")
    fcount=$(jq 'length' <<< "$files" 2>/dev/null)
    log "файлов в пакете: ${fcount}"
    
    # Выводим HTML-страницу со списком файлов
    html_head
    printf '<div class="crumbs"><a href="?">&larr; все пакеты</a> / %s @ %s</div>' "$(esc "$pname")" "$(esc "$pver")"
    
    if [ "${fcount:-0}" -gt 0 ] 2>/dev/null; then
        printf '<ul class="list">'
        jq -r '.[] | [.file_name, (.size|tostring)] | @tsv' <<< "$files" \
          | while IFS=$'\t' read -r fname fsize; do
                # Формируем ссылку на скачивание
                link="?dl=1&name=$(enc "$pname")&version=$(enc "$pver")&file=$(enc "$fname")"
                printf '<li><a href="%s"><span class="ico">&#128196;</span><span class="name">%s</span><span class="size">%s</span></a></li>' \
                    "$link" "$(esc "$fname")" "$(human "$fsize")"
            done
        printf '</ul>'
    else
        printf '<div class="empty">В пакете нет файлов</div>'
    fi
    html_foot
    exit 0
fi

# ============================================================
# СЛУЧАЙ 3: КОРЕНЬ — СПИСОК ВСЕХ ПАКЕТОВ
# ============================================================

# Получаем список всех пакетов в проекте
pkgs=$(api "/projects/${PROJECT}/packages?per_page=100")
pcount=$(jq 'length' <<< "$pkgs" 2>/dev/null)
log "пакетов найдено: ${pcount:-ошибка разбора JSON}"

# Выводим HTML-страницу со списком пакетов
html_head

if [ "${pcount:-0}" -gt 0 ] 2>/dev/null; then
    printf '<ul class="list">'
    jq -r '.[] | [.id, .name, .version] | @tsv' <<< "$pkgs" 2>/dev/null \
      | while IFS=$'\t' read -r id name version; do
            # Ссылка на просмотр пакета
            printf '<li><a href="?pkg=%s"><span class="ico">&#128193;</span><span class="name">%s</span><span class="ver">%s</span></a></li>' \
                "$id" "$(esc "$name")" "$(esc "$version")"
        done
    printf '</ul>'
else
    printf '<div class="empty">Пакетов пока нет</div>'
fi

html_foot
log "=== ответ отправлен ==="
```

## Отдельное объяснение за кодом

### Как работает вся система

```
Клиент (браузер)
    │
    │  Заходит на https://registry.runtel.org/cti/
    ▼
Nginx (front2)
    │
    │  location /cti/ → включает FastCGI
    ▼
fcgiwrap
    │
    │  Запускает скрипт /usr/lib/cgi-bin/cti-panel
    ▼
CGI-скрипт (cti-panel)
    │
    ├── 1. Логирует запрос
    ├── 2. Проверяет параметры QUERY_STRING
    │
    ├── Если есть ?dl=1 → СКАЧИВАНИЕ
    │   └── curl запрос к GitLab API → файл
    │
    ├── Если есть ?pkg=310 → ПРОСМОТР ПАКЕТА
    │   ├── Запрос к API → информация о пакете
    │   ├── Запрос к API → список файлов
    │   └── Генерация HTML → список файлов
    │
    └── Если параметров нет → КОРЕНЬ
        ├── Запрос к API → список всех пакетов
        └── Генерация HTML → список пакетов
```

### Что делает каждая часть скрипта

| Часть скрипта | Что делает |
|---------------|------------|
| `log()` | Пишет сообщения в `/var/log/cti.log` для отладки |
| `api()` | Делает запрос к GitLab API, логирует результат, возвращает JSON |
| `urldecode()` | Преобразует `%20` в пробел, `%2F` в `/` и т.д. |
| `esc()` | Экранирует спецсимволы для безопасного вывода в HTML |
| `enc()` | URL-кодирует строку для ссылок |
| `html_head()` | Выводит заголовки HTTP и начало HTML-страницы |
| `html_foot()` | Выводит закрывающие теги HTML |
| `human()` | Преобразует байты в читаемый формат (МБ, КБ) |

### Три режима работы скрипта

**Режим 1: Скачивание файла (`?dl=1`)**
```
https://registry.runtel.org/cti/?dl=1&name=CTI-desktop&version=2.3.0&file=app-v2.3.0.620.zip
```
Скрипт:
1. Формирует путь: `/projects/86/packages/generic/CTI-desktop/2.3.0/app-v2.3.0.620.zip`
2. Делает `curl` к GitLab API
3. Отдаёт файл клиенту с заголовком `Content-Disposition: attachment`

**Режим 2: Просмотр пакета (`?pkg=310`)**
```
https://registry.runtel.org/cti/?pkg=310
```
Скрипт:
1. Делает запрос к `/api/v4/projects/86/packages/310`
2. Делает запрос к `/api/v4/projects/86/packages/310/package_files`
3. Генерирует HTML-страницу со списком файлов

**Режим 3: Корень (без параметров)**
```
https://registry.runtel.org/cti/
```
Скрипт:
1. Делает запрос к `/api/v4/projects/86/packages`
2. Генерирует HTML-страницу со списком всех пакетов

## Почему это решение — отличное

| Характеристика | Описание |
|----------------|----------|
| **Универсальность** | Работает с любыми пакетами, не только с ID 310 |
| **Динамичность** | Список файлов обновляется автоматически из API |
| **Красивый дизайн** | Тёмная тема, удобные карточки, человекочитаемые размеры |
| **Логирование** | Всё пишется в `/var/log/cti.log` для отладки |
| **Безопасность** | Токен передаётся через переменную окружения, а не в скрипте |
| **Масштабируемость** | Легко добавить новые проекты или группы |

## Итог

Вы создали **полноценный реестр пакетов**, который:
1. Работает через Nginx и FastCGI.
2. Автоматически получает данные из GitLab API.
3. Показывает красивые страницы с пакетами и файлами.
4. Позволяет скачивать файлы одним кликом.
5. Ведёт логи для отладки.

Это **стабильное, масштабируемое и красивое решение**. 🚀
