## 📝 Как мы восстановили систему после удаления себя из /home/ (/etc/passwd)

### Что случилось:
Вы случайно удалили свою учётную запись `kkorablin` из системы, но сессия осталась активной. 
Из-за этого `sudo` перестал работать (`sudo: вы не существуете в базе данных passwd`), и вы не могли восстановиться обычными средствами.

---

## 🔧 Полная инструкция по восстановлению:

### 1. Загрузка с Live USB

**Шаг 1.1:** Загрузитесь с Live USB (EndeavourOS или любой Arch-based дистрибутив).

**Шаг 1.2:** Откройте терминал и определите разделы:
```bash
lsblk -f
```
**Вывод системы:**
```
nvme0n1
├─nvme0n1p1 vfat    FAT32    [загрузочный раздел, EFI]
└─nvme0n1p2 btrfs   endeavouros [корневой раздел с системой]
```

**Шаг 1.3:** Смонтируйте корневой раздел с правильным подтомом (subvolume) для btrfs:
```bash
sudo mount -o subvol=@ /dev/nvme0n1p2 /mnt
```

**Почему так:** В системе EndeavourOS (и Arch) корневой раздел btrfs содержит подтома. Основной том обычно называется `@`. Если не указать его, вы смонтируете не ту структуру.

**Шаг 1.4:** Смонтируйте загрузочный раздел:
```bash
sudo mkdir -p /mnt/boot
sudo mount /dev/nvme0n1p1 /mnt/boot
```

**Зачем:** Для работы `arch-chroot` нужна полная структура системы, включая /boot, где лежит ядро и загрузчик.

---

### 2. Вход в chroot

**Шаг 2.1:** Войдите в изолированную среду вашей системы:
```bash
sudo arch-chroot /mnt
```

**Что такое chroot:** Эта команда временно меняет корневой каталог на `/mnt`, давая вам доступ к вашей системе так, как будто вы загружены в неё. Все команды будут выполняться внутри /mnt.

---

### 3. Восстановление пользователя и группы

**Шаг 3.1:** Восстановите группу пользователя:
```bash
echo "kkorablin:x:1000:" >> /etc/group
```

**Что это:** Добавляет запись в файл `/etc/group`.
- `kkorablin` — имя группы
- `x` — пароль группы (не используется)
- `1000` — ID группы (GID)
- Пустой список пользователей в конце

**Шаг 3.2:** Восстановите пользователя в `/etc/passwd`:
```bash
echo "kkorablin:x:1000:1000:kkorablin:/home/kkorablin:/bin/bash" >> /etc/passwd
```

**Что это:** Добавляет запись в файл `/etc/passwd`:
- `kkorablin` — имя пользователя
- `x` — пароль хранится в `/etc/shadow`
- `1000` — ID пользователя (UID)
- `1000` — ID группы (GID)
- `kkorablin` — полное имя пользователя (комментарий)
- `/home/kkorablin` — домашняя директория
- `/bin/bash` — командная оболочка

**Важно:** Используйте `>>` (добавление), а не `>` (перезапись)!

**Шаг 3.3:** Установите пароль:
```bash
passwd kkorablin
```

**Что это:** Запрашивает и устанавливает пароль, записывая его в `/etc/shadow`.

**Шаг 3.4:** Добавьте пользователя в группу `wheel` (для возможности использовать sudo):
```bash
usermod -aG wheel kkorablin
```

**Что это:** Добавляет пользователя в группу `wheel`, которая по умолчанию разрешает выполнение команд через `sudo`.

**Шаг 3.5:** Проверьте, что пользователь создан:
```bash
getent passwd kkorablin
```

**Что это:** Команда `getent` показывает записи из системных баз данных (включая `/etc/passwd`). Должно вывести:
```
kkorablin:x:1000:1000:kkorablin:/home/kkorablin:/bin/bash
```

**Шаг 3.6:** Создайте домашнюю директорию, если она отсутствует:
```bash
mkdir -p /home/kkorablin
chown -R kkorablin:kkorablin /home/kkorablin
```

**Что это:** Создаёт каталог `/home/kkorablin` и делает его владельцем `kkorablin`.

---

### 4. Выход из chroot и перезагрузка

**Шаг 4.1:** Выйдите из chroot-окружения:
```bash
exit
```

**Шаг 4.2:** Отмонтируйте все разделы:
```bash
sudo umount -R /mnt
```

**Что это:** Рекурсивно размонтирует `/mnt` и все вложенные точки монтирования (включая `/mnt/boot`).

**Шаг 4.3:** Перезагрузитесь:
```bash
reboot
```

**Важно:** Не забудьте вынуть Live USB, чтобы система загрузилась с диска.

---

### 5. Проверка после перезагрузки

**Шаг 5.1:** Войдите как `kkorablin` с установленным паролем.

**Шаг 5.2:** Проверьте:
```bash
whoami
# Должно быть: kkorablin

sudo whoami
# Должно быть: root (попросит пароль)

groups
# Должна быть группа wheel
```

## 📌 Полезные советы на будущее:

### Безопасное удаление пользователей:
```bash
# Проверьте, кого вы собираетесь удалить
whoami
id

# Список пользователей с домашними директориями
ls /home/

# Безопасный цикл удаления
for user in /home/*/; do
    username=$(basename "$user")
    if [ "$username" != "kkorablin" ] && [ "$username" != "root" ]; then
        echo "Удаление: $username"
        sudo userdel -f "$username"
    fi
done
```

### Если всё же сломали sudo:
```bash
# Восстановить права sudo без sudo (если вы в группе wheel)
pkexec bash
```

### Создание резервной копии важных файлов:
```bash
sudo cp /etc/passwd /etc/passwd.backup
sudo cp /etc/group /etc/group.backup
sudo cp /etc/shadow /etc/shadow.backup
```
