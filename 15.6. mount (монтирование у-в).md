# 📚 Полное руководство по работе с файловыми системами в Linux
## Утилиты `mount`, `findmnt` и настройка `/etc/fstab`

> **Для кого:** Администраторы Linux, DevOps-инженеры, все, кто работает с монтированием дисков и сетевых хранилищ.

---

## 🎯 Часть 1. Утилита `mount` — подключение файловых систем

### 1.1 Что такое монтирование?

**Монтирование** — это процесс подключения файловой системы (раздела диска, USB-флешки, сетевой папки, ISO-образа) к определенному каталогу в вашей файловой системе. Этот каталог называется **точкой монтирования**.

### 1.2 Просмотр смонтированных систем

#### Простой просмотр всех монтирований
```bash
# Вывод всех смонтированных файловых систем
mount

# Пример вывода (ваш реальный вывод):
# /dev/nvme0n1p2 on / type ext4 (rw,noatime)
# devtmpfs on /dev type devtmpfs (rw,nosuid,size=7783192k)
# tmpfs on /run/user/1000 type tmpfs (rw,nosuid,nodev,relatime)
```

#### Фильтрация по типу файловой системы
```bash
# Только ext4 файловые системы
mount -t ext4
# Вывод: /dev/nvme0n1p2 on / type ext4 (rw,noatime)

# Только tmpfs (виртуальные ФС в оперативной памяти)
mount -t tmpfs
# Вывод: tmpfs on /run/user/1000 type tmpfs (rw,nosuid,nodev,relatime)

# Только NFS (сетевые ФС)
mount -t nfs4
# Если нет NFS — пустой вывод
```

#### Фильтрация через grep (более гибкий способ)
```bash
# Показать только реальные диски (не виртуальные ФС)
mount | grep '^/dev'

# Показать монтирования в /mnt и /media (обычно внешние диски)
mount | grep -E '/mnt|/media'

# Показать монтирования с опцией только для чтения (ro)
mount | grep -w 'ro'

# Исключить виртуальные файловые системы
mount | grep -v -E 'tmpfs|proc|sysfs|devtmpfs'
```

### 1.3 Монтирование устройств

#### Базовое монтирование
```bash
# Синтаксис: mount <устройство> <точка_монтирования>

# Пример: монтируем второй раздел первого диска SATA в /mnt/data
mount /dev/sdb1 /mnt/data

# Монтируем NVMe диск (ваш случай с /dev/nvme0n1p2 уже смонтирован в /)
mount /dev/nvme0n1p5 /mnt/backup

# Монтируем USB-флешку (обычно определяется как /dev/sdc1)
mount /dev/sdc1 /media/usb
```

#### Монтирование с указанием типа файловой системы
```bash
# Явно указываем тип (если auto не сработал)
mount -t ext4 /dev/sdb1 /mnt/data
mount -t ntfs /dev/sdc1 /mnt/windows
mount -t vfat /dev/sdd1 /mnt/fat32   # для старых флешек
mount -t exfat /dev/sde1 /mnt/exfat  # для больших файлов (>4GB)

# Просмотр доступных типов ФС
ls /sbin/mkfs.*  # показывает, какие ФС поддерживаются
```

#### Монтирование ISO-образов (очень полезно!)
```bash
# Создаем точку монтирования
mkdir /mnt/iso

# Монтируем ISO-образ (опция -o loop обязательна)
mount -o loop ubuntu-22.04.iso /mnt/iso

# Проверяем содержимое
ls /mnt/iso
# Вы увидите файлы: casper/ pool/ preseed/ README.diskdefines и т.д.

# Размонтируем после использования
umount /mnt/iso
```

#### Монтирование сетевых папок (NFS, SMB/CIFS)
```bash
# NFS монтирование
mount -t nfs4 192.168.1.100:/exported/folder /mnt/nfs

# SMB/CIFS монтирование (Windows-шары)
mount -t cifs //192.168.1.100/share /mnt/smb -o username=user,password=pass

# SMB с доменной аутентификацией
mount -t cifs //server.domain.com/share /mnt/smb -o domain=DOMAIN,username=user,password=pass
```

### 1.4 Важные опции монтирования

Опции указываются после `-o` через запятую **без пробелов**.

```bash
# Только чтение (read-only) — для защиты данных
mount -o ro /dev/sdb1 /mnt/backup

# Чтение-запись (по умолчанию, можно не указывать)
mount -o rw /dev/sdb1 /mnt/data

# Отключение exec (запрет выполнения программ из этой ФС)
mount -o noexec /dev/sdb1 /mnt/data

# Запрет доступа к устройствам из этой ФС
mount -o nodev /dev/sdb1 /mnt/data

# Отключение suid (игнорирование битов suid/sgid) — безопасность
mount -o nosuid /dev/sdb1 /mnt/data

# noatime — не обновлять время доступа к файлам (ускоряет работу)
mount -o noatime /dev/sdb1 /mnt/data

# Комбинирование опций
mount -o rw,noatime,nosuid,nodev /dev/sdb1 /mnt/data
```

### 1.5 Перемонтирование (remount)

```bash
# Перемонтировать уже смонтированную ФС с новыми опциями
# Например, переключить корень из ro в rw
mount -o remount,rw /

# Перемонтировать /boot в режим только чтения для безопасности
mount -o remount,ro /boot

# Добавить опцию noatime к существующему монтированию
mount -o remount,noatime /home

# Изменить тип ФС при перемонтировании (редко, но бывает)
mount -o remount,ro -t ext4 /dev/sdb1 /mnt/data
```

### 1.6 Монтирование через UUID (рекомендуемый способ)

UUID (Universally Unique Identifier) — уникальный идентификатор раздела, который не меняется при переподключении дисков.

```bash
# Узнаем UUID всех разделов
sudo blkid
# Вывод:
# /dev/nvme0n1p1: UUID="67E3-17ED" TYPE="vfat" PARTUUID="8f7a2b3c-01"
# /dev/nvme0n1p2: UUID="a1b2c3d4-..." TYPE="ext4" PARTUUID="8f7a2b3c-02"

# Монтируем по UUID
mount UUID="a1b2c3d4-..." /mnt/data

# Монтируем по PARTUUID (для GPT-разделов)
mount PARTUUID="8f7a2b3c-02" /mnt/data
```

### 1.7 Размонтирование (umount)

```bash
# По точке монтирования (предпочтительный способ)
umount /mnt/data

# По устройству
umount /dev/sdb1

# Принудительное размонтирование (если ФС "висит")
umount -f /mnt/nfs

# Ленивое размонтирование — отключаем сейчас, но процесс завершаем позже
umount -l /mnt/busy_share
# Полезно, когда что-то держит файл открытым

# Размонтировать все NFS-ресурсы
umount -a -t nfs4

# Проверить, смонтирована ли ФС, перед размонтированием
if mount | grep -q "/mnt/data"; then
    echo "Смонтирована, размонтируем"
    umount /mnt/data
else
    echo "Не смонтирована"
fi
```

## 🌲 Часть 2. Утилита `findmnt` — современный просмотр монтирований

`findmnt` входит в пакет `util-linux` и гораздо удобнее, чем `mount` для анализа.

### 2.1 Базовый вывод

```bash
# Древовидный вывод (показывает иерархию монтирований)
findmnt

# Пример вывода (как в вашем терминале):
# TARGET                      SOURCE         FSTYPE          OPTIONS
# /                           /dev/nvme0n1p2 ext4            rw,noatime
# ├─/tmp                      tmpfs          tmpfs           rw,noatime
# ├─/dev                      devtmpfs       devtmpfs        rw,nosuid
# │ ├─/dev/hugepages          hugetlbfs      hugetlbfs       rw,relatime
# │ └─/dev/shm                tmpfs          tmpfs           rw,nosuid
# ├─/run                      tmpfs          tmpfs           rw,nosuid
# └─/efi                      /dev/nvme0n1p1 vfat            rw,relatime

# Линейный вывод (без иерархии, как в mount)
findmnt -l

# Очень компактный вывод (без заголовков, только нужные колонки)
findmnt -n -l -o TARGET,SOURCE,FSTYPE
# Вывод:
# / /dev/nvme0n1p2 ext4
# /tmp tmpfs tmpfs
# /dev devtmpfs devtmpfs
```

### 2.2 Фильтрация по типу ФС

```bash
# Только ext4 (как в вашем примере)
findmnt -t ext4
# TARGET SOURCE         FSTYPE OPTIONS
# /      /dev/nvme0n1p2 ext4   rw,noatime

# Только tmpfs (виртуальные ФС в RAM)
findmnt -t tmpfs
# TARGET        SOURCE FSTYPE OPTIONS
# /tmp          tmpfs  tmpfs  rw,noatime
# /run          tmpfs  tmpfs  rw,nosuid
# /dev/shm      tmpfs  tmpfs  rw,nosuid
# /run/user/1000 tmpfs tmpfs  rw,nosuid

# Только pstore (ваш пример)
findmnt -t pstore
# TARGET         SOURCE FSTYPE OPTIONS
# /sys/fs/pstore none   pstore rw,nosuid,nodev,noexec,relatime

# Несколько типов через запятую
findmnt -t ext4,tmpfs
```

### 2.3 Поиск по точке монтирования или устройству

```bash
# Найти конкретную точку монтирования
findmnt /mnt/data
# Если не смонтировано — пустой вывод (код возврата 1)

# Найти по устройству
findmnt /dev/nvme0n1p2
# TARGET SOURCE         FSTYPE OPTIONS
# /      /dev/nvme0n1p2 ext4   rw,noatime

# Проверка в скриптах (важный прием!)
if findmnt /mnt/backup > /dev/null; then
    echo "Резервный раздел смонтирован"
    umount /mnt/backup
else
    echo "Раздел не смонтирован, монтируем"
    mount /dev/sdb1 /mnt/backup
fi
```

### 2.4 Работа с колонками

```bash
# Показать только определенные колонки
findmnt -o TARGET,SOURCE,FSTYPE,OPTIONS

# С размерами (полезно для дисков)
findmnt -o TARGET,SOURCE,FSTYPE,SIZE,USED,USE%

# Пример вывода с размерами:
# TARGET SOURCE         FSTYPE   SIZE   USED USE%
# /      /dev/nvme0n1p2 ext4    237.4G 45.2G  19%
# /efi   /dev/nvme0n1p1 vfat    511.0M  6.1M   2%

# Вывод без заголовков (для скриптов)
findmnt -n -o TARGET,SOURCE -t ext4
# / /dev/nvme0n1p2
```

### 2.5 Машиночитаемые форматы

```bash
# JSON-вывод (ваш пример из терминала!)
findmnt --json
# Вывод огромный, но структурированный — отлично для Python/JQ

# Парсинг JSON через jq (если установлен)
findmnt --json | jq '.filesystems[].target'
# "/"
# "/tmp"
# "/dev"
# ...

# Вывод в виде списка с кастомными разделителями
findmnt --list --output TARGET,SOURCE --noheadings | cut -d' ' -f1
```

### 2.6 Поиск в /etc/fstab

```bash
# Показать, что должно монтироваться из fstab (даже если не смонтировано)
findmnt --fstab

# Искать в fstab конкретную точку
findmnt --fstab --target /mnt/share
# Если пусто — такой записи в fstab нет

# Искать в fstab по типу ФС
findmnt --fstab -t nfs4
# Покажет все NFS-ресурсы из fstab

# Сравнить, что смонтировано vs что в fstab
comm -23 <(findmnt -n -o TARGET | sort) <(findmnt --fstab -n -o TARGET | sort)
```

### 2.7 Продвинутые примеры с findmnt

```bash
# Найти все монтирования с опцией 'noexec'
findmnt --options noexec

# Найти все монтирования на USB-устройствах (/dev/sd*)
findmnt --list | grep '^/dev/sd'

# Подсчитать количество монтирований по типам
findmnt --list -o FSTYPE | sort | uniq -c

# Показать точки монтирования, где используется >80% места
findmnt -o TARGET,USE% | awk 'NR>1 && $2 > 80'

# Найти монтирования, которые не из fstab (ручные)
comm -13 <(findmnt --fstab -n -o TARGET | sort) <(findmnt -n -o TARGET | sort)
```

## 📝 Часть 3. Файл `/etc/fstab` — автоматическое монтирование

`/etc/fstab` (file systems table) — конфигурация, которая говорит системе, **что** монтировать, **куда** и **как** при загрузке.

### 3.1 Формат записи в fstab

```
<device>  <mountpoint>  <fstype>  <options>  <dump>  <pass>
```

**Поля подробно:**

| Поле | Описание | Примеры |
|------|----------|---------|
| **device** | Устройство (диск, UUID, сетевая шара) | `/dev/sda1`, `UUID=...`, `//server/share` |
| **mountpoint** | Каталог для монтирования | `/`, `/home`, `/mnt/data` |
| **fstype** | Тип файловой системы | `ext4`, `xfs`, `ntfs`, `nfs4`, `auto` |
| **options** | Опции монтирования | `defaults`, `rw,noatime`, `ro` |
| **dump** | Резервное копирование (0 или 1) | `0` (отключено), `1` (включено) |
| **pass** | Порядок проверки fsck | `0` (не проверять), `1` (корень), `2` (остальные) |

### 3.2 Базовые примеры fstab

```bash
# Пример вашей системы (реальный fstab)
cat /etc/fstab
# /etc/fstab: static file system information.
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

# Корневой раздел (ext4)
UUID=a1b2c3d4-... /               ext4    errors=remount-ro 0       1

# EFI раздел (vfat)
UUID=67E3-17ED    /efi            vfat    umask=0077      0       2

# Раздел подкачки
UUID=xyz...       none            swap    sw              0       0

# Домашний раздел
UUID=def...       /home           ext4    defaults        0       2
```

### 3.3 Реальные примеры записей

```bash
# 1. Обычный раздел ext4
UUID=12345678-9abc-def0-1234-567890abcdef /mnt/data ext4 defaults 0 2

# 2. Раздел с NTFS (Windows)
UUID=1234ABCD /mnt/windows ntfs-3g defaults,umask=000,uid=1000,gid=1000 0 0

# 3. Раздел только для чтения
UUID=87654321-... /mnt/backup ext4 ro,noatime 0 2

# 4. NFS-шара (сетевой диск)
192.168.1.100:/export/share /mnt/nfs n4 rw,hard,intr,_netdev 0 0

# 5. SMB/CIFS (Windows-шара с авторизацией)
//192.168.1.100/share /mnt/smb cifs credentials=/etc/smbcredentials,uid=1000,gid=1000,_netdev 0 0

# 6. ISO-образ (автомонтирование при загрузке — редко)
/path/to/ubuntu.iso /mnt/iso iso9660 loop,ro 0 0

# 7. tmpfs в RAM (быстрая временная папка)
tmpfs /ramdisk tmpfs size=2G,noatime,mode=1777 0 0

# 8. Раздел с опцией noauto (монтировать только вручную)
UUID=5555-... /mnt/manual ext4 noauto,user 0 0
# mount /mnt/manual — смонтирует вручную
```

### 3.4 Создание и редактирование fstab

```bash
# ВАЖНО: Всегда делайте резервную копию перед редактированием!
cp /etc/fstab /etc/fstab.backup.$(date +%Y%m%d)

# Редактирование (требует root)
sudo nano /etc/fstab
# или
sudo vim /etc/fstab

# Проверка синтаксиса (после редактирования)
findmnt --fstab --verify
# Если ошибок нет — тишина

# Применение всех записей из fstab
mount -a
# Если есть ошибки — система покажет их

# Применить только конкретную запись
mount /mnt/data   # достаточно указать точку монтирования
```

### 3.5 Пошаговый пример: Добавление нового диска

```bash
# Шаг 1: Создаем точку монтирования
sudo mkdir -p /mnt/newdisk

# Шаг 2: Узнаем UUID нового диска
sudo blkid /dev/sdb1
# /dev/sdb1: UUID="12345678-9abc-def0-1234-567890abcdef" TYPE="ext4"

# Шаг 3: Добавляем запись в /etc/fstab
echo 'UUID=12345678-9abc-def0-1234-567890abcdef /mnt/newdisk ext4 defaults 0 2' | sudo tee -a /etc/fstab

# Шаг 4: Проверяем синтаксис
sudo findmnt --fstab --verify

# Шаг 5: Монтируем без перезагрузки
sudo mount -a

# Шаг 6: Проверяем
findmnt /mnt/newdisk
```

### 3.6 Полезные опции для fstab

| Опция | Значение | Пример использования |
|-------|----------|---------------------|
| `defaults` | rw,suid,dev,exec,auto,nouser,async | Обычные диски |
| `noauto` | Не монтировать при `mount -a` | Ручные монтирования |
| `user` | Разрешить монтирование обычным пользователям | `user,noauto` для USB |
| `nofail` | Не падать при ошибке монтирования | Сетевые диски, которые могут быть недоступны |
| `_netdev` | Сетевой ресурс (ждет сети) | NFS, SMB |
| `noatime` | Не обновлять время доступа | SSD, быстрые диски |
| `discard` | Включить TRIM для SSD | `defaults,discard` |
| `errors=remount-ro` | При ошибке перейти в read-only | Корневой раздел |

### 3.7 Примеры сложных конфигураций

```bash
# SSD с TRIM и noatime (оптимизация)
UUID=xxx / ext4 defaults,noatime,discard 0 1

# Сетевой диск, который может быть недоступен (не блокируем загрузку)
192.168.1.100:/nfs /mnt/nfs n4 rw,hard,intr,_netdev,nofail 0 0

# Внешний USB-диск (монтируется только при подключении)
UUID=YYYY /mnt/usb ext4 noauto,user,nofail 0 0

# RAM-диск для временных файлов (потеряется при выключении)
tmpfs /tmp tmpfs defaults,noatime,mode=1777,size=4G 0 0

# Windows-шара с файлом логинов (безопаснее, чем пароль в fstab)
# Файл /etc/smbcredentials: username=user, password=pass, domain=DOMAIN
# chmod 600 /etc/smbcredentials
//server/share /mnt/share cifs credentials=/etc/smbcredentials,uid=1000,gid=1000,_netdev 0 0
```

## 🔧 Часть 4. Практические сценарии

### 4.1 Сценарий: Монтирование USB-флешки

```bash
# 1. Вставляем флешку, смотрим, как определилась
sudo dmesg | tail -20
# Или
lsblk
# sdc           8:32   1  14.9G  0 disk
# └─sdc1        8:33   1  14.9G  0 part

# 2. Создаем точку монтирования
sudo mkdir -p /media/usb

# 3. Монтируем (auto определит FAT32/NTFS)
sudo mount /dev/sdc1 /media/usb

# 4. Проверяем
findmnt /media/usb
ls -la /media/usb

# 5. После работы размонтируем
sudo umount /media/usb

# 6. Безопасное извлечение
sudo eject /dev/sdc
```

### 4.2 Сценарий: Монтирование Windows-шары (SMB/CIFS)

```bash
# 1. Устанавливаем поддержку CIFS
sudo apt install cifs-utils  # Debian/Ubuntu
sudo yum install cifs-utils  # RHEL/CentOS

# 2. Создаем точку монтирования
sudo mkdir -p /mnt/windows_share

# 3. Создаем файл с паролем (безопасно)
echo 'username=myuser' > ~/.smbcredentials
echo 'password=mypass' >> ~/.smbcredentials
echo 'domain=MYDOMAIN' >> ~/.smbcredentials
chmod 600 ~/.smbcredentials

# 4. Монтируем
sudo mount -t cifs //192.168.1.100/share /mnt/windows_share -o credentials=$HOME/.smbcredentials,uid=$(id -u),gid=$(id -g)

# 5. Для постоянного монтирования добавляем в /etc/fstab
echo '//192.168.1.100/share /mnt/windows_share cifs credentials=/home/user/.smbcredentials,uid=1000,gid=1000,_netdev 0 0' | sudo tee -a /etc/fstab

# 6. Проверяем
sudo mount -a
```

### 4.3 Сценарий: Монтирование NFS-шары

```bash
# 1. Устанавливаем NFS-клиент
sudo apt install nfs-common  # Debian/Ubuntu
sudo yum install nfs-utils   # RHEL/CentOS

# 2. Создаем точку монтирования
sudo mkdir -p /mnt/nfs_share

# 3. Монтируем вручную
sudo mount -t nfs4 192.168.1.100:/exported/path /mnt/nfs_share

# 4. Добавляем в /etc/fstab
echo '192.168.1.100:/exported/path /mnt/nfs_share nfs4 rw,hard,intr,_netdev 0 0' | sudo tee -a /etc/fstab

# 5. Проверяем все NFS-монтирования
findmnt -t nfs4
```

### 4.4 Сценарий: Работа с образами дисков

```bash
# 1. Монтируем ISO-образ
sudo mount -o loop ubuntu-22.04.iso /mnt/iso

# 2. Монтируем образ диска (созданный через dd)
# Сначала смотрим структуру образа
fdisk -l disk_image.img
# Если есть разделы, монтируем с offset
sudo mount -o loop,offset=$((512*2048)) disk_image.img /mnt/image

# 3. Создаем свой образ и монтируем
dd if=/dev/zero of=myimage.img bs=1M count=100
mkfs.ext4 myimage.img
sudo mount -o loop myimage.img /mnt/myimage

# 4. Размонтируем
sudo umount /mnt/myimage
```

### 4.5 Сценарий: Диагностика проблем

```bash
# Проблема: Не удается размонтировать (device is busy)

# 1. Узнаем, какие процессы используют точку монтирования
lsof /mnt/data
# Или
fuser -v /mnt/data

# 2. Убиваем процессы (осторожно!)
fuser -km /mnt/data

# 3. Ленивое размонтирование (безопаснее)
umount -l /mnt/data

# Проблема: Ошибка "mount point does not exist"
# Решение: создать каталог
sudo mkdir -p /mnt/data

# Проблема: "unknown filesystem type 'ntfs'"
# Решение: установить ntfs-3g
sudo apt install ntfs-3g
sudo mount -t ntfs-3g /dev/sdc1 /mnt/windows

# Проблема: "mount: /mnt/data: special device /dev/sdb1 does not exist"
# Решение: проверить, видит ли система диск
lsblk
sudo fdisk -l
```

## 📋 Часть 5. Шпаргалка по командам

### Быстрый справочник

```bash
# === Просмотр ===
mount                          # Все монтирования
mount -t ext4                  # По типу ФС
findmnt                        # Древовидный вид
findmnt -l                     # Линейный вид
findmnt -t nfs4 --list         # Все NFS
findmnt --fstab                # Что в fstab
findmnt --json | jq .          # JSON-вывод

# === Монтирование ===
mount /dev/sdb1 /mnt/data      # Простое монтирование
mount -o ro /dev/sdb1 /mnt/data  # Только чтение
mount -t ntfs /dev/sdc1 /mnt/windows  # С типом ФС
mount -o loop image.iso /mnt/iso  # ISO-образ
mount -a                       # Всё из fstab

# === Перемонтирование ===
mount -o remount,rw /          # Перемонтировать корень в RW
mount -o remount,ro /boot      # Защитить /boot

# === Размонтирование ===
umount /mnt/data               # По точке монтирования
umount /dev/sdb1               # По устройству
umount -f /mnt/nfs             # Принудительно
umount -l /mnt/busy            # Ленивое

# === fstab ===
cat /etc/fstab                 # Посмотреть fstab
cp /etc/fstab /etc/fstab.bak   # Резервная копия
mount /mnt/data                # Смонтировать конкретную точку из fstab
findmnt --fstab --verify       # Проверить синтаксис

# === Полезные скрипты ===
# Проверка, смонтировано ли
if findmnt /mnt/backup >/dev/null; then echo "OK"; fi

# Все ext4 разделы
findmnt -t ext4 -o TARGET,SOURCE

# Очистка "висячих" NFS
umount -a -t nfs4 -f
```

    fi
}

## ✅ Заключение

### Ключевые выводы

1. **`mount`** — базовая утилита для монтирования, используйте её когда знаете, что монтируете
2. **`findmnt`** — мощная утилита для анализа и поиска, используйте для скриптов и отладки
3. **`/etc/fstab`** — автоматизируйте монтирование, но всегда делайте бэкап перед правкой
4. **UUID** — всегда предпочтительнее, чем `/dev/sdX`, потому что не меняется
5. **`nofail`** и **`_netdev`** — обязательные опции для сетевых дисков, чтобы система загружалась, даже если NFS недоступен

### Типичные ошибки и их решение

| Ошибка | Решение |
|--------|---------|
| `mount point does not exist` | `sudo mkdir -p /точка/монтирования` |
| `device is busy` | `fuser -km /точка` или `umount -l` |
| `unknown filesystem type` | Установить драйвер: `ntfs-3g`, `exfat-fuse` |
| `permission denied` | Добавить `user` в опции или использовать `sudo` |
| `can't find in /etc/fstab` | Указать полный путь к устройству или использовать `mount -a` |
