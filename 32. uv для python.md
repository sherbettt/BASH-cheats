# Подробное руководство по использованию UV для Python

[UV](https://astral-sh.vercel.app/blog/uv-unified-python-packaging) — это современная альтернатива pip и pip-tools, разработанная Astral (создателями Ruff). Он предлагает значительное ускорение операций с пакетами Python. 
См. исходник https://github.com/astral-sh/uv

## **Что такое uv?**

**uv** — это быстрый Python пакетный менеджер и инструмент для работы с виртуальными окружениями от создателей Ruff. Он заменяет:
- pip
- venv
- virtualenv
- pipenv
- poetry (частично)

## Полная установка UV

### Способ 1: Через pip
```bash
pip install uv
```

### Способ 2: Через pipx (рекомендуется для изоляции)
```bash
pipx install uv
```

### Проверка установки
```bash
uv --version
```

## **Исправляем проблему с PATH для pipx**

```bash
# 1. Добавляем pipx в PATH
pipx ensurepath

# 2. Обновляем текущую сессию
source ~/.bashrc
# или
exec bash

# 3. Проверяем
which uv
uv --version
```

## **Работа с виртуальными окружениями UV**

### Создание виртуального окружения
```bash
# Создать виртуальное окружение в текущей директории
uv venv .venv

# Создать с определенной версией Python
uv venv .venv --python 3.11

# Создать с системными пакетами
uv venv .venv --system-site-packages
```

### **Вход в виртуальное окружение UV**
```bash
# Linux/Mac
source .venv/bin/activate

# Проверьте что вы внутри - в приглашении будет (.venv)
(.venv) user@host:~/project#
```

### **Выход из виртуального окружения UV**
```bash
# Деактивировать окружение
deactivate
```

## Детальное использование основных функций

### 1. Работа с виртуальными окружениями

#### Установка пакетов
```bash
# Глобально (через pipx уже установили)
uv --version

# В проект (создаст venv автоматически)
uv pip install pyinstaller
uv pip install requests pandas

# Установить из requirements.txt
uv pip install -r requirements.txt
```

#### Дополнительные параметры:
```bash
uv venv --python=3.10 .venv  # Спецификация версии Python
uv venv --seed .venv  # С предустановкой pip и setuptools
```

### 2. Управление пакетами

#### Установка пакетов
```bash
uv pip install package  # Один пакет
uv pip install package1 package2  # Несколько пакетов
uv pip install -e .  # Установка в режиме разработки
```

#### Установка из requirements.txt
```bash
uv pip install -r requirements.txt
```

#### Особые варианты установки:
```bash
uv pip install "package>=1.0,<2.0"  # Спецификация версий
uv pip install --no-deps package  # Без зависимостей
uv pip install --index-url https://pypi.org/simple/ package
```

### 3. Компиляция требований (аналог pip-compile)

#### Базовое использование:
```bash
uv pip compile requirements.in -o requirements.txt
```

#### Расширенные опции:
```bash
uv pip compile pyproject.toml -o requirements.txt  # Из pyproject.toml
uv pip compile --upgrade package requirements.in  # Обновить конкретный пакет
uv pip compile --all-extras requirements.in  # Со всеми extras
```

#### Генерация разных файлов:
```bash
uv pip compile requirements.in -o requirements.txt --python-version 3.8
uv pip compile dev-requirements.in -o dev-requirements.txt
```

### 4. Синхронизация окружения

```bash
uv pip sync requirements.txt  # Точное соответствие requirements.txt
uv pip sync requirements.txt --reinstall  # Принудительная переустановка
```

### 5. uvx — запуск без установки

```bash
# Запустить pyinstaller без установки
uvx pyinstaller --onefile app.py

# Запустить любую Python утилиту
uvx black myfile.py  # Форматировщик кода
uvx pytest tests/    # Запуск тестов

# С конкретной версией
uvx "pyinstaller==5.13" --onefile app.py
```

## Рабочий процесс с UV (детальный пример)

```text

```

```bash
uv pip compile requirements.in -o requirements.txt
```

6. **Синхронизация окружения**:
```bash

```

7. **Работа в команде**:
```bash
uv pip sync requirements.txt  # Для других разработчиков
```

## 🎯 **Применение uv в нашем случае для сборки проекта**

### **Полный цикл работы с UV для сборки проекта**

Подготовка вирт. окружения
```bash
# 1. Перейти в проект
cd ~/git/runtel_opensips_sbc

# Удалить старое окружение если нужно
rm -rf .venv

# 2. Создать виртуальное окружение UV
uv venv .venv

# 3. Войти в окружение
source .venv/bin/activate
```

Установить зависимости
```bash
# 1. Установим auto-py-to-exe для анализа
uv pip install auto-py-to-exe  # системная устанвока, если python был скомпилирован
uvx pip install auto-py-to-exe  # если python был устанволен пакетным менеджером

# 2. Установим asyncpg - асинхронный драйвер PostgreSQL для Python
uv pip install asyncpg
uvx pip install asyncpg

# 3. Или используем pyi-archive_viewer для анализа
pyi-archive_viewer dist/sbc-app

# 4. Установить pyinstaller (внутри окружения)
uv pip install pyinstaller

# 5.Установить зависимости из requirements.txt
uv pip install -r requirements.txt

# 6. Или если requirements.txt нет, установить основные
uv pip install flask sqlalchemy psycopg2-binary  # предполагаемые зависимости из app.py
```

Сборка pyinstaller'ом
```bash
# 4. Быстрый анализ импортов
grep -E "^(import|from)" ваш_скрипт.py | head -20

# 5. Проверить установку
pyinstaller --version

# 6. Собрать ваш скрипт
pyinstaller --onefile ваш_скрипт.py

# Вариант 1: Простая сборка утилиты
pyinstaller --onefile --clean block_ip_fail2ban.py

# Вариант 2: С названием
pyinstaller --onefile --clean --name block-ip-util block_ip_fail2ban.py
```

Добавление недостающего модуля
```bash
# Не хватает модуля aiosqlite — это асинхронный драйвер для SQLite.
# Проверим импорт
python -c "import aiosqlite; print('aiosqlite доступен')"

# Установим aiosqlite
uv pip install aiosqlite

# Удалим старую сборку
rm -rf build/ dist/ sbc-app.spec

# Просто добавим aiosqlite и пересоберём
pyinstaller --onefile --clean --name sbc-app \
  --hidden-import "aiosqlite" \
  app.py

# Пересоберём с добавлением hidden import
pyinstaller --onefile --clean --name sbc-app \
  --hidden-import "aiosqlite" \
  --hidden-import "_cffi_backend" \
  --hidden-import "aiohttp" \
  --hidden-import "sqlalchemy" \
  --add-data "config.py:." \
  --add-data "models.py:." \
  --add-data "database.py:." \
  --add-data "auth_utils.py:." \
  app.py

# Посмотрим что импортирует database.py
head -20 database.py

# Выйти из окружения
deactivate
```

<details>
<summary>❗ Зачем нужны --hidden-import ❗</summary>

Зачем нужны --hidden-import если пакеты уже установлены? 🤔
<br/> Проблема: PyInstaller анализирует только явные импорты в коде. Но некоторые пакеты используют:
- Динамические импорты - importlib.import_module(), __import__()
- Плагины/расширения - загружаются во время выполнения
- Сonditional imports - импорты внутри функций/условий
- Вендорные копии - пакеты внутри других пакетов

```python
# В database.py импортируется create_async_engine
# SQLAlchemy динамически загружает драйвер aiosqlite во время выполнения
# PyInstaller не видит эту зависимость при статическом анализе

engine = create_async_engine(DATABASE_URL, echo=True)
# ^--- SQLAlchemy динамически импортирует aiosqlite здесь
```

**Какие `--hidden-import` нужны именно вам:**

 **1. `aiosqlite` - КРИТИЧЕСКИ ВАЖЕН**
```python
# database.py использует SQLAlchemy с aiosqlite
# SQLAlchemy загружает его динамически
DATABASE_URL = "sqlite+aiosqlite:///./app.db"
# Без --hidden-import "aiosqlite" получите ошибку:
# ModuleNotFoundError: No module named 'aiosqlite'
```

**2. `_cffi_backend` - для bcrypt**
```python
# auth_utils.py использует bcrypt для хэширования паролей
# bcrypt использует CFFI (C Foreign Function Interface)
# _cffi_backend - это нативный модуль
hashed_password = bcrypt.hashpw(password.encode(), bcrypt.gensalt())
```

**3. `aiohttp` - уже виден (можно не добавлять)**
```python
# app.py явно импортирует aiohttp
from aiohttp import web, web_request
# PyInstaller видит этот импорт
```

**4. `sqlalchemy` - уже виден (можно не добавлять)**
```python
# app.py явно импортирует sqlalchemy
from sqlalchemy import select, or_, desc, text, update, delete
```

**Почему?** Потому что:
- `aiohttp` и `sqlalchemy` уже видны (явные импорты)
- `_cffi_backend` может быть опциональным (предупреждение, но не ошибка)
- `aiosqlite` - обязателен (динамическая загрузка через SQLAlchemy)

**Проверим что действительно нужно:**
```bash
# Соберём без hidden imports для теста
pyinstaller --onefile --clean --name test-no-hidden app.py

# Запустим
./dist/test-no-hidden 2>&1 | head -20

# Увидим ошибку про aiosqlite
```

**Как узнать какие hidden imports нужны:**

1. **Соберите без hidden imports**
2. **Запустите и посмотрите ошибки**
3. **Добавьте только те модули, которых не хватает**

```bash
# Диагностика
./dist/sbc-app 2>&1 | grep -i "ModuleNotFoundError\|ImportError"
```

## Советы по оптимизации

1. Используйте `--cache-dir` для указания кастомного кэша:
```bash
uv pip install --cache-dir ~/.uv-cache package
```

2. Для CI/CD используйте флаг `--no-cache`:
```bash
uv pip install --no-cache package
```

3. Параллельная установка:
```bash
UV_PIP_INSTALL_CONCURRENCY=8 uv pip install -r requirements.txt
```

## Отличия от pip

1. UV кэширует зависимости более эффективно
2. Поддержка разрешения зависимостей в параллельном режиме
3. Более строгая проверка совместимости пакетов
4. Автоматическое использование бинарных дистрибутивов (wheels)

## Решение проблем

Если возникают ошибки:
1. Попробуйте очистить кэш:
```bash
uv clean
```

2. Используйте более подробное логирование:
```bash
UV_LOG_LEVEL=debug uv pip install package
```

3. Для проблем с совместимостью:
```bash
uv pip install --resolution=lowest-direct package
```
