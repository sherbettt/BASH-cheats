# Руководство по работе с Aptly
## Управление репозиториями, публикация и подключение клиентов

---


## 🆕 **Создание локального репозитория и публикация**

### **1. Создание нового локального репозитория**

#### 📌 **Базовое создание репозитория:**
```bash
# Простой репозиторий
aptly repo create мой-репозиторий

# С указанием дистрибутива и компонента
aptly repo create runtel-trixie -comment="Deb13 repo main/release" -distribution="trixie" -component="main"

# С несколькими компонентами
aptly repo create runtel-trixie -comment="Deb13 repo main/release" -distribution="trixie" -component="main,contrib,non-free"

# С комментарием
aptly repo create -comment="Репозиторий для проекта XYZ" проект-xyz

# Для конкретной архитектуры
aptly repo create проект-amd64 -architectures="amd64,i386"
```

#### 📌 **Редактирование созданного репозитория:**
```bash
# Изменить параметры существующего репозитория
aptly -distribution=trixie -component=dev -comment="Deb13 repo dev/unstable" repo edit runtel-trixie-dev
```

#### 📌 **Практический пример:**
```bash
# Создание репозитория для trixie
aptly -distribution=trixie -component=main -comment="Deb13 repo main/release" repo create runtel-trixie

# Создание dev-репозитория
aptly -distribution=trixie -component=dev -comment="Deb13 repo dev/unstable" repo create runtel-trixie-dev

# Проверка создания
aptly repo list
aptly repo show runtel-trixie
```

---

### **2. Добавление пакетов в новый репозиторий**

```bash
# Добавить один пакет
aptly repo add мой-репозиторий /path/to/package.deb

# Добавить все пакеты из директории
aptly repo add мой-репозиторий /path/to/debs/

# Добавить с удалением исходных файлов
aptly repo add -remove-files мой-репозиторий /path/to/debs/

# Добавить с фильтрацией
aptly repo add -filter="Priority (required)" мой-репозиторий /path/to/debs/

# Проверить добавленные пакеты
aptly repo search мой-репозиторий
aptly repo show -with-packages мой-репозиторий
```

---

### **3. Публикация репозитория**

#### 📌 **Подготовка к публикации (если уже есть публикация):**
```bash
# Проверка существующих публикаций
aptly publish list

# Удаление существующей публикации (если нужно)
aptly publish drop trixie
```

#### 📌 **Создание публикации:**
```bash
# Простая публикация
aptly publish repo -distribution=trixie -component=main runtel-trixie

# Публикация с несколькими компонентами
aptly publish repo -distribution=trixie -component=main,dev runtel-trixie runtel-trixie-dev

# Публикация с указанием архитектур
aptly publish repo -distribution=trixie -architectures="amd64,i386" -component=main runtel-trixie
```

#### 📌 **Практический пример (из вашего лога):**
```bash
# Удаляем старую публикацию, если она существует
aptly publish drop trixie

# Создаём новую публикацию с двумя компонентами
aptly publish repo -distribution=trixie -component=main,dev runtel-trixie runtel-trixie-dev

# Проверяем результат
aptly publish list | ccat
aptly publish show trixie
```

**Результат:**
```
Published repositories:
  * ./trixie [amd64] publishes {
      dev: [runtel-trixie-dev]: Deb13 repo dev/unstable,
      main: [runtel-trixie]: Deb13 repo main/release
  }
```

---

### **4. Проверка состояния после публикации**

```bash
# Список всех репозиториев
aptly repo list

# Список опубликованных репозиториев
aptly publish list

# Детальная информация о публикации
aptly publish show trixie

# Поиск пакетов в репозитории
aptly repo search runtel-trixie | ccze -A

# Просмотр структуры публикации в файловой системе
ls -la /var/www/aptly/public/trixie/dists/trixie/
```

## 🌐 **Подключение клиентов к репозиториям**

### **Строка подключения для клиента:**

```bash
# Основная строка для репозитория trixie
deb http://your-server/ trixie dev main

# Где:
# your-server    - адрес вашего сервера
# trixie         - имя дистрибутива
# dev, main      - компоненты (порядок не важен)
```

### **Пример для вашего сервера (с аутентификацией):**

```bash
# Для доступа к borishof (из текущей конфигурации)
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster borishof

# Для доступа к trixie (новый пример)
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru trixie dev main

# Или без аутентификации (если доступ открыт)
deb http://projects.runtel.ru trixie dev main
```

### **Разбор строки подключения:**
```
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster borishof
↑    ↑                                     ↑              ↑       ↑
тип  протокол с аутентификацией          сервер        дистр   компонент
```

### **Почему такая структура:**

1. **Аутентификация**: `borishof:pheiporu1ieshaeL` - логин:пароль
2. **Сервер**: `projects.runtel.ru` - адрес сервера
3. **Дистрибутив**: соответствует имени в `aptly publish list`
4. **Компонент**: соответствует компоненту в публикации

### **Действия на клиенте:**
```bash
# Добавить строку в /etc/apt/sources.list
echo "deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru trixie dev main" | sudo tee -a /etc/apt/sources.list

# Или создать отдельный файл
echo "deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru trixie dev main" | sudo tee /etc/apt/sources.list.d/runtel-trixie.list

# Обновить список пакетов
sudo apt update

# Поиск пакетов
apt search runtel

# Установка пакета
sudo apt install runtel-task-v2
```

## 🔧 **Интеграция с CI/CD (Ansible)**

### **Пример плейбука для публикации репозитория:**

```yaml
- name: Удалить старую публикацию (если есть)
  shell: |
    aptly publish drop {{ distribution_name }}
  ignore_errors: yes

- name: Создать публикацию с компонентами
  shell: |
    aptly publish repo -distribution={{ distribution_name }} \
      -component={{ components }} \
      {{ repo_name }} \
      {{ repo_name_dev }}
  register: publish_result

- name: Проверить успешность публикации
  shell: |
    aptly publish show {{ distribution_name }}
  register: verify_result
```

### **Пример для trixie:**
```yaml
- name: Публикация репозитория trixie
  shell: |
    aptly publish repo -distribution=trixie -component=main,dev runtel-trixie runtel-trixie-dev

- name: Проверка публикации
  shell: |
    aptly publish list | grep trixie
  register: check

- name: Вывод результата
  debug:
    msg: "Публикация создана: {{ check.stdout }}"
```

## 📊 **Мониторинг и обслуживание**

### **Скрипт проверки состояния:**
```bash
#!/bin/bash
echo "=== Состояние Aptly на repo-new ==="
echo

echo "1. Локальные репозитории:"
aptly repo list

echo -e "\n2. Опубликованные репозитории:"
aptly publish list

echo -e "\n3. Информация о runtel-trixie:"
aptly repo show runtel-trixie | grep -E "(Number of packages|Comment)"

echo -e "\n4. Пакеты в runtel-trixie:"
aptly repo search runtel-trixie | head -10
```

### **Обновление публикации после добавления пакетов:**
```bash
# Добавить новые пакеты
aptly repo add runtel-trixie /path/to/new/debs/

# Обновить публикацию
aptly publish update trixie

# Проверить обновление
aptly publish show trixie
```

## 🚨 **Типичные проблемы и решения**

### **1. Ошибка "prefix/distribution already used"**
```bash
# Проблема: дистрибутив уже опубликован
ERROR: prefix/distribution already used by another published repo: ./trixie

# Решение: удалить существующую публикацию
aptly publish drop trixie

# Или использовать другой дистрибутив
aptly publish repo -distribution=trixie-new -component=main,dev runtel-trixie runtel-trixie-dev
```

### **2. Клиент не видит пакеты:**
```bash
# Проверить доступность репозитория
curl -s http://projects.runtel.ru/trixie/dists/trixie/Release

# Проверить наличие Packages.gz
curl -s http://projects.runtel.ru/trixie/dists/trixie/main/binary-amd64/Packages.gz | gunzip | head -5

# Проверить обновление публикации
aptly publish show trixie
```

### **3. Ошибка при добавлении пакета:**
```bash
# Проверить права доступа
ls -la /path/to/package.deb

# Проверить целостность пакета
dpkg -I /path/to/package.deb

# Добавить с отладкой
aptly repo add -verbose runtel-trixie /path/to/package.deb
```

### **4. Конфликты версий:**
```bash
# Просмотреть все версии пакета
aptly repo search runtel-trixie | grep "runtel-task-v2" | sort -V

# Удалить конфликтующую версию
aptly repo remove runtel-trixie "runtel-task-v2 (=конфликтующая-версия)"
```

## 📝 **Краткая шпаргалка**

### **Команды для создания и публикации:**
```bash
# 1. Создать репозиторий
aptly -distribution=trixie -component=main -comment="Comment" repo create my-repo

# 2. Добавить пакеты
aptly repo add my-repo /path/to/debs/

# 3. Опубликовать
aptly publish repo -distribution=trixie -component=main,dev my-repo my-repo-dev

# 4. Проверить
aptly publish list
aptly publish show trixie
```

### **Команды для проверки:**
```bash
# Локальные репозитории
aptly repo list

# Опубликованные
aptly publish list

# Поиск пакетов
aptly repo search runtel-trixie

# Детальная информация
aptly repo show runtel-trixie
```

### **Пути к данным:**
```
Локальные репозитории: /var/lib/aptly/db
Публикации:           /var/www/aptly/public/
Конфигурация:         /etc/aptly.conf
Снимки:               /var/lib/aptly/snapshots/
```

### **Строки подключения клиентов:**
```bash
# С аутентификацией
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru trixie dev main

# Без аутентификации
deb http://projects.runtel.ru trixie dev main

# Для buster
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster borishof
```

## 🔑 **Ключевые выводы**

1. **Структура публикации**: `дистрибутив` → `компоненты` → `репозитории`
2. **Строка подключения**: `deb http://сервер дистрибутив компонент1 компонент2`
3. **Обновление**: после добавления пакетов всегда нужно выполнять `aptly publish update`
4. **Проверка**: используйте `aptly publish show` для детального просмотра
5. **Удаление**: перед созданием новой публикации удалите старую командой `aptly publish drop`

---

Эта конфигурация обеспечивает централизованное управление пакетами с поддержкой нескольких компонентов в рамках одного дистрибутива и аутентификацией доступа.
