## 🆕 **Создание локального репозитория и публикация**

### **1. Создание нового локального репозитория**

#### Базовое создание репозитория:
```bash
# Создать простой репозиторий
aptly repo create мой-репозиторий
aptly -distribution=trixie -component=main -comment="Deb13 repo main/release" repo create runtel-trixie
# или
aptly repo create runtel-trixie -comment="Deb13 repo main/release" -distribution="trixie" -component="main"
# или
aptly repo create runtel-trixie -comment="Deb13 repo main/release" -distribution="trixie" -component="main,contrib,non-free"

# Создать с комментарием
aptly repo create -comment="Репозиторий для проекта XYZ" проект-xyz

# Создать для конкретного дистрибутива
aptly repo create проект-buster -distribution="buster"

# Создать для конкретной архитектуры
aptly repo create проект-amd64 -architectures="amd64,i386"

# Если вдруг были публикации, сносим их полностью
aptly publish drop trixie

# Создаём новые публикации
aptly publish repo -distribution=trixie -component=main,dev runtel-trixie runtel-trixie-dev

# проверяем публикации
aptly publish list | ccat
aptly publish show trixie
```

#### Редактирвоание созданного репозитория
```bash
aptly -distribution=trixie -component=dev -comment="Deb13 repo dev/unstable" repo edit runtel-trixie-dev
```

#### Проверка создания:
```bash
# Проверить список репозиториев
aptly repo list
aptly repo list -raw

# Посмотреть детали нового репозитория
aptly repo show мой-репозиторий
```

### **2. Добавление пакетов в новый репозиторий**

```bash
# Добавить один пакет
aptly repo add мой-репозиторий /path/to/package.deb

# Добавить все пакеты из директории
aptly repo add мой-репозиторий /path/to/debs/

# Добавить с удалением исходных файлов
aptly repo add -remove-files мой-репозиторий /path/to/debs/

# Добавить с фильтрацией
aptly repo add -filter="Priority (required)" мой-репозиторий /path/to/debs/

# Проверить добавленные пакеты
aptly repo search мой-репозиторий
aptly repo show -with-packages мой-репозиторий
```

### **3. Создание и публикация дистрибутива**

#### Создание публикации из репозитория:
```bash
# Опубликовать репозиторий как новый дистрибутив
aptly publish repo мой-репозиторий мой-дистрибутив

# Опубликовать с компонентом
aptly publish repo -component="main" мой-репозиторий мой-дистрибутив

# Опубликовать с несколькими компонентами
aptly publish repo -component="main" -component="contrib" мой-репозиторий мой-дистрибутив

# Опубликовать в указанную директорию
aptly publish repo -distribution="stable" -architectures="amd64,i386" \
  -component="main" мой-репозиторий моя-директория
```

#### Пример создания полного рабочего процесса:
```bash
# 1. Создать репозиторий для нового проекта
aptly repo create -comment="Репозиторий проекта NewApp" newapp-buster

# 2. Добавить пакеты
aptly repo add newapp-buster /tmp/builds/newapp/*.deb

# 3. Опубликовать как новый дистрибутив
aptly publish repo -component="newapp" newapp-buster buster

# 4. Проверить публикацию
aptly publish list
aptly publish show buster
```

### **4. Работа с существующими публикациями**

#### Добавление нового компонента в существующий дистрибутив:
```bash
# Создать репозиторий для нового компонента
aptly repo create новый-компонент-buster

# Добавить пакеты
aptly repo add новый-компонент-buster /path/to/packages/

# Опубликовать как новый компонент в существующем дистрибутиве
aptly publish repo -component="новый-компонент" новый-компонент-buster buster

# Проверить обновление
aptly publish show buster
```

#### Создание нового дистрибутива на основе существующего:
```bash
# Создать снимок существующего репозитория
aptly snapshot create стабильный-снимок from repo borishof-buster

# Опубликовать снимок как новый дистрибутив
aptly publish snapshot стабильный-снимок стабильный-дистрибутив

# Или создать смешанный дистрибутив
aptly publish snapshot -component="borishof" стабильный-снимок mixed-distro
```

### **5. Управление версиями и снимками**

#### Создание снимков для бэкапа:
```bash
# Создать снимок репозитория
aptly snapshot create backup-$(date +%Y%m%d) from repo borishof-buster

# Создать снимок публикации
aptly snapshot create publish-backup-$(date +%Y%m%d) from publish buster

# Просмотреть все снимки
aptly snapshot list

# Восстановить из снимка
aptly repo import backup-20240101 новый-репозиторий "runtel-borishof-v2"
```

#### Управление версиями пакетов:
```bash
# Показать все версии пакета
aptly repo search мой-репозиторий "имя-пакета"

# Создать снимок с определенной версией
aptly snapshot create версия-1.0 from repo мой-репозиторий \
  "имя-пакета (>= 1.0), имя-пакета (<< 1.1)"
```

## 🚀 **Основные команды для работы**

### **1. Проверка состояния**

#### Показать все локальные репозитории
```bash
aptly repo list
```

#### Показать все опубликованные репозитории
```bash
aptly publish list
```

#### Проверить конкретный репозиторий borishof-buster
```bash
# Посмотреть информацию о репозитории
aptly repo show borishof-buster

# Поиск пакетов в репозитории
aptly repo search borishof-buster

# Фильтрация по dev/release версиям
aptly repo search borishof-buster | awk '/borishof/ && /release/ || /dev/ {print}'
```

#### Просмотр структуры публикации
```bash
# Детальная информация о публикации buster
aptly publish show buster

# Проверить какие компоненты доступны
ls -la /var/www/aptly/public/buster/dists/buster/
```

### **2. Добавление пакетов в репозитории**

#### Добавление пакетов в borishof-buster
```bash
# Добавить один пакет
aptly repo add -remove-files borishof-buster /path/to/package.deb

# Добавить все пакеты из директории
aptly repo add borishof-buster /path/to/debs/

# Добавить с сохранением файлов
aptly repo add -remove-files=false borishof-buster /path/to/package.deb
```

#### Обновление публикации после добавления пакетов
```bash
# Обновить публикацию buster (все компоненты)
aptly publish update buster

# Проверить обновление
aptly publish show buster
```

### **3. Поиск и управление пакетами**

#### Поиск пакетов
```bash
# Поиск всех пакетов в borishof-buster
aptly repo search borishof-buster

# Поиск конкретного пакета
aptly repo search borishof-buster "runtel-borishof-v2"

# Поиск по версии
aptly repo search borishof-buster | grep "1.5.0"
```

#### Просмотр всех пакетов
```bash
# Показать все пакеты с детальной информацией
aptly repo show -with-packages borishof-buster

# Подсчет пакетов
aptly repo show borishof-buster | grep "Number of packages"
```

#### Управление версиями пакетов
```bash
# Удалить конкретную версию
aptly repo remove borishof-buster "runtel-borishof-v2 (=1.5.0~release-487)"

# Удалить все версии пакета
aptly repo remove borishof-buster "runtel-borishof-v2"

# Удалить старые dev версии
aptly repo remove borishof-buster "runtel-borishof-v2 (<< 1.5.0)"
```

## 🌐 **Подключение клиентов к репозиториям**

### **Основной способ подключения:**

#### Для доступа к репозиторию borishof:
```bash
# Добавить в /etc/apt/sources.list или /etc/apt/sources.list.d/borishof.list
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster borishof

# или проще
deb http://projects.runtel.ru buster borishof
```

#### Разбор строки подключения:
```
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster borishof
↑    ↑                                     ↑              ↑       ↑
тип  протокол с аутентификацией          сервер        дистр   компонент
```

### **Почему именно такая строка:**

1. **Аутентификация**: `borishof:pheiporu1ieshaeL` - логин и пароль для доступа
2. **Сервер**: `projects.runtel.ru` - адрес сервера Aptly
3. **Дистрибутив**: `buster` - соответствует имени публикации в `aptly publish list`
4. **Компонент**: `borishof` - соответствует компоненту в секции `borishof: [borishof-buster]`

### **Альтернативные варианты подключения:**

#### Если нужен доступ без аутентификации (при открытом доступе):
```bash
deb http://projects.runtel.ru buster borishof
```

#### Для доступа к другим компонентам:
```bash
# Системные пакеты
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster main

# Пакеты hh
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster hh

# Пакеты kept
deb http://borishof:pheiporu1ieshaeL@projects.runtel.ru buster kept
```

### **Подключение к новому созданному дистрибутиву:**

```bash
# Если создали новый дистрибутив "мой-дистрибутив"
deb http://projects.runtel.ru мой-дистрибутив main

# Если создали новый компонент "newapp" в существующем дистрибутиве buster
deb http://projects.runtel.ru buster newapp
```

### **Действия на клиенте после настройки:**
```bash
# Обновить список пакетов
sudo apt update

# Поиск доступных пакетов
apt search runtel-borishof

# Установка пакета
sudo apt install runtel-borishof-v2

# Установка конкретной версии
sudo apt install runtel-borishof-v2=1.5.0~release-487
```

## 🔧 **Интеграция с CI/CD (Ansible)**

### **Пример плейбука для добавления пакетов:**
```yaml
- name: Добавить пакеты в репозиторий borishof-buster
  shell: |
    aptly repo add -remove-files borishof-buster /tmp/builds/{{ project_name }}/
  register: add_result
  changed_when: "'No packages found' not in add_result.stdout"

- name: Обновить публикацию
  shell: aptly publish update buster
  when: add_result.changed

- name: Проверить добавление пакетов
  shell: |
    aptly repo search borishof-buster | grep "{{ package_name }}"
  register: verify_result
```

### **Плейбук для создания нового репозитория и публикации:**
```yaml
- name: Создать новый репозиторий
  shell: |
    aptly repo create {{ repo_name }}

- name: Добавить пакеты в новый репозиторий
  shell: |
    aptly repo add {{ repo_name }} {{ deb_files_path }}

- name: Опубликовать репозиторий
  shell: |
    aptly publish repo -component="{{ component_name }}" {{ repo_name }} {{ distribution_name }}

- name: Настроить веб-доступ
  shell: |
    chmod -R 755 /var/www/aptly/public/
    systemctl reload nginx
```

## 📊 **Мониторинг и обслуживание**

### **Скрипт проверки состояния:**
```bash
#!/bin/bash
echo "=== Состояние Aptly на repo-new ==="
echo

echo "1. Локальные репозитории:"
aptly repo list

echo -e "\n2. Опубликованные репозитории:"
aptly publish list

echo -e "\n3. Статистика borishof-buster:"
aptly repo show borishof-buster | grep -E "(Number of packages|Storage)"

echo -e "\n4. Последние пакеты в borishof-buster:"
aptly repo search borishof-buster | tail -20
```

### **Очистка старых пакетов:**
```bash
#!/bin/bash
# Удалить старые dev версии (оставить последние 10)
aptly repo search borishof-buster | grep "dev" | sort -V | head -n -10 | \
while read package; do
    aptly repo remove borishof-buster "$package"
done

# Обновить публикацию
aptly publish update buster
```

## 🚨 **Типичные проблемы и решения**

### **1. Клиент не видит пакеты:**
```bash
# Проверить доступность репозитория
curl -s http://projects.runtel.ru/buster/dists/buster/borishof/binary-amd64/Packages.gz | gunzip | head -5

# Проверить аутентификацию
curl -u borishof:pheiporu1ieshaeL -s http://projects.runtel.ru/buster/dists/buster/Release

# Проверить обновление публикации
aptly publish show buster
```

### **2. Ошибка при добавлении пакета:**
```bash
# Проверить права доступа
ls -la /path/to/package.deb

# Проверить целостность пакета
dpkg -I /path/to/package.deb

# Добавить с отладкой
aptly repo add -verbose borishof-buster /path/to/package.deb
```

### **3. Конфликты версий:**
```bash
# Просмотреть все версии пакета
aptly repo search borishof-buster | grep "runtel-borishof-v2" | sort -V

# Удалить конфликтующую версию
aptly repo remove borishof-buster "runtel-borishof-v2 (=конфликтующая-версия)"
```

## 📝 **Краткая шпаргалка**

### **Создание и публикация:**
```bash
# Создание репозитория
aptly repo create новый-репозиторий

# Добавление пакетов
aptly repo add новый-репозиторий /path/to/debs/

# Публикация дистрибутива
aptly publish repo -component="main" новый-репозиторий новый-дистрибутив

# Обновление публикации
aptly publish update дистрибутив
```

### **Проверка:**
```bash
# Проверка репозиториев
aptly repo list
aptly publish list

# Поиск пакетов
aptly repo search репозиторий

# Клиентская настройка
deb http://projects.runtel.ru дистрибутив компонент
sudo apt update
```

### **Пути к данным:**
```
Локальные репозитории: /var/lib/aptly/db
Публикации: /var/www/aptly/public/
Конфигурация: /etc/aptly.conf
Снимки: /var/lib/aptly/snapshots/
```

Эта конфигурация обеспечивает централизованное управление пакетами с аутентификацией доступа и поддержкой нескольких компонентов в рамках одного дистрибутива.
