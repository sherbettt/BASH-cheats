### ⚙️ Техническое оснащение GKD Pixel 2

Это очень компактная, полностью открытая Linux-консоль для ретро-игр.

*   **Процессор (CPU)**: Rockchip RK3326S — 4 ядра Cortex-A55, частота 1.3–1.5 ГГц.
*   **Графика (GPU)**: Mali-G31 MP2.
*   **Оперативная память (RAM)**: 1 ГБ DDR3.
*   **Экран**: 2.4 дюйма, IPS, разрешение 640×480 пикселей (333 PPI).
*   **Аккумулятор**: 1800 мАч.
*   **Корпус**: Анодированный алюминиевый сплав.
*   **Система и игры**: Полностью загружаются с карты памяти **microSD** — никакой встроенной памяти для игр нет. Консоль не включится без правильно подготовленной карты .
*   **Мощность**: Этого железа достаточно для эмуляции ZX Spectrum, PlayStation 1, SNES, а также части игр для Nintendo 64 и Dreamcast.

# Инструкция: Установка Rogue OS на GKD Pixel 2

## Цель работы

Установить кастомную прошивку **Rogue OS** на портативную ретро-консоль **GKD Pixel 2**, чтобы получить доступ к расширенным возможностям эмуляции, в том числе для **ZX Spectrum**, а также к дополнительным функциям, которых нет в заводской прошивке.

Rogue OS — это популярная альтернативная прошивка, которая добавляет множество улучшений: обновленное ядро, свежие версии эмуляторов, встроенный музыкальный плеер, поддержку тем и скрейпера для обложек игр, а также PortMaster для портов ПК-игр .

---

## 📦 Что потребовалось

1.  **GKD Pixel 2** — сама консоль.
2.  **MicroSD-карта** — на 256 ГБ (в данном случае от Digma).
3.  **Кардридер** — для подключения карты к компьютеру.
4.  **Компьютер** — на Linux (дистрибутив Ximper, основанный на Alt Linux) [см. вывод `cat /etc/os-release`].
5.  **Программы**:
    *   **7-Zip** — для распаковки архивов.
    *   **balenaEtcher** — для записи образа на SD-карту .
    *   **GParted** — для управления разделами на SD-карте .

---

## 🔄 Пошаговый процесс

### Шаг 1: Скачивание прошивки

Прошивка Rogue OS распространяется на GitHub в виде двух частей: базового образа и обновлений.

**Почему так:** Процесс установки устроен двухэтапно. Сначала записывается базовый образ версии 1.0, который создает необходимую структуру разделов на карте. Затем поверх него копируются файлы обновления до актуальной версии .

1.  **Скачиваем базовый образ v1.0:**
    *   Переходим на страницу релизов: [https://github.com/Ninoh-FOX/PIXEL2-ROGUE-OS/releases/tag/GKDPIXELII_1.0](https://github.com/Ninoh-FOX/PIXEL2-ROGUE-OS/releases/tag/GKDPIXELII_1.0)
    *   Скачиваем **все** файлы: `RogueOS-v1.0.zip`, `RogueOS-v1.0.z01`, `RogueOS-v1.0.z02`, `RogueOS-v1.0.z03`, `RogueOS-v1.0.z04`, `RogueOS-v1.0.z05` .

2.  **Скачиваем файлы обновления до v1.22:**
    *   Переходим на страницу последнего релиза: [https://github.com/Ninoh-FOX/PIXEL2-ROGUE-OS/releases/tag/GKDPIXELII_1.22](https://github.com/Ninoh-FOX/PIXEL2-ROGUE-OS/releases/tag/GKDPIXELII_1.22)
    *   Скачиваем файлы: **`Image`** (ядро системы) и **`SYSTEM`** (корневая файловая система). В инструкции к релизу сказано: *"Only copy SYSTEM and Image files in EMUELEC fat32 partition on ROGUE OS V1.0"* .

> **Важно:** В папке `runtimes/`, которую вы также видели, находятся файлы для PortMaster (среды выполнения для портов игр). На этом этапе они не нужны.

### Шаг 2: Распаковка и подготовка образа

Файлы `RogueOS-v1.0.zip` и `RogueOS-v1.0.z0*` — это многотомный архив. Его нужно распаковать, чтобы получить один файл образа `.img`.

**Что делали в терминале:**
```bash
cd /home/kkorablin/Programs/PIXEL2-ROGUE-OS/
unzip RogueOS-v1.0.zip
```
В результате появился файл `RogueOS-v1.0.img` размером 8.1 ГБ.

### Шаг 3: Запись образа на SD-карту

**Почему:** Для создания загрузочной карты с правильной структурой разделов используется специальная утилита, которая побайтово копирует образ .

**Что делали:**
*   Вставили SD-карту (устройство `/dev/sdb`) в кардридер.
*   Запустили **balenaEtcher** ([https://etcher.balena.io/](https://etcher.balena.io/)) .
*   Выбрали файл `RogueOS-v1.0.img` как источник.
*   Выбрали SD-карту `/dev/sdb` как цель.
*   Нажали **Flash!**.

**Результат:** После записи на карте автоматически создались три раздела [см. вывод `lsblk -pf`]:
*   `/dev/sdb1` — **EMUELEC** (FAT32, системный). Содержит загрузчик, ядро и базовые файлы.
*   `/dev/sdb2` — **STORAGE** (ext4). Служебный раздел.
*   `/dev/sdb3` — **ROMS** (FAT32). Раздел для игр.

### Шаг 4: Расширение раздела ROMS

**Почему:** Базовый образ создает раздел ROMS небольшого размера. Чтобы использовать всю емкость карты (в вашем случае 233 ГБ), его нужно расширить, иначе большая часть места останется недоступной .

**Что делали через GParted:**
*   Запустили `sudo gparted /dev/sdb`.
*   Нашли раздел **ROMS** (`/dev/sdb3`).
*   Кликнули правой кнопкой → **Resize/Move**.
*   Перетащили правый ползунок вправо, чтобы занять всё свободное неразмеченное пространство .
*   Нажали **Apply** (зеленая галочка) для применения изменений.

**Результат:** Размер раздела ROMS увеличился до **226,8 ГБ** [см. вывод `lsblk`].

> **Альтернатива через терминал (если GParted не установлен):**
> 1. Установить GParted: `sudo apt-get install gparted` (для Alt Linux).
> 2. Или использовать `parted` в консоли, но GParted проще и нагляднее.

### Шаг 5: Обновление системы до версии 1.22

**Почему:** Файлы `Image` и `SYSTEM`, скачанные с релиза v1.22, содержат обновленное ядро, драйверы, эмуляторы и исправления. Их нужно заменить поверх версии 1.0 .

**Что делали в терминале:**
```bash
# Создали точку монтирования и примонтировали системный раздел
sudo mkdir -p /mnt/emuelec
sudo mount /dev/sdb1 /mnt/emuelec

# Скопировали новые файлы, заменив старые
sudo cp /home/kkorablin/Programs/PIXEL2-ROGUE-OS/Image /mnt/emuelec/
sudo cp /home/kkorablin/Programs/PIXEL2-ROGUE-OS/SYSTEM /mnt/emuelec/

# Проверили, что файлы скопировались
ls -lh /mnt/emuelec/ | grep -E "Image|SYSTEM"

# Отмонтировали раздел
sudo umount /mnt/emuelec
```

**Результат:** На разделе EMUELEC файлы `Image` (20 МБ) и `SYSTEM` (1,9 ГБ) заменены на новые, что соответствует версии 1.22.

### Шаг 6: Добавление игр (ROMS)

**Почему:** После установки системы нужно наполнить раздел ROMS играми. Консоль автоматически распознает их по папкам.

1.  Вставили SD-карту в компьютер.
2.  Примонтировали раздел ROMS:
    ```bash
    sudo mkdir -p /mnt/roms
    sudo mount /dev/sdb3 /mnt/roms
    ```
3.  Внутри папки `/mnt/roms` есть каталоги для разных систем. Для **ZX Spectrum** нужно найти папку `spectrum` или `zxspectrum` . Если её нет — создать.
4.  Скопировать ROM-файлы (`.z80`, `.sna`, `.tap`, `.tzx`) в эту папку .
5.  Если для эмулятора требуются BIOS-файлы, поместить их в папку `bios` в корне раздела ROMS.
6.  Отмонтировать: `sudo umount /mnt/roms`.

### Шаг 7: Первый запуск

1.  Извлекли SD-карту из компьютера.
2.  Вставили в выключенную консоль GKD Pixel 2.
3.  Включили консоль. Первый запуск может занять 1-2 минуты. Консоль может перезагрузиться один раз — это нормально, так как система доустанавливает компоненты (например, Pyxel) .

## 🌐 Где брать игры (ROMs) для эмуляторов?

Важно: консоль не поставляется с играми. Их нужно добавлять самостоятельно.

### Для ZX Spectrum

Найти игры для ZX Spectrum можно на специализированных сайтах. Вот несколько проверенных ресурсов:

1.  **Bit16 (русскоязычный сайт с большой коллекцией)**: [http://bit16.ru/](http://bit16.ru/) . На сайте есть раздел с играми для ZX Spectrum, где можно скачать файлы напрямую. Многие игры сопровождаются описаниями.
2.  **Debian package spectrum-roms**: [https://packages.debian.org/bookworm/spectrum-roms](https://packages.debian.org/bookworm/spectrum-roms) . Это пакет с образами ПЗУ (BIOS) для разных версий Spectrum (48K, 128K, +2, +3). Они могут понадобиться для корректной работы некоторых эмуляторов.
3.  **Fantadrom Ltd**: [https://www.fantadrom.narod.ru/games/index.html](https://www.fantadrom.narod.ru/games/index.html) . Сайт с играми для ZX Spectrum, переведенными на русский язык, а также оригинальными разработками. Файлы в форматах `.TRD` или `.SCL`.

### Для других систем

Для других консолей (NES, SNES, Sega, и т.д.) файлы ROM можно найти на архивах вроде **Internet Archive** (поиск по запросам "NES ROMs", "SNES ROMs") или на специализированных ретро-сайтах. Их нужно помещать в соответствующие папки внутри раздела `ROMS` на SD-карте .
