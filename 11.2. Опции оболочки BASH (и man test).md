## Введение

В Bash существует два важных механизма, которые часто путают новички: **команда `test`** (и её синоним `[ ]`) для проверки условий и **опции оболочки** (устанавливаемые через `set`) для управления поведением самого Bash. Эта статья поможет разобраться в обоих инструментах и понять, как они взаимодействуют.

## Команда `test` и её эквиваленты

### Что такое `test`?

`test` — это встроенная команда (и отдельный исполняемый файл), которая проверяет условия и возвращает код завершения:
- **0 (ноль)** — условие истинно (true)
- **1 (один)** — условие ложно (false)

### Три формы записи

```bash
# Форма 1: классическая команда test
test -f /etc/passwd

# Форма 2: синоним в квадратных скобках (обязательно пробелы!)
[ -f /etc/passwd ]

# Форма 3: расширенная версия Bash (предпочтительнее)
[[ -f /etc/passwd ]]
```

**Важно:** `[` — это имя команды, поэтому после `[` обязательно нужен пробел, и закрывающая `]` тоже требует пробела перед собой.

### Отличия `[[ ]]` от `[ ]`

| Особенность | `[ ]` | `[[ ]]` |
|------------|-------|---------|
| Поддержка регулярных выражений | Нет | Да (`=~`) |
| Сравнение строк | Нужны кавычки | Безопасно без кавычек |
| Логические операторы | `-a` (and), `-o` (or) | `&&` и `\|\|` |
| Подстановка имен файлов | Поддерживается | Отключена |

## Операторы проверки в `test`

### 1. Проверка файлов

```bash
# Существование и тип
[ -e "файл" ]    # существует (любой тип)
[ -f "файл" ]    # обычный файл
[ -d "дир" ]     # директория
[ -L "ссылка" ]  # символическая ссылка
[ -b "dev" ]     # блочное устройство
[ -c "dev" ]     # символьное устройство
[ -p "pipe" ]    # именованный канал (FIFO)
[ -S "socket" ]  # сокет

# Права доступа
[ -r "файл" ]    # доступен для чтения
[ -w "файл" ]    # доступен для записи
[ -x "файл" ]    # доступен для выполнения
[ -u "файл" ]    # установлен SUID бит
[ -g "файл" ]    # установлен SGID бит
[ -k "файл" ]    # установлен sticky бит

# Сравнение файлов
[ "файл1" -nt "файл2" ]  # новее (newer than)
[ "файл1" -ot "файл2" ]  # старше (older than)
[ "файл1" -ef "файл2" ]  # одна и та же инода (equal files)
```

### 2. Проверка строк

```bash
[ -z "$str" ]    # строка пуста (zero length)
[ -n "$str" ]    # строка не пуста (non-zero)

# Сравнение строк
[ "$str1" = "$str2" ]   # равны (в POSIX)
[ "$str1" == "$str2" ]  # равны (в Bash)
[ "$str1" != "$str2" ]  # не равны
[[ "$str1" < "$str2" ]] # лексикографически меньше
[[ "$str1" > "$str2" ]] # лексикографически больше
```

### 3. Проверка чисел

```bash
[ "$a" -eq "$b" ]  # равно (equal)
[ "$a" -ne "$b" ]  # не равно (not equal)
[ "$a" -lt "$b" ]  # меньше (less than)
[ "$a" -le "$b" ]  # меньше или равно (less or equal)
[ "$a" -gt "$b" ]  # больше (greater than)
[ "$a" -ge "$b" ]  # больше или равно (greater or equal)
```

### 4. Логические операторы

```bash
# В классическом test
[ -f "$file" -a -r "$file" ]  # AND (и)
[ -d "$dir" -o -L "$dir" ]    # OR (или)
[ ! -f "$file" ]              # NOT (не)

# В расширенном test (предпочтительно)
[[ -f "$file" && -r "$file" ]]  # AND
[[ -d "$dir" || -L "$dir" ]]    # OR
[[ ! -f "$file" ]]              # NOT
```

### 5. Дополнительные проверки

```bash
[ -v "varname" ]      # переменная установлена (set)
[ -o "optname" ]      # опция оболочки включена (например, pipefail)
[ -t 0 ]              # файловый дескриптор связан с терминалом (stdin)
[ "$var" -nt "$file" ] # переменная новее файла (для цепочек)
```

## Опции оболочки Bash (`set`)
также см. **[bash_strict_mode](https://gist.github.com/mohanpedala/1e2ff5661761d3abd0385e8223e16425)**


### Основные опции

| Команда | Название | Описание |
|---------|----------|----------|
| `set -e` | **errexit** | Выход при ошибке (ненулевой код возврата) |
| `set -u` | **nounset** | Ошибка при использовании неопределенных переменных |
| `set -x` | **xtrace** | Отладка — печатать команды перед выполнением |
| `set -o pipefail` | **pipefail** | Возвращать ошибку, если любая команда в пайпе упала |

### Управление опциями

```bash
# Включить опции
set -e
set -o errexit  # то же самое

# Отключить опции
set +e
set +o errexit

# Просмотр текущих опций
set -o
echo $-
```

### Другие полезные опции

```bash
set -v          # verbose - печатать строки ввода
set -n          # noexec - читать команды без выполнения (проверка синтаксиса)
set -C          # noclobber - не перезаписывать файлы при перенаправлении
set -B          # braceexpand - включить расширение скобок (по умолч.)
set -H          # histexpand - включить историю (стиль !)

# Опции для интерактивной оболочки
set -o emacs    # режим редактирования emacs
set -o vi       # режим редактирования vi
set -o ignoreeof # игнорировать Ctrl-D
```

## Взаимодействие `test` и опций оболочки

### Как `set -e` влияет на `test`

Самое важное: **`set -e` НЕ завершает скрипт при "ложном" результате `test` в условных конструкциях!**

```bash
#!/bin/bash
set -e  # строгий режим

# ЭТО БЕЗОПАСНО — скрипт НЕ завершится
if [ -f "/несуществующий/файл" ]; then
    echo "Файл существует"
fi

# И ЭТО БЕЗОПАСНО
[ -f "/несуществующий/файл" ] && echo "Файл существует"

# И ЭТО БЕЗОПАСНО
[ -f "/несуществующий/файл" ] || echo "Файл не существует"

# А ВОТ ЭТО ОПАСНО — скрипт завершится!
[ -f "/несуществующий/файл" ]
echo "Эта строка не выполнится"
```

### Исключения для `set -e`

Bash НЕ завершает скрипт при ошибке в следующих контекстах:
1. В условии `if`, `elif`, `while`, `until`
2. В левой части `&&` или `||` (кроме последней команды)
3. В конвейере (если не включен `pipefail`)
4. В подстановке команд `$(...)` (кроме некоторых случаев)

### Примеры взаимодействия

```bash
#!/bin/bash
set -euo pipefail

check_directory() {
    local dir="${1:-}"
    
    # Безопасная проверка
    if [ ! -d "$dir" ]; then
        echo "Ошибка: директория $dir не существует" >&2
        return 1
    fi
    
    # Опасная проверка (завершит функцию при ошибке)
    [ -w "$dir" ] || {
        echo "Нет прав на запись в $dir" >&2
        return 1
    }
    
    echo "Директория $dir существует и доступна для записи"
}

# Использование
check_directory "/tmp"      # успех
check_directory "/root"     # может упасть
check_directory             # упадет из-за set -u (пустая переменная)
```

## Практические примеры

### Пример 1: Проверка аргументов скрипта

```bash
#!/bin/bash
set -euo pipefail

usage() {
    echo "Использование: $0 [-f файл] [-d директория]"
    exit 1
}

file=""
dir=""

while getopts "f:d:" opt; do
    case $opt in
        f) file="$OPTARG" ;;
        d) dir="$OPTARG" ;;
        *) usage ;;
    esac
done

# Проверки с test
if [ -z "$file" ] && [ -z "$dir" ]; then
    echo "Ошибка: нужно указать файл или директорию"
    usage
fi

if [ -n "$file" ] && [ ! -f "$file" ]; then
    echo "Ошибка: файл $file не существует"
    exit 1
fi

if [ -n "$dir" ] && [ ! -d "$dir" ]; then
    echo "Ошибка: директория $dir не существует"
    exit 1
fi

echo "Все проверки пройдены"
```

### Пример 2: Расширенная проверка файлов

```bash
#!/bin/bash
set -euo pipefail

check_file_status() {
    local file="$1"
    
    # Используем расширенный test [[ ]] для удобства
    if [[ ! -e "$file" ]]; then
        echo "❌ Файл $file не существует"
        return 1
    fi
    
    # Проверка типа
    if [[ -f "$file" ]]; then
        echo -n "📄 Обычный файл"
    elif [[ -d "$file" ]]; then
        echo -n "📁 Директория"
    elif [[ -L "$file" ]]; then
        echo -n "🔗 Символическая ссылка"
    else
        echo -n "❓ Специальный файл"
    fi
    
    # Проверка прав (используем test -o для проверки опции)
    if [ -o pipefail ]; then
        echo -n " [pipefail включен]"
    fi
    
    # Детальные права
    echo
    echo "  Права:"
    [ -r "$file" ] && echo "    ✅ Чтение" || echo "    ❌ Чтение"
    [ -w "$file" ] && echo "    ✅ Запись" || echo "    ❌ Запись"
    [ -x "$file" ] && echo "    ✅ Выполнение" || echo "    ❌ Выполнение"
}

# Проверяем текущий скрипт
check_file_status "$0"
```

### Пример 3: Проверка переменных окружения

```bash
#!/bin/bash
set -euo pipefail

# Проверка обязательных переменных
required_vars=("HOME" "USER" "SHELL")

for var in "${required_vars[@]}"; do
    # test -v проверяет, установлена ли переменная
    if [ ! -v "$var" ]; then
        echo "Ошибка: переменная $var не установлена" >&2
        exit 1
    fi
    
    # Проверка на пустоту (если переменная может быть пустой)
    if [ -z "${!var}" ]; then
        echo "Предупреждение: переменная $var пуста" >&2
    fi
done

# Проверка с использованием [[ ]]
if [[ -v DISPLAY && -n "$DISPLAY" ]]; then
    echo "Графический интерфейс доступен (DISPLAY=$DISPLAY)"
else
    echo "Графический интерфейс не доступен"
fi
```

### Пример 4: Сравнение `[ ]` и `[[ ]]` с `set -u`

```bash
#!/bin/bash
set -u  # nounset включен

unset_var=""
empty_var=""

# ОПАСНО: [ ] может упасть с unset переменной
if [ "$unset_var" = "test" ]; then  # ОШИБКА! unset_var не определена
    echo "Никогда не выполнится"
fi

# БЕЗОПАСНО: [[ ]] обрабатывает unset как пустую строку
if [[ "$unset_var" == "test" ]]; then  # БЕЗОПАСНО, просто false
    echo "Никогда не выполнится"
fi

# Правильный способ проверки
if [ -z "${unset_var:-}" ]; then  # используем подстановку значения по умолчанию
    echo "Переменная не определена или пуста"
fi

# Сравнение с [[ ]] еще удобнее
if [[ -z "$unset_var" ]]; then  # работает без подстановки!
    echo "Переменная не определена или пуста"
fi
```

### Пример 5: Использование `test -o` для проверки опций

```bash
#!/bin/bash

check_bash_options() {
    echo "=== Текущие опции оболочки ==="
    
    # Проверяем конкретные опции
    [ -o errexit ] && echo "✅ errexit (set -e) включен" || echo "❌ errexit выключен"
    [ -o nounset ] && echo "✅ nounset (set -u) включен" || echo "❌ nounset выключен"
    [ -o xtrace ] && echo "✅ xtrace (set -x) включен" || echo "❌ xtrace выключен"
    [ -o pipefail ] && echo "✅ pipefail включен" || echo "❌ pipefail выключен"
    [ -o noclobber ] && echo "✅ noclobber (set -C) включен" || echo "❌ noclobber выключен"
    
    echo "Все опции: $-"
}

# Включаем некоторые опции
set -e
set -u

check_bash_options

# Отключаем и проверяем снова
set +e
echo
check_bash_options
```

### Пример 6: Конвейеры и `pipefail`

```bash
#!/bin/bash
# Демонстрация pipefail

# Без pipefail
set +o pipefail
echo "=== Без pipefail ==="
ls /несуществующая_директория | wc -l
echo "Статус после пайпа: $? - пайп считается успешным!"

# С pipefail
set -o pipefail
echo "=== С pipefail ==="
ls /несуществующая_директория | wc -l || true
echo "Статус после пайпа: $? - видим ошибку!"

# Корректная обработка с test
check_command() {
    if ls /несуществующая_директория 2>/dev/null | wc -l; then
        echo "Команда выполнена успешно"
    else
        echo "Команда завершилась с ошибкой"
    fi
}

check_command
```

## Bash Strict Mode

### Полный "строгий режим"

```bash
#!/bin/bash
set -euo pipefail
IFS=$'\n\t'
```

### Что делает каждая часть:

| Часть | Описание | Почему это важно |
|-------|----------|------------------|
| `set -e` | Выход при ошибке | Скрипт не продолжает работу в ошибочном состоянии |
| `set -u` | Ошибка при неопределенных переменных | Предотвращает опечатки и неожиданное поведение |
| `set -o pipefail` | Ошибка в любой части конвейера | Не маскирует ошибки в середине пайпа |
| `IFS=$'\n\t'` | Разделитель полей | Предотвращает разбиение по пробелам в именах файлов |

### Почему это работает с `test`

```bash
#!/bin/bash
set -euo pipefail
IFS=$'\n\t'

safe_function() {
    local file="$1"
    
    # Это безопасно - условие в if защищено от set -e
    if [ ! -f "$file" ]; then
        echo "Файл не найден: $file"
        return 1
    fi
    
    # Это тоже безопасно из-за ||
    [ -r "$file" ] || {
        echo "Нет прав на чтение: $file"
        return 1
    }
    
    # А вот это опасно, если файл не существует
    # [ -f "$file" ]  # НЕ ДЕЛАЙТЕ ТАК!
    
    echo "Файл $file существует и доступен для чтения"
}

# Использование
safe_function "/etc/passwd" || echo "Функция вернула ошибку"
```

### Исключения и нюансы strict mode

```bash
#!/bin/bash
set -euo pipefail

# 1. Временное отключение для опасной команды
set +e
dangerous_command
status=$?
set -e

# 2. Использование || true для игнорирования ошибок
cleanup_command || true

# 3. Проверка существования файла с test (безопасно)
if [ -f "/some/file" ]; then
    echo "Файл существует"
fi

# 4. Проверка существования команды
if command -v python3 &>/dev/null; then
    echo "Python 3 установлен"
fi
```

## Заключение

### Ключевые выводы:

1. **`test` (или `[ ]`)** — это команда для проверки условий, возвращает код 0 (истина) или 1 (ложь)
2. **`[[ ]]`** — улучшенная версия `test` в Bash с более предсказуемым поведением
3. **`set -e`** — опция оболочки для выхода при ошибке, но **не применяется** в условиях `if/while/until` и левой части `&&/||`
4. **Bash strict mode** (`set -euo pipefail`) делает скрипты надежнее, но требует понимания исключений
5. **Всегда используйте** `[[ ]]` вместо `[ ]` в Bash, если не требуется POSIX-совместимость

### Полезные команды для запоминания:

```bash
# Справка
man test
help test
help [         # встроенная справка Bash
help set       # все опции set

# Проверка опций
set -o
echo $-
[ -o pipefail ] && echo "pipefail включен"

# Безопасные шаблоны
[ -z "${var:-}" ]          # проверка на пустоту с защитой
[[ -z "$var" ]]            # то же самое, но безопаснее
[ -f "$file" ] || exit 1   # выход с ошибкой
```

### Рекомендации:

1. **Всегда начинайте скрипты с** `#!/bin/bash` и `set -euo pipefail`
2. **Используйте `[[ ]]`** вместо `[ ]` для более чистого кода
3. **Проверяйте переменные** перед использованием: `[[ -n "$var" ]]`
4. **Не бойтесь `set -e`** — он безопасен в условных конструкциях
5. **Документируйте исключения**, где намеренно игнорируете ошибки
