# Инструкция: Уменьшение размера Swap на Ximper Linux (ALT Linux)

**В случае, если вы устанавливали Ximper по умолчанию, автоматически, то следует уменьшить Swap**

**Исходная ситуация:** Swap-раздел 8.4 ГБ, не используется, нужно уменьшить до 2 ГБ

---

## 1. Диагностика текущего состояния

### 1.1. Проверка использования памяти и swap
```bash
free -h
```
**Результат:** Swap: 8.4GiB, Used: 0B

### 1.2. Определение типа swap
```bash
sudo swapon --show
```
или
```bash
sudo cat /proc/swaps
```
**Результат:** `/dev/nvme0n1p2 partition 8,4G 0B -2` — **swap-раздел**

### 1.3. Проверка файловой системы корневого раздела
```bash
df -T /
findmnt -no FSTYPE /
```
**Результат:** btrfs — **Btrfs имеет особенности работы с swap-файлами**

---

## 2. Отключение старого swap-раздела

### 2.1. Отключение swap-раздела
```bash
sudo swapoff /dev/nvme0n1p2
```

### 2.2. Редактирование /etc/fstab (отключение автозагрузки)
```bash
sudo nano /etc/fstab
```
**Найти строку с swap-разделом:**
```
UUID=527c5922-137b-485e-842b-17bbe290c5a6	swap	swap	defaults	0	0
```
**Заменить на (закомментировать):**
```
#UUID=527c5922-137b-485e-842b-17bbe290c5a6	swap	swap	defaults	0	0
```

**Проверка:**
```bash
cat -n /etc/fstab
```

---

## 3. Создание swap-файла 2 ГБ на Btrfs

**Важно:** На файловой системе Btrfs требуется специальная подготовка из-за механизма Copy-on-Write (CoW)

### 3.1. Удаление старого swap-файла (если существовал)
```bash
sudo swapoff /swapfile 2>/dev/null
sudo rm /swapfile
```

### 3.2. Создание файла с отключенным CoW
```bash
sudo truncate -s 0 /swapfile
sudo chattr +C /swapfile
```

### 3.3. Выделение места
```bash
sudo fallocate -l 2G /swapfile
```
**Альтернативный метод (если fallocate не работает):**
```bash
sudo dd if=/dev/zero of=/swapfile bs=1M count=2048 status=progress
```

### 3.4. Установка прав доступа
```bash
sudo chmod 600 /swapfile
```

### 3.5. Форматирование как swap
```bash
sudo mkswap /swapfile
```
**Результат:** `UUID=97efbe6e-aae3-4bc7-b1a2-f23a993242d0`

### 3.6. Включение swap-файла
```bash
sudo swapon /swapfile
```

---

## 4. Настройка автоматического включения при загрузке

### 4.1. Добавление записи в /etc/fstab
```bash
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

**Проверка:**
```bash
tail -n 1 /etc/fstab
```
**Результат:** `/swapfile none swap sw 0 0`

---

## 5. Проверка результатов

### 5.1. Проверка активного swap
```bash
sudo swapon --show
```
**Результат:**
```
NAME      TYPE SIZE USED PRIO
/swapfile file   2G   0B   -2
```

### 5.2. Проверка использования памяти
```bash
free -h
```
**Результат:**
```
Mem:  14Gi   6,4Gi   3,6Gi   157Mi   5,5Gi   8,6Gi
Swap: 2,0Gi   0B      2,0Gi
```

### 5.3. Финальная проверка /etc/fstab
```bash
cat -n /etc/fstab
```
**Результат (строка 8):**
```
     8	/swapfile none swap sw 0 0
```

### 5.4. Отключение старого раздела (если еще активен)
```bash
sudo swapoff /dev/nvme0n1p2 2>/dev/null
```

---

## 6. Оптимизация использования swap (рекомендуется)

### 6.1. Уменьшение "swappiness" (активности использования swap)
```bash
# Временное изменение (до перезагрузки)
sudo sysctl vm.swappiness=10

# Постоянное изменение
echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf
```

**Пояснение:** Значение 10 означает, что swap будет использоваться только при крайней необходимости (обычно значение 60).

---

## 7. Особенности и возможные проблемы

### 7.1. Btrfs и swap-файлы
На Btrfs swap-файлы требуют:
- Отключения Copy-on-Write (`chattr +C`)
- Поддержки ядром (Linux 5.0+)
- **Обязательного использования** `chattr +C` **ДО** выделения места

### 7.2. Ошибка "Недопустимый аргумент"
Возникала при попытке `swapon` без `chattr +C`. Решение — последовательность:
```bash
truncate → chattr +C → fallocate → chmod → mkswap → swapon
```

### 7.3. Старый swap-раздел
Остался на диске, но не используется:
- **Не мешает** работе системы
- Занимает 8.4 ГБ на диске
- **Для полного удаления** требуется загрузка с LiveUSB и использование GParted

# Инструкция: Преобразование swap-раздела в раздел для хранения данных

## Исходная ситуация
На диске `/dev/nvme0n1` был swap-раздел `/dev/nvme0n1p2` размером **8.4 ГБ**, который:
- Не использовался (0B used)
- Был отключен (`swapoff`)
- Был закомментирован в `/etc/fstab`
- Просто лежал мёртвым грузом

**Цель:** Использовать это пространство с пользой — создать отдельный раздел для хранения файлов.

---

## 1. Проверка текущих разделов
```bash
sudo fdisk -l /dev/nvme0n1
```
**Увидели:**
- `nvme0n1p1` - EFI (511M)
- `nvme0n1p2` - бывший swap (8.4G) ⬅️ **наша цель**
- `nvme0n1p3` - корневой раздел (468G) с системой

---

## 2. Проверка, что swap-раздел не используется
```bash
sudo swapon --show
```
**Результат:** активен только `/swapfile`, раздел `/dev/nvme0n1p2` не используется.

---

## 3. Создание файловой системы на разделе

**Зачем:** Swap-раздел имеет свой формат (сигнатуру). Чтобы использовать его для хранения файлов, нужно создать на нём обычную файловую систему.

```bash
sudo mkfs.ext4 /dev/nvme0n1p2
```
**Что сделали:**
- Создали файловую систему **ext4** (стандартная для Linux)
- Система спросила подтверждение, так как там была swap-сигнатура
- Получили новый **UUID** раздела: `cd6fd472-6d6b-4078-bb18-22d7287092b4`
- Размер: 8.2 ГБ (доступно после форматирования)

---

## 4. Создание точки монтирования

**Зачем:** В Linux каждый раздел нужно "прикрепить" (примонтировать) к папке, чтобы работать с файлами.

```bash
sudo mkdir -p /mnt/storage
```
Создали папку `/mnt/storage` — стандартное место для дополнительных разделов.

---

## 5. Монтирование раздела

```bash
sudo mount /dev/nvme0n1p2 /mnt/storage
```
**Что сделали:** Прикрепили раздел к папке `/mnt/storage`.

---

## 6. Настройка прав доступа

**Зачем:** По умолчанию раздел принадлежит root. Чтобы обычный пользователь мог туда писать файлы.

```bash
sudo chown kkorablin:kkorablin /mnt/storage
```
Теперь пользователь `kkorablin` — полный владелец раздела.

---

## 7. Автоматическое монтирование при загрузке

**Зачем:** Чтобы раздел автоматически подключался каждый раз при включении компьютера.

### 7.1. Получили UUID раздела:
```bash
UUID=$(sudo blkid -s UUID -o value /dev/nvme0n1p2)
```

### 7.2. Добавили запись в `/etc/fstab`:
```bash
echo "UUID=$UUID /mnt/storage ext4 defaults,noatime 0 2" | sudo tee -a /etc/fstab
```

**Параметры:**
- `UUID=...` — уникальный идентификатор раздела (надёжнее чем `/dev/nvme0n1p2`)
- `/mnt/storage` — куда монтировать
- `ext4` — тип файловой системы
- `defaults,noatime` — стандартные опции + не обновлять время доступа (быстрее)
- `0` — не делать dump-бэкап
- `2` — проверять файловую систему после корневого раздела

---

## 8. Создание символической ссылки для удобства

**Зачем:** Чтобы не ходить каждый раз в `/mnt/storage`, а иметь быстрый доступ из домашней папки.

```bash
ln -s /mnt/storage ~/storage
```
**Что сделали:** Создали ярлык (ссылку) `~/storage`, ведущую на `/mnt/storage`.

---

## 9. Создание рабочих папок

```bash
cd ~/storage
mkdir projects downloads backup
```
**Для чего:**
- `projects` — для рабочих проектов
- `downloads` — для загрузок (можно перенести сюда стандартную папку загрузок)
- `backup` — для бэкапов важных файлов

---

```bash

```
**Результат:** Раздел 8.2ГБ, свободно 7.8ГБ

# 📚 ПОЛНОЕ РУКОВОДСТВО ПО SWAP В LINUX

## 🔴 ЧАСТЬ 1: ОТКЛЮЧЕНИЕ И УДАЛЕНИЕ SWAP-РАЗДЕЛА

### 1.1 Проверяем текущий swap
```bash
# Посмотреть, какой swap используется
swapon --show
free -h
lsblk | grep -i swap
blkid | grep -i swap
```

### 1.2 Отключаем swap
```bash
# Отключаем все swap-разделы
swapoff -a

# Или отключаем конкретный раздел
swapoff /dev/nvme0n1pX   # замените X на номер раздела
```

### 1.3 Удаляем запись из /etc/fstab
```bash
# Редактируем fstab
mcedit /etc/fstab

# Найдите строку с swap и либо удалите её, либо закомментируйте (добавьте # в начале)
# Пример:
# UUID=f5657c6a-0eb2-4e20-9329-8bfbdad6b8b2 swap swap defaults 0 0
```

### 1.4 Перезагружаем systemd
```bash
systemctl daemon-reload
```

### 1.5 Удаляем swap-раздел (если нужно освободить место)
```bash
# Запускаем fdisk для управления разделами
fdisk /dev/nvme0n1

# В fdisk используйте команды:
# p - показать разделы (запомните номер swap-раздела)
# d - удалить раздел
# w - записать изменения и выйти

# После удаления раздела обновляем таблицу разделов
partprobe /dev/nvme0n1
```

### 1.6 Создаём файловую систему на бывшем swap-разделе
```bash
# Создаём ext4 файловую систему на освободившемся разделе
mkfs.ext4 /dev/nvme0n1pX   # X - номер бывшего swap-раздела

# Создаём точку монтирования
mkdir /mnt/data

# Монтируем временно
mount /dev/nvme0n1pX /mnt/data

# Проверяем
df -h /mnt/data
```

### 1.7 Добавляем в /etc/fstab для автоматического монтирования
```bash
# Получаем UUID нового раздела
blkid /dev/nvme0n1pX

# Добавляем строку в /etc/fstab
echo 'UUID=новый-uuid-раздела /mnt/data ext4 defaults 0 2' >> /etc/fstab

# Перезагружаем systemd
systemctl daemon-reload
```

## 🟢 ЧАСТЬ 2: СОЗДАНИЕ SWAP-ФАЙЛА

### 2.1 Выбираем размер swap-файла
Общее правило:
- Если RAM ≤ 2 ГБ: swap = RAM × 2
- Если RAM 2-8 ГБ: swap = RAM
- Если RAM > 8 ГБ: swap = RAM/2 (но не меньше 4 ГБ)

В вашем случае (14 ГБ RAM) достаточно 4-8 ГБ, но вы выбрали 512 МБ - тоже ок.

### 2.2 Создаём swap-файл

**Способ 1: fallocate (быстро, но не на всех ФС работает)**
```bash
# Создать файл размером 512 МБ
fallocate -l 512M /swapfile

# Для 2 ГБ:
fallocate -l 2G /swapfile

# Для 4 ГБ:
fallocate -l 4G /swapfile
```

**Способ 2: dd (медленнее, но надёжнее)**
```bash
# Создать файл 512 МБ
dd if=/dev/zero of=/swapfile bs=1M count=512

# Для 2 ГБ:
dd if=/dev/zero of=/swapfile bs=1M count=2048

# Для 4 ГБ:
dd if=/dev/zero of=/swapfile bs=1M count=4096
```

### 2.3 Устанавливаем правильные права доступа
```bash
# Только root может читать и писать swap-файл
chmod 600 /swapfile
```

### 2.4 Форматируем как swap
```bash
# Создаём swap-систему на файле
mkswap /swapfile

# Вы увидите что-то вроде:
# Setting up swapspace version 1, size = 512 MiB (536866816 bytes)
# без метки, UUID=6e3b75cd-d6b5-4216-96e1-43e285cf1de5
```

### 2.5 Включаем swap-файл
```bash
# Активируем swap
swapon /swapfile

# Проверяем
swapon --show
free -h
```

### 2.6 Добавляем в /etc/fstab для автозагрузки
```bash
# Редактируем fstab
mcedit /etc/fstab

# Добавляем строку:
/swapfile none swap defaults 0 0

# Или через echo:
echo '/swapfile none swap defaults 0 0' >> /etc/fstab
```

### 2.7 Перезагружаем systemd и проверяем
```bash
# Перечитываем конфиги
systemctl daemon-reload

# Проверяем синтаксис fstab
findmnt --verify
# Предупреждение про "regular file" - НОРМАЛЬНО для swap-файла!

# Проверяем статус
systemctl status swapfile.swap
systemctl list-units | grep swap
```

## 🟡 ЧАСТЬ 3: УПРАВЛЕНИЕ SWAP-ФАЙЛОМ

### 3.1 Временное отключение swap
```bash
# Отключить swap-файл
swapoff /swapfile

# Или отключить всё
swapoff -a
```

### 3.2 Включение swap
```bash
# Включить конкретный файл
swapon /swapfile

# Включить всё из /etc/fstab
swapon -a
```

### 3.3 Изменение приоритета swap
```bash
# Приоритет: чем выше число, тем приоритетнее
swapon /swapfile -p 10

# В /etc/fstab можно указать приоритет:
/swapfile none swap defaults,pri=10 0 0
```

### 3.4 Просмотр детальной информации
```bash
# Базовая информация
swapon --show

# Вся информация
swapon --show --output-all

# Использование памяти
free -h
free -m  # в мегабайтах

# Информация о swap в /proc
cat /proc/swaps
cat /proc/meminfo | grep -i swap
```

### 3.5 Изменение размера swap-файла
```bash
# Отключаем swap
swapoff /swapfile

# Удаляем старый файл
rm /swapfile

# Создаём новый нужного размера
fallocate -l 2G /swapfile
# или
dd if=/dev/zero of=/swapfile bs=1M count=2048

# Права доступа
chmod 600 /swapfile

# Форматируем
mkswap /swapfile

# Включаем
swapon /swapfile
```

### 3.6 Удаление swap-файла (если больше не нужен)
```bash
# Отключаем swap
swapoff /swapfile

# Удаляем из /etc/fstab
mcedit /etc/fstab
# Удалите или закомментируйте строку с /swapfile

# Перезагружаем systemd
systemctl daemon-reload

# Удаляем файл
rm /swapfile
```

## 🔵 ЧАСТЬ 4: ПОЛЕЗНЫЕ КОМАНДЫ И ПРОВЕРКИ

### 4.1 Диагностика проблем
```bash
# Проверка fstab на ошибки
findmnt --verify

# Проверка, видит ли система swap
lsblk | grep -i swap
blkid | grep -i swap
```

### 4.2 Настройка swappiness (как часто использовать swap)
```bash
# Посмотреть текущее значение (от 0 до 100)
cat /proc/sys/vm/swappiness

# Временно изменить (до перезагрузки)

# Постоянно изменить

# Рекомендации:
# 1-10 - для SSD, редко использовать swap
# 60 - значение по умолчанию
# 100 - активно использовать swap
```

### 4.3 Тестирование производительности
```bash
# Проверить скорость swap
time dd if=/dev/zero of=/swapfile.test bs=1M count=100
swapon /swapfile.test
time dd if=/dev/zero of=/dev/null bs=1M count=100 2>&1
```

## 📌 ВАЖНЫЕ ЗАМЕЧАНИЯ

### Преимущества swap-файла перед разделом:
1. **Гибкость** - легко изменить размер, удалить, переместить
2. **Не нужна переразметка диска**
3. **Можно создать несколько swap-файлов**
4. **Простота управления**

### Недостатки swap-файла:
1. **Чуть медленнее** (из-за файловой системы)
2. **Риск фрагментации** (но для swap не критично)
3. **Некоторые ФС не поддерживают** (btrfs, ZFS требуют настроек)

### Когда нужен swap:
- **Гибернация (suspend-to-disk)** - требует swap >= RAM
- **Недостаточно RAM** - подстраховка
- **Редко используемые данные** - можно выгрузить в swap

### Безопасность:
- Всегда ставьте `chmod 600` на swap-файл
- В swap могут попасть пароли и ключи, поэтому на серверах иногда отключают
- Для шифрования можно использовать LUKS или ecryptfs

## 🎯 ПРИМЕР ПОЛНОГО ЦИКЛА

```bash
# 1. Отключаем старый swap-раздел
swapoff -a

# 2. Правим fstab
mcedit /etc/fstab
# комментируем старую swap-запись

# 3. Создаём swap-файл на 2 ГБ
dd if=/dev/zero of=/swapfile bs=1M count=2048
chmod 600 /swapfile
mkswap /swapfile

# 4. Включаем
swapon /swapfile

# 5. Добавляем в fstab
echo '/swapfile none swap defaults 0 0' >> /etc/fstab

# 6. Проверяем
systemctl daemon-reload
findmnt --verify
swapon --show
free -h
```
