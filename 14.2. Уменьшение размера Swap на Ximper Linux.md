# Инструкция: Уменьшение размера Swap на Ximper Linux (ALT Linux)

**В случае, если вы устанавливали Ximper по умолчанию, автоматически, то следует уменьшить Swap**

**Исходная ситуация:** Swap-раздел 8.4 ГБ, не используется, нужно уменьшить до 2 ГБ

---

## 1. Диагностика текущего состояния

### 1.1. Проверка использования памяти и swap
```bash
free -h
```
**Результат:** Swap: 8.4GiB, Used: 0B

### 1.2. Определение типа swap
```bash
sudo swapon --show
```
или
```bash
sudo cat /proc/swaps
```
**Результат:** `/dev/nvme0n1p2 partition 8,4G 0B -2` — **swap-раздел**

### 1.3. Проверка файловой системы корневого раздела
```bash
df -T /
findmnt -no FSTYPE /
```
**Результат:** btrfs — **Btrfs имеет особенности работы с swap-файлами**

---

## 2. Отключение старого swap-раздела

### 2.1. Отключение swap-раздела
```bash
sudo swapoff /dev/nvme0n1p2
```

### 2.2. Редактирование /etc/fstab (отключение автозагрузки)
```bash
sudo nano /etc/fstab
```
**Найти строку с swap-разделом:**
```
UUID=527c5922-137b-485e-842b-17bbe290c5a6	swap	swap	defaults	0	0
```
**Заменить на (закомментировать):**
```
#UUID=527c5922-137b-485e-842b-17bbe290c5a6	swap	swap	defaults	0	0
```

**Проверка:**
```bash
cat -n /etc/fstab
```

---

## 3. Создание swap-файла 2 ГБ на Btrfs

**Важно:** На файловой системе Btrfs требуется специальная подготовка из-за механизма Copy-on-Write (CoW)

### 3.1. Удаление старого swap-файла (если существовал)
```bash
sudo swapoff /swapfile 2>/dev/null
sudo rm /swapfile
```

### 3.2. Создание файла с отключенным CoW
```bash
sudo truncate -s 0 /swapfile
sudo chattr +C /swapfile
```

### 3.3. Выделение места
```bash
sudo fallocate -l 2G /swapfile
```
**Альтернативный метод (если fallocate не работает):**
```bash
sudo dd if=/dev/zero of=/swapfile bs=1M count=2048 status=progress
```

### 3.4. Установка прав доступа
```bash
sudo chmod 600 /swapfile
```

### 3.5. Форматирование как swap
```bash
sudo mkswap /swapfile
```
**Результат:** `UUID=97efbe6e-aae3-4bc7-b1a2-f23a993242d0`

### 3.6. Включение swap-файла
```bash
sudo swapon /swapfile
```

---

## 4. Настройка автоматического включения при загрузке

### 4.1. Добавление записи в /etc/fstab
```bash
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

**Проверка:**
```bash
tail -n 1 /etc/fstab
```
**Результат:** `/swapfile none swap sw 0 0`

---

## 5. Проверка результатов

### 5.1. Проверка активного swap
```bash
sudo swapon --show
```
**Результат:**
```
NAME      TYPE SIZE USED PRIO
/swapfile file   2G   0B   -2
```

### 5.2. Проверка использования памяти
```bash
free -h
```
**Результат:**
```
Mem:  14Gi   6,4Gi   3,6Gi   157Mi   5,5Gi   8,6Gi
Swap: 2,0Gi   0B      2,0Gi
```

### 5.3. Финальная проверка /etc/fstab
```bash
cat -n /etc/fstab
```
**Результат (строка 8):**
```
     8	/swapfile none swap sw 0 0
```

### 5.4. Отключение старого раздела (если еще активен)
```bash
sudo swapoff /dev/nvme0n1p2 2>/dev/null
```

---

## 6. Оптимизация использования swap (рекомендуется)

### 6.1. Уменьшение "swappiness" (активности использования swap)
```bash
# Временное изменение (до перезагрузки)
sudo sysctl vm.swappiness=10

# Постоянное изменение
echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf
```

**Пояснение:** Значение 10 означает, что swap будет использоваться только при крайней необходимости (обычно значение 60).

---

## 7. Особенности и возможные проблемы

### 7.1. Btrfs и swap-файлы
На Btrfs swap-файлы требуют:
- Отключения Copy-on-Write (`chattr +C`)
- Поддержки ядром (Linux 5.0+)
- **Обязательного использования** `chattr +C` **ДО** выделения места

### 7.2. Ошибка "Недопустимый аргумент"
Возникала при попытке `swapon` без `chattr +C`. Решение — последовательность:
```bash
truncate → chattr +C → fallocate → chmod → mkswap → swapon
```

### 7.3. Старый swap-раздел
Остался на диске, но не используется:
- **Не мешает** работе системы
- Занимает 8.4 ГБ на диске
- **Для полного удаления** требуется загрузка с LiveUSB и использование GParted

# Инструкция: Преобразование swap-раздела в раздел для хранения данных

## Исходная ситуация
На диске `/dev/nvme0n1` был swap-раздел `/dev/nvme0n1p2` размером **8.4 ГБ**, который:
- Не использовался (0B used)
- Был отключен (`swapoff`)
- Был закомментирован в `/etc/fstab`
- Просто лежал мёртвым грузом

**Цель:** Использовать это пространство с пользой — создать отдельный раздел для хранения файлов.

---

## 1. Проверка текущих разделов
```bash
sudo fdisk -l /dev/nvme0n1
```
**Увидели:**
- `nvme0n1p1` - EFI (511M)
- `nvme0n1p2` - бывший swap (8.4G) ⬅️ **наша цель**
- `nvme0n1p3` - корневой раздел (468G) с системой

---

## 2. Проверка, что swap-раздел не используется
```bash
sudo swapon --show
```
**Результат:** активен только `/swapfile`, раздел `/dev/nvme0n1p2` не используется.

---

## 3. Создание файловой системы на разделе

**Зачем:** Swap-раздел имеет свой формат (сигнатуру). Чтобы использовать его для хранения файлов, нужно создать на нём обычную файловую систему.

```bash
sudo mkfs.ext4 /dev/nvme0n1p2
```
**Что сделали:**
- Создали файловую систему **ext4** (стандартная для Linux)
- Система спросила подтверждение, так как там была swap-сигнатура
- Получили новый **UUID** раздела: `cd6fd472-6d6b-4078-bb18-22d7287092b4`
- Размер: 8.2 ГБ (доступно после форматирования)

---

## 4. Создание точки монтирования

**Зачем:** В Linux каждый раздел нужно "прикрепить" (примонтировать) к папке, чтобы работать с файлами.

```bash
sudo mkdir -p /mnt/storage
```
Создали папку `/mnt/storage` — стандартное место для дополнительных разделов.

---

## 5. Монтирование раздела

```bash
sudo mount /dev/nvme0n1p2 /mnt/storage
```
**Что сделали:** Прикрепили раздел к папке `/mnt/storage`.

---

## 6. Настройка прав доступа

**Зачем:** По умолчанию раздел принадлежит root. Чтобы обычный пользователь мог туда писать файлы.

```bash
sudo chown kkorablin:kkorablin /mnt/storage
```
Теперь пользователь `kkorablin` — полный владелец раздела.

---

## 7. Автоматическое монтирование при загрузке

**Зачем:** Чтобы раздел автоматически подключался каждый раз при включении компьютера.

### 7.1. Получили UUID раздела:
```bash
UUID=$(sudo blkid -s UUID -o value /dev/nvme0n1p2)
```

### 7.2. Добавили запись в `/etc/fstab`:
```bash
echo "UUID=$UUID /mnt/storage ext4 defaults,noatime 0 2" | sudo tee -a /etc/fstab
```

**Параметры:**
- `UUID=...` — уникальный идентификатор раздела (надёжнее чем `/dev/nvme0n1p2`)
- `/mnt/storage` — куда монтировать
- `ext4` — тип файловой системы
- `defaults,noatime` — стандартные опции + не обновлять время доступа (быстрее)
- `0` — не делать dump-бэкап
- `2` — проверять файловую систему после корневого раздела

---

## 8. Создание символической ссылки для удобства

**Зачем:** Чтобы не ходить каждый раз в `/mnt/storage`, а иметь быстрый доступ из домашней папки.

```bash
ln -s /mnt/storage ~/storage
```
**Что сделали:** Создали ярлык (ссылку) `~/storage`, ведущую на `/mnt/storage`.

---

## 9. Создание рабочих папок

```bash
cd ~/storage
mkdir projects downloads backup
```
**Для чего:**
- `projects` — для рабочих проектов
- `downloads` — для загрузок (можно перенести сюда стандартную папку загрузок)
- `backup` — для бэкапов важных файлов

---

## 10. Проверка результата

```bash
df -h /mnt/storage
```
**Результат:** Раздел 8.2ГБ, свободно 7.8ГБ

```bash
lsblk -f | grep nvme0n1p2
```
**Результат:** Показывает ext4, UUID, точку монтирования

```bash
tail -n 3 /etc/fstab
```
**Результат:** Видим нашу добавленную строку


## Итог: Что мы сделали?

| Было | Стало |
|------|-------|
| 🚫 Swap-раздел 8.4ГБ (не используется) | ✅ Хранилище 8.2ГБ для файлов |
| 🚫 Бесполезная запись в fstab | ✅ Автомонтирование в /mnt/storage |
| 🚫 Только root имел доступ | ✅ Пользователь kkorablin владелец |
| 🚫 Неудобно использовать | ✅ Удобный доступ через ~/storage |
| 🚫 Пустой раздел | ✅ Готовые папки projects, downloads, backup |

## Зачем это нужно?

1. **Эффективное использование места** — 8.4 ГБ не пропадают зря
2. **Разделение данных** — отдельный раздел для пользовательских файлов
3. **Безопасность** — если система "упадёт", данные на отдельном разделе целы
4. **Удобство** — быстрое место для скачивания, бэкапов, проектов
5. **Производительность** — отдельный раздел без лишних операций
