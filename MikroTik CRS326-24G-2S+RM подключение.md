## Диагностика текущего состояния системы

На основе выполненных команд установлено:

| Команда | Результат |
|---------|----------|
| `nmcli connection show` | Активны только `runtel` (Wi-Fi) и `lo` (loopback). Проводные подключения отсутствуют. |
| `rfkill list` | Wi-Fi и Bluetooth не заблокированы. Проблем с программной блокировкой нет. |
| `ip link show` | Обнаружены только интерфейсы `lo` и `wlan0`. Ethernet-адаптер (`eth0`, `enp*`) **не обнаружен**. |
| `ip a` | Wi-Fi интерфейс `wlan0` имеет IP `192.168.97.244/23`, шлюз `192.168.96.1`. |
| `ip r s` | Маршрут по умолчанию через Wi-Fi. |

**Вывод:** Ethernet-адаптер не виден в системе. Это может означать:
1. Ethernet-кабель **не подключен** к компьютеру
2. Ethernet-адаптер **отключен в BIOS/UEFI**
3. Ethernet-адаптер **отсутствует** на данном устройстве (например, ультрабук без Ethernet-порта)

---

## Шаг 1: Подключение Ethernet-адаптера

### Вариант А: Если у ноутбука/ПК есть Ethernet-порт

1. **Физически подключите Ethernet-кабель:**
   - Один конец в любой порт **1-24** коммутатора MikroTik
   - Второй конец в Ethernet-порт компьютера

2. **После подключения кабеля проверьте появление интерфейса:**
   ```bash
   ip link show
   ip -c a s
   ```
   Должен появиться интерфейс с названием `enp*`, `eth0` или аналогичный.

3. **Если интерфейс не появился:**
   - Перезагрузите компьютер с подключенным кабелем
   - Проверьте настройки BIOS/UEFI (см. Вариант Б ниже)

### Вариант Б: Если Ethernet-адаптер отключен в BIOS/UEFI

1. **Войдите в BIOS/UEFI** при загрузке компьютера (обычно клавиши `F2`, `Del`, `Esc`)
2. **Найдите раздел с настройками устройств:**
   - `Advanced` → `Onboard Devices`
   - `Integrated Peripherals`
   - `System Configuration`
3. **Включите Ethernet/LAN Controller** (установите значение `Enabled`)
4. **Сохраните настройки** (обычно `F10`) и перезагрузитесь

### Вариант В: Если у компьютера нет Ethernet-порта

Вам потребуется **USB-to-Ethernet адаптер** (USB-LAN). Подключите его, затем:
```bash
ip link show
ip -c a s
```
Должен появиться новый интерфейс (например, `enx*` или `eth0`).

## Шаг 2: Создание нового проводного подключения

После того как Ethernet-интерфейс появился в системе (проверьте `ip link show`), выполните настройку. Вы можете выбрать **любой** из двух способов:

---

### Способ 1: Настройка через nmtui (текстовый интерфейс)

`nmtui` — это удобный текстовый менеджер с пошаговыми подсказками.

1. **Запустите текстовый интерфейс NetworkManager:**
   ```bash
   nmtui
   ```

2. **Выберите "Edit a connection"** — перемещайтесь стрелками, нажмите `Enter`

3. **Нажмите "Add"** — кнопка внизу, переключитесь на нее клавишей `Tab`, нажмите `Enter`

4. **Выберите тип "Ethernet"** и нажмите `Enter`

5. **Заполните поля:**
   - **Profile name:** `MikroTik` (или любое другое имя)
   - **Device:** выберите ваш Ethernet-интерфейс (например, `enp0s25`, `eth0` или тот, который появился после подключения кабеля)
   - **IPv4 CONFIGURATION:** измените с `<Automatic>` на **`<Manual>`** — нажмите `Enter`, выберите `Manual`, снова `Enter`
   - Нажмите **"Show"** напротив пункта, который появится ниже
   - **Addresses:** введите `192.168.88.2/24`
   - **Gateway:** введите `192.168.88.1`
   - **DNS servers:** введите `8.8.8.8, 1.1.1.1`

6. **Нажмите "OK"** внизу (переключитесь `Tab`, нажмите `Enter`), затем **"Back"** и **"Quit"**

---

### Способ 2: Настройка через nmcli (командная строка)

`nmcli` — быстрый способ настройки без дополнительных меню.

1. **Создайте новое Ethernet-подключение:**
   ```bash
   nmcli dev show
   nmcli connection show
   nmcli connection add type ethernet con-name MikroTik ifname <интерфейс> ipv4.addresses 192.168.88.2/24 ipv4.gateway 192.168.88.1 ipv4.dns "8.8.8.8 1.1.1.1" ipv4.method manual
   ```
   > **Важно:** замените `<интерфейс>` на имя Ethernet-адаптера, которое вы увидели в `ip link show` (например, `enp0s25`, `eth0`, `enx123456`).

2. **Проверьте, что подключение создано:**
   ```bash
   nmcli connection show
   ```
   В списке должна появиться строка `MikroTik` с типом `ethernet`.

3. **Просмотрите детали созданного подключения (опционально):**
   ```bash
   nmcli connection show MikroTik | grep ipv4
   ```

---

## Шаг 3: Подключение к MikroTik

1. **Отключите Wi-Fi** (чтобы не было конфликта маршрутов):
   ```bash
   nmcli connection down runtel
   ```

2. **Подключите созданное Ethernet-подключение:**
   ```bash
   nmcli connection up MikroTik
   ```
   (Если вы назвали подключение иначе, используйте это имя)

3. **Проверьте, что IP назначился:**
   ```bash
   ip addr show <интерфейс>
   ```
   Вы должны увидеть `inet 192.168.88.2/24`

4. **Проверьте маршруты:**
   ```bash
   ip route show
   ```
   Должен появиться маршрут `192.168.88.0/24 dev <интерфейс> proto static scope link`

5. **Проверьте связь с коммутатором:**
   ```bash
   ping 192.168.88.1
   ```
---

## Шаг 4: Запуск WinBox для настройки MikroTik

1. **Установите Wine** (если еще не установлен):
   ```bash
   sudo pacman -S wine
   ```
   (Для Endeavour OS/Arch Linux)

2. **Перейдите в папку с WinBox:**
   ```bash
   cd ~/Programs/WinBox-Mikrotik
   ```

3. **Запустите WinBox:**
   ```bash
   wine winbox.exe
   ```
   (или `wine winbox64.exe`, если скачали 64-битную версию)

4. **В окне WinBox:**
   - **Connect To:** `192.168.88.1`
   - **Login:** `admin`
   - **Password:** оставьте пустым (или введите пароль с наклейки на корпусе, если есть)
   - Нажмите **Connect**

---

## Шаг 5: Альтернативный способ — WebFig

Если WinBox не запускается через Wine, используйте веб-интерфейс:

1. Откройте браузер (Firefox/Chromium)
2. Перейдите по адресу:
   ```
   http://192.168.88.1
   ```
3. Введите логин `admin`, пароль оставьте пустым

## Шаг 6: Что делать, если нет ответа на ping

**Если ping 192.168.88.1 не проходит:**

1. **Убедитесь, что MikroTik включен** — на передней панели должны гореть индикаторы портов
2. **Проверьте порт подключения** — используйте порты 1-24, **не** SFP+ порты (25 и 26)
3. **Проверьте светодиоды на коммутаторе** — на порту, к которому подключен компьютер, должен гореть/мигать индикатор
4. **Проверьте настройки брандмауэра:**
   ```bash
   sudo iptables -L
   ```
   Если есть правила, блокирующие доступ, временно отключите брандмауэр для теста

5. **Сбросьте MikroTik до заводских настроек:**
   - Отключите питание коммутатора
   - Зажмите кнопку **Reset** (обычно на передней панели)
   - Включите питание, удерживая кнопку
   - Дождитесь мигания LED (обычно через 5-10 секунд) и отпустите
   - Подождите 1-2 минуты, пока устройство загрузится

---

## Шаг 7: После успешного входа

1. **Смените пароль администратора:**
   - WinBox: **System → Password**
   - WebFig: **System → Password**

2. **Обновите RouterOS:**
   - **System → Packages → Check for Updates**

3. **Настройте режим работы** (при необходимости):
   - **System → RouterBOARD → Settings**
   - Boot OS: RouterOS (полнофункциональный) или SwOS (только коммутатор)

---

## Шаг 8: Возврат к обычной сети

После завершения настройки MikroTik вернитесь к Wi-Fi:

```bash
# Отключить Ethernet-подключение
nmcli connection down MikroTik

# Включить Wi-Fi
nmcli connection up runtel
```

## Устранение неполадок: Ethernet-адаптер не появляется

Если после всех действий Ethernet-интерфейс так и не появился в `ip link show`:

1. **Проверьте наличие Ethernet-адаптера в системе:**
   ```bash
   lspci | grep -i ethernet
   ```
   Должна появиться строка с описанием контроллера (например, `Ethernet controller: Realtek ...`)

2. **Если команда ничего не вывела:**
   - Ethernet-адаптер физически отсутствует или неисправен
   - Используйте USB-to-Ethernet адаптер

3. **Загрузите драйвер Ethernet вручную:**
   ```bash
   sudo modprobe r8169    # для Realtek
   sudo modprobe e1000e   # для Intel
   ```
   (название драйвера зависит от чипа)

4. **Проверьте сообщения ядра:**
   ```bash
   dmesg | grep -i ethernet
   dmesg | grep -i eth
   ```
