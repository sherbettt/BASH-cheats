```txt
[[ -f ~/.bashrc ]] && . ~/.bashrc
```
```bash
echo '[[ -f ~/.bashrc ]] && . ~/.bashrc' > /root/.bash_profile
```

## Разница входа
---

## 1. `su -` (или `su - root`)

### Что происходит:
- Запускает **login shell** для пользователя root
- Симулирует **полный вход** в систему
- Сбрасывает **ВСЕ** переменные окружения
- Загружает `/etc/profile`, затем `~/.bash_profile`, `~/.bash_login`, `~/.profile`

### Какие переменные используются:
```bash
# Загружаются из:
/etc/profile          # системные настройки
/root/.bash_profile   # персональные настройки root
/root/.bashrc         # только если вызван из .bash_profile

# Пример переменных:
HOME=/root
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
SHELL=/bin/bash
LOGNAME=root
USER=root
TERM=linux
MAIL=/var/mail/root
```

### НЕ наследует от исходного пользователя:
- `PATH` — полный сброс
- `HOME` — становится `/root`
- `PWD` — становится `/root`
- Все пользовательские переменные (`MYVAR`, `JAVA_HOME`, и т.д.) — **теряются**

---

## 2. `sudo -i`

### Что происходит:
- Запускает **login shell** для root
- Сбрасывает **большинство** переменных
- **Безопаснее**, чем `su -` (не нужно знать пароль root)
- Использует **ваш пароль** + права из `sudoers`

### Какие переменные используются:
```bash
# Загружаются те же файлы, что и su -
/etc/profile
/root/.bash_profile

# Но с некоторыми отличиями:
HOME=/root
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
SHELL=/bin/bash
USER=root
LOGNAME=root
SUDO_USER=your_username      # кто выполнил sudo
SUDO_UID=1000                # ваш UID
SUDO_GID=1000                # ваш GID
```

### Наследует от пользователя:
- `TERM` (тип терминала)
- `DISPLAY` (если X11)
- `XAUTHORITY` (если X11)
- Переменные из `env_keep` в `/etc/sudoers`

### НЕ наследует:
- `PATH` (полный сброс)
- `JAVA_HOME`, `MAVEN_HOME` и другие пользовательские
- `http_proxy`, `https_proxy` (если не разрешено)
- `LANG`, `LC_*` (если не разрешено)

---

## 3. `sudo -s`

### Что происходит:
- Запускает **non-login shell** для root
- **Сохраняет текущую директорию**
- Наследует **большинство** переменных окружения
- Использует ваш пароль

### Какие переменные используются:
```bash
# Загружается только:
/root/.bashrc   # и /etc/bash.bashrc

# Переменные от пользователя (сохраняются):
PATH=/home/your_user/bin:/usr/local/bin:/usr/bin:/bin
HOME=/root    # меняется на /root
PWD=/home/your_user/project   # остаётся текущая директория
JAVA_HOME=/usr/lib/jvm/java-11  # наследуется!
MAVEN_HOME=/opt/maven           # наследуется!
http_proxy=http://proxy:8080    # наследуется!
TERM=xterm-256color
LANG=en_US.UTF-8

# Системные переменные (добавляются от root):
USER=root
LOGNAME=root
SUDO_USER=your_username
```

### Наследует практически всё, кроме:
- `HOME` — принудительно `/root`
- `SHELL` — может быть изменён
- `USER`/`LOGNAME` — становятся root
- Некоторые безопасные переменные сбрасываются

---

## Сравнительная таблица окружения

| Переменная | `su -` | `sudo -i` | `sudo -s` |
|------------|--------|-----------|-----------|
| `HOME` | `/root` | `/root` | `/root` |
| `USER` | `root` | `root` | `root` |
| `PATH` | Системный | Системный | **Ваш** PATH |
| `PWD` | `/root` | `/root` | **Ваша** директория |
| `JAVA_HOME` | ❌ Потерян | ❌ Потерян | ✅ **Сохранён** |
| `http_proxy` | ❌ Потерян | ❌ Потерян | ✅ **Сохранён** |
| `LANG` | Системный | Системный | ✅ **Ваш** |
| `SUDO_USER` | ❌ Нет | ✅ `your_user` | ✅ `your_user` |
| Загруженные файлы | `.bash_profile`<br>`.profile` | `.bash_profile`<br>`.profile` | `.bashrc` |

## Какой вход лучше?

### Для повседневной работы — **`sudo -s`**
```bash
sudo -s
```
**Плюсы:**
- ✅ Читает `.bashrc` автоматически
- ✅ Сохраняет ваши переменные (`JAVA_HOME`, `MAVEN_HOME`)
- ✅ Остаётся в текущей директории
- ✅ Быстрый вход

**Минусы:**
- ⚠️ Может быть небезопасно, если есть вредоносные переменные
- ⚠️ Некоторые команды могут работать непредсказуемо из-за сохранённого PATH

---

### Для системного администрирования — **`sudo -i`**
```bash
sudo -i
```
**Плюсы:**
- ✅ Чистое окружение root
- ✅ Безопасно (сбрасывает опасные переменные)
- ✅ Предсказуемое поведение
- ✅ Не требует пароля root

**Минусы:**
- ⚠️ Не читает `.bashrc` (нужно исправлять)
- ⚠️ Потеря пользовательских переменных

---

### Для автоматизации и скриптов — **`su -`**
```bash
su -
```
**Плюсы:**
- ✅ Полностью чистое окружение
- ✅ Предсказуемое поведение

**Минусы:**
- ❌ Требует пароль root
- ❌ Менее безопасно (пароль передаётся в открытом виде)
- ❌ Не используется в современных системах

## Мои рекомендации

### Если вы разработчик и часто работаете с Java/Maven:
```bash
# Создать /root/.bash_profile:
echo '[[ -f ~/.bashrc ]] && . ~/.bashrc' > /root/.bash_profile

# В /root/.bashrc добавить:
export JAVA_HOME=/usr/lib/jvm/java-11
export MAVEN_HOME=/opt/maven
export PATH=$JAVA_HOME/bin:$MAVEN_HOME/bin:$PATH

# Использовать:
sudo -i   # теперь будет работать как надо
```

### Если вы часто переключаетесь:
```bash
# Алиасы в вашем .bashrc:
alias root='sudo -i'          # полный доступ
alias sroot='sudo -s'         # быстрый доступ с вашими переменными
```

### Для безопасности:
- Используйте **`sudo -i`** для критических операций
- Используйте **`sudo -s`** для быстрых команд
- **Никогда** не используйте `su -` в производственных системах

## Практический тест

Запустите и сравните:
```bash
# В вашей сессии:
export TEST_VAR="hello"
export JAVA_HOME="/my/java"

# Проверьте:
sudo -s -c 'echo $TEST_VAR $JAVA_HOME $PWD'
# Результат: hello /my/java /home/your_user/project

sudo -i -c 'echo $TEST_VAR $JAVA_HOME $PWD'
# Результат: (пусто) (пусто) /root

su - -c 'echo $TEST_VAR $JAVA_HOME $PWD'
# Результат: (пусто) (пусто) /root
```
