Утилита `mkfs` (от англ. *make filesystem*) — это стандартный инструмент командной строки в Linux для создания файловой системы на диске или разделе, проще говоря — для его форматирования. 
Это обёртка, которая вызывает специализированные утилиты, такие как `mkfs.ext4`, `mkfs.xfs` или `mkfs.btrfs`.

### 📝 Основной синтаксис и опции

Базовый синтаксис команды выглядит так:

```bash
mkfs [опции] [-t тип] [оп-ции_фс] устройство [размер]
```

Самые часто используемые опции приведены в таблице:

| Опция | Описание |
| :--- | :--- |
| `-t, --type тип` | Указывает тип создаваемой файловой системы (например, `ext4`, `xfs`, `btrfs`). По умолчанию используется `ext2`. |
| `-c, --check` | Проверяет устройство на наличие "плохих" (bad) блоков перед созданием файловой системы. |
| `-l, --badblocks-file файл` | Читает список уже известных "плохих" блоков из указанного файла. |
| `-L, --label метка` | Устанавливает метку (имя) для тома, что удобно для его последующей идентификации. |
| `-V, --verbose` | Включает подробный вывод, показывающий, какая именно команда и с какими параметрами выполняется. |
| `-V, --version` | Выводит информацию о версии утилиты `mkfs` и завершает работу. |
| `-h, --help` | Показывает краткую справочную информацию. |

### 💻 Создание файловых систем

Самый распространённый способ использования `mkfs` — с указанием типа файловой системы через опцию `-t` и целевого устройства:

```bash
sudo mkfs -t ext4 /dev/sdb1
```

Это эквивалентно вызову специализированной команды `mkfs.ext4 /dev/sdb1`. Поскольку создание файловой системы требует прав суперпользователя, перед командой обязательно используется `sudo`.

#### Примеры для популярных типов ФС

*   **ext4**: Надёжная и производительная файловая система, используемая по умолчанию в большинстве дистрибутивов Linux. Подходит для общих рабочих станций и серверов.
    ```bash
    sudo mkfs -t ext4 /dev/sdb1
    ```
    Более продвинутый пример с заданием метки тома:
    ```bash
    sudo mkfs -t ext4 -L DataDisk /dev/sdb1
    ```

*   **XFS**: Высокопроизводительная файловая система, хорошо зарекомендовавшая себя при работе с очень большими файлами (например, в системах видеонаблюдения или базах данных).
    ```bash
    sudo mkfs -t xfs /dev/sdb1
    ```

*   **Btrfs**: Современная файловая система с расширенными возможностями, такими как создание "моментальных снимков" (снимков состояния) и встроенное сжатие данных.
    ```bash
    sudo mkfs -t btrfs /dev/sdb1
    ```

### ⚠️ Важные практические рекомендации

*   **Идентификация устройства**: Перед форматированием убедитесь, что выбранное устройство верно. Используйте команду `lsblk`, чтобы увидеть список всех дисков и разделов. Традиционные имена для SATA/SSD-дисков — `/dev/sda`, `/dev/sdb`, а для разделов на них — `/dev/sda1`, `/dev/sdb2` и т.д.
*   **Подготовка устройства**: `mkfs` работает с блочным устройством (диском или его разделом). Если вы создаёте файловую систему на новом диске, сначала, скорее всего, потребуется создать на нём разделы с помощью утилит типа `fdisk` или `parted`.
*   **Монтирование**: После создания файловой системы её необходимо смонтировать, чтобы получить доступ к данным. Сначала создайте точку монтирования (директорию) и затем примонтируйте устройство в неё:
    ```bash
    sudo mkdir /mnt/mydrive
    sudo mount /dev/sdb1 /mnt/mydrive
    ```
    Чтобы диск монтировался автоматически при загрузке системы, нужно добавить соответствующую запись в файл `/etc/fstab`, используя `blkid` для получения уникального идентификатора (UUID) раздела.

*   **Безопасность**: Создание файловой системы — это **необратимая операция**, которая удаляет все данные на устройстве. Всегда дважды проверяйте имя устройства и, если сомневаетесь, используйте `lsblk` для подтверждения. `mkfs` часто используется только на вновь созданных разделах или дисках, которые вы намерены полностью стереть.

### ❓ Частые ошибки и их решение

*   **`mkfs: command not found`**: Утилита не установлена. В большинстве дистрибутивов она входит в пакет `util-linux` (обычно установлен по умолчанию).
*   **`/dev/sdb1: No such file or directory`**: Указанного устройства или раздела не существует. Проверьте правильность имени командами `lsblk` или `fdisk -l`.
*   **`/dev/sdb1 is apparently in use by the system`**: Раздел уже используется (например, смонтирован). Размонтируйте его командой `sudo umount /dev/sdb1` перед попыткой форматирования.
*   **`mkfs: /dev/sdb1 contains a ... filesystem`**: На разделе уже есть файловая система. Это предупреждение, и `mkfs` всё равно выполнит форматирование, если вы подтвердите действие.
