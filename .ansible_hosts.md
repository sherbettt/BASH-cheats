## Пример файла с хостами для Ansible.

Файл требуется созлать вручную, и он может располагаться в:
- `/etc/ansible/hosts`  ALT Linux;
- в текущей директории;
- `~/.ansible/hosts` - домашняя директория пользователя.

Права поменять для вашего пользователя, если центарлизовано решили создать файл
```bash
sudo chmod 644 /etc/ansible/hosts
```

Если не хотите трогать /etc, можно использовать переменную:
```bash
# Добавьте в ~/.bashrc или ~/.zshrc
export ANSIBLE_INVENTORY=/home/kkorablin/.ansible/hosts

# Создайте файл
mkdir -p ~/.ansible
touch ~/.ansible/hosts
```


Смотри, где расположен сам Ansible
```bash
ansible --version
```

Проверить текущий inventory
```bash
ansible-config dump | grep -i inventory 
```

Проверка работы inventory
```bash
# Проверить, какой инвентори использует Ansible
ansible-config dump | grep DEFAULT_HOST_LIST

# Проверим, что inventory загружается
ansible-inventory --list

# Проверим конкретную группу
ansible-inventory --graph

# Пинг localhost
ansible local -m ping

# Пинг всех хостов
ansible all -m ping

# Пинг конкретной группы
ansible stepanov -m ping
ansible uprdor -m ping

# Пинг конкретного хоста
ansible 178.176.229.202 -m ping

# Явно укажите путь
ansible -i /etc/ansible/hosts all --list-hosts

# Проверить доступность хостов с явным указанием инвентори
ansible -i /etc/ansible/hosts all -m ping

# Или проверьте конфигурацию
ansible-config dump | grep DEFAULT_HOST_LIST
```

**Рекомендация**: Для Endevour OS рекомендуется использовать инвентори в домашней директории, так как:
- Не требует прав sudo для редактирования
- Легче бэкапить и версионировать (можно добавить в git)
- Не конфликтует с системными файлами
```bash
# Настройка удобного инвентори
mkdir -p ~/.ansible
cp /etc/ansible/hosts ~/.ansible/hosts
echo 'export ANSIBLE_INVENTORY=~/.ansible/hosts' >> ~/.bashrc
source ~/.bashrc

# Проверка
ansible all --list-hosts
ansible localhost -m ping
```


Создать симлинку в системной директории ролей
```bash
sudo ln -s /home/<user_name>/projects/git/installer_pbxv2_cluster /etc/ansible/roles/installer_pbxv2_cluster
sudo ln -s /home/kkorablin/git/runtel/installer_pbxv2_singlenode /etc/ansible/roles/installer_pbxv2_singlenode
```
