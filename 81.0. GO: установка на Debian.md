# Установка нескольких версий Go на Debian 13

## 🛠️ Часть 1: Установка GVM (Go Version Manager)

### 1.1 Установка зависимостей

```bash
sudo apt update -y
sudo apt install -y curl git mercurial make binutils bison gcc build-essential
```

### 1.2 Установка GVM

```bash
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)
```

После выполнения вы увидите сообщение:
```
Cloning from https://github.com/moovweb/gvm.git to /root/.gvm
No existing Go versions detected
Installed GVM v1.0.22
```

### 1.3 Настройка окружения

Добавьте GVM в автозагрузку при каждом входе в систему:

```bash
echo '[[ -s "$HOME/.gvm/scripts/gvm" ]] && source "$HOME/.gvm/scripts/gvm"' >> ~/.bashrc
source ~/.bashrc
```

**Важно:** Проверьте, что строка добавлена только один раз, чтобы избежать дублирования:

```bash
grep "gvm" ~/.bashrc
```

Если строка продублировалась, удалите лишнюю через редактор:
```bash
mcedit ~/.bashrc   # или nano ~/.bashrc
```

## 📦 Часть 2: Установка версий Go

### 2.1 Установка из бинарных файлов (рекомендуется)

GVM по умолчанию компилирует Go из исходников, что занимает много времени и может зависнуть. **Используйте флаг `-B`** для установки из готовых бинарников:

```bash
# Установка конкретной версии (примеры)
gvm install go1.19.8 -B
gvm install go1.23.4 -B
gvm install go1.24.0 -B
```

**Внимание:** Если вы указываете версию, которой не существует (например, go1.26.5), установка завершится с ошибкой. Проверяйте актуальные версии на [официальном сайте Go](https://go.dev/dl/).

### 2.2 Альтернативный способ: установка из исходников (медленно)

Если вам нужна кастомная сборка или у вас нет доступа к бинарникам:

```bash
gvm install go1.19.8
```

**Предупреждение:** Этот процесс может занять 10-30 минут и часто подвисает. **Рекомендуется использовать флаг `-B`.**

### 2.3 Просмотр установленных версий

```bash
gvm list
```

Вывод должен показывать все установленные версии:
```
gvm gos (installed)

   go1.19.8
   go1.23.4
   go1.24.0
```

## 🔄 Часть 3: Переключение между версиями

### 3.1 Временное переключение (для текущей сессии)

```bash
gvm use go1.23.4
```

После этого проверьте версию:
```bash
go version
# go version go1.23.4 linux/amd64
```

### 3.2 Установка версии по умолчанию

```bash
gvm use go1.24.0 --default
```

Теперь при каждом новом открытии терминала будет использоваться эта версия.

### 3.3 Переключение для конкретного проекта

Перейдите в директорию проекта и установите локальную версию:

```bash
cd /path/to/your/project
gvm use go1.19.8
```

GVM запомнит эту версию для текущей директории, и при переходе в неё версия будет меняться автоматически.

## ✅ Часть 4: Проверка и диагностика

### 4.1 Проверка текущей версии Go

```bash
go version
```

### 4.2 Проверка окружения

```bash
go env GOPATH   # Путь к рабочей директории Go
go env GOROOT   # Путь к установленной версии Go
echo $PATH      # Проверка, что путь к GVM в начале
```

### 4.3 Проверка, что GVM работает

```bash
gvm list
```

### 4.4 Удаление ненужной версии

```bash
gvm uninstall go1.19.8
```

## ⚠️ Часть 5: Важные замечания и устранение проблем

### 5.1 Избегайте системного пакета golang-go

Не устанавливайте Go через `apt install golang-go`, так как он может конфликтовать с GVM. Если пакет уже установлен:

```bash
sudo apt remove golang-go
```

### 5.2 Команда `go` не найдена после установки GVM

Если после установки GVM команда `go` не работает:

1. Убедитесь, что вы выполнили `source ~/.bashrc`
2. Проверьте, что GVM загружен:
   ```bash
   type gvm
   # должно показывать: gvm is a function
   ```
3. Перезапустите терминал или выполните:
   ```bash
   exec $SHELL
   ```

### 5.3 Установка зависла на "Downloading Go source..."

Прервите процесс (Ctrl+C) и используйте бинарную установку с флагом `-B`:

```bash
gvm install go1.24.0 -B
```

### 5.4 Ошибка "No system Go version detected"

Это нормально для свежей установки GVM. Просто установите версию Go через GVM.

### 5.5 Некорректное отображение версии

Если `go version` показывает не ту версию, которую вы выбрали через `gvm use`:

1. Проверьте, что нет конфликтов в `PATH`:
   ```bash
   which go
   # должно показывать путь внутри /root/.gvm/
   ```
2. Перезагрузите сессию: `exec $SHELL`

## Ручеая установка Go
Смотрим [доступные версии GO](https://go.dev/dl/) и качаем.
```bash
cd /usr/local/
wget https://go.dev/dl/go1.25.1.linux-amd64.tar.gz
tar -xvf go1.25.1.linux-amd64.tar.gz
```
Редактируем **`/etc/profile`**, добавляя вниз:
```ini
export PATH=$PATH:/usr/local/go/bin
echo 'export PATH=$PATH:/usr/local/go/bin' | sudo tee -a /etc/profile
#echo 'export PATH=$PATH:/usr/local/go/bin' | sudo cat >> /etc/profile
#printf '\nexport PATH=$PATH:/usr/local/go/bin\n' | sudo tee -a /etc/profile
```
И один раз выполняем данную команду: ***`export PATH=$PATH:/usr/local/go/bin`***
```sh
root@app /usr/local/go
14:04:20 # go version
go version go1.25.1 linux/amd64
```
