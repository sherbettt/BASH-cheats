## Для зеркал EndeavourOS лучше использовать `eos-rankmirrors`

Это родная утилита дистрибутива, которая специально создана для работы с зеркалами EndeavourOS. Она уже предустановлена и отлично справляется со своей задачей .

---

## Можно ли добавить фильтр по РФ в eos-rankmirrors?

**Короткий ответ: нет, прямой фильтрации по странам нет.**

`eos-rankmirrors` не поддерживает выборку зеркал по географическому признаку (в отличие от `reflector` с параметром `--country`). Утилита работает только со списком зеркал, которые предоставлены в пакете `endeavouros-mirrorlist` .

**Но есть обходные пути!**

### Способ 1: Ручное добавление РФ зеркал (если они существуют)

Если вы знаете адреса РФ зеркал для EndeavourOS, вы можете добавить их в начало списка принудительно:

```bash
sudo eos-rankmirrors --prefer='https://ваше-рф-зеркало/repo/$repo/$arch' --sort rate
```

Параметр `--prefer` добавляет указанные зеркала в начало итогового списка .

**Проблема:** На данный момент в официальном списке зеркал EndeavourOS **нет зеркал из России**. Это видно из вашего вывода — ни одного `.ru` домена в списке. Поэтому добавить их неоткуда.

### Способ 2: Игнорирование проблемных зеркал

Вы можете исключить зеркала, которые выдают ошибки (как `funami`, `meowsmp` и другие в вашем выводе):

```bash
sudo eos-rankmirrors --ignore='funami|meowsmp|distrohub|del2|diyarciftci|c0urier' --sort rate
```

Или прописать их в конфиг `/etc/eos-rankmirrors.conf` :

```bash
EOS_IGNORED_MIRRORS=(funami meowsmp distrohub)
```

### Способ 3: Изменение порядка сортировки (рекомендую)

Вы уже использовали `--sort rate`, что дало отличный результат. Самые быстрые зеркала для вас:
- Швеция (`mirror.retropc.se`) — **0.197 секунды**
- Швейцария (`pkg.adfinis-on-exoscale.ch`) — **0.199 секунды**
- Германия (`mirror.moson.org`) — **0.214 секунды**

Это **отличные показатели** — быстрее, чем большинство локальных зеркал в других странах.

## Настройка eos-rankmirrors для постоянного использования

Чтобы `eos-rankmirrors` всегда использовал сортировку по скорости (а не по умолчанию — по свежести), отредактируйте конфиг :

```bash
sudo nano /etc/eos-rankmirrors.conf
```

Добавьте или раскомментируйте строку:

```bash
CMD_OPTIONS=(
    --sort rate
)
```

Теперь при каждом обновлении зеркал (вручную или автоматически) будет использоваться сортировка по скорости.

# Статья: Настройка зеркал в EndeavourOS — полный гайд

## Введение

EndeavourOS использует два независимых репозитория:
1. **Собственные репозитории EndeavourOS** — небольшие, содержат специфичные для дистрибутива пакеты (Welcome app, темы, утилиты)
2. **Репозитории Arch Linux** — основные, содержат подавляющее большинство пакетов 

Для каждого из них существуют свои инструменты настройки.

## Часть 1. Зеркала EndeavourOS (eos-rankmirrors)

### Что это и зачем нужно

`eos-rankmirrors` — встроенная утилита EndeavourOS для сортировки зеркал дистрибутива. Она работает только со списком зеркал из пакета `endeavouros-mirrorlist` .

**Как работает ранжирование:**

По умолчанию сортировка происходит **сначала по свежести (age), затем по скорости (rate)**. Это важно, потому что самое быстрое зеркало может быть давно не обновляться — тогда вы получите устаревшие пакеты. Сначала отбираются зеркала с актуальным `update-level`, а уже среди них — самые быстрые .

### Базовые команды

**Сортировка по скорости (как у вас уже есть):**

```bash
eos-rankmirrors --sort rate
```

**Сортировка по свежести (по умолчанию):**

```bash
eos-rankmirrors
```

**Игнорирование проблемных зеркал:**

```bash
eos-rankmirrors --ignore='funami|meowsmp|distrohub'
```

**Принудительное добавление зеркал в начало списка:**

```bash
eos-rankmirrors --prefer='https://ваше-зеркало/repo/$repo/$arch' --sort rate
```

### Продвинутая настройка

**Файл конфигурации:** `/etc/eos-rankmirrors.conf`

Пример настройки для постоянного использования сортировки по скорости :

```bash
# /etc/eos-rankmirrors.conf
CMD_OPTIONS=(
    --sort rate
)

# Игнорируем проблемные зеркала
EOS_IGNORED_MIRRORS=(
    funami
    meowsmp
    distrohub
    del2
    diyarciftci
    c0urier
)
```

### Почему нет РФ зеркал?

На момент написания статьи в официальном списке зеркал EndeavourOS **отсутствуют зеркала на территории России**. Поэтому единственный способ получить "локальные" зеркала — дождаться их появления в официальном списке или поднять своё зеркало.

**Но это не проблема!** Как показал ваш тест, зеркала в Европе (Швеция, Швейцария, Германия) дают скорость **0.2-0.3 секунды** — это отличный результат.

## Часть 2. Зеркала Arch Linux (reflector)

### Что это

`reflector` — официальный инструмент Arch Linux для работы с зеркалами основных репозиториев. Он значительно гибче `eos-rankmirrors` и поддерживает :
- Фильтрацию по странам
- Фильтрацию по протоколу
- Сортировку по разным критериям
- Ограничение количества зеркал

### Ваша успешная команда

```bash
sudo reflector --verbose \
    --country Russia \
    --country Germany \
    --country Netherlands \
    --country 'United States' \
    --latest 10 \
    --sort rate \
    --protocol https \
    --connection-timeout 10 \
    --download-timeout 10 \
    --save /etc/pacman.d/mirrorlist
```

**Что она сделала:**
- Искала зеркала в РФ (их не оказалось)
- Взяла зеркала из Германии, Нидерландов, США
- Выбрала 10 самых свежих из каждой страны
- Отсортировала по скорости
- Сохранила результат

**Результат:** Mirrorlist создан, в нём 10 зеркал, включая `berlin.mirror.pkgbuild.com` и `arch.phinau.de`.

## Часть 3. Альтернативные инструменты

В экосистеме EndeavourOS/Arch существует несколько инструментов для работы с зеркалами :

| Инструмент | Для каких репозиториев | Особенности |
|------------|----------------------|-------------|
| `eos-rankmirrors` | Только EndeavourOS | Простой, встроенный |
| `reflector` | Arch Linux | Гибкий, официальный |
| `rate-mirrors` | Arch + EndeavourOS | Быстрый, из AUR |
| `rami` | Arch Linux | Новый, много опций |

## Часть 4. Практические рекомендации

### Как часто обновлять зеркала

- **EndeavourOS зеркала:** при каждой синхронизации пакетов или раз в 1-2 недели
- **Arch зеркала:** раз в 1-2 недели

### Порядок действий (важно!)

```bash
# 1. Обновляем зеркала EndeavourOS
sudo eos-rankmirrors --sort rate

# 2. Обновляем зеркала Arch
sudo reflector --latest 10 --sort rate --protocol https --save /etc/pacman.d/mirrorlist

# 3. Обновляем систему
sudo pacman -Syyu
```

Порядок не имеет значения — файлы разные и независимые. Главное — сделать оба обновления **перед** использованием `pacman` .

### Как проверить результат

```bash
# Просмотр зеркал EndeavourOS
ccat /etc/pacman.d/endeavouros-mirrorlist

# Просмотр зеркал Arch
ccat /etc/pacman.d/mirrorlist
```
