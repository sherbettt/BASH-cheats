# Настройка Яндекс.Диска в EndeavourOS: полное руководство

## Введение

В операционной системе EndeavourOS (дистрибутив на базе Arch Linux) возникла необходимость настроить синхронизацию с облачным хранилищем Яндекс.Диск. Задача не сложная, но важная для регулярной работы с документами, фотографиями и другими данными.

В этой статье описан весь путь: от диагностики проблемы до финальной настройки и проверки работы клиента.

---

## Часть 1. Исходные данные и диагностика

### Текущее окружение
- **ОС**: EndeavourOS (ядро на базе Arch Linux)
- **Пользователь**: `kkorablin`
- **Локальная папка для синхронизации**: `/home/kkorablin/Yandex.Disk`

### Что было обнаружено при проверке?

При выполнении команды `ll ~/` в домашней директории была видна папка `Yandex.Disk`, но внутри неё не было файлов.

При проверке конфигурации клиента `~/.config/yandex-disk/config.cfg` выяснилось, что:

1. **Файл авторизации не указан**. В конфиге была строка `auth=/home/kkorablin/.config/yandex-disk/passwd`, но самого файла `passwd` не существовало.
2. **Папка синхронизации была закомментирована**. Строка `#dir=/path/to/sync/dir` осталась по умолчанию.
3. Не были заданы никакие параметры прокси, исключаемые папки или другие настройки.

**Вывод**: клиент установлен, но первичная настройка (`setup`) не была выполнена.

## Часть 2. Выполнение первоначальной настройки

### Запуск мастера настройки

```bash
yandex-disk setup
```

### Пошаговый проход по диалогу

#### 1. Вопрос о прокси-сервере
```
Использовать прокси-сервер? [y/N]: n
```
*Пояснение*: прокси не используется, домашняя сеть прямо выходит в интернет.

#### 2. Авторизация в Яндексе
```
Авторизация
Если у вас ещё нет аккаунта на Яндексе, его можно завести по ссылке 
https://passport.yandex.ru/passport?mode=register

Откройте страницу 'https://ya.ru/device' и введите код ‘kebm7ekf’ в течение 300 секунд.
Убедитесь, что вы авторизованы в браузере с нужным аккаунтом.
```
*Что было сделано*: в браузере открыта страница `ya.ru/device`, введён полученный код. После этого Яндекс выдал разрешение для приложения.

```
Токен сохранен в /home/kkorablin/.config/yandex-disk/passwd
```

**Важно**: Токен — это ключ доступа к вашему Диску. Он хранится в зашифрованном виде. Без него клиент не может авторизоваться на сервере.

#### 3. Выбор папки синхронизации
```
Введите путь к папке Яндекс.Диска 
(Оставьте пустым для использования папки '/home/kkorablin/Yandex.Disk'): 
```
*Действие*: просто нажат Enter, выбран путь по умолчанию.

#### 4. Автозапуск клиента
```
Запускать Яндекс.Диск при входе в систему? [Y/n]: y
```
*Пояснение*: клиент будет автоматически стартовать при каждом входе пользователя. Это удобно — не нужно помнить о ручном запуске.

#### 5. Запуск демона
```
Запуск демона... Готово
```
Демон (`yandex-disk`) — это фоновый процесс, который постоянно отслеживает изменения в папке и синхронизирует их с облаком.

### Первый статус синхронизации


Вывод:
```

Статус ядра синхронизации: синхронизация
Путь к папке Яндекс.Диска: '/home/kkorablin/Yandex.Disk'
        Всего: 10 GB
        Занято: 531.03 MB
        Свободно: 9.48 GB
        Максимальный размер файла: 1024 MB
        Размер корзины: 0 B

Последние синхронизированные пути:
        файл: 'Save Photos/люк-рыбы.jpg'
        файл: 'Docs/Pnevmatika инструкции/MP-46-instructions.pdf'
        ...
```

**Что мы видим**:
- Клиент начал синхронизацию (загружено 4% данных).
- Общий объём данных в облаке — 531 МБ.
- Свободно на Диске — 9.48 ГБ (из 10 ГБ).
- Показывается история последних 10 синхронизированных файлов.

### Повторная проверка спустя время

```bash
yandex-disk status
```

Вывод:
```
Статус синхронизации: 389.80 MB/ 531.03 MB (73 %)
Статус ядра синхронизации: синхронизация
...
```

**Прогресс**: 73% данных уже синхронизировано. Клиент работает стабильно.

### Завершающая синхронизация

```bash
yandex-disk sync
```

Вывод:
```
Папка синхронизирована
```

**Пояснение**: команда `sync` дожидается окончания всех текущих операций и завершает работу (в отличие от демона, который работает постоянно).

## Часть 4. Исследование возможностей клиента

В ходе работы была изучена справочная информация о клиенте:

```bash
yandex-disk --help
```

**Основные команды**:

| Команда | Назначение |
|---------|------------|
| `start` | Запустить демон и начать синхронизацию в фоне |
| `stop` | Остановить демон |
| `status` | Показать текущий статус, прогресс и последние файлы |
| `sync` | Синхронизировать папку и завершиться |
| `token` | Получить OAuth-токен и сохранить его |
| `publish` | Сделать файл или папку публичными и получить ссылку |
| `unpublish` | Удалить публичный доступ |
| `setup` | Запустить мастер первоначальной настройки |

**Важные опции**:
- `--dir` — указать другую папку для синхронизации
- `--exclude-dirs` — исключить определённые папки
- `--read-only` — режим «только чтение» (локальные изменения не загружаются)
- `--no-daemon` — запуск без возврата управления в консоль

### Что нельзя сделать?

В версии 0.1.6.1080 нет команды `queue` (просмотр очереди синхронизации). Для отслеживания прогресса используется `status` и просмотр лога.

---

## Часть 5. Полезные команды для повседневной работы

### Прогресс в реальном времени

```bash
watch -n 2 yandex-disk status
```

Обновление статуса каждые 2 секунды.

### Просмотр лога демона

```bash
tail -f ~/.config/yandex-disk/daemon.log
```

Отслеживание всех операций клиента в реальном времени.

### Остановка и запуск демона

```bash
yandex-disk stop
yandex-disk start
```

### Автоматическая синхронизация с ожиданием (для скриптов)

```bash
yandex-disk sync --wait
```

1. **Диагностика состояния клиента** — обнаружено отсутствие первоначальной настройки.
2. **Запуск мастера настройки (`yandex-disk setup`)**:

5. **Проверка статуса** — подтверждение корректной работы на всех этапах (4%, 73%, 100%).
6. **Изучение документации** — основные команды и опции клиента.

### В чём была основная проблема?

Клиент был установлен, но **не был выполнен обязательный шаг первичной настройки**. Без этого он не может получить токен, узнать папку синхронизации и начать работу. После выполнения `yandex-disk setup` всё заработало штатно, без дополнительных исправлений.

### Текущее состояние

- Яндекс.Диск работает в фоновом режиме.
- Папка `/home/kkorablin/Yandex.Disk` синхронизирована со всеми файлами из облака.
- Клиент запускается автоматически при входе в систему.
- Для управления доступны команды `status`, `stop`, `start`, `sync`.
