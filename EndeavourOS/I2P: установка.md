## 1. Принципиальная разница между i2pd и i2p-bin

### **i2pd (C++)** - "Легкий демон"
- **Философия**: Минималистичный, эффективный, делает только основное
- **Язык**: C++ (компилируемый, быстрый)
- **Память**: ~50-100 MB RAM
- **Структура**: Один двоичный файл + конфиг
- **Веб-морда**: Есть, но простая `http://127.0.0.1:7070/`
- **Основное**: Только роутер + базовые сервисы
- **Пример**: Как легковой автомобиль - довезет, но без излишеств

### **i2p-bin (Java)** - "Полнофункциональный комбайн"
- **Философия**: Всё включено, официальная эталонная реализация
- **Язык**: Java (интерпретируемый, через JVM)
- **Память**: ~400-800 MB RAM
- **Структура**: Много JAR файлов + wrapper
- **Веб-морда**: Полноценная, красивая `http://127.0.0.1:7657/`
- **Основное**: Роутер + SAM/BOB + встроенные приложения
- **Пример**: Как большой внедорожник - комфортно, но прожорливо

## 2. Сравнение веб-интерфейсов

### i2pd (C++) - простая веб-морда
```bash
# После запуска открываем
http://127.0.0.1:7070/
```
**Что есть:**
- Основная статистика (пиры, туннели)
- Небольшой мониторинг
- Минимум настроек (в основном через конфиг)
- Быстро загружается

### i2p (Java) - полноценная веб-морда
```bash
# После запуска открываем
http://127.0.0.1:7657/
```
**Что есть:**
- Полная админ-панель
- Настройка туннелей
- Управление бандвитчем
- Статистика в графиках
- Менеджер встроенных приложений
- Консоль и логи

## 3. Что лучше написано и оптимизировано?

### **i2pd (C++)** - лучше оптимизирован
**Плюсы:**
- Быстрее стартует
- Меньше жрет память
- Никаких зависимостей кроме стандартных библиотек
- Работает на слабом железе
- Быстрее обрабатывает пакеты

**Минусы:**
- Меньше функций
- Может не поддерживать новые фичи I2P
- Веб-морда простая

### **i2p-bin (Java)** - функциональнее
**Плюсы:**
- Полная поддержка всех функций I2P
- Стабильнее (зрелый код, годами тестировался)
- Больше встроенных утилит
- Красивый веб-интерфейс

**Минусы:**
- Медленный старт (JVM)
- Много памяти
- Зависит от Java
- Сложнее конфигурировать

## 4. Как лучше выходить в I2P сеть?

### Выбор метода доступа:

| Метод | Описание | Плюсы | Минусы |
|-------|----------|-------|--------|
| **Веб-браузер через HTTP прокси** | Настраиваете браузер на `127.0.0.1:4444` (HTTP) или `127.0.0.1:4445` (SOCKS) | Просто, привычно | Не все сайты работают через HTTP прокси |
| **I2P Browser Bundle** | Специализированный браузер с настройками I2P | Настроено из коробки | Занимает место |
| **SOCKS прокси** | Настройка приложений через SOCKS5 `127.0.0.1:4447` (i2pd) или `127.0.0.1:4444` (i2p) | Работает с любым приложением | Требует настройки приложений |
| **SAM (Simple Anonymous Messaging)** | API для разработчиков, приложения подключаются через SAM | Гибкость | Сложно для обычных пользователей |

### Рекомендация для начала:
1. **i2pd** - настройте SOCKS5 прокси в браузере
2. **i2p** - используйте встроенный браузер или HTTP прокси

## 5. Настройка и запуск

### **Настройка i2pd (рекомендую начать с него)**

#### Запуск:
```bash
# Запустить сервис
sudo systemctl start i2pd.service

# Автозапуск
sudo systemctl enable i2pd.service

# Проверить статус
sudo systemctl status i2pd.service

# Посмотреть логи (полезно для отладки)
sudo journalctl -u i2pd.service -f
```

#### Настройка конфига:
```bash
# Открыть конфиг
sudo nano /var/lib/i2pd/i2pd.conf
```

**Базовые настройки** (раскомментируйте и измените):
```ini
# Путь для логов
log = /var/log/i2pd/i2pd.log
loglevel = info

# HTTP прокси (для браузера)
[http]
address = 127.0.0.1
port = 4444

# HTTPS прокси
[https]
address = 127.0.0.1
port = 4445

# SOCKS5 прокси (рекомендую)
[socks]
address = 127.0.0.1
port = 4447

# SAM для приложений
[sam]
address = 127.0.0.1
port = 7656

# Веб-консоль
[webconsole]
address = 127.0.0.1
port = 7070
```

#### Проверка работы:
```bash
# Проверить, что сервис работает
curl --socks5-hostname 127.0.0.1:4447 http://check.torproject.org/

# Или просто открыть веб-консоль
firefox http://127.0.0.1:7070/
```

### **Настройка i2p (Java)**

#### Запуск:
```bash
# Запустить сервис
sudo systemctl start i2prouter.service

# Автозапуск
sudo systemctl enable i2prouter.service

# Проверить статус
sudo systemctl status i2prouter.service

# Посмотреть логи
sudo journalctl -u i2prouter.service -f
```

#### Основные конфиги:
```bash
# Основной конфиг
sudo nano /opt/i2p/router.config

# Настройки wrapper (JVM параметры)
sudo nano /opt/i2p/wrapper.config
```

#### Проверка работы:
```bash
# Открыть веб-консоль
firefox http://127.0.0.1:7657/
```

## 6. Настройка браузера для работы через I2P

### **Firefox:**
1. Откройте Настройки -> Сеть -> Настройка прокси
2. Выберите "Ручная настройка"
3. Настройте:

**Для i2pd:**
```
HTTP прокси: 127.0.0.1:4444
HTTPS прокси: 127.0.0.1:4445
SOCKS прокси: 127.0.0.1:4447
SOCKS v5
```

**Для i2p:**
```
HTTP прокси: 127.0.0.1:4444
HTTPS прокси: 127.0.0.1:4445
```

4. В `about:config` установите:
```
network.proxy.socks_remote_dns = true
```

### **Chromium/Google Chrome:**
```bash
# Запуск с прокси
chromium --proxy-server="socks5://127.0.0.1:4447" \
         --host-resolver-rules="MAP * ~NOTFOUND , EXCLUDE localhost"

# Или через расширение Proxy SwitchyOmega
```

## 9. Вывод и рекомендация

### Для обычного пользователя:
**Выбирайте i2pd (C++)**, если:
- У вас не супер-мощный компьютер
- Хотите быстро запуститься
- Не нужны все навороты Java-версии
- Любите минимализм

**Выбирайте i2p (Java)**, если:
- У вас мощный компьютер (>= 8GB RAM)
- Хотите красивый веб-интерфейс
- Нужны все функции I2P
- Планируете использовать встроенные приложения

### Для сервера/VPS:
Однозначно **i2pd** - меньше ресурсов, стабильнее, проще в управлении.

## 10. Продвинутая настройка: Сборка портативного браузера i2pdbrowser для i2pd

Если вы не хотите настраивать прокси вручную или хотите получить полностью изолированный браузер для работы с I2P, вы можете использовать проект **i2pdbrowser** от PurpleI2P. Это набор скриптов, который скачивает свежий Firefox ESR и настраивает его для работы через i2pd.

### 10.1. Скачивание архива

Существует два способа получить i2pdbrowser: скачать готовый портативный архив или клонировать репозиторий со скриптами для сборки.

**Способ 1: Скачать готовый портативный архив (рекомендуется для быстрого старта)**

Перейдите на страницу релизов проекта: https://github.com/PurpleI2P/i2pdbrowser/releases/tag/latest

Найдите в разделе "Assets" файл с именем, содержащим `linux`, например `I2PdBrowserPortable_473cdfe-linux.tar.gz`, и скачайте его.

В терминале это можно сделать с помощью `wget`:
```bash
# Перейдите в папку, где вы храните программы
cd ~/Programs

# Скачайте архив с последней версией
wget https://github.com/PurpleI2P/i2pdbrowser/releases/download/latest/I2PdBrowserPortable_473cdfe-linux.tar.gz
```

**Способ 2: Клонировать репозиторий (для сборки из исходников)**

Если вы хотите собрать браузер самостоятельно (например, для получения самой свежей версии Firefox), клонируйте репозиторий:
```bash
git clone https://github.com/PurpleI2P/i2pdbrowser.git
```

### 10.2. Просмотр содержимого архива перед распаковкой

Перед распаковкой полезно посмотреть, что находится внутри архива. Это делается с помощью команды `tar` с флагом `-t`:

```bash
# Просмотр содержимого архива (вывод первых 20 строк для краткости)
tar -tzf I2PdBrowserPortable_473cdfe-linux.tar.gz | head -20
```

Вы увидите структуру архива, которая обычно содержит две основные папки:
- `build/` - содержит скрипты сборки, настройки и профиль Firefox
- `i2pd/` - содержит исполняемый файл i2pd и его конфигурацию

### 10.3. Распаковка архива

```bash
# Распаковка в текущую папку
tar -xzf I2PdBrowserPortable_473cdfe-linux.tar.gz

# Или распаковка в отдельную папку (рекомендуется)
mkdir -p i2p-browser
tar -xzf I2PdBrowserPortable_473cdfe-linux.tar.gz -C i2p-browser
```

После распаковки у вас будет папка с двумя каталогами: `build/` и `i2pd/`.

### 10.4. Сборка браузера (если вы клонировали репозиторий)

Если вы скачали исходники, а не готовый портативный архив, вам нужно собрать браузер с помощью скрипта `build.sh`:

```bash
# Перейдите в папку с исходниками
cd ~/Programs/i2pdbrowser

# Запустите скрипт сборки
cd build
./build.sh
```

Этот скрипт скачает последнюю версию Firefox ESR, язык packages, расширение NoScript и настроит всё для работы через I2P.

### 10.5. Установка необходимых зависимостей

При первом запуске собранного Firefox вы можете столкнуться с ошибкой о недостающей библиотеке `libdbus-glib-1.so.2`.

**Решение для EndeavourOS / Arch Linux:**
```bash
sudo pacman -S dbus-glib
```

Для других дистрибутивов (Debian/Ubuntu):
```bash
sudo apt-get install libdbus-glib-1-2
```

### 10.6. Запуск i2pd из портативной сборки

Портативная сборка i2pdbrowser включает в себя i2pd, который не конфликтует с системным. Запустите его в отдельном терминале:

```bash
# Перейдите в папку с i2pd из сборки
cd ~/Programs/i2p-browser/i2pd

# Запустите демон
./i2pd --datadir=.
```

**Важно:** Оставьте этот терминал открытым. i2pd должен работать всё время, пока вы пользуетесь I2P. Для удобства можно использовать `screen`:
```bash
# Установка screen (если не установлен)
sudo pacman -S screen

# Запуск i2pd в фоновой сессии screen
screen -dmS i2p ./i2pd --datadir=.
```

### 10.7. Запуск браузера из портативной сборки

Для запуска браузера используйте скрипт `start-i2pd-browser` из папки `build/scripts`:

```bash
cd ~/Programs/i2p-browser/build
./scripts/start-i2pd-browser
```

**Если скрипт не работает или выдает ошибку**, попробуйте запустить Firefox напрямую с указанием профиля:

```bash
cd ~/Programs/i2p-browser/build/firefox
./firefox --profile ../profile
```

### 10.8. Проверка работы и решение проблем

**Проверка, что i2pd слушает порты:**
```bash
ss -tulpn | grep -E "4444|4447|7070"
```

**Проверка, что HTTP прокси работает:**
```bash
curl -v -x http://127.0.0.1:4444 http://i2p-projekt.i2p/
```

Если команда `curl` возвращает HTML-код или редирект (код 302), значит прокси работает. Если вы получаете ошибку "Empty reply from server", проверьте, что i2pd запущен и подключился к сети (в логах должно быть много строк с "Added new routers").

**Распространенная ошибка:** Если браузер не может найти адрес, откройте веб-консоль i2pd `http://127.0.0.1:7070/` и посмотрите на количество роутеров. Если их меньше 20, подождите 5-10 минут, пока i2pd загрузит сетевую базу.

### 10.9. Использование системного i2pd вместо портативного

Если вы предпочитаете использовать системный i2pd (установленный через пакетный менеджер), вы можете сделать это, но вам нужно будет либо перенастроить порты, либо указать браузеру на системные порты.

**Для Arch/EndeavourOS системный i2pd устанавливается и запускается так:**
```bash
# Установка
sudo pacman -S i2pd

# Запуск и включение в автозагрузку
sudo systemctl enable --now i2pd.service

# Проверка статуса
sudo systemctl status i2pd.service
```

**Внимание:** Если системный i2pd уже использует порты 4444 и 4447, портативный i2pd из сборки не сможет их занять. Выберите что-то одно: либо используйте портативный i2pd из папки `~/Programs/i2p-browser/i2pd`, либо системный.

**Чтобы использовать портативный браузер с системным i2pd:**
```bash
# Остановите портативный i2pd
# Запустите браузер из сборки
cd ~/Programs/i2p-browser/build/firefox
./firefox --profile ../profile
```
И в настройках браузера вручную укажите прокси `127.0.0.1:4444`.

### 10.10. Итоговая инструкция по ежедневному использованию

После того как всё настроено, каждый раз для выхода в I2P делайте следующее:

1.  **Запустите i2pd** (если он не запущен как системный сервис):
    ```bash
    cd ~/Programs/i2p-browser/i2pd && ./i2pd --datadir=.
    ```

2.  **Запустите браузер**:
    ```bash
    cd ~/Programs/i2p-browser/build/firefox && ./firefox --profile ../profile
    ```

3.  **Откройте сайт** в браузере, например, `http://i2p-projekt.i2p/` или `http://forum.i2p/`.

### 10.11. Создание ярлыка для быстрого запуска (необязательно)

Создайте `.desktop` файл, чтобы запускать браузер из меню приложений:

```bash
nano ~/.local/share/applications/i2p-browser.desktop
```

Вставьте следующее содержимое:
```ini
[Desktop Entry]
Name=I2P Browser
Comment=Anonymous browser for I2P network
Exec=/home/kkorablin/Programs/i2p-browser/build/firefox/firefox --profile /home/kkorablin/Programs/i2p-browser/build/profile
Icon=/home/kkorablin/Programs/i2p-browser/build/preferences/browser/chrome/icons/default/default128.png
Terminal=false
Type=Application
Categories=Network;
```
