# Установка утилиты dysk на Debian 12 (LXC)

## 1. Скачать готовый бинарник
```bash
curl -LO https://github.com/Canop/dysk/releases/download/v3.6.0b/dysk_3.6.0.zip
#или
wget https://github.com/Canop/dysk/releases/download/v3.6.0b/dysk_3.6.0.zip
unzip dysk_3.6.0.zip
```

## 2. Установить в систему
```bash
sudo cp build/x86_64-unknown-linux-musl/dysk /usr/local/bin/
sudo chmod +x /usr/local/bin/dysk
```

## 3. Опционально: man и автодополнение
```bash
# man-страница
sudo mkdir -p /usr/local/share/man/man1
sudo cp build/man/dysk.1 /usr/local/share/man/man1/

# автодополнение для bash
sudo cp build/completion/dysk.bash /etc/bash_completion.d/
source /etc/bash_completion.d/dysk.bash
```

## 4. Проверка
```bash
dysk --version
dysk
```

**Примечание**: Не требуется обновление Rust/cargo — используем готовую статическую сборку `x86_64-unknown-linux-musl`, работает на любой системе Linux.

## 5. Расширенные примеры использования dysk
```bash
dysk -a                    # показать все диски (включая tmpfs, devtmpfs)
dysk /home                 # информация только о конкретной точке монтирования
dysk -s                    # сортировка по использованию (самые загруженные сверху)
dysk -S                    # сортировка по размеру
dysk -m                    # показать в мегабайтах (по умолчанию человеко-читаемый)
dysk -j                    # вывод в JSON для скриптов
dysk -h                    # справка по всем опциям
dysk -d 2                  # глубина дерева монтирования (показывает вложенные)
dysk --no-color            # отключить цвета для скриптов
dysk --no-graph            # отключить графическую полосу
```

# Установка Rust-утилит для дисков на Debian (LXC)

## 0. Компилятор C
```bash
sudo apt install build-essential pkg-config
cc --version
```


## 1. Базовое обновление Rust (рекомендуется)
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env
rustup install stable
rustup default stable
```
```bash
#см.
https://crates.io/
```

**Важно: Добавьте ~/.cargo/bin в PATH:**
```bash
# Для bash
echo 'export PATH="$HOME/.cargo/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Для zsh
echo 'export PATH="$HOME/.cargo/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# Для устранения надоедливого "correct" в zsh (для procs)
echo 'alias procs="nocorrect procs"' >> ~/.zshrc
```
Читай про переменные [Параметры_Cargo](https://www.altlinux.org/Участник:HihinRuslan/Параметры_Cargo)

## 5. Расширенные примеры для дисковых утилит

### dust — анализ размера папок
```bash
dust                      # анализ текущей директории
dust ~/Downloads          # анализ конкретной папки
dust -d 2                 # глубина 2 уровня вложенности
dust -r                   # обратная сортировка (меньшие сверху)
dust -p                   # показать проценты от общего размера
dust -c                   # цветной вывод (по умолчанию)
dust -s                   # агрегировать одинаковые имена
dust -z 10                # показать только 10 записей
dust -x                   # не выходить за пределы файловой системы
dust -o                   # не показывать скрытые файлы
dust -t 10                # минимальный размер 10 МБ для показа
```

### dua — интерактивный анализ (TUI)
```bash
# Интерактивный режим
dua i                     # запустить интерактивный режим в текущей папке
dua i /var/log            # анализ конкретной папки
dua i -t 4                # использовать 4 потока для ускорения

# Хоткеи в интерактивном режиме:
# ↑/↓ или j/k         — навигация
# → или l              — войти в папку
# ← или h              — выйти из папки
# /                    — поиск
# s                    — сортировка (по размеру, имени, количеству)
# d                    — удалить элемент (с подтверждением)
# c                    — показать график использования
# q или Ctrl+c         — выход
# ?                    — справка

# Неинтерактивные режимы
dua                    # показать размер текущей папки (агрегированный)
dua -A                 # показать размеры всех подпапок
dua -f                 # форматировать как дерево
dua -b                 # использовать байты вместо человеко-читаемого
```

### diskus — быстрый подсчет
```bash
diskus                  # размер текущей папки
diskus /home/user       # размер конкретной папки
diskus -b               # вывод в байтах
diskus -v               # подробный вывод
diskus -o json          # JSON-вывод
diskus --threads 8      # использовать 8 потоков
```

### gdu — альтернатива ncdu (TUI)
```bash
gdu                     # интерактивный режим в текущей папке
gdu /home               # анализ /home
gdu -l                  # следовать симлинкам
gdu -i .git             # игнорировать папки .git
gdu -n                  # показать только файлы/папки с размером > 0

# Хоткеи в gdu:
# ↑/↓                    — навигация
# Enter или →            — войти в папку
# ←                      — выйти из папки
# d                      — удалить файл/папку (осторожно!)
# c                      — показать график использования
# s                      — сортировка
# i                      — информация о файле/папке
# r                      — обновить
# q или Ctrl+c           — выход
```

Все утилиты используют статическую сборку (`musl`) и работают на любой системе Linux без зависимостей.

### 📁 **Расширенные примеры для файловых утилит**

**fd — поиск файлов**
```bash
fd -H                     # включая скрытые файлы
fd -t d                   # только директории
fd -t f                   # только файлы
fd -x rm {}               # выполнить команду для каждого найденного
fd -X vim                 # выполнить команду со всеми найденными сразу
fd --size +10M            # файлы больше 10 МБ
fd --changed-within 10m   # измененные за последние 10 минут
fd --executable           # только исполняемые файлы
```
практические примеры
```bash
# 1. Удалить все .tmp файлы
fd -e tmp -x rm {}

# 2. Удалить все пустые директории
fd -t d -e empty -x rmdir {}

# 3. Удалить все файлы и папки .git
fd -H '^.git$' -t f -t d -x rm -rf {}

# 4. БЕЗОПАСНЫЙ способ — сначала посмотреть, что будет удалено
fd -x echo {}                # показать список
# или
fd --exec-batch echo {}      # показать батчем

# 5. Подтверждение перед удалением
fd -x rm -i {}               # -i запросит подтверждение

# 6. Вместо прямого удаления — использовать trash-cli
fd -x trash-put {}           # переместить в корзину

# 7. Удалить file1, file2, ..., file8
fd 'file[1-8]$' -x rm {}

# 8. Отсортировать и взять первые 8
fd '^file' | head -8 | xargs rm

# 9. Использовать --batch для групповой обработки
fd '^file' --batch-size 8 -x rm {}

# 10. Удалить все, кроме file9 и file10
fd '^file' -x rm {} --exclude 'file9' --exclude 'file10'
```

# Ошибка OOM killer

Если возникает следующая ошибка:
```bash
   Compiling zellij-client v0.43.1
   Compiling zellij v0.43.1
error: could not compile `zellij` (bin "zellij")

Caused by:
  process didn't exit successfully: `rustc ...` (signal: 9, SIGKILL: kill)
error: failed to compile `zellij v0.43.1`
```

Это **SIGKILL (signal 9)** — компилятор убит системой OOM Killer из-за нехватки памяти.

### Решение 1: Уменьшить нагрузку на память
```bash
# Ограничить количество параллельных заданий
cargo install zellij --jobs 1

# Или с меньшим уровнем оптимизации
CARGO_PROFILE_RELEASE_OPT_LEVEL=0 cargo install zellij --jobs 1
```

### Решение 2: Увеличить swap (временно)
```bash
# Создать swap-файл 2GB
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# Установка
cargo install zellij

# После установки можно отключить
sudo swapoff /swapfile
sudo rm /swapfile
```

### Решение 3: Готовый бинарник (рекомендуется при нехватке памяти)
```bash
# Скачать предкомпилированный бинарник
wget https://github.com/zellij-org/zellij/releases/latest/download/zellij-x86_64-unknown-linux-musl.tar.gz
tar -xzf zellij-x86_64-unknown-linux-musl.tar.gz
sudo cp zellij /usr/local/bin/
```

### Решение 4: Через системный пакет (для Arch/Manjaro)
```bash
sudo pacman -S zellij
```

# Обновление всех установленных пакетов через `cargo install`

## 1. **Самый простой способ — cargo-update**

Установите специальную утилиту для обновления:
```bash
# Установка cargo-update
cargo install cargo-update

# Обновление всех установленных пакетов
cargo install-update --all

# Или сокращенно
cargo install-update -a
```

## 2. **Список установленных пакетов с последующей переустановкой**

```bash
# Посмотреть все установленные через cargo пакеты
cargo install --list

# Обновить конкретный пакет
cargo install <package_name> --force

# Обновить все (через цикл в bash)
for pkg in $(cargo install --list | grep -E '^[a-z0-9_-]+ v' | awk '{print $1}'); do
    cargo install $pkg --force
done
```

## 3. **Использование cargo-upgrades (альтернатива)**

```bash
# Установка
cargo install cargo-upgrades

# Проверка наличия обновлений
cargo upgrades

# Обновление всех пакетов
cargo upgrades --apply
```

## 4. **Скрипт для обновления с проверкой версий**

Создайте скрипт `update-cargo-pkgs.sh`:
```bash
#!/bin/bash
echo "Checking installed cargo packages..."

# Получаем список установленных пакетов
cargo install --list | grep -E '^[a-z0-9_-]+ v' | while read -r line; do
    pkg=$(echo "$line" | awk '{print $1}')
    current_version=$(echo "$line" | grep -oE 'v[0-9]+\.[0-9]+\.[0-9]+[a-z0-9.-]*' | sed 's/v//')
    
    echo "Checking $pkg (current: $current_version)..."
    
    # Получаем последнюю версию из crates.io
    latest_version=$(cargo search "$pkg" --limit 1 | grep "^$pkg" | grep -oE '[0-9]+\.[0-9]+\.[0-9]+[a-z0-9.-]*' | head -1)
    
    if [ -n "$latest_version" ] && [ "$current_version" != "$latest_version" ]; then
        echo "  ↳ Updating $pkg $current_version → $latest_version"
        cargo install "$pkg" --force
    else
        echo "  ↳ Already up to date"
    fi
done

echo "Done!"
```

Сделайте скрипт исполняемым и запустите:
```bash
chmod +x update-cargo-pkgs.sh
./update-cargo-pkgs.sh
```

## 5. **Проверка только наличия обновлений (без установки)**

```bash
# Показать, какие пакеты можно обновить
cargo install --list | awk '{print $1}' | while read pkg; do
    current=$(cargo install --list | grep "^$pkg" | grep -oE 'v[0-9.]+')
    latest=$(cargo search "$pkg" --limit 1 | grep "^$pkg" | grep -oE '[0-9.]+' | head -1)
    if [[ "$current" != *"$latest"* ]]; then
        echo "$pkg: $current → $latest"
    fi
done
```

## 6. **Утилита cargo-outdated для проектов (не для глобальных пакетов)**

**Важно:** `cargo outdated` работает только в директории с проектом (где есть `Cargo.toml`), **не** для глобально установленных пакетов.

```bash
# Для проектов (если нужно обновить зависимости)
cargo install cargo-outdated
cd ~/my-project
cargo outdated
cargo update
```

## Рекомендация

**Самый удобный способ** — использовать `cargo-update`:

```bash
# Один раз установить
cargo install cargo-update

# Периодически обновлять все
cargo install-update --all
```

**Преимущества:**
- Простая команда
- Показывает версии до и после обновления
- Не требует нахождения в проекте
- Работает только с пакетами, установленными через `cargo install`

**Примечание:** При обновлении некоторых пакетов может потребоваться память (как в случае с zellij). Если возникает ошибка OOM, используйте `--jobs 1` или скачайте готовый бинарник.
