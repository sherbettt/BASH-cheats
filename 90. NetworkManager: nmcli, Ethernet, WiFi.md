# NetworkManager nmcli: полное руководство по WiFi, Ethernet и статическим маршрутам

`nmcli` (NetworkManager Command Line Interface) — мощный инструмент для управления сетями в Linux. Это руководство охватывает всё: от базовых команд до продвинутой настройки маршрутов.

## 1. Основные команды и просмотр состояния
```bash
# Статус всех устройств (интерфейсов)
nmcli device status
# или сокращённо
nmcli d

# Список всех профилей подключений
nmcli connection show
# или сокращённо
nmcli c

# Только активные подключения
nmcli connection show --active

# Подробная информация о конкретном профиле
nmcli connection show "Имя_профиля"

# Общий статус NetworkManager
nmcli general status

# Состояние WiFi-радио (включено/выключено)
nmcli radio wifi

# Включить/выключить WiFi адаптер
nmcli radio wifi on
nmcli radio wifi off
```

## 2. Управление подключениями и профилями
```bash
# проверка автоподключения на интерфейсах
nmcli -f name,autoconnect connection

# Активировать (подключить) существующий профиль
nmcli connection up "Имя_профиля"

# Деактивировать (отключить) профиль
nmcli connection down "Имя_профиля"

# Отключиться от текущей сети (по интерфейсу)
nmcli device disconnect wlan0

# Удалить (забыть) профиль
nmcli connection delete "Имя_профиля"

# Включить автоподключение для профиля
nmcli connection modify "Имя_профиля" connection.autoconnect yes

# Отключить автоподключение
nmcli connection modify "Имя_профиля" connection.autoconnect no

# Перезапуск NetworkManager (если изменения не применяются)
sudo systemctl restart NetworkManager
```

## 3. Настройка проводного (Ethernet) подключения
```bash
# Создать новое Ethernet-подключение (DHCP)
nmcli connection add type ethernet ifname eth0 con-name "my-eth"

# Создать статическое Ethernet-подключение (IP, шлюз, DNS)
nmcli connection add type ethernet ifname eth0 con-name "my-static-eth" \
ip4 192.168.1.100/24 gw4 192.168.1.1
nmcli connection modify "my-static-eth" ipv4.dns "8.8.8.8,8.8.4.4"

# Активировать созданный профиль
nmcli connection up "my-static-eth"
```

## 4. Работа с WiFi сетями
```bash
### Просмотр доступных сетей
# Простой список сетей
nmcli device wifi list

# Принудительное сканирование перед выводом
nmcli device wifi list --rescan yes

# Вывод только нужных полей (SSID, BSSID, сигнал, защита)
nmcli -f SSID,BSSID,SIGNAL,SECURITY device wifi list

# Принудительное сканирование (без вывода)
nmcli device wifi rescan

# Показать информацию о сетях для конкретного интерфейса
nmcli device wifi list ifname wlan0
```

### Подключение к WiFi

**Важно:** В командах подключения `nmcli device wifi connect` указывается **имя интерфейса** (например, `wlan0`), а не имя профиля.

### Способ 1: С паролем в команде (самый частый)
```bash
nmcli device wifi connect "Имя_сети" password "ваш_пароль"
```

### Способ 2: С запросом пароля (пароль не остаётся в истории)
```bash
nmcli device wifi connect "Имя_сети" --ask
```

### Способ 3: Если сеть скрыта (не вещает SSID)
```bash
nmcli device wifi connect "Имя_сети" password "ваш_пароль" hidden yes
```

### Способ 4: Подключение к конкретной точке по BSSID (MAC-адресу)
#### Когда несколько точек с одинаковым SSID (2.4ГГц и 5ГГц)
```bash
nmcli device wifi connect "Имя_сети" password "пароль" bssid AA:BB:CC:DD:EE:FF
```

### Способ 5: Подключение только по BSSID (без указания SSID)
```bash
nmcli device wifi connect AA:BB:CC:DD:EE:FF password "пароль"
```

### Создать профиль WiFi без немедленного подключения
```bash
nmcli connection add type wifi ifname wlan0 con-name "мой-wifi" ssid "SSID"
nmcli connection modify "мой-wifi" wifi-sec.key-mgmt wpa-psk
nmcli connection modify "мой-wifi" wifi-sec.psk "пароль"
```

### Работа с несколькими точками доступа (одинаковый SSID)
#### Посмотреть все точки с одинаковым именем
```bash
nmcli -f SSID,BSSID,CHAN,SIGNAL device wifi list | grep "runtel"
```

#### Создать отдельный профиль для конкретной точки (например, 5ГГц)
```bash
nmcli connection add type wifi con-name "runtel-5GHz" ifname wlan0 ssid "runtel"
nmcli connection modify "runtel-5GHz" wifi.bssid BC:07:1D:FB:52:1B
nmcli connection modify "runtel-5GHz" wifi-sec.key-mgmt wpa-psk
nmcli connection modify "runtel-5GHz" wifi-sec.psk "мой_пароль"
nmcli connection up "runtel-5GHz"
```

#### Заблокировать профиль для подключения только к конкретному BSSID
```bash
nmcli connection modify "Имя_профиля" wifi.bssid AA:BB:CC:DD:EE:FF
```

#### Узнать, через какую точку (BSSID) вы подключены сейчас
```bash
iwconfig wlan0 | grep "Access Point"
# или
nmcli -f GENERAL,WIFI-PROPERTIES device show wlan0
```

## 5. Статические маршруты через NetworkManager

Этот раздел особенно полезен, когда ваш основной шлюз (например, WiFi-роутер) не знает маршруты до удалённых подсетей, но есть промежуточный хост (шлюз), который эти подсети знает.

### Просмотр маршрутов
```bash
# Посмотреть таблицу маршрутизации ядра
ip route show
# или сокращённо
ip r
```

### Посмотреть маршруты, настроенные в конкретном профиле NetworkManager
```bash
nmcli connection show "Имя_профиля" | grep -E "routes"
```

### Добавление постоянного маршрута

**Критически важно:** В `nmcli connection modify` нужно указывать не имя интерфейса (`wlan0`), а **имя профиля подключения** (например, `runtel`). Имя профиля можно узнать через `nmcli connection show`.

#### Шаг 1: Узнайте точное имя профиля
```bash
nmcli connection show
```

#### Шаг 2: Добавление маршрута в профиль
##### Формат: +ipv4.routes "сеть/маска шлюз"
```bash
sudo nmcli connection modify "Имя_профиля" +ipv4.routes "192.168.45.0/24 192.168.87.2"
sudo nmcli connection modify "Имя_профиля" +ipv4.routes "192.168.46.0/24 192.168.87.2"
```

#### Шаг 3: Применить изменения (переподключить профиль)
```bash
sudo nmcli connection down "Имя_профиля" && sudo nmcli connection up "Имя_профиля"
```

#### Шаг 4: Проверить, что маршруты появились в таблице ядра
```bash
ip r | grep -E "45|46"
```

### Удаление маршрута из профиля
##### Удаление маршрута (используйте - вместо + и точную запись)
```bash
sudo nmcli connection modify "Имя_профиля" -ipv4.routes "192.168.45.0/24 192.168.87.2"
```

### Решение проблем: игнорирование маршрутов от DHCP

Если DHCP-сервер присылает свои маршруты и они перезаписывают ваши статические, нужно отключить их получение:

##### Проверить текущее значение (должно быть "нет")
```bash
nmcli connection show "Имя_профиля" | grep ignore-auto-routes
```
```c
kkorablin@runtel-re ~ $ nmcli connection show "runtel" | grep ignore-auto-routes
ipv4.ignore-auto-routes:                нет
ipv6.ignore-auto-routes:                нет
```

#### Отключить получение маршрутов от DHCP
```bash
sudo nmcli connection modify "Имя_профиля" ipv4.ignore-auto-routes yes
```

#### Переподключить профиль
```bash
sudo nmcli connection down "Имя_профиля" && sudo nmcli connection up "Имя_профиля"
```

### Альтернативный способ: скрипт dispatcher

Если NetworkManager упорно сбрасывает маршруты даже после `ignore-auto-routes`, используйте скрипт, который будет добавлять маршруты при каждом поднятии интерфейса.

```bash
# Создайте скрипт
sudo nano /etc/NetworkManager/dispatcher.d/99-my-routes

## Содержимое скрипта (замените `wlan0`, `192.168.45.0/24` и `192.168.87.2` на свои значения):

```bash
#!/bin/bash

INTERFACE="$1"
ACTION="$2"

if [ "$INTERFACE" = "wlan0" ] && [ "$ACTION" = "up" ]; then
    # Небольшая задержка для стабилизации интерфейса
    sleep 2
    ip route add 192.168.45.0/24 via 192.168.87.2 dev wlan0
    ip route add 192.168.46.0/24 via 192.168.87.2 dev wlan0
fi
```

Сделайте скрипт исполняемым:
```bash
sudo chmod +x /etc/NetworkManager/dispatcher.d/99-my-routes
```

## 6. Примеры полного процесса настройки

### Пример 1: Подключение к новой WiFi сети
```bash
# Сканируем сети
nmcli device wifi list

# Подключаемся (создаётся профиль автоматически)
nmcli device wifi connect "MyWiFiNetwork" password "mypassword123"

# Проверяем статус
nmcli connection show --active

# Узнаём BSSID текущей точки
iwconfig wlan0 | grep "Access Point"
```

### Пример 2: Настройка статического IP на Ethernet
```bash
# Создаём профиль со статикой
nmcli connection add type ethernet ifname eth0 con-name "static-eth" \
ip4 192.168.1.50/24 gw4 192.168.1.1 ipv4.dns "8.8.8.8"

# Включаем автоподключение
nmcli connection modify "static-eth" connection.autoconnect yes

# Подключаем
nmcli connection up "static-eth"
```

### Пример 3: Добавление постоянных маршрутов для офисной сети
```bash
# Смотрим имя профиля WiFi
nmcli connection show

# Добавляем маршруты до офисных подсетей
sudo nmcli connection modify "runtel" +ipv4.routes "10.0.0.0/8 192.168.1.1"
sudo nmcli connection modify "runtel" +ipv4.routes "172.16.0.0/12 192.168.1.1"

# Отключаем получение маршрутов от DHCP
sudo nmcli connection modify "runtel" ipv4.ignore-auto-routes yes

# Переподключаемся
sudo nmcli connection down "runtel" && sudo nmcli connection up "runtel"

# Проверяем
ip r | grep -E "10\.|172\.16"
```

## 7. Диагностика и устранение проблем

### Просмотр логов NetworkManager в реальном времени
```bash
journalctl -u NetworkManager -f
```

### Проверить качество соединения с текущей точкой доступа
```bash
sudo iw dev wlan0 link
```

### Подробная информация об интерфейсе
```bash
nmcli device show wlan0
nmcli device show wlp1s0   # для ноутбуков с Intel/Qualcomm
```

### Проверка скорости и трафика на интерфейсе
```bash
nmcli device show wlan0 | grep -i speed
```

### Тест связи с новым шлюзом после добавления маршрута
```bash
ping 192.168.87.2
```

### Просмотр ARP-таблицы (кто есть в локальной сети)
```bash
ip neigh show
```

### Просмотр всех маршрутов ядра
```bash
ip route show
```

## Важные моменты

- **Имя интерфейса vs имя профиля:** В командах `nmcli device wifi connect` используется **имя интерфейса** (`wlan0`). В командах `nmcli connection modify` используется **имя профиля** (например, `runtel`). Имя профиля можно узнать через `nmcli connection show`.
- **BSSID (MAC-адрес) точки доступа** позволяет выбрать конкретную точку, если несколько точек имеют одинаковое имя (SSID). Узнать BSSID можно через `nmcli device wifi list`.
- **Статические маршруты** через `nmcli` теряются после перезагрузки, если не настроены правильно. Используйте `ignore-auto-routes` или скрипт dispatcher для сохранения.
- Для некоторых операций может потребоваться `sudo`.
- При подключении по BSSID убедитесь, что вы находитесь в зоне уверенного приёма сигнала от этой конкретной точки.
- После изменения конфигурации профиля всегда переподключайте его: `nmcli connection down "имя" && nmcli connection up "имя"`.
